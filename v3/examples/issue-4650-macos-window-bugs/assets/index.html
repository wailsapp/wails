macOS Window Behavior Bugs (v3.0.0-alpha)
BUG #1

Window.hide() Removes Tray Icon

On macOS, when hiding the window, the system tray icon also disappears, making the application completely inaccessible.

✓ Expected (Windows): Window hides, tray icon remains visible and clickable
✗ Actual (macOS): Both window AND tray icon disappear

Try using the system tray menu to restore the window after hiding.

BUG #2

ToggleMaximise White Flicker

On macOS with Frameless=true, maximizing the window causes a noticeable white flash before the dark content appears.

✓ Expected: Smooth transition maintaining dark background color
✗ Actual (macOS): White background flashes during zoom animation

Click the button and watch carefully - you'll see a white flash as the window expands.

Platform Note: Both issues are macOS-specific and do not occur on Windows. The issues are particularly noticeable on macOS Sequoia 15.5 with Apple Silicon (M4).

Root Causes:
  • Issue #1: Related to NSApplicationActivationPolicy handling
  • Issue #2: NSWindow's default white background visible during zoom animation
