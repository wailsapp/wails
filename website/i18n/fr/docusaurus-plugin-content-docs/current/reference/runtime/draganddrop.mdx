# Glisser-déposer

:::

Cette partie du runtime gère le glisser-déposer des fichiers et des dossiers dans la fenêtre.<br/>

Pour activer cette fonctionnalité, vous devez définir [EnableFileDrop](../../reference/options.mdx#enablefiledrop) à `true`
dans [Options de l'application](../../reference/options.mdx#drag-and-drop).

### OnFileDrop

Cette méthode gère l'événement drop sur la fenêtre.

Go: `OnFileDrop(ctx context.Context, callback func(x, y int, paths []string))`<br/>
Appelle la fonction callback avec les coordonnées à l'intérieur de la fenêtre où le glissement a été libéré et un tableau contenant les chemins absolus des fichiers.

JS: `OnFileDrop(callback: (x: number, y: number, paths: string[]) => void, useDropTarget: boolean) :void`<br/>
Appelle la fonction callback avec les coordonnées à l'intérieur de la fenêtre où le glissement a été libéré et un tableau contenant les chemins absolus des fichiers.

Quand `useDropTarget` est à `true`, en plus d'appeler le callback quand la drop arrive, il enregistre les event listeners sur
la fenêtre qui écoutent les coordonnées de glissement et vérifie si la souris est sur un élément qui a le style
[CSSDropProperty](../../reference/options.mdx#cssdropproperty). Si l'élément possède la propriété requise,
il ajoute la classe `wails-drop-target-active` à la liste des classes de l'élément et le supprime lorsque la souris s'en éloigne.

### OnFileDropOff

Cette méthode supprime tous les auditeurs et gestionnaires enregistrés pour les événements de glisser-déposer.

Go: `OnFileDropOff(ctx context.Context)`<br/>
N'a pas de valeur de retour.

JS: `OnFileDropOff(): void`<br/>
N'a pas de valeur de retour.
