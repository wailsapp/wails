Dateiassoziationen ermöglichen es Ihrer Anwendung, bestimmte Dateitypen zu verarbeiten, wenn
Benutzer sie öffnen. Dies ist besonders nützlich für Texteditoren, Bildbetrachter oder
jede Anwendung, die mit bestimmten Dateiformaten arbeitet. Dieser Leitfaden erklärt, wie
Sie Dateiassoziationen in Ihrer Wails v3-Anwendung implementieren.

## Übersicht

Die Unterstützung für Dateiassoziationen ist in Wails v3 derzeit verfügbar für:

- Windows (NSIS-Installationspakete)
- macOS (Anwendungs-Bundles)

## Konfiguration

Dateiassoziationen werden in der Datei `config.yml` im `build`-Verzeichnis Ihres
Projekts konfiguriert.

### Grundlegende Konfiguration

So richten Sie Dateiassoziationen ein:

1. Öffnen Sie `build/config.yml`
2. Fügen Sie Ihre Dateiassoziationen unter dem Abschnitt `fileAssociations` hinzu
3. Führen Sie `wails3 update build-assets` aus, um die Build-Assets zu aktualisieren
4. Legen Sie das Feld `FileAssociations` in den Anwendungsoptionen fest
5. Packen Sie Ihre Anwendung mit `wails3 package`

Hier ist ein Beispiel für eine Konfiguration:

```yaml
fileAssociations:
  - ext: myapp
    name: MyApp-Dokument
    description: MyApp-Dokumentdatei
    iconName: myappFileIcon
    role: Editor
  - ext: custom
    name: Benutzerdefiniertes Format
    description: Benutzerdefiniertes Dateiformat
    iconName: customFileIcon
    role: Editor
```

### Konfigurationseigenschaften

| Eigenschaft    | Beschreibung                                                      | Plattform |
|-------------|------------------------------------------------------------------|----------|
| ext         | Dateierweiterung ohne führenden Punkt (z. B. `txt`)          | Alle     |
| name        | Anzeigename für den Dateityp                                   | Alle     |
| description | Beschreibung, die in den Dateieigenschaften angezeigt wird     | Windows  |
| iconName    | Name der Icon-Datei (ohne Erweiterung) im Build-Ordner         | Alle     |
| role        | Rolle der Anwendung für diesen Dateityp (z. B. `Editor`, `Viewer`) | macOS    |
| mimeType    | MIME-Typ der Datei (z. B. `image/jpeg`)                      | macOS    |

## Lauschen auf Datei-Öffnen-Ereignisse

Um Datei-Öffnen-Ereignisse in Ihrer Anwendung zu verarbeiten, können Sie auf das
Ereignis `events.Common.ApplicationOpenedWithFile` lauschen:

```go title="main.go"
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "github.com/wailsapp/wails/v3/pkg/events"
)

func main() {
    app := application.New(application.Options{
        Name: "MyApp",
        FileAssociations: []string{".txt", ".md"}, // Unterstützte Erweiterungen angeben
    })

    // Auf Dateien lauschen, die zum Öffnen der Anwendung verwendet werden
    app.Event.OnApplicationEvent(events.Common.ApplicationOpenedWithFile, func(event *application.ApplicationEvent) {
        associatedFile := event.Context().Filename()
        application.InfoDialog().SetMessage("Anwendung mit Datei geöffnet: " + associatedFile).Show()
    })

    // Erstellen Sie Ihr Fenster und starten Sie die App...
}

```

## Schritt-für-Schritt-Tutorial

Gehen wir Schritt für Schritt durch das Einrichten von Dateiassoziationen für einen einfachen Texteditor:

<Steps>

1. ### Erstellen Sie Icons

    - Erstellen Sie Icons für Ihren Dateityp (empfohlene Größen: 16x16, 32x32, 48x48,
      256x256)
    - Speichern Sie die Icons im `build`-Ordner Ihres Projekts
    - Benennen Sie sie entsprechend Ihrer `iconName`-Konfiguration (z. B.
      `textFileIcon.png`)

    :::tip
    Sie können `wails3 generate icons` verwenden, um die erforderlichen Icons für Sie zu generieren.
    Führen Sie `wails3 generate icons --help` für weitere Informationen aus.
    :::

    - Fügen Sie für macOS eine Kopieranweisung wie `cp build/darwin/documenticon.icns {{.BIN_DIR}}/{{.APP_NAME}}.app/Contents/Resources` in die Aufgabe `create:app:bundle:` ein.

2. ### Konfigurieren Sie Dateiassoziationen

    Bearbeiten Sie die Datei `build/config.yml`, um Ihre Dateiassoziationen hinzuzufügen:

    ```yaml
    # build/config.yml
    fileAssociations:
      - ext: txt
        name: Textdokument
        description: Reintextdokument
        iconName: textFileIcon
        role: Editor
    ```

3. ### Aktualisieren Sie die Build-Assets

    Führen Sie den folgenden Befehl aus, um die Build-Assets zu aktualisieren:

    ```bash
    wails3 update build-assets
    ```

4. ### Legen Sie Dateiassoziationen in den Anwendungsoptionen fest

    Legen Sie in Ihrer Datei `main.go` das Feld `FileAssociations` in den
    Anwendungsoptionen fest:

    ```go
    app := application.New(application.Options{
      Name: "MyApp",
      FileAssociations: []string{".txt", ".md"}, // Unterstützte Erweiterungen angeben
    })
    ```

    :::tip[Warum sind Dateierweiterungen sowohl in der Anwendungskonfiguration als auch in config.yml erforderlich?]

    Unter Windows wird die Anwendung gestartet, wenn eine Datei mit einer Dateiassoziation geöffnet wird,
    wobei der Dateiname als erstes Argument an die Anwendung übergeben wird. Die
    Anwendung hat keine Möglichkeit zu wissen, ob das erste Argument eine Datei oder ein
    Befehlszeilenargument ist, daher verwendet sie das Feld `FileAssociations` in den
    Anwendungsoptionen, um zu bestimmen, ob das erste Argument eine zugeordnete Datei ist oder nicht.

    :::

5. ### Packen Sie Ihre Anwendung

    Packen Sie Ihre Anwendung mit dem folgenden Befehl:

    ```bash
    wails3 package
    ```

    Die gepackte Anwendung wird im Verzeichnis `bin` erstellt. Sie können die Anwendung
    dann installieren und testen.

    ## Zusätzliche Hinweise

    - Icons sollten im PNG-Format im Build-Ordner bereitgestellt werden
    - Das Testen von Dateiassoziationen erfordert die Installation der gepackten Anwendung

</Steps>
