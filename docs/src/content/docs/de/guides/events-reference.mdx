# Leitfaden für Events

Events sind das Herzstück der Kommunikation in Wails-Anwendungen. Sie ermöglichen es, verschiedene Teile Ihrer Anwendung miteinander kommunizieren zu lassen, ohne stark gekoppelt zu sein. Dieser Leitfaden führt Sie durch alles, was Sie wissen müssen, um Events effektiv in Ihrer Wails-Anwendung zu nutzen.

## Verständnis von Wails-Events

Stellen Sie sich Events als Nachrichten vor, die in Ihrer gesamten Anwendung verbreitet werden. Jeder Teil Ihrer Anwendung kann auf diese Nachrichten hören und entsprechend reagieren. Dies ist besonders nützlich für:

- **Reagieren auf Fensteränderungen**: Wissen, wann Ihr Fenster minimiert, maximiert oder verschoben wird
- **Verarbeiten von Systemereignissen**: Auf Themenänderungen oder Energieereignisse reagieren
- **Benutzerdefinierte Anwendungslogik**: Erstellen Sie Ihre eigenen Events für Funktionen wie Datenaktualisierungen oder Benutzeraktionen
- **Kommunikation zwischen Komponenten**: Lassen Sie verschiedene Teile Ihrer App kommunizieren, ohne direkte Abhängigkeiten

## Namenskonvention für Events

Alle Wails-Events folgen einem Namespace-Muster, um ihren Ursprung klar anzugeben:

- `common:` - Plattformübergreifende Events, die auf Windows, macOS und Linux funktionieren
- `windows:` - Windows-spezifische Events
- `mac:` - macOS-spezifische Events  
- `linux:` - Linux-spezifische Events

Zum Beispiel:
- `common:WindowFocus` - Fenster hat Fokus erhalten (funktioniert überall)
- `windows:APMSuspend` - System wird suspendiert (nur Windows)
- `mac:ApplicationDidBecomeActive` - App wurde aktiv (nur macOS)

## Erste Schritte mit Events

### Lauschen auf Events (Frontend)

Der häufigste Anwendungsfall ist das Lauschen auf Events in Ihrem Frontend-Code:

```javascript
import { Events } from '@wailsio/runtime';

// Lauschen, wenn das Fenster den Fokus erhält
Events.On('common:WindowFocus', () => {
    console.log('Fenster hat nun Fokus!');
    // Vielleicht einige Daten aktualisieren oder Animationen fortsetzen
});

// Lauschen auf Themenänderungen
Events.On('common:ThemeChanged', (event) => {
    console.log('Thema geändert:', event.data);
    // Passen Sie das Thema Ihrer App entsprechend an
});

// Lauschen auf benutzerdefinierte Events von Ihrem Go-Backend
Events.On('my-app:data-updated', (event) => {
    console.log('Daten aktualisiert:', event.data);
    // Aktualisieren Sie Ihre Benutzeroberfläche mit den neuen Daten
});
```

### Senden von Events (Backend)

Von Ihrem Go-Code aus können Sie Events senden, auf die Ihr Frontend hören kann:

```go
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "time"
)

func (s *Service) UpdateData() {
    // Führen Sie einige Datenverarbeitungen durch...

    // Benachrichtigen Sie das Frontend
    app := application.Get()
    app.Event.Emit("my-app:data-updated",
        map[string]interface{}{
            "timestamp": time.Now(),
            "count": 42,
        },
	)
}
```

### Senden von Events (Frontend)

Obwohl weniger häufig verwendet, können Sie auch Events von Ihrem Frontend senden, auf die Ihr Go-Code hören kann:

```javascript
import { Events } from '@wailsio/runtime';

// Event ohne Daten
Events.Emit('myapp:close-window')

// Event mit Daten
Events.Emit('myapp:disconnect-requested', 'id-123')
```

Wenn Sie TypeScript in Ihrem Frontend verwenden und [typisierte Events](#typed-events-with-typecheck) in Ihrem Go-Code registrieren, erhalten Sie Autovervollständigung/Prüfung der Event-Namen und Datentypprüfung.

### Entfernen von Event-Listenern

Räumen Sie Ihre Event-Listener immer auf, wenn sie nicht mehr benötigt werden:

```javascript
import { Events } from '@wailsio/runtime';

// Speichern Sie den Handler-Verweis
const focusHandler = () => {
    console.log('Fenster fokussiert');
};

// Fügen Sie den Listener hinzu
Events.On('common:WindowFocus', focusHandler);

// Entfernen Sie ihn später, wenn er nicht mehr benötigt wird
Events.Off('common:WindowFocus', focusHandler);

// Oder entfernen Sie alle Listener für ein Event
Events.Off('common:WindowFocus');
```

## Häufige Anwendungsfälle

### 1. Pause/Wiederaufnahme bei Fensterfokus

Viele Anwendungen müssen bestimmte Aktivitäten pausieren, wenn das Fenster den Fokus verliert:

```javascript
import { Events } from '@wailsio/runtime';

let animationRunning = true;

Events.On('common:WindowLostFocus', () => {
    animationRunning = false;
    pauseBackgroundTasks();
});

Events.On('common:WindowFocus', () => {
    animationRunning = true;
    resumeBackgroundTasks();
});
```

### 2. Reagieren auf Themenänderungen

Halten Sie Ihre App mit dem Systemthema synchron:

```javascript
import { Events } from '@wailsio/runtime';

Events.On('common:ThemeChanged', (event) => {
    const isDarkMode = event.data.isDark;

    if (isDarkMode) {
        document.body.classList.add('dark-theme');
        document.body.classList.remove('light-theme');
    } else {
        document.body.classList.add('light-theme');
        document.body.classList.remove('dark-theme');
    }
});
```

### 3. Verarbeiten von Datei-Drops

Lassen Sie Ihre App gezogene Dateien akzeptieren:

```javascript
import { Events } from '@wailsio/runtime';

Events.On('common:WindowFilesDropped', (event) => {
    const files = event.data.files;

    files.forEach(file => {
        console.log('Datei abgelegt:', file);
        // Verarbeiten Sie die abgelegten Dateien
        handleFileUpload(file);
    });
});
```

### 4. Fenster-Lebenszyklus-Management

Reagieren Sie auf Fensterzustandsänderungen:

```javascript
import { Events } from '@wailsio/runtime';

Events.On('common:WindowClosing', () => {
    // Speichern Sie Benutzerdaten vor dem Schließen
    saveApplicationState();

    // Sie könnten das Schließen auch verhindern, indem Sie false zurückgeben
    // von einem registrierten Fenster-Schließen-Handler
});

Events.On('common:WindowMaximise', () => {
    // Passen Sie die Benutzeroberfläche für den maximierten Anzeigemodus an
    adjustLayoutForMaximized();
});

Events.On('common:WindowRestore', () => {
    // Kehren Sie den normalen Zustand der Benutzeroberfläche zurück
    adjustLayoutForNormal();
});
```

### 5. Plattform-spezifische Funktionen

Verarbeiten Sie plattformspezifische Events, wenn nötig:

```javascript
import { Events } from '@wailsio/runtime';

// Windows-spezifisches Energiemanagement
Events.On('windows:APMSuspend', () => {
    console.log('System geht in den Ruhezustand');
    saveState();
});

Events.On('windows:APMResumeSuspend', () => {
    console.log('System ist aufgewacht');
    refreshData();
});

// macOS-spezifischer App-Lebenszyklus
Events.On('mac:ApplicationWillTerminate', () => {
    console.log('App wird beendet');
    performCleanup();
});
```

## Erstellen benutzerdefinierter Events

Sie können Ihre eigenen Events für anwendungsspezifische Bedürfnisse erstellen.

### Backend (Go)

```go
// Senden Sie ein benutzerdefiniertes Event, wenn sich Daten ändern

func (s *Service) ProcessUserData(userData UserData) error {
    // Verarbeiten Sie die Daten...

    app := application.Get()
    // Benachrichtigen Sie alle Listener
    app.Event.Emit("user:data-processed",
        map[string]interface{}{
            "userId": userData.ID,
            "status": "completed",
            "timestamp": time.Now(),
        },
    )
    return nil
}

// Senden Sie periodische Updates
func (s *Service) StartMonitoring() {
    app := application.Get()
    ticker := time.NewTicker(5 * time.Second)
    go func() {
        for range ticker.C {
            stats := s.collectStats()
            app.Event.Emit("monitor:stats-updated", stats)
        }
    }()
}
```

### Frontend (JavaScript)

```javascript
import { Events } from '@wailsio/runtime';

// Lauschen Sie auf Ihre benutzerdefinierten Events
Events.On('user:data-processed', (event) => {
    const { userId, status, timestamp } = event.data;

    showNotification(`Benutzer ${userId} Verarbeitung ${status}`);
    updateUIWithNewData();
});

Events.On('monitor:stats-updated', (event) => {
    updateDashboard(event.data);
});
```

## Typisierte Events mit Typsicherheit

Wails v3 unterstützt typisierte Events mit voller TypeScript-Typsicherheit durch Event-Registrierung und automatische Bindungsgenerierung.

### Registrieren benutzerdefinierter Events

Rufen Sie `application.RegisterEvent` zur Initialisierungszeit auf, um benutzerdefinierte Event-Namen mit ihren Datentypen zu registrieren:

```go
package main

import "github.com/wailsapp/wails/v3/pkg/application"

type UserLoginData struct {
    UserID   string
    Username string
    LoginTime string
}

type MonitorStats struct {
    CPUUsage    float64
    MemoryUsage float64
}-------|-------|------|
| `common:ApplicationStarted` | Anwendung wurde vollständig gestartet | App initialisieren, gespeicherten Zustand laden |
| `common:WindowRuntimeReady` | Wails-Laufzeit ist bereit | Wails-API-Aufrufe starten |
| `common:ThemeChanged` | Systemthema hat sich geändert | App-Design aktualisieren |
| `common:WindowFocus` | Fenster hat Fokus erhalten | Aktivitäten fortsetzen, Daten aktualisieren |
| `common:WindowLostFocus` | Fenster hat Fokus verloren | Aktivitäten pausieren, Zustand speichern |
| `common:WindowMinimise` | Fenster wurde minimiert | Rendering pausieren, Ressourcennutzung reduzieren |
| `common:WindowMaximise` | Fenster wurde maximiert | Layout für Vollbild anpassen |
| `common:WindowRestore` | Fenster wurde aus Min/Max wiederhergestellt | Zum normalen Layout zurückkehren |
| `common:WindowClosing` | Fenster wird gleich geschlossen | Daten speichern, Ressourcen bereinigen |
| `common:WindowFilesDropped` | Dateien wurden auf das Fenster abgelegt | Dateiimporte verarbeiten |
| `common:WindowDidResize` | Fenster wurde in der Größe geändert | Layout anpassen, Diagramme neu rendern |
| `common:WindowDidMove` | Fenster wurde verschoben | Positionsabhängige Funktionen aktualisieren |

### Plattform-spezifische Events

#### Windows-Events

Wichtige Events für Windows-Anwendungen:

| Event | Beschreibung | Anwendungsfall |
|-------|-------|------|
| `windows:SystemThemeChanged` | Windows-Design hat sich geändert | App-Farben aktualisieren |
| `windows:APMSuspend` | System wird in den Ruhezustand versetzt | Zustand speichern, Operationen pausieren |
| `windows:APMResumeSuspend` | System wurde aus dem Ruhezustand wiederhergestellt | Zustand wiederherstellen, Daten aktualisieren |
| `windows:APMPowerStatusChange` | Stromversorgungsstatus hat sich geändert | Leistungseinstellungen anpassen |

#### macOS-Events

Wichtige macOS-Anwendungs-Events:

| Event | Beschreibung | Anwendungsfall |
|-------|-------|------|
| `mac:ApplicationDidBecomeActive` | App wurde aktiv | Operationen fortsetzen |
| `mac:ApplicationDidResignActive` | App wurde inaktiv | Operationen pausieren |
| `mac:ApplicationWillTerminate` | App wird beendet | Letzte Bereinigung |
| `mac:WindowDidEnterFullScreen` | Vollbildmodus wurde betreten | UI für Vollbild anpassen |
| `mac:WindowDidExitFullScreen` | Vollbildmodus wurde verlassen | Normale UI wiederherstellen |

#### Linux-Events

Wichtige Linux-Fenster-Events:

| Event | Beschreibung | Anwendungsfall |
|-------|-------|------|
| `linux:SystemThemeChanged` | Desktop-Design hat sich geändert | App-Design aktualisieren |
| `linux:WindowFocusIn` | Fenster hat Fokus erhalten | Aktivitäten fortsetzen |
| `linux:WindowFocusOut` | Fenster hat Fokus verloren | Aktivitäten pausieren |
| `linux:WindowLoadStarted` | WebView hat mit dem Laden begonnen | Ladeindikator anzeigen |
| `linux:WindowLoadRedirected` | WebView wurde umgeleitet | Navigationsumleitungen verfolgen |
| `linux:WindowLoadCommitted` | WebView hat das Laden übernommen | Inhalte werden empfangen |
| `linux:WindowLoadFinished` | WebView hat das Laden abgeschlossen | Ladeindikator ausblenden, JS/CSS injizieren |

## Best Practices

### 1. Event-Namensräume verwenden

Verwenden Sie bei der Erstellung benutzerdefinierter Events Namensräume, um Konflikte zu vermeiden:

```javascript
import { Events } from '@wailsio/runtime';

// Gut - benannte Events
Events.Emit('myapp:user:login');
Events.Emit('myapp:data:updated');
Events.Emit('myapp:network:connected');

// Vermeiden - generische Namen, die Konflikte verursachen könnten
Events.Emit('login');
Events.Emit('update');
```

### 2. Listener bereinigen

Entfernen Sie immer Event-Listener, wenn Komponenten unmounten:

```javascript
import { Events } from '@wailsio/runtime';

// React-Beispiel
useEffect(() => {
    const handler = (event) => {
        // Event verarbeiten
    };

    Events.On('common:WindowResize', handler);

    // Bereinigung
    return () => {
        Events.Off('common:WindowResize', handler);
    };
}, []);
```

### 3. Plattformunterschiede behandeln

Prüfen Sie die Plattformverfügbarkeit bei der Verwendung plattformspezifischer Events:

```javascript
import { Events } from '@wailsio/runtime';

// Plattformspezifische Events können bedingungslos registriert werden;
// sie feuern einfach nie auf nicht unterstützten Plattformen.
Events.On('windows:APMSuspend', handleSuspend);
Events.On('mac:ApplicationWillTerminate', handleTerminate);
```

### 4. Events nicht übermäßig verwenden

Obwohl Events leistungsstark sind, sollten Sie sie nicht für alles verwenden:

- ✅ Verwenden Sie Events für: Systembenachrichtigungen, Lebenszyklusänderungen, Broadcast-Aktualisierungen
- ❌ Vermeiden Sie Events für: Direkte Funktionsrückgaben, Einzelkomponentenaktualisierungen, synchrone Operationen

## Debugging von Events

Um Event-Probleme zu debuggen:

```javascript
import { Events } from '@wailsio/runtime';

// Alle Events protokollieren (nur für die Entwicklung)
if (isDevelopment) {
    const originalOn = Events.On;
    Events.On = function(eventName, handler) {
        console.log(`[Event Registriert] ${eventName}`);
        return originalOn.call(this, eventName, function(event) {
            console.log(`[Event Gefeuert] ${eventName}`, event);
            return handler(event);
        });
    };
}
```

## Zusammenfassung

Events in Wails bieten eine leistungsstarke, entkoppelte Möglichkeit zur Kommunikation in Ihrer Anwendung. Durch die Befolgung der Muster und Praktiken in diesem Leitfaden können Sie reaktionsschnelle, plattformbewusste Anwendungen erstellen, die nahtlos auf Systemänderungen und Benutzerinteraktionen reagieren.

Denken Sie daran: Beginnen Sie mit allgemeinen Events für die plattformübergreifende Kompatibilität, fügen Sie bei Bedarf plattformspezifische Events hinzu, und räumen Sie Ihre Event-Listener stets auf, um Speicherlecks zu vermeiden.
