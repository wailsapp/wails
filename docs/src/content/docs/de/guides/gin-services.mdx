# Gin für Services verwenden

Das Gin-Webframework ist eine beliebte Wahl zum Erstellen von HTTP-Services in Go. Mit Wails v3 können Sie Gin-basierte Services einfach in Ihre Anwendung integrieren, was eine leistungsstarke Möglichkeit bietet, HTTP-Anfragen zu verarbeiten, RESTful-APIs zu implementieren und Webinhalte bereitzustellen.

Diese Anleitung führt Sie durch das Erstellen eines Gin-basierten Services, der an einer bestimmten Route in Ihrer Wails-Anwendung eingebunden werden kann. Wir erstellen ein vollständiges Beispiel, das demonstriert, wie Sie:

1. Einen Gin-basierten Service erstellen
2. Die Wails Service-Schnittstelle implementieren
3. Routen und Middleware einrichten
4. Mit dem Wails-Ereignissystem interagieren
5. Vom Frontend aus mit dem Service kommunizieren

## Voraussetzungen

Stellen Sie vor dem Start sicher, dass Sie Folgendes haben:

- Wails v3 installiert
- Grundlegende Kenntnisse in Go und dem Gin-Framework
- Vertrautheit mit HTTP-Konzepten und RESTful-APIs

Sie müssen das Gin-Framework zu Ihrem Projekt hinzufügen:

```bash
go get github.com/gin-gonic/gin
```

## Erstellen eines Gin-basierten Services

Beginnen wir mit der Erstellung eines Gin-Services, der die Wails Service-Schnittstelle implementiert. Unser Service verwaltet eine Sammlung von Benutzern und stellt API-Endpunkte zum Abrufen und Erstellen von Benutzerdatensätzen bereit.

### 1. Definieren Sie Ihre Datenmodelle

Definieren Sie zunächst die Datenstrukturen, mit denen Ihr Service arbeiten soll:

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User repräsentiert einen Benutzer im System
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData repräsentiert Daten, die in Ereignissen gesendet werden
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. Erstellen Sie Ihre Service-Struktur

Definieren Sie anschließend die Service-Struktur, die Ihren Gin-Router und jeden Zustand, den Ihr Service aufrechterhalten muss, enthält:

```go
// GinService implementiert einen Wails-Service, der Gin für die HTTP-Verarbeitung verwendet
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService erstellt eine neue GinService-Instanz
func NewGinService() *GinService {
	// Erstellen Sie einen neuen Gin-Router
	ginEngine := gin.New()

	// Fügen Sie Middlewares hinzu
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// Definieren Sie Routen
	service.setupRoutes()

	return service
}
```

### 3. Implementieren Sie die Service-Schnittstelle

Implementieren Sie die erforderlichen Methoden für die Wails Service-Schnittstelle:

```go
// ServiceName gibt den Namen des Services zurück
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup wird aufgerufen, wenn der Service startet
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// Speichern Sie die Anwendungsinstanz für die spätere Verwendung
	s.app = application.Get()

	// Registrieren Sie einen Ereignishandler, der vom Frontend ausgelöst werden kann
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// Protokollieren Sie die Ereignisdaten
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// Sende ein Ereignis zurück an das Frontend
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown wird aufgerufen, wenn der Service heruntergefahren wird
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// Bereinigen Sie Ressourcen, falls erforderlich
	return nil
}
```

### 3. Implementieren Sie die http.Handler-Schnittstelle

Um Ihren Service an einer bestimmten Route einbinden zu können, implementieren Sie die `http.Handler`-Schnittstelle. Diese einzelne Methode `ServeHTTP` ist das Tor für alle HTTP-Anfragen an Ihren Service. Sie delegiert die Anfragenverarbeitung an den Gin-Router, sodass Sie alle leistungsstarken Funktionen von Gin für Routing und Middleware nutzen können.

```go
// ServeHTTP implementiert die http.Handler-Schnittstelle
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// Alle Anfragen gehen an den Gin-Router
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. Richten Sie Ihre Routen ein

Definieren Sie Ihre API-Routen in einer separaten Methode für eine bessere Organisation. Dieser Ansatz hält Ihren Code sauber und erleichtert das Verständnis der Struktur Ihrer API. Der Gin-Router bietet eine fluide API zur Definition von Routen, einschließlich Unterstützung für Routengruppen, die dabei helfen, verwandte Endpunkte zu organisieren.

```go
// setupRoutes konfiguriert die API-Routen
func (s *GinService) setupRoutes() {
	// Endpunkt für Basisinformationen
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// Benutzer-Gruppe
	users := s.ginEngine.Group("/users")
	{
		// Alle Benutzer abrufen
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// Benutzer nach ID abrufen
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// Einen neuen Benutzer erstellen
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// Setzen Sie die ID und die Erstellungszeit
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// Zum Benutzer-Slice hinzufügen
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// Sende ein Ereignis, um über den neuen Benutzer zu informieren
			s.app.Event.Emit("user-created", newUser)
		})

		// Einen Benutzer löschen
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// Entferne den Benutzer aus dem Slice
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. Erstellen Sie benutzerdefinierte Middleware

Sie können benutzerdefinierte Gin-Middleware erstellen, um Ihren Service zu erweitern. Middleware-Funktionen in Gin werden in der Reihenfolge ausgeführt, in der sie zum Router hinzugefügt werden, und können Aufgaben wie Protokollierung, Authentifizierung und Fehlerbehandlung durchführen. Dieses Beispiel zeigt eine einfache Protokollierungs-Middleware, die Details der Anfrage aufzeichnet.

```go
// LoggingMiddleware ist eine Gin-Middleware, die Details der Anfrage protokolliert
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// Timer starten
		start := time.Now()

		// Anfrage verarbeiten
		c.Next()

		// Latenz berechnen
		latency := time.Since(start)

		// Details der Anfrage protokollieren
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## Registrieren Ihres ServicesUm Ihren Gin-basierten Dienst in einer Wails-Anwendung zu verwenden, müssen Sie ihn bei der Anwendung registrieren und den Pfad angeben, unter dem er eingebunden werden soll. Dies geschieht bei der Erstellung der Wails-Anwendungsinstanz. Der von Ihnen angegebene Pfad wird zum Basispfad für alle in Ihrem Gin-Router definierten Endpunkte.

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

In diesem Beispiel wird der Gin-Dienst unter dem Pfad `/api` eingebunden. Das bedeutet, dass wenn Ihr Gin-Router einen Endpunkt als `/info` definiert, dieser in Ihrer Anwendung unter `/api/info` erreichbar ist. Dieser Ansatz ermöglicht es Ihnen, Ihre API-Endpunkte logisch zu organisieren und Konflikte mit anderen Teilen Ihrer Anwendung zu vermeiden.

## Integration in das Wails-Ereignissystem

Eine der leistungsstarken Funktionen der Verwendung von Gin mit Wails-Diensten ist die nahtlose Integration in das Wails-Ereignissystem. Dies ermöglicht eine Echtzeitkommunikation zwischen Ihrem Backend-Dienst und dem Frontend.

In der `ServiceStartup`-Methode Ihres Dienstes können Sie Ereignis-Handler registrieren, um Ereignisse vom Frontend zu verarbeiten:

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

Sie können auch Ereignisse vom Frontend aus Ihren Gin-Routen oder anderen Teilen Ihres Dienstes aus senden:

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## Frontend-Integration

Um mit Ihrem Gin-Dienst vom Frontend aus zu interagieren, müssen Sie die Wails-Laufzeitumgebung importieren, HTTP-Anfragen an Ihre API-Endpunkte senden und das Wails-Ereignissystem für die Echtzeitkommunikation verwenden.

Für den Produktionsbetrieb wird empfohlen, das Paket `@wailsio/runtime` anstelle des direkten Imports von `/wails/runtime.js` zu verwenden. Dies gewährleistet Typsicherheit, bessere IDE-Unterstützung, Versionsverwaltung über npm und Kompatibilität mit modernen JavaScript-Tools.

Installieren Sie das Paket:
```bash
npm install @wailsio/runtime
```

Verwenden Sie es dann in Ihrem Code:
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## Abschließende Gedanken

Die Integration des Gin-Webframeworks mit Wails v3 Services bietet einen leistungsstarken und flexiblen Ansatz zum Aufbau modularer und wartbarer Webanwendungen. Durch die Nutzung von Gins Routing- und Middleware-Fähigkeiten in Kombination mit dem Wails-Ereignissystem können Sie reichhaltige, interaktive Anwendungen mit einer sauberen Trennung der Verantwortlichkeiten erstellen.

Den vollständigen Beispielcode für diesen Leitfaden finden Sie im Wails-Repository unter `v3/examples/gin-service`.
