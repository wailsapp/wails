## Überblick

Sicherheit ist entscheidend für Desktop-Anwendungen. Befolgen Sie diese Richtlinien, um Ihre Anwendung sicher zu halten.

## Eingabevalidierung

### Immer validieren

```go
func (s *UserService) CreateUser(email, password string) (*User, error) {
    // E-Mail validieren
    if !isValidEmail(email) {
        return nil, errors.New("invalid email")

    // Passwortstärke validieren
    if len(password) < 8 {
        return nil, errors.New("password too short")
    }
    
    // Eingabe bereinigen
    email = strings.TrimSpace(email)
    email = html.EscapeString(email)
    
    // Weiter...
}
```

### HTML bereinigen

```go
import "html"

func (s *Service) SaveComment(text string) error {
    // HTML escapen
    text = html.EscapeString(text)
    
    // Länge validieren
    if len(text) > 1000 {
        return errors.New("comment too long")
    }
    
    return s.db.Save(text)
}
```

## Authentifizierung

### Sichere Passwortspeicherung

```go
import "golang.org/x/crypto/bcrypt"

func hashPassword(password string) (string, error) {
    hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    return string(hash), err
}

func verifyPassword(hash, password string) bool {
    err := bcrypt.CompareHashAndPassword([]byte(hash), []byte(password))
    return err == nil
}
```

### Sitzungsverwaltung

```go
type Session struct {

func (a *AuthService) CreateSession(userID int) (*Session, error) {
```

## Datenschutz

### Sensible Daten verschlüsseln

```go
import "crypto/aes"
import "crypto/cipher"

func encrypt(data []byte, key []byte) ([]byte, error) {
    block, err := aes.NewCipher(key)
    if err != nil {
        return nil, err
    }
    
    gcm, err := cipher.NewGCM(block)
    if err != nil {
        return nil, err
    }
    
    nonce := make([]byte, gcm.NonceSize())
    return gcm.Seal(nonce, nonce, data, nil), nil
}
```

## Netzwerksicherheit

### HTTPS verwenden

```go
func makeAPICall(url string) (*Response, error) {
    // Immer HTTPS verwenden
    if !strings.HasPrefix(url, "https://") {
        return nil, errors.New("only HTTPS allowed")
    }
```

### Zertifikate überprüfen

```go
import "crypto/tls"

func secureClient() *http.Client {
    return &http.Client{
        Transport: &http.Transport{
            TLSClientConfig: &tls.Config{
                MinVersion: tls.VersionTLS12,
            },
        },
    }
}
```

### Pfade validieren

```go
func readFile(path string) ([]byte, error) {
    // Pfadtraversal verhindern
    if strings.Contains(path, "..") {
        return nil, errors.New("invalid path")
    }
    
    // Prüfen, ob die Datei im erlaubten Verzeichnis existiert
    absPath, err := filepath.Abs(path)

    }
    
    if !strings.HasPrefix(absPath, allowedDir) {
        return nil, errors.New("access denied")
    }
    
    return os.ReadFile(absPath)
}
```

## Ratenbegrenzung

```go
type RateLimiter struct {
    requests map[string][]time.Time
    mu       sync.Mutex
    limit    int
    window   time.Duration
}

func (r *RateLimiter) Allow(key string) bool {
    r.mu.Lock()
    defer r.mu.Unlock()
    
    now := time.Now()
    
    // Alte Anfragen bereinigen
    var recent []time.Time
    for _, t := range r.requests[key] {
        if now.Sub(t) < r.window {
            recent = append(recent, t)
        }
    }
    
    if len(recent) >= r.limit {
        return false
    }
    
    r.requests[key] = append(recent, now)
    return true
}
```

## Best-Practices

### ✅ Tun

- Alle Eingaben validieren
- HTTPS für Netzwerkaufrufe verwenden
- Sensible Daten verschlüsseln
- Sichere Passwort-Hashing-Verfahren verwenden
- Ratenbegrenzung implementieren
- Abhängigkeiten aktuell halten
- Sicherheitsereignisse protokollieren
- OS-Schlüsselbünde verwenden

### ❌ Nicht tun

- Benutzereingaben nicht vertrauen
- Passwörter nicht im Klartext speichern
- Keine Secrets hartkodieren
- Zertifikatsüberprüfung nicht überspringen
- Keine sensiblen Daten in Logs offenlegen
- Keine schwache Verschlüsselung verwenden
- Sicherheitsupdates nicht ignorieren

## Sicherheits-Checkliste

- [ ] Alle Benutzereingaben validiert
- [ ] Passwörter mit bcrypt gehasht
- [ ] Sensible Daten verschlüsselt
- [ ] HTTPS für alle Netzwerkaufrufe verwendet
- [ ] Ratenbegrenzung implementiert
- [ ] Dateipfade validiert
- [ ] Abhängigkeiten aktuell
- [ ] Sicherheitsprotokollierung aktiviert
- [ ] Fehlermeldungen geben keine Informationen preis
- [ ] Code auf Sicherheitslücken überprüft
