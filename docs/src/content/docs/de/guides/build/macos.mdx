## Anwendungs-Bundle

Paketieren Sie Ihre App als standardmäßiges macOS `.app`-Bundle:

```bash
wails3 package GOOS=darwin
```

Dies erstellt `bin/<AppName>.app` mit folgenden Inhalten:
- Die kompilierte Binärdatei in `Contents/MacOS/`
- Das App-Symbol in `Contents/Resources/` (aus `icons.icns` oder, falls vorhanden, aus einem Asset-Katalog `Assets.car`)
- `Info.plist` mit den App-Metadaten

### Universal-Binary

Erstellen Sie für Apple Silicon und Intel-Macs:

```bash
wails3 task darwin:package:universal
```

Dies erstellt eine einzelne `.app`, die nativ auf beiden Architekturen läuft. Universal-Binaries können auf jeder Plattform erstellt werden — unter Linux und Windows wird `wails3 tool lipo` automatisch verwendet.

## Anpassen des Bundles

Bearbeiten Sie `build/darwin/Info.plist`, um anzupassen:

- Bundle-Identifier (`CFBundleIdentifier`)
- App-Name und -Version
- Mindestens macOS-Version
- Dateizuordnungen
- URL-Schemata

Das App-Symbol wird aus den Assets im Verzeichnis `build/` generiert. Verwenden Sie die Aufgabe `generate:icons`:

```bash
wails3 task generate:icons
```

Dies verwendet `build/appicon.png`, um `darwin/icons.icns` und `windows/icon.ico` zu erzeugen. Unter macOS können Sie auch `build/appicon.icon` (Icon Composer-Format) bereitstellen: Die Aufgabe übergibt `-iconcomposerinput appicon.icon -macassetdir darwin`, wodurch `Assets.car` und `darwin/icons.icns` aus der `.icon`-Datei erzeugt werden (auf Nicht-macOS-Plattformen übersprungen). Wenn `Assets.car` vorhanden ist, führen Sie die Aufgabe `update:build-assets` aus, damit `Info.plist` und `CFBundleIconName` entsprechend aktualisiert werden:

```bash
wails3 task update:build-assets
```

Um den Icon-Befehl manuell aus dem Verzeichnis `build/` auszuführen:

```bash
cd build
wails3 generate icons -input appicon.png -macfilename darwin/icons.icns -windowsfilename windows/icon.ico -iconcomposerinput appicon.icon -macassetdir darwin
```

## Code-Signierung

Signieren Sie Ihre App für die Verteilung:

```bash
# Mit dem Wrapper (automatische Plattformerkennung)
wails3 sign GOOS=darwin

# Oder direkt mit der Aufgabe
wails3 task darwin:sign
```

Konfigurieren Sie die Signierung in `build/darwin/Taskfile.yml`:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  ENTITLEMENTS: "build/darwin/entitlements.plist"
```

### Notarisierung

Für Apps, die außerhalb des Mac App Stores verteilt werden, erfordert Apple eine Notarisierung:

```bash
wails3 task darwin:sign:notarize
```

Speichern Sie zunächst Ihre Anmeldedaten:

```bash
wails3 signing credentials \
  --apple-id "you@email.com" \
  --team-id "TEAMID" \
  --password "app-specific-password" \
  --profile "my-notarize-profile"
```

Konfigurieren Sie in `build/darwin/Taskfile.yml`:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  KEYCHAIN_PROFILE: "my-notarize-profile"
```

Weitere Details finden Sie unter [Signing Applications](/de/guides/build/signing).

## DMG-Installer

Erstellen Sie ein DMG-Disk-Image für die Verteilung:

```bash
wails3 task darwin:create:dmg
```

<Aside type="tip">
Passen Sie den DMG-Hintergrund und das Layout an, indem Sie die Assets in `build/darwin/dmg/` bearbeiten.
</Aside>

## Fehlerbehebung

### „App ist beschädigt und kann nicht geöffnet werden“

Die App ist nicht signiert. Entweder signieren Sie sie mit einem Developer-ID-Zertifikat, oder die Nutzer können Gatekeeper umgehen:

```bash
xattr -cr /path/to/YourApp.app
```

### Notarisierung schlägt fehl

Häufige Probleme:
- **Ungültige Anmeldedaten**: Führen Sie `wails3 signing credentials` erneut aus
- **Hardened runtime erforderlich**: Stellen Sie sicher, dass die Berechtigungen `com.apple.security.cs.allow-unsigned-executable-memory` enthalten, falls erforderlich
- **Fehlender Zeitstempel**: Der Signiervorgang sollte automatisch einen Zeitstempel einschließen

### Cross-kompilierte App läuft nicht

Cross-kompilierte macOS-Binaries sind nicht signiert. Übertragen Sie sie auf einen Mac und signieren Sie sie vor dem Testen:

```bash
codesign --force --deep --sign - YourApp.app
```
