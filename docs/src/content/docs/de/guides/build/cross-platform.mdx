## Schnellstart

Wails v3 unterstützt das Bauen für Windows, macOS und Linux von jedem Host-Betriebssystem aus. Das Build-System erkennt Ihre Umgebung automatisch und wählt die richtige Kompilierungsmethode.

**Möchten Sie für macOS und Linux cross-compilieren?** Führen Sie diesen Befehl einmal aus, um die Docker-Images einzurichten (~800 MB Download):

```bash
wails3 task setup:docker
```

Dann bauen Sie für jede beliebige Plattform:

```bash
# Build for current platform (production by default)
wails3 build

# Build for specific platforms
wails3 build GOOS=windows
wails3 build GOOS=darwin
wails3 build GOOS=linux

# Build for ARM64 architecture
wails3 build GOOS=windows GOARCH=arm64
wails3 build GOOS=darwin GOARCH=arm64
wails3 build GOOS=linux GOARCH=arm64

# Environment variable style also works
GOOS=darwin GOARCH=arm64 wails3 build
```

### Windows

Windows ist das einfachste Ziel für die Cross-Kompilierung, da es standardmäßig kein CGO erfordert.

```bash
wails3 build GOOS=windows
```

Dies funktioniert von jedem Host-Betriebssystem aus ohne zusätzliche Einrichtung. Go's integrierte Cross-Kompilierung erledigt alles.

**Wenn Ihre App CGO erfordert** (z. B. wenn Sie eine C-Bibliothek oder ein CGO-abhängiges Paket verwenden), benötigen Sie Docker beim Bauen von macOS oder Linux aus:

```bash
# One-time setup
wails3 task setup:docker

# Build with CGO enabled
wails3 task windows:build CGO_ENABLED=1
```

Die Taskfile erkennt `CGO_ENABLED=1` auf Nicht-Windows-Hosts und verwendet automatisch das Docker-Image.

### macOS

macOS-Builds erfordern CGO für die WebView-Integration, was bedeutet, dass die Cross-Kompilierung spezielle Tools benötigt.

```bash
# Build for Apple Silicon (arm64) - default
wails3 build GOOS=darwin

# Build for Intel (amd64)
wails3 build GOOS=darwin GOARCH=amd64

# Build universal binary (both architectures)
wails3 task darwin:build:universal
```

**Von Linux oder Windows aus** müssen Sie zuerst Docker einrichten:

```bash
wails3 task setup:docker
```

Sobald die Images erstellt sind, erkennt das Build-System, dass Sie nicht auf macOS sind, und verwendet automatisch Docker. Sie müssen Ihre Build-Befehle nicht ändern.

Beachten Sie, dass cross-kompilierte macOS-Binärdateien nicht signiert sind. Sie müssen sie auf macOS oder in CI vor der Verteilung signieren.

### Linux

Linux-Builds erfordern CGO für die WebView-Integration.

```bash
wails3 build GOOS=linux

# Build for specific architecture
wails3 build GOOS=linux GOARCH=amd64
wails3 build GOOS=linux GOARCH=arm64
```

**Von macOS oder Windows aus** müssen Sie zuerst Docker einrichten:

```bash
wails3 task setup:docker
```

Das Build-System erkennt, dass Sie nicht auf Linux sind, und verwendet automatisch Docker.

**Auf Linux ohne C-Compiler** prüft das Build-System auf `gcc` oder `clang`. Wenn keines gefunden wird, fällt es auf Docker zurück. Dies ist nützlich für minimale Container oder Systeme, auf denen keine Build-Tools installiert sind. Sie können entweder:

1. Einen C-Compiler installieren: `sudo apt install build-essential` (Debian/Ubuntu) oder `sudo pacman -S base-devel` (Arch)
2. Das Docker-Image erstellen und automatisch verwenden lassen

### ARM-Architektur

Alle Plattformen unterstützen die ARM64-Cross-Kompilierung mit `GOARCH`:

```bash
# Windows ARM64 (Surface Pro X, Windows on ARM)
wails3 build GOOS=windows GOARCH=arm64

# Linux ARM64 (Raspberry Pi 4/5, AWS Graviton)
wails3 build GOOS=linux GOARCH=arm64

# macOS ARM64 (Apple Silicon - this is the default on macOS)
wails3 build GOOS=darwin GOARCH=arm64

# macOS Intel (amd64)
wails3 build GOOS=darwin GOARCH=amd64
```

Das Docker-Image enthält Zig-Cross-Compiler-Ziele für sowohl amd64 als auch arm64 auf allen Plattformen, sodass ARM-Builds von jedem Host aus funktionieren:

| Build ARM64 für | Von Windows | Von macOS | Von Linux |
|-----------------|--------------|------------|------------|
| **Windows ARM64** | Native Go | Native Go | Native Go |
| **macOS ARM64** | Docker | Native | Docker |
| **Linux ARM64** | Docker | Docker | Docker* |

*Linux ARM64 von Linux x86_64 aus verwendet Docker, da CGO-Cross-Kompilierung eine andere Toolchain erfordert.

## Funktionsweise

### Cross-Kompilierungs-Matrix

| Host → Ziel | Windows | macOS | Linux |
|---------------|---------|-------|-------|
| **Windows**   | Native  | Docker | Docker |
| **macOS**     | Native Go | Native | Docker |
| **Linux**     | Native Go | Docker | Native |

- **Native** = Plattform-eigenes Toolchain, keine zusätzliche Einrichtung
- **Native Go** = Go's integrierte Cross-Kompilierung (`CGO_ENABLED=0`)
- **Docker** = Docker-Image mit Zig-Cross-Compiler

### CGO-Anforderungen

| Ziel | CGO erforderlich | Cross-Kompilierungsmethode |
|--------|--------------|--------------------------|
| Windows | Nein (Standard) | Native Go. Nur Docker, wenn `CGO_ENABLED=1` |
| macOS | Ja | Docker mit macOS SDK |
| Linux | Ja | Docker, oder nativ, wenn C-Compiler verfügbar |

### Automatische Erkennung

Die Taskfiles wählen automatisch die richtige Build-Methode basierend auf Ihrer Umgebung:

- **Windows-Ziel:** Verwendet standardmäßig native Go-Cross-Kompilierung. Wenn Sie `CGO_ENABLED=1` explizit auf einem Nicht-Windows-Host setzen, wechselt es zu Docker.
- **macOS-Ziel:** Verwendet automatisch Docker, wenn nicht auf macOS. Keine manuelle Eingabe erforderlich.
- **Linux-Ziel:** Prüft auf `gcc` oder `clang`. Verwendet native Kompilierung, wenn gefunden, sonst fällt es auf Docker zurück.

### Docker-Image

Wails verwendet ein einzelnes Docker-Image (`wails-cross`), das für alle Plattformen bauen kann. Es verwendet [Zig](https://ziglang.org/) als Cross-Compiler, der von jedem Host aus jede Plattform anvisieren kann. Das macOS SDK ist für Darwin-Ziele enthalten.

```bash
wails3 task setup:docker
```

Sie können prüfen, ob das Image bereit ist, indem Sie `wails3 doctor` ausführen.

### macOS SDK

Das Docker-Image lädt das macOS SDK von [joseluisq/macosx-sdks](https://github.com/joseluisq/macosx-sdks) während des Image-Erstellungsprozesses herunter. Dies ist erforderlich, da macOS-Header für die CGO-Kompilierung benötigt werden.

Standardmäßig verwendet das Image **macOS SDK 14.5** (Sonoma). Um eine andere Version zu verwenden, bauen Sie es neu mit:

```bash
docker build -t wails-cross \
  --build-arg MACOS_SDK_VERSION=15.0 \
  -f build/docker/Dockerfile.cross build/docker/
```

Siehe [verfügbare SDK-Versionen](https://github.com/joseluisq/macosx-sdks/releases) für Optionen.

**Wichtig:** Wails verteilt das macOS SDK nicht. Die Benutzer sind dafür verantwortlich, die Lizenzbedingungen des Apple SDK vor der Nutzung dieser Funktion zu überprüfen.

## CI/CD-Integration

Für Produktionsveröffentlichungen empfehlen wir die Verwendung von CI/CD mit nativen Runnern für jede Plattform. Dies vermeidet die Cross-Kompilierung vollständig und stellt sicher, dass Sie korrekt signierte Binärdateien erhalten.

```yaml
name: Build

on:
  push:
    branches: [main]

jobs:
  build:
    strategy:
      matrix:
        include:
          - os: ubuntu-latest
            goos: linux
          - os: macos-latest
            goos: darwin
          - os: windows-latest
            goos: windows

    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-go@v5
        with:
          go-version: '1.24'

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Wails CLI
        run: go install github.com/wailsapp/wails/v3/cmd/wails3@latest

      - name: Install Task
        uses: arduino/setup-task@v2

      - name: Build
        run: wails3 build

      - uses: actions/upload-artifact@v4
        with:
          name: app-${{ matrix.goos }}
          path: bin/
```

## Fehlerbehebung

### Docker-Image nicht gefunden

```
Docker image 'wails-cross' not found.
```

Führen Sie `wails3 task setup:docker` aus, um das Docker-Image zu erstellen. Sie müssen dies nur einmal tun.

### Docker-Daemon läuft nicht

```
Docker is required for cross-compilation. Please install Docker.
```

Starten Sie Docker Desktop oder den Docker-Daemon. Unter Linux müssen Sie möglicherweise `sudo systemctl start docker` ausführen.### Kein C-Compiler unter Linux

Wenn Sie beim Bauen unter Linux CGO-bezogene Fehler sehen, haben Sie zwei Optionen:

1. **Installieren Sie einen C-Compiler:**
   - Debian/Ubuntu: `sudo apt install build-essential`
   - Arch Linux: `sudo pacman -S base-devel`
   - Fedora: `sudo dnf install gcc`

2. **Verwenden Sie stattdessen Docker:** Führen Sie `wails3 task setup:docker` aus, und die Taskfile wird automatisch Docker verwenden, wenn kein Compiler erkannt wird.

### macOS-Binärdateien nicht signiert

Cross-kompilierte macOS-Binärdateien sind nicht code-signiert. Apple erfordert Code-Signierung für die Verteilung, daher müssen Sie:

1. Die Binärdatei auf einem macOS-Gerät signieren, oder
2. In CI mit einem macOS-Läufer signieren

Siehe [Applications signieren](/de/guides/build/signing) für Details.

### Erstellung universeller Binärdateien

Universelle Binärdateien (arm64 + amd64 kombiniert) können auf jeder Plattform erstellt werden:

```bash
wails3 task darwin:build:universal
```

Unter Linux und Windows verwendet Wails seinen integrierten Befehl `wails3 tool lipo` (angetrieben von [konoui/lipo](https://github.com/konoui/lipo)), um die Binärdateien zu kombinieren. Dies erstellt eine einzelne Binärdatei, die nativ auf Apple Silicon und Intel-Macs läuft.
