# Codesignierung Ihrer Anwendung

Diese Anleitung behandelt, wie Sie Ihre Wails-Anwendungen für macOS, Windows und Linux signieren. Wails v3 bietet integrierte CLI-Tools für die Codesignierung, Notarisierung und PGP-Schlüsselverwaltung.

- **macOS** - Signieren und notarisieren Sie Ihre macOS-Anwendungen
- **Windows** - Signieren Sie Ihre Windows-Executables und Pakete
- **Linux** - Signieren Sie DEB- und RPM-Pakete mit PGP-Schlüsseln

## Plattformübergreifende Signier-Matrix

Diese Matrix zeigt, was Sie von jeder Quellplattform aus signieren können:

| Zielformat | Von Windows | Von macOS | Von Linux |
|---------------|:------------:|:----------:|:----------:|
| Windows EXE/MSI | ✅ | ✅ | ✅ |
| macOS .app Bundle | ❌ | ✅ | ❌ |
| macOS Notarisierung | ❌ | ✅ | ❌ |
| Linux DEB | ✅ | ✅ | ✅ |
| Linux RPM | ✅ | ✅ | ✅ |

<Aside type="tip">
Windows- und Linux-Pakete können von **jeder Plattform** aus signiert werden. Das Signieren von macOS-Anwendungen erfordert einen Mac aufgrund der Tooling-Anforderungen von Apple.
</Aside>

### Signier-Backends

Wails wählt automatisch das bestverfügbare Signier-Backend aus:

| Plattform | Native Backend | Plattformübergreifende Backend |
|----------|----------------|------------------------|
| Windows | `signtool.exe` (Windows SDK) | Eingebaut |
| macOS | `codesign` (Xcode) | Nicht verfügbar |
| Linux | N/A | Eingebaut |

Wenn Sie auf der nativen Plattform arbeiten, verwendet Wails die nativen Tools für maximale Kompatibilität. Beim Cross-Compiling wird die integrierte Signierunterstützung verwendet.

## Schnellstart

Der einfachste Weg zur Konfiguration der Signierung ist der interaktive Einrichtungsassistent:

```bash
wails3 setup signing
```

Dieser Befehl:
- Führt Sie durch die Konfiguration der Signierungsanmeldeinformationen für jede Plattform
- Listet auf macOS verfügbare Developer ID-Zertifikate aus Ihrem Schlüsselbund auf
- Kann für Linux einen neuen PGP-Schlüssel generieren, falls Sie keinen haben
- **Speichert Passwörter sicher in Ihrem System-Schlüsselbund** (nicht in Taskfiles)
- Aktualisiert den `vars`-Abschnitt in den Taskfiles jeder Plattform mit nicht-sensiblen Konfigurationen

<Aside type="tip">
Passwörter werden im nativen Anmeldeinformationsspeicher Ihres Systems gespeichert (macOS Keychain, Windows Credential Manager oder Linux Secret Service). Das bedeutet, dass Ihre Signierungskonfiguration sicher ist und über alle Ihre Wails-Projekte hinweg funktioniert.
</Aside>

Um nur bestimmte Plattformen zu konfigurieren:

```bash
wails3 setup signing --platform darwin
wails3 setup signing --platform windows --platform linux
```

### Manuelle Konfiguration

Alternativ können Sie die plattformspezifischen Taskfiles manuell bearbeiten. Bearbeiten Sie den `vars`-Abschnitt oben in jeder Datei:

<Tabs>
<TabItem label="macOS">
Bearbeiten Sie `build/darwin/Taskfile.yml`:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  KEYCHAIN_PROFILE: "my-notarize-profile"
  # ENTITLEMENTS: "build/darwin/entitlements.plist"
```

Führen Sie dann aus:

```bash
wails3 task darwin:sign           # Nur signieren
wails3 task darwin:sign:notarize  # Signieren und notarisieren
```
</TabItem>
<TabItem label="Windows">
Bearbeiten Sie `build/windows/Taskfile.yml`:

```yaml
vars:
  SIGN_CERTIFICATE: "path/to/certificate.pfx"
  # Oder stattdessen den Fingerabdruck verwenden:
  # SIGN_THUMBPRINT: "certificate-thumbprint"
  # TIMESTAMP_SERVER: "http://timestamp.digicert.com"
```

Das Passwort wird aus dem System-Schlüsselbund abgerufen (führen Sie `wails3 setup signing` zur Konfiguration aus).

Führen Sie dann aus:

```bash
wails3 task windows:sign           # Executable signieren
wails3 task windows:sign:installer # NSIS-Installer signieren
```
</TabItem>
<TabItem label="Linux">
Bearbeiten Sie `build/linux/Taskfile.yml`:

```yaml
vars:
  PGP_KEY: "path/to/signing-key.asc"
  # SIGN_ROLE: "builder"  # Optionen: origin, maint, archive, builder
```

Das Passwort wird aus dem System-Schlüsselbund abgerufen (führen Sie `wails3 setup signing` zur Konfiguration aus).

Führen Sie dann aus:

```bash
wails3 task linux:sign:deb       # DEB-Paket signieren
wails3 task linux:sign:rpm       # RPM-Paket signieren
wails3 task linux:sign:packages  # Alle Pakete signieren
```
</TabItem>
</Tabs>

Sie können auch die CLI direkt verwenden:

```bash
# Prüfen der Signierfähigkeiten auf Ihrem System
wails3 signing info

# Verfügbare Signier-Identitäten auflisten (macOS/Windows)
wails3 signing list
```

## macOS Codesignierung

### Voraussetzungen

- Apple Developer Account ($99/Jahr)
- Developer ID Application-Zertifikat
- Installierte Xcode Command Line Tools

### Signier-Identitäten

Verfügbare Signier-Identitäten prüfen:

```bash
wails3 signing list
```

Ausgabe:

```
Found 2 signing identities:

  Developer ID Application: Your Company (ABCD1234) [valid]
    Hash: ABC123DEF456...

  Apple Development: your@email.com (XYZ789) [valid]
    Hash: DEF789ABC123...
```

<Aside type="tip">
Für die Verteilung außerhalb des App Store benötigen Sie ein **Developer ID Application**-Zertifikat.
</Aside>

### Konfiguration

Bearbeiten Sie `build/darwin/Taskfile.yml` und legen Sie die Signierungsvariablen fest:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  KEYCHAIN_PROFILE: "my-notarize-profile"
  ENTITLEMENTS: "build/darwin/entitlements.plist"
```

| Variable | Erforderlich | Beschreibung |
|----------|----------|-------------|
| `SIGN_IDENTITY` | Ja | Ihre Developer ID (z.B. "Developer ID Application: Your Company (TEAMID)") |
| `KEYCHAIN_PROFILE` | Für Notarisierung | Name des Keychain-Profils mit gespeicherten Anmeldeinformationen |
| `ENTITLEMENTS` | Nein | Pfad zur Entitlements-Datei |

Führen Sie dann aus:

```bash
wails3 task darwin:sign           # Bauen, paketieren und signieren
wails3 task darwin:sign:notarize  # Bauen, paketieren, signieren und notarisieren
```

### Entitlements

Entitlements steuern, auf welche Fähigkeiten Ihre Anwendung zugreifen kann. Wails-Anwendungen benötigen typischerweise unterschiedliche Entitlements für Entwicklung vs. Produktion:

- **Entwicklung**: Erfordert JIT, ungespeicherten Speicher und Debugging-Entitlements
- **Produktion**: Minimale Entitlements (nur Netzwerkzugriff)

Verwenden Sie den interaktiven Einrichtungsassistenten, um beide Dateien zu generieren:

```bash
wails3 setup entitlements
```

Dies erstellt:
- `build/darwin/entitlements.dev.plist` - Für Entwicklungsbauten
- `build/darwin/entitlements.plist` - Für Produktions-/signierte Bauten

**Vorlagen verfügbar:**
| Vorlage | Beschreibung |
|--------|-------------|
| Entwicklung | JIT, ungespeicherter Speicher, Debugging, Netzwerk |
| Produktion | Nur Netzwerk (minimal, am sichersten) |
| Beide | Erstellt sowohl Dev- als auch Produktionsdateien (empfohlen) |
| App Store | Sandbox aktiviert mit Netzwerk- und Dateizugriff |
| Benutzerdefiniert | Wählen Sie einzelne Entitlements |

<Aside type="note">
Die `run`-Aufgabe in der darwin Taskfile verwendet automatisch `entitlements.dev.plist`. Die `sign`-Aufgaben verwenden `entitlements.plist` für Produktionsbauten.
</Aside>

Legen Sie dann `ENTITLEMENTS` in den Taskfile-Variablen fest, um auf die entsprechende Datei zu verweisen.

### Notarisierung

Apple verlangt, dass alle verteilten Apps notarisiert werden.

<Steps>
1. **Speichern Sie Ihre Anmeldeinformationen im Schlüsselbund** (Einmalige Einrichtung):
   ```bash
   wails3 signing credentials \
     --apple-id "your@email.com" \
     --team-id "ABCD1234" \
     --password "app-specific-password" \
     --profile "my-notarize-profile"
   ```

2. **Legen Sie KEYCHAIN_PROFILE in Ihrer Taskfile** fest, um mit dem oben genannten Profilnamen übereinzustimmen.

3. **Signieren und notarisieren Sie Ihre App**:
   ```bash
   wails3 task darwin:sign:notarize
   ```

4. **Notarisierung verifizieren**:
   ```bash
   spctl --assess --verbose=2 bin/MyApp.app
   ```
</Steps>

<Aside type="note">
Die Notarisierung dauert in der Regel 1-2 Minuten. Das Ticket wird automatisch an Ihre App angeheftet.
</Aside>

## Windows Codesignierung

### Voraussetzungen

- Codesignierungszertifikat (von DigiCert, Sectigo usw.)
- Für natives Signieren: Windows SDK installiert (für signtool.exe)
- Für plattformübergreifend: Nur die Zertifikatsdatei

| Variable | Erforderlich | Beschreibung |
|----------|--------------|--------------|
| `SIGN_CERTIFICATE` | Eine dieser | Pfad zur .pfx/.p12-Zertifikatsdatei |
| `SIGN_THUMBPRINT` | Eine dieser | Zertifikats-Fingerabdruck im Windows-Zertifikatsspeicher |
| `TIMESTAMP_SERVER` | Nein | URL des Zeitstempelservers (Standard: http://timestamp.digicert.com) |

<Aside type="note">
Das Zertifikatspasswort wird in Ihrem System-Schlüsselbund gespeichert, nicht in der Taskfile. Führen Sie `wails3 setup signing` aus, um es zu konfigurieren, oder setzen Sie die Umgebungsvariable `WAILS_WINDOWS_CERT_PASSWORD` in CI.
</Aside>

Führen Sie dann Folgendes aus:

```bash
wails3 task windows:sign           # Build und signiere ausführbare Datei
wails3 task windows:sign:installer # Build und signiere NSIS-Installer
```

### Plattformübergreifende Signierung

Windows-Ausführbare Dateien können von jeder Plattform aus signiert werden. Dieselbe Taskfile-Konfiguration und dieselben Befehle funktionieren auf macOS und Linux.

### Unterstützte Windows-Formate

| Format | Erweiterung | Hinweise |
|--------|-----------|-------|
| Ausführbare Dateien | .exe | Standard-PE-Signierung |
| Installer | .msi | Windows-Installer-Pakete |
| App-Pakete | .msix, .appx | Moderne Windows-Apps |

## Linux-Paket-Signierung

Linux-Pakete (DEB und RPM) werden mit PGP/GPG-Schlüsseln signiert. Im Gegensatz zur Codesignierung unter Windows und macOS beweist die Linux-Paket-Signierung, dass das Paket von einer vertrauenswürdigen Quelle stammt, anstatt dass der Code vom Betriebssystem vertraut wird.

### Voraussetzungen

- PGP-Schlüsselpaar (kann mit Wails generiert werden)

### Generieren eines PGP-Schlüssels

Wenn Sie keinen PGP-Schlüssel haben, kann Wails einen für Sie generieren:

```bash
wails3 signing generate-key \
  --name "Ihr Name" \
  --email "ihre@email.com" \
  --comment "Paket-Signierungsschlüssel" \
  --output-private signing-key.asc \
  --output-public signing-key.pub.asc
```

Optionen:
- `--bits`: Schlüsselgröße (Standard: 4096)
- `--expiry`: Ablaufdauer des Schlüssels (z. B. "1y", "6m", "0" für kein Ablaufdatum)
- `--password`: Passwort zum Schutz des privaten Schlüssels

<Aside type="caution">
Bewahren Sie Ihren privaten Schlüssel sicher auf! Speichern Sie ihn verschlüsselt und sichern Sie ihn ordnungsgemäß.
</Aside>

### Konfiguration

Bearbeiten Sie `build/linux/Taskfile.yml` und legen Sie die Signierungsvariablen fest:

```yaml
vars:
  PGP_KEY: "path/to/signing-key.asc"
  # SIGN_ROLE: "builder"  # Optionen: origin, maint, archive, builder
```

| Variable | Erforderlich | Beschreibung |
|----------|----------|-------------|
| `PGP_KEY` | Ja | Pfad zur PGP-privaten Schlüsseldatei |
| `SIGN_ROLE` | Nein | DEB-Signierungsrolle (Standard: builder) |

<Aside type="note">
Das PGP-Schlüssel-Passwort wird in Ihrem System-Schlüsselbund gespeichert, nicht in der Taskfile. Führen Sie `wails3 setup signing` aus, um es zu konfigurieren, oder setzen Sie die Umgebungsvariable `WAILS_PGP_PASSWORD` in CI.
</Aside>

Führen Sie dann Folgendes aus:

```bash
wails3 task linux:sign:deb       # Build und signiere DEB-Paket
wails3 task linux:sign:rpm       # Build und signiere RPM-Paket
wails3 task linux:sign:packages  # Build und signiere alle Pakete
```

### DEB-Signierungsrollen

Für DEB-Pakete können Sie die Signierungsrolle über `SIGN_ROLE` angeben:

- `origin`: Signatur vom Paketursprung
- `maint`: Signatur vom Paketbetreuer
- `archive`: Signatur vom Archivbetreuer
- `builder`: Signatur vom Paketbauer (Standard)

### Plattformübergreifende Signierung

Linux-Pakete können von jeder Plattform aus signiert werden. Dieselbe Taskfile-Konfiguration und dieselben Befehle funktionieren unter Windows und macOS.

### Anzeigen von Schlüsselinformationen

```bash
wails3 signing key-info --key signing-key.asc
```

Ausgabe:

```
PGP-Schlüsselinformationen:
  Schlüssel-ID:       ABC123DEF456
  Fingerabdruck:      1234 5678 90AB CDEF ...
  Benutzer-IDs:       Ihr Name <ihre@email.com>
  Erstellt:           2024-01-15
  Läuft ab:           2025-01-15
  Hat Privaten:       Ja
  Verschlüsselt:      Ja
```

### Überprüfen von Linux-Paketen

```bash
# DEB-Signatur überprüfen
dpkg-sig --verify myapp_1.0.0_amd64.deb

# RPM-Signatur überprüfen
rpm --checksig myapp-1.0.0.x86_64.rpm
```

### Verteilen Ihres öffentlichen Schlüssels

Benutzer benötigen Ihren öffentlichen Schlüssel, um Pakete zu überprüfen:

```bash
# Öffentlichen Schlüssel zur Verteilung exportieren
wails3 signing key-info --key signing-key.asc --export-public > myapp-signing.pub.asc

# Benutzer können ihn importieren:
# Für DEB (apt):
sudo apt-key add myapp-signing.pub.asc
# Oder für modernes apt:
sudo cp myapp-signing.pub.asc /etc/apt/trusted.gpg.d/

# Für RPM:
sudo rpm --import myapp-signing.pub.asc
```

## GitHub Actions-Integration

In CI-Umgebungen werden Passwörter über Umgebungsvariablen bereitgestellt, anstatt den System-Schlüsselbund zu verwenden:

| Umgebungsvariable | Beschreibung |
|---------------------|-------------|
| `WAILS_WINDOWS_CERT_PASSWORD` | Windows-Zertifikatspasswort |
| `WAILS_PGP_PASSWORD` | PGP-Schlüssel-Passwort für Linux-Pakete |

Sie können Taskfile-Variablen auch direkt übergeben:

```bash
wails3 task darwin:sign SIGN_IDENTITY="$SIGN_IDENTITY" KEYCHAIN_PROFILE="$KEYCHAIN_PROFILE"
```

### macOS-Workflow

```yaml
name: Build und signiere macOS

on:
  push:
    tags: ['v*']

jobs:
  build:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4

      - name: Go einrichten
        uses: actions/setup-go@v5
        with:
          go-version: '1.23'

      - name: Wails installieren
        run: go install github.com/wailsapp/wails/v3/cmd/wails3@latest

      - name: Zertifikat importieren
        env:
          CERTIFICATE_BASE64: ${{ secrets.MACOS_CERTIFICATE }}
          CERTIFICATE_PASSWORD: ${{ secrets.MACOS_CERTIFICATE_PASSWORD }}
        run: |
          echo $CERTIFICATE_BASE64 | base64 --decode > certificate.p12
          security create-keychain -p "" build.keychain
          security default-keychain -s build.keychain
          security unlock-keychain -p "" build.keychain
          security import certificate.p12 -k build.keychain -P "$CERTIFICATE_PASSWORD" -T /usr/bin/codesign
          security set-key-partition-list -S apple-tool:,apple:,codesign: -s -k "" build.keychain

      - name: Notarisierungs-Anmeldedaten speichern
        env:
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
          APPLE_APP_PASSWORD: ${{ secrets.APPLE_APP_PASSWORD }}
        run: |
          wails3 signing credentials \
            --apple-id "$APPLE_ID" \
            --team-id "$APPLE_TEAM_ID" \
            --password "$APPLE_APP_PASSWORD" \
            --profile "notarize-profile"

      - name: Build, signieren und notarisieren
        env:
          SIGN_IDENTITY: ${{ secrets.MACOS_SIGN_IDENTITY }}
        run: |
          wails3 task darwin:sign:notarize \
            SIGN_IDENTITY="$SIGN_IDENTITY" \
            KEYCHAIN_PROFILE="notarize-profile"

      - name: Artefakt hochladen
        uses: actions/upload-artifact@v4
        with:
          name: MyApp-macOS
          path: bin/*.app
```

### Windows-Workflow

```yaml
name: Build und signiere Windows

on:
  push:
    tags: ['v*']

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Go einrichten
        uses: actions/setup-go@v5
        with:
          go-version: '1.23'

      - name: Wails installieren
        run: go install github.com/wailsapp/wails/v3/cmd/wails3@latest

      - name: Zertifikat importieren
        env:
          CERTIFICATE_BASE64: ${{ secrets.WINDOWS_CERTIFICATE }}
        run: |
          $certBytes = [Convert]::FromBase64String($env:CERTIFICATE_BASE64)
          [IO.File]::WriteAllBytes("certificate.pfx", $certBytes)

      - name: Build und signieren
        env:
          WAILS_WINDOWS_CERT_PASSWORD: ${{ secrets.WINDOWS_CERTIFICATE_PASSWORD }}
        run: |
          wails3 task windows:sign SIGN_CERTIFICATE=certificate.pfx

      - name: Artefakt hochladen
        uses: actions/upload-artifact@v4
        with:
          name: MyApp-Windows
          path: bin/*.exe
```

### Plattformübergreifender Workflow (Linux-Läufer)

Signieren Sie Windows- und Linux-Pakete von einem einzigen Linux-Läufer aus:

```yaml
name: Build und signieren (Plattformübergreifend)

on:
  push:
    tags: ['v*']
```

| Flag | Beschreibung |
|------|--------------|
| `--input` | Pfad zur zu signierenden Datei |
| `--output` | Ausgabepfad (optional, Standard: Überschreiben der Quelldatei) |
| `--verbose` | Aktiviert detaillierte Ausgabe |

**Windows/macOS-Flags:**
| Flag | Beschreibung |
|------|------|
| `--certificate` | Pfad zum PKCS#12-Zertifikat (.pfx/.p12) |
| `--password` | Zertifikatspasswort |
| `--timestamp` | URL des Zeitstempelservers |

**macOS-spezifische Flags:**
| Flag | Beschreibung |
|------|------|
| `--identity` | Signieridentität (verwenden Sie '-', für ad-hoc-Signierung) |
| `--entitlements` | Pfad zur Entitlements-plist |
| `--hardened-runtime` | Aktiviert den gehärteten Laufzeitmodus (Standard: true) |
| `--notarize` | Zur Notarisierung einreichen |
| `--keychain-profile` | Schlüsselbundprofil für die Notarisierung |

**Windows-spezifische Flags:**
| Flag | Beschreibung |
|------|------|
| `--thumbprint` | Zertifikatsfingerabdruck im Windows-Speicher |
| `--description` | Anwendungsbeschreibung |
| `--url` | Anwendungs-URL |

**Linux-spezifische Flags:**
| Flag | Beschreibung |
|------|------|
| `--pgp-key` | Pfad zum privaten PGP-Schlüssel |
| `--pgp-password` | PGP-Schlüssel-Passwort |
| `--role` | DEB-Signierungsrolle (origin/maint/archive/builder) |

### wails3 signing info

Zeigt die verfügbaren Signierungsfunktionen auf dem aktuellen System an.

```bash
wails3 signing info
```

### wails3 signing list

Listet verfügbare Signieridentitäten auf.

```bash
wails3 signing list
```

### wails3 signing credentials

Speichert Notarisierungsanmeldeinformationen im Schlüsselbund (nur macOS).

```bash
wails3 signing credentials [flags]

Flags:
  --apple-id      Apple ID-E-Mail
  --team-id       Apple Developer Team ID
  --password      App-spezifisches Passwort
  --profile       Schlüsselbundprofilname
```

### wails3 signing generate-key

Generiert ein PGP-Schlüsselpaar für die Linux-Paketsignierung.

```bash
wails3 signing generate-key [flags]

Flags:
  --name            Name für den Schlüssel (erforderlich)
  --email           E-Mail für den Schlüssel (erforderlich)
  --comment         Kommentar für den Schlüssel
  --bits            Schlüsselgröße in Bits (Standard: 4096)
  --expiry          Schlüsselablauf (z. B. "1y", "6m", "0" für nie)
  --password        Passwort zur Verschlüsselung des privaten Schlüssels
  --output-private  Pfad für die Ausgabe des privaten Schlüssels
  --output-public   Pfad für die Ausgabe des öffentlichen Schlüssels
```

### wails3 signing key-info

Zeigt Informationen über einen PGP-Schlüssel an.

```bash
wails3 signing key-info --key <path-to-key>
```

## Fehlerbehebung

### macOS-Probleme

**"No Developer ID certificate found"**
- Stellen Sie sicher, dass Ihr Zertifikat im Schlüsselbund installiert ist
- Prüfen Sie mit `wails3 signing list`, ob es abgelaufen ist
- Stellen Sie sicher, dass Sie ein "Developer ID Application"-Zertifikat haben (nicht nur "Apple Development")

**"Notarization failed"**
- Prüfen Sie das Notarisierungsprotokoll: `xcrun notarytool log <submission-id> --keychain-profile <profile>`
- Stellen Sie sicher, dass der gehärtete Laufzeitmodus aktiviert ist
- Überprüfen Sie, ob Ihre App keine unsignierten Binärdateien enthält

**"Codesign failed"**
- Stellen Sie sicher, dass der Schlüsselbund entsperrt ist: `security unlock-keychain`
- Prüfen Sie die Dateiberechtigungen des App-Bundles

### Windows-Probleme

**"Certificate not found"**
- Überprüfen Sie, ob der Zertifikatspfad korrekt ist
- Prüfen Sie das Zertifikatspasswort
- Stellen Sie sicher, dass das Zertifikat gültig ist (nicht abgelaufen oder widerrufen)

**"Timestamp server error"**
- Versuchen Sie einen anderen Zeitstempelserver:
  - `http://timestamp.digicert.com`
  - `http://timestamp.sectigo.com`
  - `http://timestamp.comodoca.com`

### Linux-Probleme

**"Invalid PGP key"**
- Stellen Sie sicher, dass die Schlüsseldatei im ASCII-armored-Format vorliegt
- Prüfen Sie mit `wails3 signing key-info`, ob der Schlüssel abgelaufen ist
- Überprüfen Sie, ob das Passwort korrekt ist

**"Signature verification failed"**
- Stellen Sie sicher, dass der öffentliche Schlüssel ordnungsgemäß importiert wurde
- Prüfen Sie, ob das Paket nach der Signierung nicht verändert wurde
