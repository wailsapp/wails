Benutzerdefinierte URL-Protokolle (auch URL-Schemata genannt) ermöglichen es Ihrer Anwendung, gestartet zu werden, wenn Benutzer auf Links mit Ihrem benutzerdefinierten Protokoll klicken, wie z. B. `myapp://action` oder `myapp://open/document`.

## Übersicht

Benutzerdefinierte Protokolle ermöglichen:
- **Deep Linking**: Starten Sie Ihre App mit spezifischen Daten
- **Browser-Integration**: Behandeln von Links aus Webseiten
- **E-Mail-Links**: Öffnen Sie Ihre App aus E-Mail-Clients
- **App-übergreifende Kommunikation**: Starten Sie aus anderen Anwendungen

**Beispiel**: `myapp://open/document?id=123` startet Ihre App und öffnet Dokument 123.

## Konfiguration

Definieren Sie benutzerdefinierte Protokolle in Ihren Anwendungsoptionen:

```go
package main

import "github.com/wailsapp/wails/v3/pkg/application"

func main() {
    app := application.New(application.Options{
        Name:        "My Application",
        Description: "My awesome application",
        Protocols: []application.Protocol{
            {
                Scheme:      "myapp",
                Description: "My Application Protocol",
                Role:        "Editor",  // macOS only
            },
        },
    })

    // Register handler for protocol events
    app.Event.On(application.Events.ApplicationOpenedWithURL, func(event *application.ApplicationEvent) {
        url := event.Context().ClickedURL()
        handleCustomURL(url)
    })

    app.Run()
}

func handleCustomURL(url string) {
    // Parse and handle the custom URL
    // Example: myapp://open/document?id=123
    println("Received URL:", url)
}
```

## Protokoll-Handler

Lauschen Sie auf Protokollereignisse, um eingehende URLs zu verarbeiten:

```go
app.Event.On(application.Events.ApplicationOpenedWithURL, func(event *application.ApplicationEvent) {
    url := event.Context().ClickedURL()

    // Parse the URL
    parsedURL, err := parseCustomURL(url)
    if err != nil {
        app.Logger.Error("Failed to parse URL:", err)
        return
    }

    // Handle different actions
    switch parsedURL.Action {
    case "open":
        openDocument(parsedURL.DocumentID)
    case "settings":
        showSettings()
    case "user":
        showUser Profile(parsedURL.UserID)
    default:
        app.Logger.Warn("Unknown action:", parsedURL.Action)
    }
})
```

## URL-Struktur

Entwerfen Sie klare, hierarchische URL-Strukturen:

```
myapp://action/resource?param=value

Examples:
myapp://open/document?id=123
myapp://settings/theme?mode=dark
myapp://user/profile?username=john
```

**Best Practices:**
- Verwenden Sie Kleinbuchstaben für Schemanamen
- Halten Sie Schemata kurz und einprägsam
- Verwenden Sie hierarchische Pfade für Ressourcen
- Fügen Sie Abfrageparameter für optionale Daten hinzu
- URL-coden Sie Sonderzeichen

## Plattform-Registrierung

Benutzerdefinierte Protokolle werden auf jeder Plattform unterschiedlich registriert.

<Tabs syncKey="platform">
  <TabItem label="Windows" icon="seti:windows">

### Windows NSIS-Installer

**Wails v3 registriert benutzerdefinierte Protokolle automatisch**, wenn NSIS-Installer verwendet werden.

#### Automatische Registrierung

Wenn Sie Ihre Anwendung mit `wails3 build` erstellen, führt der NSIS-Installer Folgendes aus:
1. Registriert automatisch alle in `application.Options.Protocols` definierten Protokolle
2. Verknüpft Protokolle mit Ihrer Anwendungs-Executable
3. Richtet entsprechende Registry-Einträge ein
4. Entfernt Protokollzuordnungen während der Deinstallation

**Keine zusätzliche Konfiguration erforderlich!**

#### Funktionsweise

Die NSIS-Vorlage enthält integrierte Makros:
- `wails.associateCustomProtocols` - Registriert Protokolle während der Installation
- `wails.unassociateCustomProtocols` - Entfernt Protokolle während der Deinstallation

Diese Makros werden automatisch basierend auf Ihrer `Protocols`-Konfiguration aufgerufen.

#### Manuelle Registry (Erweitert)

Falls Sie eine manuelle Registrierung benötigen (außerhalb von NSIS):

```batch
@echo off
REM Register custom protocol
REG ADD "HKEY_CURRENT_USER\SOFTWARE\Classes\myapp" /ve /d "URL:My Application Protocol" /f
REG ADD "HKEY_CURRENT_USER\SOFTWARE\Classes\myapp" /v "URL Protocol" /t REG_SZ /d "" /f
REG ADD "HKEY_CURRENT_USER\SOFTWARE\Classes\myapp\shell\open\command" /ve /d "\"%1\"" /f
```

#### Testen

Testen Sie Ihre Protokollregistrierung:

```powershell
# Open protocol URL from PowerShell
Start-Process "myapp://test/action"

# Or from command prompt
start myapp://test/action
```

### Windows MSIX-Paket

Benutzerdefinierte Protokolle werden auch automatisch registriert, wenn MSIX-Paketierung verwendet wird.

#### Automatische Registrierung

Wenn Sie Ihre Anwendung mit MSIX erstellen, enthält das Manifest automatisch Protokollregistrierungen aus Ihrer `build/config.yml`-Protokollkonfiguration.

Das generierte Manifest enthält:

```xml
<uap:Extension Category="windows.protocol">
  <uap:Protocol Name="myapp">
    <uap:DisplayName>My Application Protocol</uap:DisplayName>
  </uap:Protocol>
</uap:Extension>
```

#### Universal Links (Web-zu-App-Verknüpfung)

Windows unterstützt **Web-zu-App-Verknüpfungen**, die ähnlich wie Universal Links unter macOS funktionieren. Wenn Sie Ihre Anwendung als MSIX-Paket bereitstellen, können Sie HTTPS-Links aktivieren, um Ihre App direkt zu starten.

<Aside type="note">
Web-zu-App-Verknüpfungen erfordern eine manuelle Manifestkonfiguration. Benutzerdefinierte Protokoll-Schemata werden automatisch aus `build/config.yml` konfiguriert, aber zugeordnete Domänen müssen manuell zu Ihrem MSIX-Manifest hinzugefügt werden.
</Aside>

Um Web-zu-App-Verknüpfungen zu aktivieren, folgen Sie dem [Microsoft-Leitfaden zur Web-zu-App-Verknüpfung](https://learn.microsoft.com/en-us/windows/apps/develop/launch/web-to-app-linking). Sie müssen:

1. **App URI Handler manuell zu Ihrem MSIX-Manifest hinzufügen** (`build/windows/msix/app_manifest.xml`):
   ```xml
   <uap3:Extension Category="windows.appUriHandler">
     <uap3:AppUriHandler>
       <uap3:Host Name="myawesomeapp.com"/>
     </uap3:AppUriHandler>
   </uap3:Extension>
   ```

2. **`windows-app-web-link` auf Ihrer Website konfigurieren:** Hosten Sie eine `windows-app-web-link`-Datei unter `https://myawesomeapp.com/.well-known/windows-app-web-link`. Diese Datei sollte die Paketinformationen Ihrer App und die von ihr verwalteten Pfade enthalten.

Wenn ein Web-zu-App-Link Ihre Anwendung startet, erhalten Sie dasselbe `ApplicationOpenedWithURL`-Ereignis wie bei benutzerdefinierten Protokoll-Schemata.

### Info.plist-Konfiguration

Unter macOS werden Protokolle über Ihre `Info.plist`-Datei registriert.

#### Automatische Konfiguration

Wails generiert automatisch die `Info.plist` mit Ihren Protokollen, wenn Sie mit `wails3 build` erstellen.

Die Protokolle aus `application.Options.Protocols` werden hinzugefügt zu:

```xml
<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLName</key>
        <string>My Application Protocol</string>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>myapp</string>
        </array>
        <key>CFBundleTypeRole</key>
        <string>Editor</string>
    </dict>
</array>
```

#### Testen

```bash
# Open protocol URL from terminal
open "myapp://test/action"

# Check registered handlers
/System/Library/Frameworks/CoreServices.framework/Versions/A/Frameworks/LaunchServices.framework/Versions/A/Support/lsregister -dump | grep myapp
```

#### Universal Links

Zusätzlich zu benutzerdefinierten Protokoll-Schemata unterstützt macOS auch **Universal Links**, die es Ihrer App ermöglichen, durch reguläre HTTPS-Links (z. B. `https://myawesomeapp.com/path`) gestartet zu werden. Universal Links bieten ein nahtloses Benutzererlebnis zwischen Ihrer Web- und Desktop-App.

<Aside type="caution">
Universal Links erfordern, dass Ihre macOS-App mit einem gültigen Apple Developer-Zertifikat und Provisioning-Profil **code-signiert** ist. Nicht signierte oder ad-hoc signierte Builds können keine Universal Links öffnen. Stellen Sie sicher, dass Ihre App ordnungsgemäß signiert ist, bevor Sie testen.
</Aside>

Um Universal Links zu aktivieren, folgen Sie dem [Apple-Leitfaden zur Unterstützung von Universal Links in Ihrer App](https://developer.apple.com/documentation/xcode/supporting-universal-links-in-your-app). Sie müssen:

1. **Berechtigungen (Entitlements) in Ihrer `entitlements.plist` hinzufügen**:
   ```xml```xml
   <key>com.apple.developer.associated-domains</key>
   <array>
     <string>applinks:myawesomeapp.com</string>
   </array>
   ```

2. **Fügen Sie `NSUserActivityTypes` zu `Info.plist` hinzu**:
   ```xml
   <key>NSUserActivityTypes</key>
   <array>
     <string>NSUserActivityTypeBrowsingWeb</string>
   </array>
   ```

3. **Konfigurieren Sie `apple-app-site-association` auf Ihrer Website**: Stellen Sie eine `apple-app-site-association`-Datei unter `https://myawesomeapp.com/.well-known/apple-app-site-association` bereit.

Wenn ein Universal Link Ihre App auslöst, erhalten Sie dasselbe `ApplicationOpenedWithURL`-Ereignis, sodass der Verarbeitungscode identisch zu benutzerdefinierten Protokoll-Schemata ist.

### Desktop-Eintrag

Unter Linux werden Protokolle über `.desktop`-Dateien registriert.

#### Automatische Konfiguration

Wails generiert eine Desktop-Eintragsdatei mit Protokoll-Handlern, wenn Sie mit `wails3 build` bauen.

**In v3 behoben**: Der Linux-Desktop-Vorlage werden nun Protokoll-Handler korrekt hinzugefügt.

Die generierte Desktop-Datei enthält:

```ini
[Desktop Entry]
Type=Application
Name=My Application
Exec=/usr/bin/myapp %u
MimeType=x-scheme-handler/myapp;
```

#### Manuelle Registrierung

Falls erforderlich, installieren Sie die Desktop-Datei manuell:

```bash
# Desktop-Datei kopieren
cp myapp.desktop ~/.local/share/applications/

# Desktop-Datenbank aktualisieren
update-desktop-database ~/.local/share/applications/

# Protokoll-Handler registrieren
xdg-mime default myapp.desktop x-scheme-handler/myapp
```

#### Testen

```bash
# Protokoll-URL öffnen
xdg-open "myapp://test/action"

# Registrierten Handler überprüfen
xdg-mime query default x-scheme-handler/myapp
```

## Vollständiges Beispiel

Hier ist ein vollständiges Beispiel, das mehrere Protokoll-Aktionen verarbeitet:

```go
package main

import (
    "fmt"
    "net/url"
    "strings"
    "github.com/wailsapp/wails/v3/pkg/application"
)

type App struct {
    app    *application.Application
    window *application.WebviewWindow
}

func main() {
    app := application.New(application.Options{
        Name:        "DeepLink Demo",
        Description: "Custom protocol demonstration",
        Protocols: []application.Protocol{
            {
                Scheme:      "deeplink",
                Description: "DeepLink Demo Protocol",
                Role:        "Editor",
            },
        },
    })

    myApp := &App{app: app}
    myApp.setup()

    app.Run()
}

func (a *App) setup() {
    // Fenster erstellen
    a.window = a.app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:  "DeepLink Demo",
        Width:  800,
        Height: 600,
        URL:    "http://wails.localhost/",
    })

    // Benutzerdefinierte Protokoll-URLs verarbeiten
    a.app.Event.On(application.Events.ApplicationOpenedWithURL, func(event *application.ApplicationEvent) {
        customURL := event.Context().ClickedURL()
        a.handleDeepLink(customURL)
    })
}

func (a *App) handleDeepLink(rawURL string) {
    // URL parsen
    parsedURL, err := url.Parse(rawURL)
    if err != nil {
        a.app.Logger.Error("Failed to parse URL:", err)
        return
    }

    // Fenster in den Vordergrund holen
    a.window.Show()
    a.window.SetFocus()

    // Pfad und Abfrage extrahieren
    path := strings.Trim(parsedURL.Path, "/")
    query := parsedURL.Query()

    // Verschiedene Aktionen verarbeiten
    parts := strings.Split(path, "/")
    if len(parts) == 0 {
        return
    }

    action := parts[0]

    switch action {
    case "open":
        if len(parts) >= 2 {
            resource := parts[1]
            id := query.Get("id")
            a.openResource(resource, id)
        }

    case "settings":
        section := ""
        if len(parts) >= 2 {
            section = parts[1]
        }
        a.openSettings(section)

    case "user":
        if len(parts) >= 2 {
            username := parts[1]
            a.openUserProfile(username)
        }

    default:
        a.app.Logger.Warn("Unknown action:", action)
    }
}

func (a *App) openResource(resourceType, id string) {
    fmt.Printf("Opening %s with ID: %s\n", resourceType, id)
    // Ereignis an Frontend senden
    a.app.Event.Emit("navigate", map[string]string{
        "type": resourceType,
        "id":   id,
    })
}

func (a *App) openSettings(section string) {
    fmt.Printf("Opening settings section: %s\n", section)
    a.app.Event.Emit("navigate", map[string]string{
        "page":    "settings",
        "section": section,
    })
}

func (a *App) openUserProfile(username string) {
    fmt.Printf("Opening user profile: %s\n", username)
    a.app.Event.Emit("navigate", map[string]string{
        "page": "user",
        "user": username,
    })
}
```

## Frontend-Integration

Verarbeiten Sie Navigationsereignisse in Ihrem Frontend:

```javascript
import { Events } from '@wailsio/runtime'

// Auf Navigationsereignisse vom Protokoll-Handler hören
Events.On('navigate', (event) => {
    const { type, id, page, section, user } = event.data

    if (type === 'document') {
        // Dokument mit ID öffnen
        router.push(`/document/${id}`)
    } else if (page === 'settings') {
        // Einstellungen öffnen
        router.push(`/settings/${section}`)
    } else if (page === 'user') {
        // Benutzerprofil öffnen
        router.push(`/user/${user}`)
    }
})
```

## Sicherheitsüberlegungen

### Alle Eingaben validieren

Validieren und bereinigen Sie URLs von externen Quellen immer:

```go
func (a *App) handleDeepLink(rawURL string) {
    // URL parsen
    parsedURL, err := url.Parse(rawURL)
    if err != nil {
        a.app.Logger.Error("Invalid URL:", err)
        return
    }

    // Schema validieren
    if parsedURL.Scheme != "myapp" {
        a.app.Logger.Warn("Invalid scheme:", parsedURL.Scheme)
        return
    }

    // Pfad validieren
    path := strings.Trim(parsedURL.Path, "/")
    if !isValidPath(path) {
        a.app.Logger.Warn("Invalid path:", path)
        return
    }

    // Parameter bereinigen
    params := sanitizeQueryParams(parsedURL.Query())

    // Gültige URL verarbeiten
    a.processDeepLink(path, params)
}

func isValidPath(path string) bool {
    // Nur alphanumerische Zeichen und Schrägstriche erlauben
    validPath := regexp.MustCompile(`^[a-zA-Z0-9/]+$`)
    return validPath.MatchString(path)
}

func sanitizeQueryParams(query url.Values) map[string]string {
    sanitized := make(map[string]string)
    for key, values := range query {
        if len(values) > 0 {
            // Ersten Wert nehmen und bereinigen
            sanitized[key] = sanitizeString(values[0])
        }
    }
    return sanitized
}
```

### Injection-Angriffe verhindern

Führen Sie URLs niemals direkt als Code oder SQL aus:

```go
// ❌ NICHT tun: URL-Inhalt ausführen
func badHandler(url string) {
    exec.Command("sh", "-c", url).Run() // GEFÄHRLICH!
}

// ✅ Tun: Parsen und validieren
func goodHandler(url string) {
    parsed, _ := url.Parse(url)
    action := parsed.Query().Get("action")

    // Whitelist für erlaubte Aktionen
    allowed := map[string]bool{
        "open":     true,
        "settings": true,
        "help":     true,
    }

    if allowed[action] {
        handleAction(action)
    }
}
```
