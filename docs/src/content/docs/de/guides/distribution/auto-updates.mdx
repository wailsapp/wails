# Automatische Updates

Wails v3 bietet ein integriertes Updater-System, das automatische Aktualisierungsprüfungen, Downloads und Installationen unterstützt. Der Updater unterstützt auch binäre Delta-Updates (Patches) für minimale Download-Größen.

## Schnellstart

Fügen Sie den Updater-Dienst zu Ihrer Anwendung hinzu:

```go title="main.go"
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "time"
)

func main() {
    // Erstellen Sie den Updater-Dienst
    updater, err := application.CreateUpdaterService(
        "1.0.0", // Aktuelle Version
        application.WithUpdateURL("https://updates.example.com/myapp/"),
        application.WithCheckInterval(24 * time.Hour),
    )
    if err != nil {
        panic(err)
    }

    app := application.New(application.Options{
        Name: "MyApp",
        Services: []application.Service{
            application.NewService(updater),
        },
    })

    // ... Rest Ihrer App
    app.Run()
}
```

Verwenden Sie ihn dann aus Ihrem Frontend:

```typescript title="App.tsx"
import { updater } from './bindings/myapp';

async function checkForUpdates() {
  const update = await updater.CheckForUpdate();

  if (update) {
    console.log(`Neue Version verfügbar: ${update.version}`);
    console.log(`Release-Notizen: ${update.releaseNotes}`);

    // Herunterladen und installieren
    await updater.DownloadAndApply();
  }
}
```

## Konfigurationsoptionen

Der Updater unterstützt verschiedene Konfigurationsoptionen:

```go
updater, err := application.CreateUpdaterService(
    "1.0.0",
    // Erforderlich: URL, unter der die Update-Manifeste gehostet werden
    application.WithUpdateURL("https://updates.example.com/myapp/"),

    // Optional: Alle 24 Stunden automatisch nach Updates suchen
    application.WithCheckInterval(24 * time.Hour),

    // Optional: Pre-Release-Versionen zulassen
    application.WithAllowPrerelease(true),

    // Optional: Update-Kanal (stable, beta, canary)
    application.WithChannel("stable"),

    // Optional: Signierte Updates erfordern
    application.WithRequireSignature(true),
    application.WithPublicKey("your-ed25519-public-key"),
)
```

| Plattform | Schlüssel |
|----------|-----|
| macOS (Apple Silicon) | `macos-arm64` |
| macOS (Intel) | `macos-amd64` |
| Windows (64-Bit) | `windows-amd64` |
| Linux (64-Bit) | `linux-amd64` |
| Linux (ARM64) | `linux-arm64` |

## Frontend-API

Der Updater stellt Methoden bereit, die automatisch an Ihr Frontend gebunden werden:

### TypeScript-Typen

```typescript
interface UpdateInfo {
  version: string;
  releaseDate: Date;
  releaseNotes: string;
  size: number;
  patchSize?: number;
  mandatory: boolean;
  hasPatch: boolean;
}

interface Updater {
  // Die aktuelle Anwendungsversion abrufen
  GetCurrentVersion(): string;

  // Prüfen, ob ein Update verfügbar ist
  CheckForUpdate(): Promise<UpdateInfo | null>;

  // Das Update herunterladen (sendet Fortschrittsereignisse)
  DownloadUpdate(): Promise<void>;

  // Das heruntergeladene Update anwenden (startet die App neu)
  ApplyUpdate(): Promise<void>;

  // Herunterladen und in einem Aufruf anwenden
  DownloadAndApply(): Promise<void>;

  // Den aktuellen Status abrufen: idle, checking, available, downloading, ready, installing, error
  GetState(): string;

  // Die verfügbaren Update-Informationen abrufen
  GetUpdateInfo(): UpdateInfo | null;

  // Die letzte Fehlermeldung abrufen
  GetLastError(): string;

  // Den Updater-Status zurücksetzen
  Reset(): void;
}
```

### Fortschrittsereignisse

Hören Sie auf Fortschrittsereignisse beim Download:

```typescript
import { Events } from '@wailsio/runtime';

Events.On('updater:progress', (data) => {
  console.log(`Heruntergeladen: ${data.downloaded} / ${data.total}`);
  console.log(`Fortschritt: ${data.percentage.toFixed(1)}%`);
  console.log(`Geschwindigkeit: ${(data.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s`);
});
```

## Vollständiges Beispiel

Hier ist ein vollständiges Beispiel mit einer React-Komponente:

```tsx title="UpdateChecker.tsx"
import { useState, useEffect } from 'react';
import { updater } from './bindings/myapp';
import { Events } from '@wailsio/runtime';

interface Progress {
  downloaded: number;
  total: number;
  percentage: number;
  bytesPerSecond: number;
}

export function UpdateChecker() {
  const [checking, setChecking] = useState(false);
  const [updateInfo, setUpdateInfo] = useState<any>(null);
  const [downloading, setDownloading] = useState(false);
  const [progress, setProgress] = useState<Progress | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // Auf Fortschrittsereignisse hören
    const cleanup = Events.On('updater:progress', (data: Progress) => {
      setProgress(data);
    });

    // Beim Laden nach Updates suchen
    checkForUpdates();

    return () => cleanup();
  }, []);

  async function checkForUpdates() {
    setChecking(true);
    setError(null);

    try {
      const info = await updater.CheckForUpdate();
      setUpdateInfo(info);
    } catch (err) {
      setError(err.message);
    } finally {
      setChecking(false);
    }
  }

  async function downloadAndInstall() {
    setDownloading(true);
    setError(null);

    try {
      await updater.DownloadAndApply();
      // Die App wird automatisch neu gestartet
    } catch (err) {
      setError(err.message);
      setDownloading(false);
    }
  }

  if (checking) {
    return <div>Suche nach Updates...</div>;
  }

  if (error) {
    return (
      <div>
        <p>Fehler: {error}</p>
        <button onClick={checkForUpdates}>Wiederholen</button>
      </div>
    );
  }

  if (!updateInfo) {
    return (
      <div>
        <p>Sie sind auf dem neuesten Stand! (v{updater.GetCurrentVersion()})</p>
        <button onClick={checkForUpdates}>Erneut prüfen</button>
      </div>
    );
  }

  if (downloading) {
    return (
      <div>
        <p>Update wird heruntergeladen...</p>
        {progress && (
          <div>
            <progress value={progress.percentage} max={100} />
            <p>{progress.percentage.toFixed(1)}%</p>
            <p>{(progress.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s</p>
          </div>
        )}
      </div>
    );
  }return (
    <div>
      <h3>Update verfügbar!</h3>
      <p>Version {updateInfo.version} ist verfügbar</p>
      <p>Größe: {updateInfo.hasPatch
        ? `${(updateInfo.patchSize / 1024).toFixed(0)} KB (Patch)`
        : `${(updateInfo.size / 1024 / 1024).toFixed(1)} MB`}
      </p>
      <div dangerouslySetInnerHTML={{ __html: updateInfo.releaseNotes }} />
      <button onClick={downloadAndInstall}>Herunterladen & Installieren</button>
      <button onClick={() => setUpdateInfo(null)}>Überspringen</button>
    </div>
  );
}
```

## Update-Strategien

### Prüfung beim Start

```go
func (a *App) OnStartup(ctx context.Context) {
    // Prüfe nach einer kurzen Verzögerung auf Updates
    go func() {
        time.Sleep(5 * time.Second)
        info, err := a.updater.CheckForUpdate()
        if err == nil && info != nil {
            // Sende Event an das Frontend
            application.Get().EmitEvent("update-available", info)
        }
    }()
}
```

### Hintergrundprüfung

Konfiguriere automatische Hintergrundprüfungen:

```go
updater, _ := application.CreateUpdaterService(
    "1.0.0",
    application.WithUpdateURL("https://updates.example.com/myapp/"),
    application.WithCheckInterval(6 * time.Hour), // Prüfe alle 6 Stunden
)
```

### Manuelles Menüelement zur Prüfung

```go
menu := application.NewMenu()
menu.Add("Nach Updates suchen...").OnClick(func(ctx *application.Context) {
    info, err := updater.CheckForUpdate()
    if err != nil {
        application.InfoDialog().SetMessage("Fehler bei der Update-Prüfung").Show()
        return
    }
    if info == nil {
        application.InfoDialog().SetMessage("Sie sind auf dem neuesten Stand!").Show()
        return
    }
    // Zeige Update-Dialog an...
})
```

## Delta-Updates (Patches)

Delta-Updates (Patches) ermöglichen es Nutzern, nur die Änderungen zwischen Versionen herunterzuladen, was die Download-Größe drastisch reduziert.

### Funktionsweise

1. Erzeuge beim Bauen einer neuen Version Patches aus vorherigen Versionen
2. Stelle Patches zusammen mit vollständigen Updates auf deinem Server bereit
3. Der Updater lädt automatisch Patches herunter, wenn verfügbar
4. Wenn das Patchen fehlschlägt, wird auf den vollständigen Download zurückgegriffen

### Generieren von Patches

Patches werden mit dem `bsdiff`-Algorithmus generiert. Du benötigst das Tool `bsdiff`:

```bash
# Installiere bsdiff (macOS)
brew install bsdiff

# Installiere bsdiff (Ubuntu/Debian)
sudo apt-get install bsdiff

# Generiere einen Patch
bsdiff old-binary new-binary patch.bsdiff
```

### Benennung der Patch-Dateien

Organisiere deine Patches in deinem Update-Verzeichnis:

```
updates/
├── update.json
├── myapp-1.2.0-macos-arm64.tar.gz
├── myapp-1.2.0-windows-amd64.zip
└── patches/
    ├── 1.0.0-to-1.2.0-macos-arm64.patch
    ├── 1.1.0-to-1.2.0-macos-arm64.patch
    ├── 1.0.0-to-1.2.0-windows-amd64.patch
    └── 1.1.0-to-1.2.0-windows-amd64.patch
```

<Aside type="tip">
Behalte Patches der letzten wenigen Versionen. Nutzer mit sehr alten Versionen laden automatisch das vollständige Update herunter.
</Aside>

## Updates hosten

### Statisches Dateihosting

Updates können auf jedem statischen Dateiserver gehostet werden:

- **Amazon S3** / **Cloudflare R2**
- **Google Cloud Storage**
- **GitHub Releases**
- **Jedes CDN oder Webserver**

Beispielhafte Ordnerstruktur eines S3-Buckets:

```
s3://my-updates-bucket/myapp/
├── stable/
│   ├── update.json
│   ├── myapp-1.2.0-macos-arm64.tar.gz
│   └── patches/
│       └── 1.1.0-to-1.2.0-macos-arm64.patch
└── beta/
    └── update.json
```

## Sicherheit

### Prüfsummen-Verifizierung

Alle Downloads werden gegen SHA256-Prüfsummen im Manifest verifiziert:

```json
{
  "checksum": "sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
}
```

### Signaturen-Verifizierung

Für zusätzliche Sicherheit aktiviere die Signaturen-Verifizierung:

<Steps>
1. **Generiere ein Schlüsselpaar**:
   ```bash
   # Generiere Ed25519-Schlüsselpaar
   openssl genpkey -algorithm Ed25519 -out private.pem
   openssl pkey -in private.pem -pubout -out public.pem
   ```

2. **Signiere dein Update-Manifest**:
   ```bash
   openssl pkeyutl -sign -inkey private.pem -in update.json -out update.json.sig
   ```

3. **Konfiguriere den Updater**:
   ```go
   updater, _ := application.CreateUpdaterService(
       "1.0.0",
       application.WithUpdateURL("https://updates.example.com/myapp/"),
       application.WithRequireSignature(true),
       application.WithPublicKey("MCowBQYDK2VwAyEA..."), // Base64-kodierter öffentlicher Schlüssel
   )
   ```
</Steps>

## Best Practices

### Empfohlen

- Teste Updates gründlich, bevor du sie bereitstellst
- Halte vorherige Versionen für Rollbacks verfügbar
- Zeige Release Notes für Nutzer an
- Ermögliche es Nutzern, nicht-zwingende Updates zu überspringen
- Verwende HTTPS für alle Downloads
- Verifiziere Prüfsummen, bevor du Updates anwendest
- Behandle Fehler angemessen

### Nicht empfohlen

- Erzwingen Sie sofortige Neustarts ohne Warnung
- Überspringen Sie die Prüfsummen-Verifizierung
- Unterbrechen Sie Nutzer während wichtiger Arbeiten
- Löschen Sie die vorherige Version sofort
- Ignorieren Sie Update-Fehler

## Fehlerbehebung

### Update nicht gefunden

- Überprüfe, ob die Manifest-URL korrekt ist
- Prüfe, ob der Plattform-Schlüssel übereinstimmt (z. B. `macos-arm64`)
- Stelle sicher, dass die Version im Manifest neuer ist

### Download schlägt fehl

- Prüfe die Netzwerkverbindung
- Überprüfe, ob die Download-URL erreichbar ist
- Prüfe die CORS-Konfiguration bei cross-origin-Anfragen

### Patch schlägt fehl

- Der Updater greift automatisch auf den vollständigen Download zurück
- Stelle sicher, dass `bspatch` auf dem System verfügbar ist
- Überprüfe, ob die Patch-Prüfsumme korrekt ist

### Anwendung startet nicht neu

- Stelle auf macOS sicher, dass die App korrekt signiert ist
- Prüfe auf Windows nach Dateisperren
- Überprüfe auf Linux die Dateiberechtigungen
