## Übersicht

Tests stellen sicher, dass Ihre Anwendung korrekt funktioniert und Regressionen verhindert werden.

## Unit-Tests

### Tests von Diensten

```go
func TestUserService_Create(t *testing.T) {
    service := &UserService{
        users: make(map[string]*User),
    }
    
    user, err := service.Create("john@example.com", "password123")
    if err != nil {
        t.Fatalf("unexpected error: %v", err)
    }
    
    if user.Email != "john@example.com" {
        t.Errorf("expected email john@example.com, got %s", user.Email)
    }
}
```

### Tests mit Mocks

```go
type MockDB struct {
    users map[string]*User
}

func (m *MockDB) Create(user *User) error {
    m.users[user.ID] = user
    return nil
}

func TestUserService_WithMock(t *testing.T) {
    mockDB := &MockDB{users: make(map[string]*User)}
    service := &UserService{db: mockDB}
    
    user, err := service.Create("test@example.com", "pass")
    if err != nil {
        t.Fatal(err)
    }
    
    if len(mockDB.users) != 1 {
        t.Error("expected 1 user in mock")
    }
}
```

## Integrationstests

### Tests mit echten Abhängigkeiten

```go
func TestIntegration(t *testing.T) {
    // Testdatenbank einrichten
    db, err := sql.Open("sqlite3", ":memory:")
    if err != nil {
        t.Fatal(err)
    }
    defer db.Close()
    
    // Schema erstellen
    _, err = db.Exec(`CREATE TABLE users (...)`)
    if err != nil {
        t.Fatal(err)
    }
    
    // Dienst testen
    service := &UserService{db: db}
    user, err := service.Create("test@example.com", "password")
    if err != nil {
        t.Fatal(err)
    }
    
    // In der Datenbank überprüfen
    var count int
    db.QueryRow("SELECT COUNT(*) FROM users").Scan(&count)
    if count != 1 {
        t.Errorf("expected 1 user, got %d", count)
    }
}
```

## Frontend-Tests

### JavaScript-Unit-Tests

```javascript
// Using Vitest
import { describe, it, expect } from 'vitest'
import { formatDate } from './utils'

describe('formatDate', () => {
    it('formats date correctly', () => {
        const date = new Date('2024-01-01')
        expect(formatDate(date)).toBe('2024-01-01')
    })
})
```

### Tests von Bindings

```javascript
import { vi } from 'vitest'
import { GetUser } from './bindings/myapp/userservice'

// Mock the binding
vi.mock('./bindings/myapp/userservice', () => ({
    GetUser: vi.fn()
}))

describe('User Component', () => {
    it('loads user data', async () => {
        GetUser.mockResolvedValue({ name: 'John', email: 'john@example.com' })
        
        // Test your component
        const user = await GetUser(1)
        expect(user.name).toBe('John')
    })
})
```

## Best Practices

### ✅ Tun

- Schreiben Sie Tests, bevor Sie Bugs beheben
- Testen Sie Randfälle
- Verwenden Sie tabellengetriebene Tests
- Mocken Sie externe Abhängigkeiten
- Testen Sie die Fehlerbehandlung
- Halten Sie Tests schnell

### ❌ Nicht tun

- Überspringen Sie keine Fehlerfälle
- Testen Sie keine Implementierungsdetails
- Schreiben Sie keine instabilen Tests
- Ignorieren Sie keine Testfehler
- Überspringen Sie keine Integrationstests

## Tests ausführen

```bash
# Go-Tests ausführen
go test ./...

# Mit Abdeckungsanalyse ausführen
go test -cover ./...

# Bestimmten Test ausführen
go test -run TestUserService

# Frontend-Tests ausführen
cd frontend && npm test

# Mit Watch-Modus ausführen
cd frontend && npm test -- --watch
```
