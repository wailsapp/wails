Ein **Dienst** (Service) in Wails ist eine Go-Struktur, die Geschäftslogik enthält, die Sie für Ihr Frontend verfügbar machen möchten. Dienste halten Ihren Code organisiert, indem sie zusammenhängende Funktionen zusammenfassen.

Stellen Sie sich einen Dienst als eine Sammlung von Methoden vor, die Ihr JavaScript-Code aufrufen kann. Jede öffentliche Methode im Dienst wird nach dem Generieren der Bindings vom Frontend aus aufrufbar.

In diesem Tutorial erstellen wir einen QR-Code-Generator-Dienst, um diese Konzepte zu veranschaulichen. Am Ende werden Sie verstehen, wie man Dienste erstellt, Abhängigkeiten verwaltet und Ihren Go-Code mit dem Frontend verbindet.

<br/>

<Steps>

1. ## Erstellen Sie die QR-Dienst-Datei

   Erstellen Sie eine neue Datei namens `qrservice.go` in Ihrem Anwendungsverzeichnis:

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **Was hier passiert:**

   - `QRService` ist eine leere Struktur, die unsere QR-Code-Generierungsmethoden enthält
   - `NewQRService()` ist eine Konstruktorfunktion, die eine neue Instanz unseres Dienstes erstellt
   - `Generate()` ist eine Methode, die Text und eine Größe entgegennimmt und den QR-Code als PNG-Byte-Array zurückgibt
   - Die Methode gibt `([]byte, error)` zurück, was der Go-Konvention entspricht, Fehler als letzten Wert zurückzugeben
   - Wir verwenden das Paket `github.com/skip2/go-qrcode`, um die eigentliche QR-Code-Generierung zu handhaben

    <br/>

2. ## Registrieren des Dienstes

   Das Erstellen eines Dienstes reicht nicht aus – wir müssen ihn bei der Wails-Anwendung **registrieren**, damit diese weiß, dass der Dienst existiert und Bindungen dafür generieren kann.

   Die Registrierung erfolgt in `main.go`, wenn Sie Ihre Anwendung erstellen. Sie übergeben Ihre Dienstinstanzen an die Option `Services`:

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **Was hier passiert:**

   - `application.NewService()` wickelt Ihren Dienst ein, damit Wails ihn verwalten kann
   - Wir rufen `NewQRService()` auf, um eine Instanz unseres Dienstes zu erstellen
   - Der Dienst wird dem `Services`-Slice in den Anwendungsoptionen hinzugefügt
   - Wails scannt diesen Dienst nun nach öffentlichen Methoden, um sie für das Frontend verfügbar zu machen

    <br/>

3. ## Abhängigkeiten installieren

   Wir haben das Paket `github.com/skip2/go-qrcode` in unserem Code referenziert, aber es noch nicht tatsächlich heruntergeladen. Go muss von dieser Abhängigkeit wissen und sie in Ihr Projekt herunterladen.

   Führen Sie diesen Befehl in Ihrem Terminal von Ihrem Projektverzeichnis aus aus:

   ```bash
   go mod tidy
   ```

   **Was hier passiert:**

   - `go mod tidy` scannt Ihre Go-Dateien nach Import-Anweisungen
   - Es lädt fehlende Pakete (wie `go-qrcode`) herunter und fügt sie `go.mod` hinzu
   - Es entfernt auch Abhängigkeiten, die nicht mehr verwendet werden
   - Dies stellt sicher, dass Ihr Projekt über allen benötigten Code verfügt, um erfolgreich kompiliert zu werden

   Sie sollten eine Ausgabe sehen, die anzeigt, dass das QR-Code-Paket heruntergeladen und zu Ihrem Projekt hinzugefügt wurde.

    <br/>

4. ## Bindings generieren

   Um diese Methoden von Ihrem Frontend aus aufrufen zu können, müssen wir Bindings generieren.
   Dies können Sie tun, indem Sie `wails generate bindings` im Stammverzeichnis Ihres Projekts ausführen.

   :::note
   Beim ersten Mal, wenn Sie dies in einem Projekt ausführen, führt der Bindings-Generator eine gründliche Analyse Ihres Codes und Ihrer Abhängigkeiten durch. Dies kann manchmal etwas länger dauern als erwartet, nachfolgende Ausführungen sind jedoch viel schneller.
   :::

   Sobald Sie dies ausgeführt haben, sollten Sie etwas Ähnliches wie das Folgende in Ihrem Terminal sehen:

   ```bash
    % wails3 generate bindings
    INFO  Processed: 337 Packages, 1 Service, 1 Method, 0 Enums, 0 Models in 740.196125ms.
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

   Sie werden feststellen, dass es im Frontend-Verzeichnis ein neues Verzeichnis namens `bindings` gibt:

   ```bash
   frontend/
   └── bindings
       └── changeme
           ├── index.js
           └── qrservice.js
   ```

    :::tip[Profi-Tipp]
        Wenn Sie Ihre Anwendung mit `wails3 build` erstellen, werden automatisch Bindings für Sie generiert und auf dem neuesten Stand gehalten.
    :::
    <br/>

5. ## Verständnis der Bindings

   Schauen wir uns die generierten Bindings in `bindings/changeme/qrservice.js` an:

   ```js title="bindings/changeme/qrservice.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT

    /**
     * QRService handles QR code generation
     * @module
     */

    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
    // @ts-ignore: Unused imports
    import {Call as $Call, Create as $Create} from "@wailsio/runtime";

    /**
     * Generate creates a QR code from the given text
     * @param {string} text
     * @param {number} size
     * @returns {Promise<string> & { cancel(): void }}
     */
    export function Generate(text, size) {
        let $resultPromise = /** @type {any} */($Call.ByID(3576998831, text, size));
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
            return $Create.ByteSlice($result);
        }));
        $typingPromise.cancel = $resultPromise.cancel.bind($resultPromise);
        return $typingPromise;
    }
   ```

    Wir können sehen, dass die Bindings für die Methode `Generate` generiert wurden. Die Parameternamen wurden beibehalten,
    ebenso wie die Kommentare. JSDoc wurde ebenfalls für die Methode generiert, um Ihrer IDE Typinformationen bereitzustellen.

    :::note
    Es ist nicht notwendig, die generierten Bindings vollständig zu verstehen, aber es ist wichtig zu verstehen, wie sie funktionieren.
    :::

    Die Bindings bieten:
    - Funktionen, die Ihren Go-Methoden entsprechen
    - Automatische Konvertierung zwischen Go- und JavaScript-Typen
    - Promise-basierte asynchrone Operationen
    - Typinformationen als JSDoc-Kommentare

    :::tip[Typescript]
    Der Bindings-Generator unterstützt auch die Generierung von Typescript-Bindings. Dies können Sie tun, indem Sie `wails3 generate bindings -ts` ausführen.
    :::

    Der generierte Dienst wird von einer `index.js`-Datei neu exportiert:

   ```js title="bindings/changeme/index.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT
    import * as QRService from "./qrservice.js";
    export {
        QRService
    };
   ```

Anschließend können Sie es über den vereinfachten Importpfad `./bindings/changeme` aufrufen, der nur aus Ihrem Go-Paketpfad besteht, ohne dass ein Dateiname angegeben werden muss.

:::note
Vereinfachte Importpfade sind nur verfügbar, wenn Frontend-Bundler verwendet werden.
Wenn Sie ein Vanilla-Frontend bevorzugen, das keinen Bundler einsetzt,
müssen Sie entweder `index.js` oder `qrservice.js` manuell importieren.
:::

6. ## Bindings im Frontend verwenden

Nun können wir unseren Go-Dienst aus JavaScript heraus aufrufen! Die generierten Bindings machen dies einfach und typsicher.

Aktualisieren Sie `frontend/src/main.js`, um die neuen Bindings zu verwenden:

```js title="frontend/src/main.js"
import { QRService } from './bindings/changeme';

async function generateQR() {
    const text = document.getElementById('text').value;
    if (!text) {
        alert('Bitte geben Sie einen Text ein');
        return;
    }

    try {
        // QR-Code als Base64 generieren
        const qrCodeBase64 = await QRService.Generate(text, 256);

        // QR-Code anzeigen
        const qrDiv = document.getElementById('qrcode');
        qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

    } catch (err) {
        console.error('QR-Code konnte nicht generiert werden:', err);
        alert('QR-Code konnte nicht generiert werden: ' + err);
    }
}

export function initializeQRGenerator() {
    const button = document.getElementById('generateButton');
    button.addEventListener('click', generateQR);
}
```

**Was hier passiert:**

- Wir importieren `QRService` aus den generierten Bindings
- `QRService.Generate()` ruft unsere Go-Methode auf – sie gibt ein Promise zurück, daher verwenden wir `await`
- Die Go-Methode gibt `[]byte` zurück, das Wails automatisch in einen Base64-String für JavaScript konvertiert
- Wir erstellen eine Data-URL mit dem Base64-String, um das PNG-Bild anzuzeigen
- Der `try/catch`-Block behandelt alle Fehler von der Go-Seite (z. B. ungültige Eingaben)
- Wenn unser Go-Code einen Fehler zurückgibt, wird das Promise abgelehnt und wir fangen es hier ab

7. ## Alternative Vorgehensweise: HTTP-Handler

Bisher haben wir folgende Bereiche behandelt:
- Erstellen eines neuen Dienstes
- Generieren von Bindings
- Verwenden der Bindings in unserem Frontend-Code

**Warum einen HTTP-Handler verwenden?**

Methodenbindungen funktionieren hervorragend für Datenoperationen, aber es gibt einen alternativen Ansatz zum Bereitstellen von Dateien, Bildern oder anderen Medien. Anstatt alles in Base64 umzuwandeln und durch die Bindings zu senden, kann Ihr Dienst wie ein mini-webserver agieren.

Dies ist nützlich, wenn:
- Sie Bilder, Videos oder große Dateien bereitstellen
- Sie Standard-HTML-Tags `<img>` oder `<video>` mit `src`-Attributen verwenden möchten
- Sie direkten URL-Zugriff auf Ressourcen benötigen

Wenn Ihr Dienst Go's Standardmethode `ServeHTTP(w http.ResponseWriter, r *http.Request)` implementiert, kann Wails sie als HTTP-Endpunkt verfügbar machen. Erweitern wir unseren QR-Code-Dienst, um dies zu unterstützen:

```go title="qrservice.go" ins={4-5,37-65}
package main

import (
    "net/http"
    "strconv"

    "github.com/skip2/go-qrcode"
)

// QRService behandelt die QR-Code-Generierung
type QRService struct {
    // Wir können hier Status hinzufügen, falls erforderlich
}

// NewQRService erstellt einen neuen QR-Dienst
func NewQRService() *QRService {
    return &QRService{}
}

// Generate erstellt einen QR-Code aus dem gegebenen Text
func (s *QRService) Generate(text string, size int) ([]byte, error) {
    // QR-Code generieren
    qr, err := qrcode.New(text, qrcode.Medium)
    if err != nil {
        return nil, err
    }

    // In PNG konvertieren
    png, err := qr.PNG(size)
    if err != nil {
        return nil, err
    }

    return png, nil
}

func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // Extrahieren Sie den Text-Parameter aus der Anfrage
    text := r.URL.Query().Get("text")
    if text == "" {
        http.Error(w, "Fehlender 'text'-Parameter", http.StatusBadRequest)
        return
    }
    // Extrahieren Sie den Size-Parameter aus der Anfrage
    sizeText := r.URL.Query().Get("size")
    if sizeText == "" {
        sizeText = "256"
    }
    size, err := strconv.Atoi(sizeText)
    if err != nil {
        http.Error(w, "Ungültiger 'size'-Parameter", http.StatusBadRequest)
        return
    }

    // Generieren Sie den QR-Code
    qrCodeData, err := s.Generate(text, size)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // Schreiben Sie die QR-Code-Daten in die Antwort
    w.Header().Set("Content-Type", "image/png")
    w.Write(qrCodeData)
}
```

**Was hier passiert:**

- `ServeHTTP` ist Go's Standard-Schnittstelle zum Behandeln von HTTP-Anfragen
- Wir parsen Query-Parameter aus der URL (`?text=hello&size=256`)
- Wir rufen unsere bestehende `Generate()`-Methode auf, um den QR-Code zu erstellen
- Wir setzen den Content-Type auf `image/png`, damit Browser wissen, dass es sich um ein Bild handelt
- Wir schreiben die rohen PNG-Bytes direkt in die Antwort – kein Base64 erforderlich!

Aktualisieren Sie nun `main.go`, um den Pfad anzugeben, unter dem der QR-Code-Dienst verfügbar sein soll:

```go title="main.go" ins={8-10}
func main() {
    app := application.New(application.Options{
        Name:        "myproject",
        Description: "A demo of using raw HTML & CSS",
        LogLevel:    slog.LevelDebug,
        Services: []application.Service{
            application.NewService(NewQRService(), application.ServiceOptions{
                Route: "/qrservice",
            }),
        },
        Assets: application.AssetOptions{
            Handler: application.AssetFileServerFS(assets),
        },
        Mac: application.MacOptions{
            ApplicationShouldTerminateAfterLastWindowClosed: true,
        },
    })

    app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:  "myproject",
        Width:  600,
        Height: 400,
    })

    // Run the application. This blocks until the application has been exited.
    err := app.Run()

    // If an error occurred while running the application, log it and exit.
    if err != nil {
        log.Fatal(err)
    }
}
```

**Was hier passiert:**

- Wir fügen `application.ServiceOptions` hinzu, um zu konfigurieren, wie der Service exponiert wird.
- `Route: "/qrservice"` macht den HTTP-Handler unter `/qrservice` zugänglich.
- Jede Anfrage an `/qrservice?text=hello` ruft nun unsere `ServeHTTP`-Methode auf.
- Ohne Festlegen von `Route` ist die HTTP-Handler-Funktionalität deaktiviert.

:::note
Wenn Sie die Option `Route` nicht explizit festlegen,
ist der HTTP-Handler nicht vom Frontend aus zugänglich.
:::

Aktualisieren Sie abschließend `main.js`, um ein einfaches Bild-`src` anstelle der Base64-Codierung zu verwenden:

```js title="frontend/src/main.js"
async function generateQR() {
    const text = document.getElementById('text').value;
    if (!text) {
        alert('Please enter some text');
        return;
    }

    const img = document.getElementById('qrcode');
    // Make the image source the path to the QR code service, passing the text
    img.src = `/qrservice?text=${encodeURIComponent(text)}`
}

export function initializeQRGenerator() {
    const button = document.getElementById('generateButton');
    if (button) {
        button.addEventListener('click', generateQR);
    } else {
        console.error('Generate button not found');
    }
}
```

**Was hier passiert:**

- Wir haben den Import und den Aufruf `await QRService.Generate()` entfernt.
- Stattdessen setzen wir `img.src` einfach auf unseren HTTP-Endpunkt.
- `encodeURIComponent()` maskiert Sonderzeichen in der URL sicher.
- Der Browser führt automatisch eine HTTP-GET-Anfrage aus, wenn wir `src` setzen.
- Dies ist einfacher und effizienter für Bilder – keine Base64-Konvertierung nötig!

8. ## Unterstützung dynamischer Konfigurationen

**Das Problem mit hartkodierten Routen:**

Im obigen Beispiel haben wir eine hartkodierte Route `/qrservice` in unserem JavaScript-Code verwendet.
Dies erzeugt eine enge Kopplung zwischen Ihrer Go-Konfiguration und Ihrem Frontend-Code.

Wenn Sie `main.go` bearbeiten und die Option `Route` ändern, ohne `main.js` zu aktualisieren,
wird die Anwendung fehlschlagen:

```go title="main.go" ins={3}
        // ...
            application.NewService(NewQRService(), application.ServiceOptions{
                Route: "/services/qr",
            }),
        // ...
```

Hartkodierte Routen funktionieren für einfache Anwendungen, machen Ihren Code jedoch spröde und schwerer zu warten.

**Die Lösung: Dynamische Konfiguration**

Methodenbindungen und HTTP-Handler können zusammenarbeiten! Wir können Bindungen verwenden, um dem Frontend mitzuteilen, welche Route verwendet werden soll, wodurch die Konfiguration dynamisch wird und der hartkodierte Pfad entfällt.

So funktioniert es:
1. Die Lifecycle-Methode `ServiceStartup` wird beim Start Ihrer App ausgeführt.
2. Wir speichern die konfigurierte Route aus den Optionen.
3. Wir fügen eine Methode `URL()` hinzu, die das Frontend aufrufen kann, um die korrekte Route zu erhalten.
4. Jetzt fragt das Frontend den Go-Service nach seiner Route, anstatt zu raten.

Implementieren Sie zunächst die `ServiceStartup`-Schnittstelle und fügen Sie eine neue `URL`-Methode hinzu:

```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
package main

import (
    "context"
    "net/http"
    "net/url"
    "strconv"

    "github.com/skip2/go-qrcode"
    "github.com/wailsapp/wails/v3/pkg/application"
)

// QRService handles QR code generation
type QRService struct {
    route string
}

// NewQRService creates a new QR service
func NewQRService() *QRService {
    return &QRService{}
}

// ServiceStartup runs at application startup.
func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
    s.route = options.Route
    return nil
}

// Generate creates a QR code from the given text
func (s *QRService) Generate(text string, size int) ([]byte, error) {
    // Generate the QR code
    qr, err := qrcode.New(text, qrcode.Medium)
    if err != nil {
        return nil, err
    }

    // Convert to PNG
    png, err := qr.PNG(size)
    if err != nil {
        return nil, err
    }

    return png, nil
}

// URL returns an URL that may be used to fetch
// a QR code with the given text and size.
// It returns an error if the HTTP handler is not available.
func (s *QRService) URL(text string, size int) (string, error) {
    if s.route == "" {
        return "", errors.New("http handler unavailable")
    }

    return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
}

func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // Extract the text parameter from the request
    text := r.URL.Query().Get("text")
    if text == "" {
        http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
        return
    }
    // Extract Size parameter from the request
    sizeText := r.URL.Query().Get("size")
    if sizeText == "" {
        sizeText = "256"
    }
    size, err := strconv.Atoi(sizeText)
    if err != nil {
        http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
        return
    }

    // Generate the QR code
    qrCodeData, err := s.Generate(text, size)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // Write the QR code data to the response
    w.Header().Set("Content-Type", "image/png")
    w.Write(qrCodeData)
}
```

**Was hier passiert:**

- Wir haben ein Feld `route` hinzugefügt, um die konfigurierte Route aus `ServiceStartup` zu speichern.
- `ServiceStartup(ctx, options)` wird beim Start der App aufgerufen – hier speichern wir die Route.
- Die Methode `URL()` baut die vollständige URL mit Query-Parametern auf.
- Wenn keine Route konfiguriert ist (route ist leer), geben wir einen Fehler zurück.
- `url.QueryEscape()` kodiert den Text sicher für die Verwendung in einer URL.
- Diese Methode ist über Bindungen für das Frontend verfügbar.

Aktualisieren Sie nun `main.js`, um die Methode `URL` anstelle eines hartkodierten Pfads zu verwenden:

```js title="frontend/src/main.js" ins={1,11-12}

    const img = document.getElementById('qrcode');

```

**Was hier passiert:**

- Wir importieren `QRService`, um die Bindings erneut zu verwenden
- Anstatt `/qrservice` hart zu codieren, rufen wir `await QRService.URL(text, 256)` auf
- Der Go-Service erstellt die URL mit der korrekten Route und den Parametern
- Wenn Sie nun die Route in `main.go` ändern, verwendet das Frontend automatisch die neue Route
- Keine manuelle Synchronisation zwischen Go-Konfiguration und Frontend-Code mehr!

Es sollte genau wie im vorherigen Beispiel funktionieren,
aber das Ändern der Service-Route in `main.go`
bricht das Frontend nicht mehr.
