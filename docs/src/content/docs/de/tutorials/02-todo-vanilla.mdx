1. **Projekt generieren**

   Erstellen Sie zunächst ein neues Wails-Projekt. Wir verwenden die Standard-Vanilla-Vorlage, die uns einen sauberen Ausgangspunkt bietet:


2. **Erstellen Sie den TODO-Service**

   Der TODO-Service verwaltet den Zustand unserer Anwendung und stellt Methoden für CRUD-Operationen bereit. Im Gegensatz zu einem Webserver, bei dem jede Anfrage isoliert ist, können Desktop-Apps mehrere gleichzeitige Operationen haben, daher benötigen wir eine thread-sichere Zustandsverwaltung.

   Löschen Sie `greetservice.go` und erstellen Sie eine neue Datei `todoservice.go`:

   ```go title="todoservice.go"
   package main

   import (
       "errors"
       "sync"
   )

   type Todo struct {
       ID        int    `json:"id"`
       Title     string `json:"title"`
       Completed bool   `json:"completed"`
   }

   type TodoService struct {
       todos  []Todo
       nextID int
       mu     sync.RWMutex
   }

   func NewTodoService() *TodoService {
       return &TodoService{
           todos:  []Todo{},
           nextID: 1,
       }
   }

   func (t *TodoService) GetAll() []Todo {
       t.mu.RLock()
       defer t.mu.RUnlock()
       return t.todos
   }

   func (t *TodoService) Add(title string) (*Todo, error) {
       if title == "" {
           return nil, errors.New("title cannot be empty")
       }

       t.mu.Lock()
       defer t.mu.Unlock()

       todo := Todo{
           ID:        t.nextID,
           Title:     title,
           Completed: false,
       }
       t.todos = append(t.todos, todo)
       t.nextID++

       return &todo, nil
   }

   func (t *TodoService) Toggle(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i := range t.todos {
           if t.todos[i].ID == id {
               t.todos[i].Completed = !t.todos[i].Completed
               return nil
           }
       }
       return errors.New("todo not found")
   }

   func (t *TodoService) Delete(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i, todo := range t.todos {
           if todo.ID == id {
               t.todos = append(t.todos[:i], t.todos[i+1:]...)
               return nil
           }
       }
       return errors.New("todo not found")
   }
   ```

   **Was hier passiert:**

   **Die `Todo`-Struktur:**
   - Definiert die Form unserer Daten mit den Feldern ID, Title und Completed
   - Die `json:`-Tags sagen Go, wie diese Struktur in JSON für das Frontend konvertiert werden soll
   - Jedes Feld ist exportiert (großgeschrieben), damit der Bindings-Generator es sehen kann

   **Die `TodoService`-Struktur:**
   - `todos []Todo` – ein Slice, das alle unsere TODO-Einträge enthält
   - `nextID int` – verfolgt die nächste zuzuweisende ID (simuliert Auto-Inkrement)
   - `mu sync.RWMutex` – ein Lese-/Schreib-Mutex für thread-sicheren Zugriff

   **Threadsicherheit mit `sync.RWMutex`:**
   - Desktop-Apps können mehrere gleichzeitige Operationen von der Benutzeroberfläche haben
   - `RLock()` ermöglicht mehrere Leser gleichzeitig (z. B. mehrere `GetAll`-Aufrufe)
   - `Lock()` gibt exklusiven Zugriff für Schreiboperationen (z. B. `Add`, `Toggle`, `Delete`)
   - `defer` stellt sicher, dass Sperren freigegeben werden, auch wenn die Funktion frühzeitig zurückkehrt oder abstürzt

   **Die Methoden:**
   - `GetAll()` – Gibt alle Todos zurück (verwendet Lesesperre, da wir die Daten nicht ändern)
   - `Add(title)` – Erstellt einen neuen Todo, validiert die Eingabe, inkrementiert die ID
   - `Toggle(id)` – Kehrt den abgeschlossenen-Status eines Todos um
   - `Delete(id)` – Entfernt einen Todo aus dem Slice

   **Fehlerbehandlung:**
   - Wir geben `error` als letzten Wert zurück, entsprechend den Go-Konventionen
   - Leere Titel werden abgelehnt
   - Operationen auf nicht existierende Todos geben Fehler zurück
   - Diese Fehler werden im Frontend zu JavaScript-Ausnahmen

3. **main.go aktualisieren**

   **Was hier passiert:**
   - Wir entfernen den Standard-GreetService und fügen stattdessen unseren TodoService hinzu

4. **Erstellen Sie die Frontend-Benutzeroberfläche**

   Jetzt erstellen wir das Frontend. Hier rufen wir unsere Go-Methoden auf und zeigen die Benutzeroberfläche an. Wir verwenden Vanilla JavaScript, um die Dinge einfach zu halten und Ihnen zu zeigen, wie die Bindings direkt funktionieren.

   Ersetzen Sie `frontend/src/main.js`:

   ```javascript title="frontend/src/main.js"
   import {TodoService} from "../bindings/changeme";

   async function loadTodos() {
       const todos = await TodoService.GetAll();
       const list = document.getElementById('todo-list');

       list.innerHTML = todos.map(todo => `
           <div class="todo ${todo.completed ? 'completed' : ''}">
               <input type="checkbox"
                      ${todo.completed ? 'checked' : ''}
                      onchange="toggleTodo(${todo.id})">
               <span>${todo.title}</span>
               <button onclick="deleteTodo(${todo.id})">Delete</button>
           </div>
       `).join('');
   }

   window.addTodo = async () => {
       const input = document.getElementById('todo-input');
       const title = input.value.trim();

       if (title) {
           await TodoService.Add(title);
           input.value = '';
           await loadTodos();
       }
   }

   window.toggleTodo = async (id) => {
       await TodoService.Toggle(id);
       await loadTodos();
   }

   window.deleteTodo = async (id) => {
       await TodoService.Delete(id);
       await loadTodos();
   }

   // Load todos on startup
   loadTodos();
   ```

   **Was hier passiert:**

   **Importieren der Bindings:**
   - `import {TodoService} from "../bindings/changeme"` – holt die automatisch generierten Go-Bindings herein
   - Hinweis: `changeme` wird Ihr tatsächlicher Modulname aus `go.mod` sein

   **Die Funktion `loadTodos()`:**
   - Ruft `TodoService.GetAll()` auf, um alle Todos von Go abzurufen
   - Erstellt HTML für jeden Todo unter Verwendung von Template-Literalen
   - Fügt dynamisch die Klasse `completed` hinzu/entfernt sie für das Styling
   - Verwendet `onclick`-Attribute, um Buttons mit unseren Funktionen zu verbinden
   - Verknüpft das gesamte HTML und injiziert es in das DOM

   **Die CRUD-Funktionen:**
   - `addTodo()` – Validiert die Eingabe, ruft die Go-Methode `Add` auf, aktualisiert die Liste
   - `toggleTodo(id)` – Ruft die Go-Methode `Toggle` auf, aktualisiert die Liste
   - `deleteTodo(id)` – Ruft die Go-Methode `Delete` auf, aktualisiert die Liste
   - Alle Funktionen sind asynchron, da Go-Aufrufe Promises zurückgeben**Warum an das Fenster anhängen:**
- `window.addTodo = ...` macht Funktionen über HTML `onclick`-Attribute zugänglich
- Dies ist ein einfaches Muster für Vanilla JS (Frameworks handhaben dies anders)
- In der Produktion könnten Sie stattdessen ein ordentliches Event Delegation verwenden

**Das Refresh-Muster:**
- Nach jeder Mutation (Hinzufügen/Aktivieren/Löschen) rufen wir `loadTodos()` erneut auf
- Dies stellt sicher, dass die UI mit dem Go-Zustand synchronisiert bleibt
- Alternative: Go-Methoden können den neuen Zustand zurückgeben, um den zweiten Aufruf zu vermeiden

## Funktionsweise

### Thread-sichere Zustandsverwaltung

Das `sync.RWMutex` bietet sicheren gleichzeitigen Zugriff:

```go
func (t *TodoService) GetAll() []Todo {
    t.mu.RLock()  // Lesesperre – mehrere Leser erlaubt
    defer t.mu.RUnlock()
    return t.todos
}

func (t *TodoService) Add(title string) (*Todo, error) {
    t.mu.Lock()  // Schreibsperre – exklusiver Zugriff
    defer t.mu.Unlock()
    // ... Mutationen
}
```

**Warum das wichtig ist:**
- Mehrere Frontend-Aufrufe können gleichzeitig erfolgen
- Lesevorgänge blockieren sich nicht gegenseitig
- Schreibvorgänge erhalten exklusiven Zugriff
- `defer` stellt sicher, dass Sperren immer freigegeben werden

### Fehlerbehandlung

Der Dienst gibt Fehler für ungültige Operationen zurück:

```go
func (t *TodoService) Add(title string) (*Todo, error) {
    if title == "" {
        return nil, errors.New("title cannot be empty")
    }
    // ...
}
```

Im Frontend können Sie diese abfangen:

```javascript
try {
    await TodoService.Add(title);
} catch (err) {
    alert('Error: ' + err);
}
```

### Zustands-Synchronisation

Nach jeder Mutation laden wir die vollständige Liste neu:

```javascript
window.addTodo = async () => {
    await TodoService.Add(title);  // Mutation
    await loadTodos();             // Refresh
}
```

**Alternative Vorgehensweise:** Geben Sie die aktualisierte Liste aus jeder Methode zurück, um den zweiten Aufruf zu vermeiden.

## Erweiterungen

### Statistiken hinzufügen

Fügen Sie dies zu `todoservice.go` hinzu:

```go
type TodoStats struct {
    Total     int `json:"total"`
    Completed int `json:"completed"`
    Active    int `json:"active"`
}

func (t *TodoService) GetStats() TodoStats {
    t.mu.RLock()
    defer t.mu.RUnlock()

    stats := TodoStats{
        Total: len(t.todos),
    }

    for _, todo := range t.todos {
        if todo.Completed {
            stats.Completed++
        } else {
            stats.Active++
        }
    }

    return stats
}
```

Im Frontend anzeigen:

```javascript
async function loadTodos() {
    const [todos, stats] = await Promise.all([
        TodoService.GetAll(),
        TodoService.GetStats()
    ]);

    // Statistiken anzeigen
    document.getElementById('stats').textContent =
        `${stats.active} active, ${stats.completed} completed`;

    // ... Todos rendern
}
```

### "Abgeschlossene löschen" hinzufügen

```go
func (t *TodoService) ClearCompleted() int {
    t.mu.Lock()
    defer t.mu.Unlock()

    removed := 0
    newTodos := []Todo{}

    for _, todo := range t.todos {
        if !todo.Completed {
            newTodos = append(newTodos, todo)
        } else {
            removed++
        }
    }

    t.todos = newTodos
    return removed
}
```
