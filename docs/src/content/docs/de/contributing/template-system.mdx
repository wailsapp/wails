Wails liefert ein **Templatesystem** mit, das `wails3 init` ermöglicht, ein startbereites
Projekt für **jeden** modernen Frontend-Stack (React, Svelte, Vue, Solid, Vanilla
JS …) zu erzeugen.

Diese Seite behandelt:

1. Verzeichnisstruktur der Vorlagen  
2. Wie die CLI Vorlagen auswählt und rendert  
3. Schritt-für-Schritt-Erstellung einer neuen Vorlage  
4. Aktualisieren oder Überschreiben bestehender Vorlagen  
5. Fehlerbehebung und Best Practices  

---

## 1. Wo Vorlagen gespeichert sind

```
v3/
└── internal/templates/
    ├── _common/           # Dateien, die in JEDEN Projekt kopiert werden (main.go, .gitignore)
    ├── base/              # Backend-nur „Plain Go“-Vorlage
    ├── react/             # React + Vite
    ├── react-ts/
    ├── svelte/
    ├── vue/
    ├── ... (andere)
    └── templates.go       # Registrierung + Hilfsfunktionen
```

* **`_common/`** – universeller Boilerplate (Go-Module, Taskfile, README-Stub,  
  VS Code-Einstellungen).  
* **Framework-Ordner** – enthalten **frontend/** und alle stack-spezifischen Konfigurationen.  
* Ordnernamen entsprechen der **Vorlagen-ID**, die Sie an die CLI übergeben  
  (`wails3 init -t react-ts`).

> Das gesamte Verzeichnis wird über `go:embed` in die CLI-Binärdatei kompiliert, sodass Benutzer
> Projekte auch offline erstellen können.

## 2. Wie `wails3 init` Vorlagen verwendet

Aufrufkette:

```
cmd/wails3/init.go
       │
       ▼
internal/commands/init.go  ← parst Flags, Zielverzeichnis
       │
       ▼
internal/templates.Load()  ← templates.go
       │
       ▼
Template.CopyTo(dest)      ← Dateisystem-Kopie + Textersetzung
```

### Ersetzungen

Platzhalter, die in `{{ }}` eingeschlossen sind, werden beim Kopieren ersetzt:

| Platzhalter | Beispiel | Quelle |
|-------------|---------|--------|
| `{{ProjectName}}` | `myapp` | CLI-Flag/Verzeichnisname |
| `{{ModulePath}}`  | `github.com/me/myapp` | wenn `--module` angegeben |
| `{{WailsVersion}}`| `v3.0.0` | kompilierte Konstante |

Sie können **beliebige** Platzhalter hinzufügen; stellen Sie einfach sicher, dass sie in
`internal/commands/init.go` einen Wert erhalten.

### Post-Copy-Hooks

Jede Vorlage kann eine **Taskfile** mit einer `deps`-Aufgabe mitbringen.
Nach dem Kopieren führt die CLI aus:

```
task --taskfile Taskfile.yml deps
```

Typische Arbeiten:

* `go mod tidy`
* `npm install`
* git init & erster Commit (optional)

Das Hook-Verhalten wird in `_common/Taskfile.yml` definiert, kann aber pro Vorlage überschrieben werden.

## 3. Erstellen einer neuen Vorlage

> Beispiel: **Qwik-TS**-Vorlage hinzufügen

### 3.1 Ordner & ID

```
internal/templates/qwik-ts/
```

Der Ordnername = Vorlagen-ID. Halten Sie sie im **kebab-case**.

### 3.2 Minimaler Satz an Dateien

```
qwik-ts/
├── README.md              # Wird in der CLI mit -list angezeigt
├── frontend/              # Ihr Webprojekt
│   ├── src/
│   ├── package.json
│   └── vite.config.ts
├── Taskfile.yml           # deps & dev-Aufgaben
└── go/                    # Optionale zusätzliche Go-Dateien
```

Beginnen Sie, indem Sie `react-ts` kopieren und Dateien entfernen.

### 3.3 Platzhalter aktualisieren

Suchen/Ersetzen:

* `myapp` → `{{ProjectName}}`
* `github.com/you/myapp` → `{{ModulePath}}`

### 3.4 Registrieren (optional)

Wenn der Ordner existiert, **erkennt templates.go ihn automatisch** über `embed`.
Fügen Sie nur dann expliziten Code hinzu, wenn Sie eine **benutzerdefinierte Validierung** benötigen:

```go
func (t *Template) Validate() error {
    // prüfe Node ≥ 20 für diese Vorlage
}
```

### 3.5 Testen

```bash
wails3 init -n demo -t qwik-ts
cd demo
wails3 dev
```

Stellen Sie sicher, dass:

* Der Dev-Server startet  
* `wailsjs`-Bindings generiert werden  
* Hot-Reload funktioniert

## 4. Ändern bestehender Vorlagen

1. Bearbeiten Sie Dateien unter `internal/templates/<id>/`.
2. Führen Sie `go generate ./v3/...` **oder** einfach `go run ./v3/cmd/wails3 –help` aus; die
   embed-Direktive kompiliert automatisch neu.
3. Erhöhen Sie **Abhängigkeitsversionen** in `frontend/package.json` & `Taskfile.yml`.
4. Aktualisieren Sie `README.md` innerhalb der Vorlage – Benutzer sehen sie über  
   `wails3 init -t react --help`.

### Häufige Anpassungen

| Aufgabe | Wo |
|------|-------|
| Dev-Server-Port ändern | `frontend/Taskfile.yml` (`vite dev --port {{DevPort}}`) |
| Umgebungsvariablen hinzufügen | dieselbe Taskfile oder `.env` |
| JS-Paketmanager ersetzen | Wechseln Sie `npm` → `pnpm` in der Taskfile |

## 5. Tipps zur Vorlagenerstellung

* **Halten Sie das Frontend generisch** – vermeiden Sie Verweise auf wails-spezifische Globals; die
  Laufzeit wird automatisch injiziert.
* **Keine kompilierten Artefakte** – schließen Sie `node_modules`, `dist`, `.DS_Store` über
  `.tmplignore` aus.
* **Dokumentieren Sie Voraussetzungen** – Node-Version, zusätzliche CLI-Tools usw.
* **Vermeiden Sie Breaking Changes** – erstellen Sie eine *neue* Vorlagen-ID, wenn die Überarbeitung groß ist.

## 6. Fehlerbehebung

| Symptom | Ursache | Lösung |
|---------|-------|-----|
| `unknown template id` | Ordnername stimmt nicht überein | Prüfen Sie `wails3 init -list` |
| Platzhalter nicht ersetzt | Fehlender Eintrag in `Data`-Map | Bearbeiten Sie `internal/commands/init.go` |
| Dev-Server öffnet leere Seite | Falsche `DevPort`-Umgebung | Stellen Sie sicher, dass die Taskfile den Port an stdout ausgibt |
| Frontend baut im Prod-Fehler | Vergessener vite `base`-Pfad | Setzen Sie `base: "./"` in `vite.config.ts` |

Fügen Sie `--verbose` zu `wails3 init` hinzu, um jede kopierte Datei und jede Ersetzung auszugeben.

## 7. Wichtige Quelldatei-Übersicht

| Datei | Verantwortung |
|------|----------------|
| `internal/templates/templates.go` | Einbetten des Template-Dateisystems, gibt `[]Template` zurück |
| `internal/templates/<id>/**` | Tatsächlicher Vorlageninhalt |
| `internal/commands/init.go` | CLI-Kleber: wählt Vorlage aus, füllt Platzhalter |
| `internal/commands/generate_template.go` | Dienstprogramm zum *Exportieren* eines live-Projekts zurück in eine Vorlage (nützlich für Updates) |

---

## 8. Zusammenfassung

* Vorlagen leben in **`internal/templates/`** und sind in die CLI eingebaut.
* `wails3 init -t <id>` kopiert `_common` + ausgewählte Vorlage, führt
  Ersetzungen durch und führt dann einen **deps**-Taskfile-Hook aus.
* Das Erstellen einer Vorlage ist so einfach wie **Erstellen eines Ordners**, Hinzufügen von Dateien und
  Verwenden von Platzhaltern.
* Das System ist **erweiterbar** und **selbstständig** – perfekt zum Teilen
  benutzerdefinierter Stacks mit Ihrem Team oder der Community.

Viel Spaß beim Templating!
