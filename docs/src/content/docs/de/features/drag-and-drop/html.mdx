HTML5 Drag & Drop ermöglicht es Nutzern, Elemente innerhalb der Benutzeroberfläche Ihrer App zu ziehen und abzulegen – beispielsweise zum Neuordnen einer Liste oder zum Verschieben von Elementen zwischen Spalten. Dies ist eine Standard-Webfunktionalität, die in Wails ohne spezielle Konfiguration funktioniert.

## Ein Element ziehbar machen

Standardmäßig können die meisten Elemente nicht gezogen werden. Um ein Element ziehbar zu machen, fügen Sie `draggable="true"` hinzu:

```html
<div class="item" draggable="true">Zieh mich</div>
```

Das Element zeigt nun eine Drag-Vorschau an, wenn der Nutzer es anklickt und zieht.

## Eine Drop-Zone definieren

Elemente akzeptieren standardmäßig keine Drops. Um ein Element Drop-Zone zu machen, müssen Sie das Standardverhalten bei `dragover` verhindern:

```html
<div class="drop-zone" id="target">Hier ablegen</div>

<script>
const target = document.getElementById('target');

target.addEventListener('dragover', (e) => {
    e.preventDefault(); // Drop erlauben
});

target.addEventListener('drop', (e) => {
    e.preventDefault();
    // Drop verarbeiten
});
</script>
```

Das Aufrufen von `preventDefault()` bei `dragover` ist erforderlich – es signalisiert, dass dieses Element Drops akzeptiert. Ohne diesen Schritt wird das Drop-Ereignis nicht ausgelöst.

## Drag-Hover stylen

Um Nutzern visuell zu zeigen, wo sie Elemente ablegen können, fügen Sie visuelles Feedback hinzu, wenn über eine Drop-Zone gezogen wird. Das Ereignis `dragenter` wird ausgelöst, wenn ein Element in die Zone eintritt, und `dragleave`, wenn es sie verlässt:

```css
.drop-zone {
    border: 2px dashed #ccc;
    padding: 40px;
    transition: all 0.2s ease;
}

.drop-zone.drag-over {
    border-color: #007bff;
    background-color: rgba(0, 123, 255, 0.1);
}
```

```javascript
const target = document.getElementById('target');

target.addEventListener('dragenter', () => {
    target.classList.add('drag-over');
});

target.addEventListener('dragleave', () => {
    target.classList.remove('drag-over');
});

target.addEventListener('drop', (e) => {
    e.preventDefault();
    target.classList.remove('drag-over');
    // Drop verarbeiten
});
```

Hinweis: `dragleave` wird auch ausgelöst, wenn man in ein untergeordnetes Element eintritt, was zu Flackern führen kann. Das vollständige Beispiel unten zeigt, wie man damit umgeht.

## Vollständiges Beispiel

Eine Aufgabenliste, bei der Elemente zwischen Prioritätsspalten gezogen werden können. Dabei wird das gezogene Element in einer Variablen gespeichert – der einfachste Ansatz, wenn sich alles auf derselben Seite befindet:

```html
<div class="tasks">
    <div class="item" draggable="true">Login-Bug beheben</div>
    <div class="item" draggable="true">Dokumentation aktualisieren</div>
    <div class="item" draggable="true">Dark Mode hinzufügen</div>
</div>

<div class="columns">
    <div class="drop-zone" data-priority="high">
        <h3>Hohe Priorität</h3>
        <ul></ul>
    </div>
    <div class="drop-zone" data-priority="low">
        <h3>Niedrige Priorität</h3>
        <ul></ul>
    </div>
</div>

<script>
let draggedItem = null;

// Verfolgen, welches Element gerade gezogen wird
document.querySelectorAll('.item').forEach(item => {
    item.addEventListener('dragstart', () => {
        draggedItem = item;
        item.classList.add('dragging');
    });
    
    item.addEventListener('dragend', () => {
        item.classList.remove('dragging');
    });
});

// Drops auf jeder Zone verarbeiten
document.querySelectorAll('.drop-zone').forEach(zone => {
    zone.addEventListener('dragover', (e) => {
        e.preventDefault();
    });
    
    zone.addEventListener('dragenter', () => {
        zone.classList.add('drag-over');
    });
    
    zone.addEventListener('dragleave', (e) => {
        // Die Klasse nur entfernen, wenn wir die Zone vollständig verlassen,
        // und nicht nur in ein untergeordnetes Element eintreten
        if (!zone.contains(e.relatedTarget)) {
            zone.classList.remove('drag-over');
        }
    });
    
    zone.addEventListener('drop', (e) => {
        e.preventDefault();
        zone.classList.remove('drag-over');
        
        if (draggedItem) {
            const li = document.createElement('li');
            li.textContent = draggedItem.textContent;
            zone.querySelector('ul').appendChild(li);
            draggedItem.remove();
        }
    });
});
</script>

<style>
.item {
    padding: 12px 16px;
    background: #f0f0f0;
    margin: 8px 0;
    border-radius: 8px;
    cursor: grab;
}

.item.dragging {
    opacity: 0.5;
}

.drop-zone {
    min-height: 150px;
    border: 2px dashed #ccc;
    border-radius: 8px;
    padding: 15px;
    transition: all 0.2s ease;
}

.drop-zone.drag-over {
    border-color: #007bff;
    background: rgba(0, 123, 255, 0.1);
}
</style>
```

## Kombination mit File Drop

Wenn Ihre App sowohl HTML Drag & Drop als auch [File Drop](./files) verwendet, erhalten Ihre HTML-Drop-Zonen auch Ereignisse, wenn Nutzer Dateien aus dem Betriebssystem in die App ziehen. Um Verwirrung zu vermeiden, filtern Sie Datei-Drag-Ereignisse in Ihren Handlern heraus:

```javascript
zone.addEventListener('dragenter', (e) => {
    // Externe Datei-Drag-Ereignisse ignorieren
    if (e.dataTransfer?.types.includes('Files')) return;
    
    zone.classList.add('drag-over');
});

zone.addEventListener('dragover', (e) => {
    // Externe Datei-Drag-Ereignisse ignorieren
    if (e.dataTransfer?.types.includes('Files')) return;
    
    e.preventDefault();
});

zone.addEventListener('drop', (e) => {
    // Externe Datei-Drag-Ereignisse ignorieren
    if (e.dataTransfer?.types.includes('Files')) return;
    
    e.preventDefault();
    zone.classList.remove('drag-over');
    // Internen Drop verarbeiten
});
```

Das Array `dataTransfer.types` enthält `'Files'`, wenn der Nutzer Dateien aus dem Betriebssystem zieht, aber Typen wie `'text/plain'` für interne HTML-Drag-Ereignisse. So können Sie zwischen beiden unterscheiden.

## Daten mit dataTransfer übergeben

Das obige Beispiel speichert das gezogene Element in einer JavaScript-Variablen. Das funktioniert gut, wenn sich alles auf derselben Seite befindet. Wenn Sie jedoch zwischen iframes ziehen oder Daten übergeben möchten, die nicht an ein DOM-Element gebunden sind, verwenden Sie die `dataTransfer`-API:

```javascript
// Beim Start des Ziehens Daten speichern
item.addEventListener('dragstart', (e) => {
    e.dataTransfer.setData('text/plain', item.id);
});

// Beim Ablegen Daten abrufen
target.addEventListener('drop', (e) => {
    e.preventDefault();
    const itemId = e.dataTransfer.getData('text/plain');
    const item = document.getElementById(itemId);
    // Element verschieben oder kopieren
});
```

Die Daten werden als Strings gespeichert. Wenn Sie Objekte übergeben möchten, müssen Sie diese ggf. mit `JSON.stringify()` serialisieren.
