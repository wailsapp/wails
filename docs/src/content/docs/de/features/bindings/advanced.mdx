Dieser Leitfaden behandelt fortgeschrittene Techniken zur Anpassung und Optimierung des Bindungsgenerierungsprozesses in Wails v3.

## Anpassen des generierten Codes mit Direktiven

### Einfügen von benutzerdefiniertem Code

Die Direktive `//wails:inject` ermöglicht es Ihnen, benutzerdefinierten JavaScript/TypeScript-Code in die generierten Bindungen einzufügen:

```go
//wails:inject console.log("Hello from Wails!");
type MyService struct {}

func (s *MyService) Greet(name string) string {
    return "Hello, " + name
}
```

Dies fügt den angegebenen Code in die generierte JavaScript/TypeScript-Datei für den Dienst `MyService` ein.

Sie können auch bedingtes Einfügen verwenden, um bestimmte Ausgabeformate zu targeten:

```go
//wails:inject j*:console.log("Hello JS!");  // Nur JavaScript
//wails:inject t*:console.log("Hello TS!");  // Nur TypeScript
```

### Einschließen zusätzlicher Dateien

Die Direktive `//wails:include` ermöglicht es Ihnen, zusätzliche Dateien mit den generierten Bindungen einzuschließen:

```go
//wails:include js/*.js
package mypackage
```

Diese Direktive wird typischerweise in Paketdokumentationskommentaren verwendet, um zusätzliche JavaScript/TypeScript-Dateien mit den generierten Bindungen einzuschließen.

### Markieren interner Typen und Methoden

Die Direktive `//wails:internal` markiert einen Typ oder eine Methode als intern und verhindert, dass er/sie an das Frontend exportiert wird:

```go
//wails:internal
type InternalModel struct {
    Field string
}

//wails:internal
func (s *MyService) InternalMethod() {}
```

Dies ist nützlich für Typen und Methoden, die nur intern von Ihrem Go-Code verwendet werden und nicht nach außen exposed werden sollten.

### Ignorieren von Methoden

Die Direktive `//wails:ignore` ignoriert eine Methode während der Bindungsgenerierung vollständig:

```go
//wails:ignore
func (s *MyService) IgnoredMethod() {}
```

Dies ist ähnlich wie `//wails:internal`, ignoriert die Methode jedoch vollständig, anstatt sie nur als intern zu markieren.

### Benutzerdefinierte Methoden-IDs

Die Direktive `//wails:id` gibt eine benutzerdefinierte ID für eine Methode an und überschreibt die standardmäßige hashbasierte ID:

```go
//wails:id 42
func (s *MyService) CustomIDMethod() {}
```

Dies kann nützlich sein, um die Kompatibilität beim Refaktorisieren des Codes aufrechtzuerhalten.

## Arbeiten mit komplexen Typen

### Verschachtelte Structs

Der Bindungsgenerator verarbeitet verschachtelte Structs automatisch:

```go
type Address struct {
    Street string
    City   string
    State  string
    Zip    string
}

type Person struct {
    Name    string
    Address Address
}

func (s *MyService) GetPerson() Person {
    return Person{
        Name: "John Doe",
        Address: Address{
            Street: "123 Main St",
            City:   "Anytown",
            State:  "CA",
            Zip:    "12345",
        },
    }
}
```

Der generierte JavaScript/TypeScript-Code enthält Klassen für sowohl `Person` als auch `Address`.

### Maps und Slices

Maps und Slices werden ebenfalls automatisch verarbeitet:

```go
type Person struct {
    Name       string
    Attributes map[string]string
    Friends    []string
}

func (s *MyService) GetPerson() Person {
    return Person{
        Name: "John Doe",
        Attributes: map[string]string{
            "hair": "brown",
            "eyes": "blue",
        },
        Friends: []string{"Jane", "Bob", "Alice"},
    }
}
```

In JavaScript werden Maps als Objekte und Slices als Arrays dargestellt. In TypeScript werden Maps als `Record<K, V>` und Slices als `T[]` dargestellt.

### Generische Typen

Der Bindungsgenerator unterstützt generische Typen:

```go
type Result[T any] struct {
    Data  T
    Error string
}

func (s *MyService) GetResult() Result[string] {
    return Result[string]{
        Data:  "Hello, World!",
        Error: "",
    }
}
```

Der generierte TypeScript-Code enthält eine generische Klasse für `Result`:

```typescript
export class Result<T> {
    "Data": T;
    "Error": string;

    constructor(source: Partial<Result<T>> = {}) {
        if (!("Data" in source)) {
            this["Data"] = null as any;
        }
        if (!("Error" in source)) {
            this["Error"] = "";
        }

        Object.assign(this, source);
    }

    static createFrom<T>(source: string | object = {}): Result<T> {
        let parsedSource = typeof source === "string" ? JSON.parse(source) : source;
        return new Result<T>(parsedSource as Partial<Result<T>>);
    }
}
```

### Interfaces

Der Bindungsgenerator kann TypeScript-Interfaces anstelle von Klassen generieren, indem die Flag `-i` verwendet wird:

```bash
wails3 generate bindings -ts -i
```

Dies generiert TypeScript-Interfaces für alle Modelle:

```typescript
export interface Person {
    Name: string;
    Attributes: Record<string, string>;
    Friends: string[];
}
```

## Optimieren der Bindungsgenerierung

### Verwendung von Namen statt IDs

Standardmäßig verwendet der Bindungsgenerator hashbasierte IDs für Methodenaufrufe. Sie können die Flag `-names` verwenden, um Namen zu verwenden:

```bash
wails3 generate bindings -names
```

Dies generiert Code, der Methodennamen anstelle von IDs verwendet:

```javascript
export function Greet(name) {
    let $resultPromise = $Call.ByName("Greet", name);
    return $resultPromise;
}
```

Dies kann den generierten Code lesbarer und einfacher zu debuggen machen, ist jedoch möglicherweise etwas weniger effizient.

### Bündeln der Laufzeit

Standardmäßig importiert der generierte Code die Wails-Laufzeit aus dem npm-Paket `@wailsio/runtime`. Sie können die Flag `-b` verwenden, um die Laufzeit mit dem generierten Code zu bündeln:

```bash
wails3 generate bindings -b
```

Dies schließt den Laufzeitcode direkt in die generierten Dateien ein und eliminiert die Notwendigkeit des npm-Pakets.

### Deaktivieren von Index-Dateien

Wenn Sie die Index-Dateien nicht benötigen, können Sie die Flag `-noindex` verwenden, um deren Generierung zu deaktivieren:

```bash
wails3 generate bindings -noindex
```

Dies kann nützlich sein, wenn Sie es vorziehen, Dienste und Modelle direkt aus ihren jeweiligen Dateien zu importieren.

## Praxisbeispiele

### Authentifizierungsdienst

Hier ist ein Beispiel für einen Authentifizierungsdienst mit benutzerdefinierten Direktiven:

```go
package auth

//wails:inject console.log("Auth service initialized");
type AuthService struct {
    // Private Felder
    users map[string]User
}

type User struct {
    Username string
    Email    string
    Role     string
}

type LoginRequest struct {
    Username string
    Password string
}

type LoginResponse struct {
    Success bool
    User    User
    Token   string
    Error   string
}

// Login authentifiziert einen Benutzer
func (s *AuthService) Login(req LoginRequest) LoginResponse {
    // Implementierung...
}

// GetCurrentUser gibt den aktuellen Benutzer zurück
func (s *AuthService) GetCurrentUser() User {
    // Implementierung...
}

// Interne Hilfsmethode
//wails:internal
func (s *AuthService) validateCredentials(username, password string) bool {
    // Implementierung...
}
```

### Datenverarbeitungsdienst

Hier ist ein Beispiel für einen Datenverarbeitungsdienst mit generischen Typen:

```go
package data

type ProcessingResult[T any] struct {
    Data  T
    Error string
}

type DataService struct {}

// Process verarbeitet Daten und gibt ein Ergebnis zurück
func (s *DataService) Process(data string) ProcessingResult[map[string]int] {
    // Implementierung...
}

// ProcessBatch verarbeitet mehrere Datenelemente
func (s *DataService) ProcessBatch(data []string) ProcessingResult[[]map[string]int] {
    // Implementierung...
}

// Interne Hilfsmethode
//wails:internal
func (s *DataService) parseData(data string) (map[string]int, error) {
    // Implementierung...
}
```

### Bedingtes Code-Injection

Hier ist ein Beispiel für bedingtes Code-Injection für verschiedene Ausgabeformate:

```go
//wails:inject j*:/**
//wails:inject j*: * @param {string} arg
//wails:inject j*: * @returns {Promise<void>}
//wails:inject j*: */
//wails:inject j*:export async function CustomMethod(arg) {
//wails:inject t*:export async function CustomMethod(arg: string): Promise<void> {
//wails:inject     await InternalMethod("Hello " + arg + "!");//wails:inject }
type Service struct{}
```

Dies injiziert unterschiedlichen Code für JavaScript- und TypeScript-Ausgaben und stellt für jede Sprache die entsprechenden Typannotationen bereit.
