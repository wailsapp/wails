## Best Practices für Bindings

Folgen Sie **bewährten Mustern** für das Binding-Design, um saubere, performante und sichere Bindings zu erstellen. Dieser Leitfaden behandelt Prinzipien der API-Gestaltung, Leistungsoptimierung, Sicherheitsmuster, Fehlerbehandlung und Teststrategien für wartbare Anwendungen.

## Prinzipien der API-Gestaltung

### 1. Einzelne Verantwortung

Jeder Dienst sollte einen klaren Zweck haben:

```go
// ❌ Schlecht: God-Objekt
type AppService struct {
    // Macht alles
}

func (a *AppService) SaveFile(path string, data []byte) error
func (a *AppService) GetUser(id int) (*User, error)
func (a *AppService) SendEmail(to, subject, body string) error
func (a *AppService) ProcessPayment(amount float64) error

// ✅ Gut: Fokussierte Dienste
type FileService struct{}
func (f *FileService) Save(path string, data []byte) error

type UserService struct{}
func (u *UserService) GetByID(id int) (*User, error)

type EmailService struct{}
func (e *EmailService) Send(to, subject, body string) error

type PaymentService struct{}
func (p *PaymentService) Process(amount float64) error
```

### 2. Klare Methodennamen

Verwenden Sie beschreibende, handlungsorientierte Namen:

```go
// ❌ Schlecht: Unklare Namen
func (s *Service) Do(x string) error
func (s *Service) Handle(data interface{}) interface{}
func (s *Service) Process(input map[string]interface{}) bool

// ✅ Gut: Klare Namen
func (s *FileService) SaveDocument(path string, content string) error
func (s *UserService) AuthenticateUser(email, password string) (*User, error)
func (s *OrderService) CreateOrder(items []Item) (*Order, error)
```

### 3. Konsistente Rückgabetypen

Geben Sie Fehler immer explizit zurück:

```go
// ❌ Schlecht: Inkonsistente Fehlerbehandlung
func (s *Service) GetData() interface{}  // Wie mit Fehlern umgehen?
func (s *Service) SaveData(data string)  // Stille Fehler?

// ✅ Gut: Explizite Fehler
func (s *Service) GetData() (Data, error)
func (s *Service) SaveData(data string) error
```

### 4. Eingabevalidierung

Validieren Sie alle Eingaben auf der Go-Seite:

```go
// ❌ Schlecht: Keine Validierung
func (s *UserService) CreateUser(email, password string) (*User, error) {
    user := &User{Email: email, Password: password}
    return s.db.Create(user)
}

// ✅ Gut: Zuerst validieren
func (s *UserService) CreateUser(email, password string) (*User, error) {
    // E-Mail validieren
    if !isValidEmail(email) {
        return nil, errors.New("ungültige E-Mail-Adresse")
    }
    
    // Passwort validieren
    if len(password) < 8 {
        return nil, errors.New("Passwort muss mindestens 8 Zeichen lang sein")
    }
    
    // Passwort hashen
    hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
    if err != nil {
        return nil, err
    }
    
    user := &User{
        Email:        email,
        PasswordHash: string(hash),
    }
    
    return s.db.Create(user)
}
```

## Leistungsmuster

### 1. Stapelverarbeitung

Reduzieren Sie Bridge-Aufrufe durch Stapelverarbeitung:

```go
// ❌ Schlecht: N Aufrufe
// JavaScript
for (const item of items) {
    await ProcessItem(item)  // N Bridge-Aufrufe
}

// ✅ Gut: 1 Aufruf
// Go
func (s *Service) ProcessItems(items []Item) ([]Result, error) {
    results := make([]Result, len(items))
    for i, item := range items {
        results[i] = s.processItem(item)
    }
    return results, nil
}

// JavaScript
const results = await ProcessItems(items)  // 1 Bridge-Aufruf
```

### 2. Paginierung

Geben Sie keine riesigen Datensätze zurück:

```go
// ❌ Schlecht: Gibt alles zurück
func (s *Service) GetAllUsers() ([]User, error) {
    return s.db.FindAll()  // Könnten Millionen sein
}

// ✅ Gut: Paginiert
type PageRequest struct {
    Page     int `json:"page"`
    PageSize int `json:"pageSize"`
}

type PageResponse struct {
    Items      []User `json:"items"`
    TotalItems int    `json:"totalItems"`
    TotalPages int    `json:"totalPages"`
    Page       int    `json:"page"`
}

func (s *Service) GetUsers(req PageRequest) (*PageResponse, error) {
    // Validieren
    if req.Page < 1 {
        req.Page = 1
    }
    if req.PageSize < 1 || req.PageSize > 100 {
        req.PageSize = 20
    }
    
    // Gesamtanzahl ermitteln
    total, err := s.db.Count()
    if err != nil {
        return nil, err
    }
    
    // Seite abrufen
    offset := (req.Page - 1) * req.PageSize
    users, err := s.db.Find(offset, req.PageSize)
    if err != nil {
        return nil, err
    }
    
    return &PageResponse{
        Items:      users,
        TotalItems: total,
        TotalPages: (total + req.PageSize - 1) / req.PageSize,
        Page:       req.Page,
    }, nil
}
```

### 3. Caching

Speichern Sie aufwändige Operationen im Cache:

```go
type CachedService struct {
    cache map[string]interface{}
    mu    sync.RWMutex
    ttl   time.Duration
}

func (s *CachedService) GetData(key string) (interface{}, error) {
    // Cache prüfen
    s.mu.RLock()
    if data, ok := s.cache[key]; ok {
        s.mu.RUnlock()
        return data, nil
    }
    s.mu.RUnlock()
    
    // Daten abrufen
    data, err := s.fetchData(key)
    if err != nil {
        return nil, err
    }
    
    // Im Cache speichern
    s.mu.Lock()
    s.cache[key] = data
    s.mu.Unlock()
    
    // Ablauf planen
    go func() {
        time.Sleep(s.ttl)
        s.mu.Lock()
        delete(s.cache, key)
        s.mu.Unlock()
    }()
    
    return data, nil
}
```

### 4. Streaming mit Events

Verwenden Sie Events für Daten-Streaming:

```go
// ❌ Schlecht: Polling
func (s *Service) GetProgress() int {
    return s.progress
}

// JavaScript pollt
setInterval(async () => {
    const progress = await GetProgress()
    updateUI(progress)
}, 100)

// ✅ Gut: Events
func (s *Service) ProcessLargeFile(path string) error {
    file, err := os.Open(path)
    if err != nil {
        return err
    }
    defer file.Close()
    
    scanner := bufio.NewScanner(file)
    total := 0
    processed := 0
    
    // Zeilen zählen
    for scanner.Scan() {
        total++
    }
    
    // Verarbeiten
    file.Seek(0, 0)
    scanner = bufio.NewScanner(file)
    
    for scanner.Scan() {
        s.processLine(scanner.Text())
        processed++
        
        // Fortschritt senden
        s.app.Event.Emit("progress", map[string]interface{}{
            "processed": processed,
            "total":     total,
            "percent":   int(float64(processed) / float64(total) * 100),
        })
    }
    
    return scanner.Err()
}

// JavaScript hört zu
OnEvent("progress", (data) => {
    updateProgress(data.percent)
})
```

## Sicherheitsmuster

### 1. Eingabesäuberung

Säubern Sie immer Benutzereingaben:

```go
import (
    "html"
    "strings"
)

func (s *Service) SaveComment(text string) error {
    // Säubern
    text = strings.TrimSpace(text)
    text = html.EscapeString(text)
    
    // Länge validieren
    if len(text) == 0 {
        return errors.New("Kommentar darf nicht leer sein")
    }
    if len(text) > 1000 {
        return errors.New("Kommentar zu lang")
    }
    
    return s.db.SaveComment(text)
}
```
