## Rahmlose Fenster

Wails bietet **Unterstützung für rahmlose Fenster** mit CSS-basierten Ziehbereichen und plattformnativem Verhalten. Entfernen Sie die plattformnative Titelleiste, um vollständige Kontrolle über die Fensterdekoration, benutzerdefinierte Designs und einzigartige Benutzererfahrungen zu erhalten, während wesentliche Funktionen wie Ziehen, Ändern der Größe und Systemsteuerungen erhalten bleiben.

## Schnellstart

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:     "Frameless App",
    Width:     800,
    Height:    600,
    Frameless: true,
})
```

**CSS für ziehbare Titelleiste:**

```css
.titlebar {
    --wails-draggable: drag;
    height: 40px;
    background: #333;
}

.titlebar button {
    --wails-draggable: no-drag;
}
```

**HTML:**

```html
<div class="titlebar">
    <span>My Application</span>
    <button onclick="window.close()">×</button>
</div>
```

**Das war's!** Sie haben eine benutzerdefinierte Titelleiste.

## Erstellen von rahmlosen Fenstern

### Einfaches rahmloses Fenster

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Frameless: true,
    Width:     800,
    Height:    600,
})
```

**Was Sie erhalten:**
- Keine Titelleiste
- Keine Fensterränder
- Keine Systembuttons
- Transparenter Hintergrund (optional)

**Was Sie implementieren müssen:**
- Ziehbereich
- Buttons zum Schließen/Minimieren/Maximieren
- Größengriff (falls veränderbar)

### Mit transparentem Hintergrund

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Frameless:      true,
    BackgroundType: application.BackgroundTypeTransparent,
})
```

**Anwendungsfälle:**
- Abgerundete Ecken
- Benutzerdefinierte Formen
- Overlay-Fenster
- Startbildschirme

## Ziehbereiche

### CSS-basiertes Ziehen

Verwenden Sie die CSS-Eigenschaft `--wails-draggable`:

```css
/* Ziehbereich */
.titlebar {
    --wails-draggable: drag;
}

/* Nicht-ziehbare Elemente innerhalb des ziehbaren Bereichs */
.titlebar button {
    --wails-draggable: no-drag;
}
```

**Werte:**
- `drag` - Bereich ist ziehbar
- `no-drag` - Bereich ist nicht ziehbar (auch wenn das übergeordnete Element ziehbar ist)

### Vollständiges Beispiel für eine Titelleiste

```html
<div class="titlebar">
    <div class="title">My Application</div>
    <div class="controls">
        <button class="minimize">−</button>
        <button class="maximize">□</button>
        <button class="close">×</button>
    </div>
</div>
```



## Systembuttons

### Implementieren von Schließen/Minimieren/Maximieren

**Go-Seite:**

```go
type WindowControls struct {
    window *application.WebviewWindow
}

func (wc *WindowControls) Minimise() {
    wc.window.Minimise()
}

func (wc *WindowControls) Maximise() {
    if wc.window.IsMaximised() {
        wc.window.UnMaximise()
    } else {
        wc.window.Maximise()
    }
}

func (wc *WindowControls) Close() {
    wc.window.Close()
}
```

**JavaScript-Seite:**

```javascript
import { Minimise, Maximise, Close } from './bindings/WindowControls'

document.querySelector('.minimize').addEventListener('click', Minimise)
document.querySelector('.maximize').addEventListener('click', Maximise)
document.querySelector('.close').addEventListener('click', Close)
```

**Oder verwenden Sie Runtime-Methoden:**

```javascript
import { Window } from '@wailsio/runtime'

document.querySelector('.minimize').addEventListener('click', () => Window.Minimise())
document.querySelector('.maximize').addEventListener('click', () => Window.Maximise())
document.querySelector('.close').addEventListener('click', () => Window.Close())
```

### Maximierungsstatus umschalten

Verfolgen Sie den Maximierungsstatus für das Button-Symbol:

```javascript
import { Window } from '@wailsio/runtime'

async function toggleMaximise() {
    const isMaximised = await Window.IsMaximised()

    if (isMaximised) {
        await Window.Restore()
    } else {
        await Window.Maximise()
    }

    updateMaximiseButton()
}

async function updateMaximiseButton() {
    const isMaximised = await Window.IsMaximised()
    const button = document.querySelector('.maximize')
    button.textContent = isMaximised ? '❐' : '□'
}
```

## Größengriffe

### CSS-basiertes Ändern der Größe

Wails bietet automatische Größengriffe für rahmlose Fenster:

```css
/* Ändern der Größe an allen Kanten aktivieren */
body {
    --wails-resize: all;
}

/* Oder spezifische Kanten */
.resize-top {
    --wails-resize: top;
}

.resize-bottom {
    --wails-resize: bottom;
}

.resize-left {
    --wails-resize: left;
}

.resize-right {
    --wails-resize: right;
}

/* Ecken */
.resize-top-left {
    --wails-resize: top-left;
}

.resize-top-right {
    --wails-resize: top-right;
}

.resize-bottom-left {
    --wails-resize: bottom-left;
}

.resize-bottom-right {
    --wails-resize: bottom-right;
}
```

**Werte:**
- `all` - Ändern der Größe von allen Kanten
- `top`, `bottom`, `left`, `right` - Spezifische Kanten
- `top-left`, `top-right`, `bottom-left`, `bottom-right` - Ecken
- `none` - Keine Größenänderung

### Beispiel für einen Größengriff

```html
<div class="window">
    <div class="titlebar">...</div>
    <div class="content">...</div>
    <div class="resize-handle resize-bottom-right"></div>
</div>
```

```css
.resize-handle {
    position: absolute;
    width: 16px;
    height: 16px;
}

.resize-bottom-right {
    --wails-resize: bottom-right;
    bottom: 0;
    right: 0;
    cursor: nwse-resize;
}
```

    Windows erkennt Ziehbereiche automatisch aus CSS.

    ```go
            InvisibleTitleBarHeight:    40,
        },
    })
    ```

    - Native Vollbildunterstützung
    - Traffic-Light-Buttons (optional)

    ```go
    Mac: application.MacOptions{
        InvisibleTitleBarHeight: 40,
    },
    ```
  </TabItem>

    - Basis rahmlose Unterstützung

## Vollständiges Beispiel

Hier ist ein produktionsreifes rahmloses Fenster:

**Go:**

```go
package main

import (
    _ "embed"
    "github.com/wailsapp/wails/v3/pkg/application"
)

//go:embed frontend/dist
var assets embed.FS

func main() {
    app := application.New(application.Options{
        Name: "Frameless App",
    })

    window := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:     "Frameless Application",
        Width:     1000,
        Height:    700,
        MinWidth:  800,
        MinHeight: 600,
        Frameless: true,
        
        Assets: application.AssetOptions{
            Handler: application.AssetFileServerFS(assets),
        },
        
        Mac: application.MacOptions{
            TitleBarAppearsTransparent: true,
            InvisibleTitleBarHeight:    40,
        },
        
        Windows: application.WindowsOptions{
            DisableFramelessWindowDecorations: false,
        },
    })

    window.Center()
    window.Show()

    app.Run()
}
```

```javascript
import { Window } from '@wailsio/runtime'

// Minimise button
document.querySelector('.minimize').addEventListener('click', () => {
    Window.Minimise()
})

// Maximise/restore button
const maximiseBtn = document.querySelector('.maximize')
maximiseBtn.addEventListener('click', async () => {
    const isMaximised = await Window.IsMaximised()

    if (isMaximised) {
        await Window.Restore()
    } else {
        await Window.Maximise()
    }

    updateMaximiseButton()
})

// Close button
document.querySelector('.close').addEventListener('click', () => {
    Window.Close()
})

// Update maximise button icon

    maximiseBtn.textContent = isMaximised ? '❐' : '□'
    maximiseBtn.title = isMaximised ? 'Restore' : 'Maximise'
}

// Initial state
updateMaximiseButton()
```

## Best Practices

### ✅ Tun

- **Ziehbaren Bereich bereitstellen** - Benutzer müssen das Fenster verschieben können
- **System-Schaltflächen implementieren** - Schließen, minimieren, maximieren
- **Mindestgröße festlegen** - Unbenutzbare Layouts verhindern
- **Auf allen Plattformen testen** - Das Verhalten variiert
- **CSS für Ziehbereiche verwenden** - Flexibel und wartbar
- **Visuelles Feedback bereitstellen** - Hover-Zustände für Schaltflächen

### ❌ Nicht tun

- **Nicht die Resize-Handles vergessen** - Wenn das Fenster skalierbar ist
- **Nicht das gesamte Fenster ziehbar machen** - Verhindert Interaktion
- **Nicht `no-drag` für Schaltflächen vergessen** - Sie funktionieren sonst nicht---
