## Über diese Referenz

Dies ist die vollständige API-Referenz für Wails v3. Sie dokumentiert jeden öffentlichen Typ, jede Methode und jede Option, die im Framework verfügbar ist.

**Organisation:**

## API-Konventionen

<details>
<summary><strong>Go-API-Konventionen</strong> - Für Entwickler, die neu in Go sind</summary>

#### Namensgebung

- **Typen**: PascalCase (z. B. `WebviewWindow`)
- **Methoden**: PascalCase (z. B. `SetTitle()`)
- **Optionen**: PascalCase-Strukturen (z. B. `WindowOptions`)
- **Konstanten**: PascalCase (z. B. `WindowStartStateMaximised`)

#### Fehlerbehandlung

Die meisten Methoden, die fehlschlagen können, geben `error` als letzten Rückgabewert zurück:

```go
window, err := app.Window.New()
if err != nil {
    log.Fatal(err)
}
```

#### Kontext

Methoden, die blockieren können oder eine Abbruchmöglichkeit benötigen, akzeptieren `context.Context`:

```go
err := app.RunWithContext(ctx)
```

#### Optionen-Muster

Die Konfiguration verwendet Options-Strukturen:

```go
app := application.New(application.Options{
    Name: "My App",
    Description: "A demo application",
    Services: []application.Service{
        application.NewService(&MyService{}),
    },
})
```

</details>

### JavaScript-API-Konventionen

#### Namensgebung

- **Funktionen**: camelCase (z. B. `setTitle()`)
- **Konstanten**: SCREAMING_SNAKE_CASE (z. B. `WINDOW_EVENT_FOCUS`)

#### Standardmäßig asynchron

Alle Go-Methodenaufrufe geben Promises zurück:

```javascript
// Async/await (empfohlen)
const result = await MyService.DoSomething()

// Promise-Kette
MyService.DoSomething()
    .then(result => console.log(result))
    .catch(error => console.error(error))
```

#### Fehlerbehandlung

Go-Fehler werden zu JavaScript-Ausnahmen:

```javascript
try {
    await MyService.MightFail()
} catch (error) {
    console.error('Go error:', error)
}
```

#### Typsicherheit

TypeScript-Definitionen werden automatisch generiert:

```typescript
// Vollständig typisiert
import { Greet } from './bindings/GreetService'

const message: string = await Greet("World")
```

## Plattformunterschiede

Einige APIs verhalten sich auf verschiedenen Plattformen unterschiedlich:

| Feature | Windows | macOS | Linux |
|---------|---------|-------|-------|
| **Anwendungsmenü** | Fenster-Menüleiste | Globale Menüleiste | Fenster-Menüleiste |
| **Systemablaufleiste** | Benachrichtigungsbereich | Menüleiste | Systemablaufleiste |
| **Dock** | N/A | ✅ Verfügbar | N/A |
| **Dateidialoge** | Native | Native | Native (GTK) |
| **Transparenz** | ✅ Vollständig | ✅ Vollständig | ⚠️ Eingeschränkt |

Plattformspezifisches Verhalten wird in jedem API-Abschnitt dokumentiert.

## Versionierung

Wails v3 folgt der semantischen Versionierung:

- **Major** (v3.x.x): Brechende Änderungen
- **Minor** (v3.x.x): Neue Funktionen, abwärtskompatibel
- **Patch** (v3.x.x): Fehlerbehebungen, abwärtskompatibel

**Aktueller Status:** Alpha (API stabil, Verfeinerungen laufen)

## Deprecation-Richtlinie

Wenn APIs als veraltet markiert werden:

1. **In der Dokumentation markiert** mit Deprecation-Hinweis
2. **Alternative bereitgestellt** mit Migrationsanleitung
3. **Für 1 Major-Version beibehalten** vor der Entfernung
4. **Compiler-Warnungen** (wo möglich)

## API-Stabilität

### Stabile APIs ✅

Diese APIs sind stabil und sicher für den Produktionsgebrauch:

- Kernanwendungs-APIs
- Fensterverwaltung
- Menüsystem
- Ereignissystem
- Dateidialoge
- Dienstbindungen

### Instabile APIs ⚠️

Diese APIs können sich vor der endgültigen Veröffentlichung ändern:

- Einige erweiterte Fensteroptionen
- Plattformspezifische Funktionen
- Experimentelle Funktionen

Instabile APIs sind in der Dokumentation markiert.
