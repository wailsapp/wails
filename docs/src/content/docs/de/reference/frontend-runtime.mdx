Die Wails Frontend-Laufzeitumgebung ist die Standardbibliothek für Wails-Anwendungen. Sie bietet
eine Reihe von Funktionen, die in Ihren Anwendungen verwendet werden können, darunter:

- Fensterverwaltung
- Dialoge
- Browser-Integration
- Zwischenablage
- Menüs
- Systeminformationen
- Ereignisse
- Kontextmenüs
- Bildschirme
- WML (Wails Markup Language)

Die Laufzeitumgebung ist für die Integration zwischen Go und dem Frontend erforderlich. Es gibt 2
Wege, die Laufzeitumgebung zu integrieren:

- Verwendung des Pakets `@wailsio/runtime`
- Verwendung eines vorgefertigten Bundles

## Verwendung des npm-Pakets

Das Paket `@wailsio/runtime` ist ein JavaScript-Paket, das den Zugriff auf die
Wails-Laufzeitumgebung vom Frontend aus ermöglicht. Es wird von allen Standardvorlagen
verwendet und ist der empfohlene Weg, die Laufzeitumgebung in Ihre Anwendung zu integrieren.
Durch die Verwendung des Pakets `@wailsio/runtime` werden nur die Teile der Laufzeitumgebung eingebunden, die Sie tatsächlich nutzen.

Das Paket ist auf npm verfügbar und kann mit folgendem Befehl installiert werden:

```shell
npm install --save @wailsio/runtime
```

## Verwendung eines vorgefertigten Bundles

Einige Projekte verwenden keinen JavaScript-Bundler und bevorzugen möglicherweise eine
vorgefertigte gebündelte Version der Laufzeitumgebung. Diese Version kann lokal
mit dem folgenden Befehl generiert werden:

```shell
wails3 generate runtime
```

Der Befehl gibt eine Datei `runtime.js` (und `runtime.debug.js`) im aktuellen
Verzeichnis aus. Diese Datei ist ein ES-Modul, das von Ihren Anwendungsskripten
genau wie das npm-Paket importiert werden kann, aber die API wird auch im globalen window-Objekt exportiert,
sodass Sie es für einfachere Anwendungen wie folgt verwenden können:

```html
<html>
    <head>
        <script type="module" src="./runtime.js"></script>
        <script>
            window.onload = function () {
                wails.Window.SetTitle("A new window title");
            }
        </script>
    </head>
    <!--- ... -->
</html>
```

:::caution
Es ist wichtig, das Attribut `type="module"` im `<script>`-Tag, das die Laufzeitumgebung lädt,
einzufügen und abzuwarten, bis die Seite vollständig geladen ist, bevor Sie die API aufrufen,
da Skripte mit dem Attribut `type="module"` asynchron ausgeführt werden.
:::

## Initialisierung

Neben den API-Funktionen bietet die Laufzeitumgebung Unterstützung für Kontextmenüs und das Ziehen von Fenstern.
Diese Funktionen funktionieren nur wie erwartet, nachdem die Laufzeitumgebung initialisiert wurde.
Stellen Sie auch dann, wenn Sie die API nicht verwenden, sicher, dass Sie eine Side-Effect-Import-Anweisung
irgendwo in Ihrem Frontend-Code einfügen:

```javascript
import "@wailsio/runtime";
```

Ihr Bundler sollte das Vorhandensein von Side-Effects erkennen und den gesamten
erforderlichen Initialisierungscode in den Build einbeziehen.

:::note
Wenn Sie das vorgefertigte Bundle bevorzugen, reicht das Hinzufügen eines Script-Tags wie oben gezeigt aus.
:::

## Vite-Plugin für typisierte Ereignisse

Die Laufzeitumgebung enthält ein Vite-Plugin, das HMR (Hot Module Replacement)-Unterstützung für typisierte Ereignisse während der Entwicklung ermöglicht.

### Einrichtung

Fügen Sie das Plugin zu Ihrer `vite.config.ts` hinzu:

```typescript
import { defineConfig } from 'vite'
import wails from '@wailsio/runtime/plugins/vite'

export default defineConfig({
  plugins: [wails()],
})
```

### Vorteile

- **Automatisches Neuladen**: Ereignisbindungen werden automatisch neu generiert und neu geladen, wenn Sie `wails3 generate bindings` ausführen
- **Entwicklungsmodus**: Funktioniert nahtlos mit `wails3 dev` für sofortige Updates
- **Typsicherheit**: Vollständige TypeScript-Unterstützung mit Autovervollständigung und Typüberprüfung

### Verwendung mit Ereignisregistrierung

Registrieren Sie Ihre Ereignisse in Go:

```go
type UserData struct {
    ID   string
    Name string
}

func init() {
    application.RegisterEvent[UserData]("user-updated")
}
```

Generieren Sie Bindings:

```bash
wails3 generate bindings

# Oder, um TypeScript-Definitionen einzuschließen
wails3 generate bindings -ts

# Weitere Optionen finden Sie unter:
wails3 generate bindings -help
```

Verwenden Sie typisierte Ereignisse in Ihrem Frontend:

```typescript
import { Events } from '@wailsio/runtime'
import { UserUpdated } from './bindings/events'

// Typsicheres Ereignis mit Autovervollständigung
Events.Emit(UserUpdated({
    ID: "123",
    Name: "John Doe"
}))
```

## API-Referenz

Die Laufzeitumgebung ist in Module unterteilt, die jeweils spezifische Funktionen bereitstellen. Importieren Sie nur das, was Sie benötigen:

```javascript
import { Events, Window, Clipboard } from '@wailsio/runtime'
```

### Ereignisse

Ereignissystem zur Kommunikation zwischen Go und JavaScript.

#### On()

Registriert einen Callback für ein Ereignis.

```typescript
function On(eventName: string, callback: (event: WailsEvent) => void): () => void
function On<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**Rückgabe:** Unsubscribe-Funktion

**Beispiel:**

```javascript
import { Events } from '@wailsio/runtime'

// Basis-Ereignisabfrage
const unsubscribe = Events.On('user-logged-in', (event) => {
    console.log('Benutzer:', event.data.username)
})

// Mit typisierten Ereignissen (TypeScript)
import { UserLogin } from './bindings/events'

Events.On(UserLogin, (event) => {
    // event.data ist vom Typ UserLoginData
    console.log('Benutzer:', event.data.username)
})

// Später: unsubscribe()
```

#### Once()

Registriert einen Callback, der nur einmal ausgeführt wird.

```typescript
function Once(eventName: string, callback: (event: WailsEvent) => void): () => void
function Once<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**Beispiel:**

```javascript
import { Events } from '@wailsio/runtime'

Events.Once('app-ready', () => {
    console.log('App initialisiert')
})
```

#### Emit()

Sendet ein Ereignis an das Go-Backend oder andere Fenster.

```typescript
function Emit(name: string, data?: any): Promise<boolean>
function Emit<T>(event: Event<T>): Promise<boolean>
```

**Rückgabe:** Promise, das zu `true` aufgelöst wird, wenn das Ereignis abgebrochen wurde, andernfalls zu `false`

**Beispiel:**

```javascript
import { Events } from '@wailsio/runtime'

// Basis-Ereignisübertragung
const wasCancelled = await Events.Emit('button-clicked', { buttonId: 'submit' })

// Mit typisierten Ereignissen (TypeScript)
import { UserLogin } from './bindings/events'

const cancelled = await Events.Emit(UserLogin({
    UserID: "123",
    Username: "john_doe",
    LoginTime: new Date().toISOString()
}))

if (cancelled) {
    console.log('Anmeldung wurde von einem Hook abgebrochen')
}
```

:::note
Der Rückgabewert gibt an, ob das Ereignis von einem Hook abgebrochen wurde. Die meisten Ereignisse können nicht abgebrochen werden und geben immer `false` zurück.
:::

**Beispiel:**


### System

Systemnahe Methoden für die direkte Kommunikation mit dem Backend.

#### invoke()

Sendet eine rohe Nachricht direkt an das Backend. Dies umgeht das Standard-Bindungssystem und wird vom `RawMessageHandler` in Ihren Anwendungsoptionen verarbeitet.

```typescript
function invoke(message: any): void
```

**Beispiel:**

```javascript
import { System } from '@wailsio/runtime'

// Sendet eine rohe Nachricht an das Backend
System.invoke('my-custom-message')

// Sendet strukturierte Daten als JSON
System.invoke(JSON.stringify({ action: 'update', value: 42 }))
```

:::caution
Dies ist eine „Fire-and-Forget"-Funktion ohne Rückgabewert. Verwenden Sie Events, um Antworten vom Backend zu empfangen.
:::

Weitere Details finden Sie im [Leitfaden für Raw Messages](/de/guides/raw-messages).

### WML (Wails Markup Language)

WML bietet deklarative Attribute für häufige Aktionen. Fügen Sie Attribute zu HTML-Elementen hinzu:

#### Attribute

**wml-event** - Löst ein Event beim Klicken aus

```html
<button wml-event="save-clicked">Speichern</button>
```

**wml-window** - Ruft eine Fenstermethode auf

```html
<button wml-window="Close">Fenster schließen</button>
<button wml-window="Minimise">Minimieren</button>
```

**wml-target-window** - Gibt das Zielfenster für wml-window an

```html
<button wml-window="Show" wml-target-window="settings">
    Einstellungen anzeigen
</button>
```

**wml-openurl** - Öffnet eine URL im Browser

```html
<a href="#" wml-openurl="https://wails.io">Wails besuchen</a>
```

**wml-confirm** - Zeigt einen Bestätigungsdialog vor der Aktion an

```html
<button wml-window="Close" wml-confirm="Are you sure you want to close?">
    Schließen
</button>
```

**Beispiel:**

```html
<div>
    <button wml-event="save-clicked">Speichern</button>
    <button wml-window="Minimise">Minimieren</button>
    <button wml-window="Close" wml-confirm="Fenster schließen?">Schließen</button>
    <a href="#" wml-openurl="https://github.com/wailsapp/wails">GitHub</a>
</div>
```

## Vollständiges Beispiel

```javascript
import { Events, Window, Clipboard, Dialogs, Screens } from '@wailsio/runtime'

// Auf Events aus Go hören
Events.On('data-updated', (event) => {
    console.log('Daten:', event.data)
    updateUI(event.data)
})

// Fenster-Verwaltung
document.getElementById('center-btn').addEventListener('click', async () => {
    await Window.Center()
})

document.getElementById('fullscreen-btn').addEventListener('click', async () => {
    const isFullscreen = await Window.IsFullscreen()
    if (isFullscreen) {
        await Window.UnFullscreen()
    } else {
        await Window.Fullscreen()
    }
})

// Zwischenablage-Operationen
document.getElementById('copy-btn').addEventListener('click', async () => {
    await Clipboard.SetText('Kopiert von Wails!')
})

// Dialog mit Bestätigung
document.getElementById('delete-btn').addEventListener('click', async () => {
    const result = await Dialogs.Question({
        Title: 'Bestätigen',
        Message: 'Diesen Artikel löschen?',
        Buttons: [
            { Label: 'Löschen' },
            { Label: 'Abbrechen', IsDefault: true }
        ]
    })

    if (result === 'Löschen') {
        await Events.Emit('delete-item', { id: currentItemId })
    }
})

// Bildschirm-Informationen
const screens = await Screens.GetAll()
console.log(`Erkannt ${screens.length} Bildschirm(e)`)
screens.forEach(screen => {
    console.log(`- ${screen.Name}: ${screen.Size.Width}x${screen.Size.Height}`)
})
```

## Best Practices

### ✅ Tun

- **Selektiv importieren** - Importieren Sie nur, was Sie benötigen
- **Promises behandeln** - Alle Methoden geben Promises zurück
- **WML für einfache Aktionen verwenden** - Sauberer als JavaScript
- **Rückgabewerte prüfen** - Besonders bei Dialogen
- **Events abbestellen** - Aufräumen, wenn fertig

### ❌ Nicht tun

- **await nicht vergessen** - Die meisten Methoden sind asynchron
- **UI nicht blockieren** - Verwenden Sie async/await korrekt
- **Fehler nicht ignorieren** - Behandeln Sie immer Rejections

## TypeScript-Unterstützung

Das Runtime-Paket enthält vollständige TypeScript-Definitionen:

```typescript
import { Events, Window } from '@wailsio/runtime'

Events.On('custom-event', (event) => {
    // TypeScript kennt event.data, event.name, event.sender
    console.log(event.data)
})

// Alle Methoden sind vollständig typisiert
const size: { width: number, height: number } = await Window.Size()
```
