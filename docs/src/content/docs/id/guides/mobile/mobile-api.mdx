<Aside type="caution" title="Fitur Eksperimental">
Dukungan mobile bersifat eksperimental dan mungkin berubah di rilis mendatang.
</Aside>

Kemampuan mobile native diekspos dengan dua cara:

- **Manager per platform** â€” `application.IOS` (dalam file `//go:build ios`) dan
  `application.Android` (dalam file `//go:build android`). Gunakan ini untuk hal-hal yang bersifat
  platform-spesifik. Lihat referensi [iOS](/id/guides/mobile/ios) dan
  [Android](/id/guides/mobile/android) untuk permukaan per-platform selengkapnya.
- **`application.Mobile`** â€” satu manager terjaga build yang mencakup subset
  kemampuan yang berperilaku identik di kedua platform. Gunakan ini saat Anda
  ingin satu code path yang dikompilasi dan berjalan di mana saja.

## `application.Mobile`

`application.Mobile` mengirimkan ke `IOS` di iOS, ke `Android` di Android, dan ke
stub no-op di desktop. Karena tidak memiliki build constraint, Anda dapat memanggilnya dari
Go biasa yang agnostik platform â€” tidak perlu file `//go:build` Anda sendiri:

```go
// Bekerja di file mana pun, pada target mana pun.
// Di desktop ini mengembalikan "" (no-op); di perangkat mengembalikan path yang sebenarnya.
dbDir := application.Mobile.StoragePath()
if dbDir == "" {
    // Di luar perangkat, atau direktori tidak dapat dibuat â€” tangani sesuai.
    return
}
db, _ := sql.Open("sqlite", filepath.Join(dbDir, "app.db"))
```

`StoragePath()` mengembalikan path absolut ke direktori file privat aplikasi â€”
`getFilesDir()` di Android, direktori Application Support di iOS â€” tempat yang direkomendasikan
untuk database dan file persisten lainnya. Mengembalikan string kosong di desktop, dan di perangkat
jika direktori tidak tersedia (di iOS, jika tidak dapat dibuat), jadi periksa `""` sebelum menggunakannya.

<Aside type="note">
Di luar perangkat (build desktop) setiap method `Mobile` adalah no-op dan setiap query
mengembalikan nilai zero-nya (`""` untuk string). Inilah yang memungkinkan kode lintas platform
memanggil `application.Mobile.*` tanpa syarat. Saat Anda membutuhkan path yang sebenarnya
di desktop juga, bercabang berdasarkan platform dan gunakan `os.UserConfigDir()` atau serupa.
</Aside>

## Yang tetap bersifat platform-spesifik

Kemampuan yang berbeda bentuknya antara iOS dan Android **tidak** ada di
`Mobile`; panggil melalui `application.IOS` / `application.Android` dari file bertag build:

| Keperluan | iOS | Android |
|---|---|---|
| Kecerahan (set) | `IOS.SetBrightness(0.0-1.0)` | `Android.SetBrightness(0-100)` |
| Kecerahan / orientasi (get) | `IOS.GetBrightness()` / `IOS.GetOrientation()` | `Android.BrightnessJSON()` / `Android.OrientationJSON()` |
| Notifikasi lokal | `IOS.PostNotification(json)` | `Android.Notify(json)` |
| Penyimpanan aman (tulis) | `IOS.SecureSet(key, value)` | `Android.SecureSet(json)` |
| Eksekusi latar belakang | `IOS.BeginBackgroundTask` / `EndBackgroundTask` | `Android.StartForegroundService` / `StopForegroundService` |

<Aside type="tip">
Antarmuka `MobileManager` adalah kontrak di balik `application.Mobile`.
Karena kedua manager platform harus memenuhinya, setiap method yang tercantum di atas dijamin
memiliki tanda tangan yang identik di iOS dan Android â€” jika pernah berbeda, build platform gagal dikompilasi.
</Aside>
