Wails v3 adalah **framework desktop full-stack** yang terdiri dari runtime Go,
bridge JavaScript, tool-chain berbasis task, dan koleksi template yang
memungkinkan Anda mengirim aplikasi native yang didukung teknologi web modern.

Halaman ini menyajikan *gambaran besar* dalam empat diagram:

1. **Arsitektur Keseluruhan** – bagaimana setiap subsistem terhubung  
2. **Alur Runtime** – apa yang terjadi saat JS memanggil Go dan sebaliknya  
3. **Development vs Production** – dua mode asset server  
4. **Implementasi Platform** – di mana kode spesifik OS berada

## 4 · Pemisahan Runtime Spesifik Platform

**File Runtime Per-OS**

**[Placeholder Diagram Pemisahan Platform]**

Setiap fitur mengikuti pola ini:

1. **Interface umum** di `pkg/application`  
2. **Entry message processor** di `pkg/application/messageprocessor_*.go`  
3. **Implementasi per-OS** di `pkg/application/*_{darwin,linux,windows}.go` (mis.
   `webview_window_darwin.go`, `clipboard_linux.go`, `dialogs_windows.go`,
   `systemtray_*.go`, `mainthread_*.go`), dilindungi build tag. Linux additionally
   memiliki cgo bridge di `linux_cgo.go` / `linux_cgo_gtk4.{go,c,h}`.

`internal/runtime/` hanya membawa glue build-tag kecil `runtime{_darwin,_linux,_windows,_android,_dev,_prod}.go`
plus runtime JS embedded di bawah `internal/runtime/desktop/`.

`internal/capabilities/` ada untuk mendeklarasikan set capability per-platform, tetapi
tidak ada sentinel `ErrCapability` — feature gating dilakukan via
build tag biasa dan stub return spesifik platform (mis. `nil` atau error spesifik fitur).
