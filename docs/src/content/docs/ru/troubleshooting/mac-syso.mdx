---
title: Syso-файлы на macOS
description: Устранение ошибок сборки Syso-файлов на macOS
---

## Проблема

При попытке собрать приложение Wails на macOS сборка завершается ошибкой, похожей на следующую:

```
Error: Users/runner/go/pkg/mod/golang.org/toolchain@v0.0.1-go1.23.1.darwin-arm64/pkg/tool/darwin_arm64/link: running clang failed: exit status 1
ld: unknown file type in '/private/var/folders/ml/x_tvfgn50_s7p67dm1ypcqqm0000gn/T/go-link-774134794/000000.o'
clang: error: linker command failed with exit code 1 (use -v to see invocation)
```

## Почему это происходит?

При сборке для Windows Wails генерирует `.syso`-файлы для иконки приложения, иконки окна и иконки меню. Эти файлы необходимы для сборки приложения в Windows.
Эти `.syso`-файлы можно найти в корневой директории проекта, и они могут вызывать проблемы при сборке для macOS.

## Решение

Чтобы исправить эту проблему, вы можете удалить syso-файлы из корневой директории проекта или, если вы генерируете их самостоятельно, назвать их `wails_windows_<arch>.syso`, например `wails_windows_arm64.syso`.
