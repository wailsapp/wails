Мы приветствуем (и поощряем) вашу обратную связь! Пожалуйста, поищите существующие тикеты или
сообщения перед созданием новых. Вот различные способы предоставить обратную связь:

<Tabs>
  <TabItem label="Ошибки" icon="error">

    Если вы нашли ошибку, пожалуйста, сообщите нам, опубликовав сообщение в канале [v3 Alpha Feedback](https://discord.gg/bdj28QNHmT) в Discord.

    - Сообщение должно четко описывать, в чем заключается ошибка, и содержать простой воспроизводимый пример. Если в документации неясно, что *должно* происходить, пожалуйста, включите это в сообщение.
    - Сообщение должно иметь тег `Bug`.
    - Пожалуйста, включите вывод команды `wails doctor` в ваше сообщение.
    - Если ошибка заключается в поведении, которое не соответствует текущей документации, например, окно не изменяет размер должным образом, пожалуйста, сделайте следующее:
      - Обновите существующий пример в директории `v3/example` или создайте новый пример в папке `v3/examples`, который четко демонстрирует проблему.
      - Откройте [PR](https://github.com/wailsapp/wails/pulls) с заголовком `[v3 alpha test] <описание ошибки>`.
      - Пожалуйста, включите ссылку на PR в ваше сообщение.

    :::caution

    _Помните_, неожиданное поведение не обязательно является ошибкой — возможно, оно просто не делает то, что вы ожидаете. Для этого используйте тег `Suggestions`.

    :::

  </TabItem>

  <TabItem label="Исправления" icon="approve-check">

    Если у вас есть исправление для ошибки или обновление документации, пожалуйста, сделайте следующее:

    - Откройте pull request в [репозитории Wails](https://github.com/wailsapp/wails). Заголовок PR должен начинаться с `[v3 alpha]`.
    - Создайте сообщение в канале [v3 Alpha Feedback](https://discord.gg/bdj28QNHmT).
    - Сообщение должно иметь тег `PR`.
    - Пожалуйста, включите ссылку на PR в ваше сообщение.

  </TabItem>

  <TabItem label="Предложения" icon="puzzle" id="abc">

    Если у вас есть предложение, пожалуйста, сообщите нам, опубликовав сообщение в канале [v3 Alpha Feedback](https://discord.gg/bdj28QNHmT) в Discord:

    - Сообщение должно иметь тег `Suggestion`.

    Не стесняйтесь обращаться к нам в [Discord](https://discord.gg/bdj28QNHmT), если у вас есть какие-либо вопросы.

  </TabItem>

  <TabItem label="Голосование" icon="star">

    - Сообщения можно "проголосовать" с помощью эмодзи :thumbsup:. Пожалуйста, голосуйте за любые сообщения, которые являются для вас приоритетными.
    - Пожалуйста, *не* просто добавляйте комментарии вроде "+1" или "я тоже".
    - Не стесняйтесь комментировать, если есть что добавить к сообщению, например, "эта ошибка также затрагивает сборки для ARM" или "Другой вариант — это ....."

  </TabItem>

</Tabs>

Список известных проблем и работ в процессе можно найти
[здесь](https://github.com/orgs/wailsapp/projects/6).

## Вопросы, по которым мы ищем обратную связь

- API
  - Легко ли им пользоваться?
  - Делает ли он то, что вы ожидаете?
  - Чего-то не хватает?
  - Есть ли что-то, что следует удалить?
  - Он последователен между Go и JS?
- Система сборки
  - Легко ли ей пользоваться?
  - Можем ли мы ее улучшить?
- Примеры
  - Они понятны?
  - Охватывают ли они основы?
- Функции
  - Какие функции отсутствуют?
  - Какие функции не нужны?
- Документация
  - Что могло бы быть понятнее?
