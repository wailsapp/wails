2. **Создайте сервис TODO**

   Сервис TODO будет управлять состоянием нашего приложения и предоставлять методы для операций CRUD. В отличие от веб-сервера, где каждый запрос изолирован, десктопные приложения могут иметь несколько одновременных операций, поэтому нам нужно потокобезопасное управление состоянием.

   Удалите `greetservice.go` и создайте новый файл `todoservice.go`:

   ```go title="todoservice.go"
   package main

   import (
       "errors"
       "sync"
   )

   type Todo struct {
       ID        int    `json:"id"`
       Title     string `json:"title"`
       Completed bool   `json:"completed"`
   }

   type TodoService struct {
       todos  []Todo
       nextID int
       mu     sync.RWMutex
   }

   func NewTodoService() *TodoService {
       return &TodoService{
           todos:  []Todo{},
           nextID: 1,
       }
   }

   func (t *TodoService) GetAll() []Todo {
       t.mu.RLock()
       defer t.mu.RUnlock()
       return t.todos
   }

   func (t *TodoService) Add(title string) (*Todo, error) {
       if title == "" {
           return nil, errors.New("title cannot be empty")
       }

       t.mu.Lock()
       defer t.mu.Unlock()

       todo := Todo{
           ID:        t.nextID,
           Title:     title,
           Completed: false,
       }
       t.todos = append(t.todos, todo)
       t.nextID++

       return &todo, nil
   }

   func (t *TodoService) Toggle(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i := range t.todos {
           if t.todos[i].ID == id {
               t.todos[i].Completed = !t.todos[i].Completed
               return nil
           }
       }
       return errors.New("todo not found")
   }

   func (t *TodoService) Delete(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i, todo := range t.todos {
           if todo.ID == id {
               t.todos = append(t.todos[:i], t.todos[i+1:]...)
               return nil
           }
       }
       return errors.New("todo not found")
   }
   ```

   **Что здесь происходит:**

   **Структура `Todo`:**
   - Определяет форму наших данных с полями ID, Title и Completed
   - Теги `json:` говорят Go, как преобразовать эту структуру в JSON для фронтенда
   - Каждое поле экспортировано (с заглавной буквы), чтобы генератор биндингов мог его видеть

   **Структура `TodoService`:**
   - `todos []Todo` — срез, хранящий все наши элементы TODO
   - `nextID int` — отслеживает следующий ID для назначения (симулирует автоинкремент)
   - `mu sync.RWMutex` — мьютекс чтения/записи для потокобезопасного доступа

   **Потокобезопасность с помощью `sync.RWMutex`:**
   - Десктопные приложения могут иметь несколько одновременных операций от интерфейса
   - `RLock()` позволяет нескольким читателям одновременно (например, несколько вызовов `GetAll`)
   - `Lock()` даёт эксклюзивный доступ для записи (например, `Add`, `Toggle`, `Delete`)
   - `defer` гарантирует, что блоки будут сняты, даже если функция вернётся досрочно или вызовет панику

   **Методы:**
   - `GetAll()` — возвращает все задачи (использует блокировку чтения, так как мы не изменяем данные)
   - `Add(title)` — создаёт новую задачу, проверяет ввод, увеличивает ID
   - `Toggle(id)` — переключает статус выполнения задачи
   - `Delete(id)` — удаляет задачу из среза

   **Обработка ошибок:**
   - Мы возвращаем `error` последним значением в соответствии с соглашениями Go
   - Пустые заголовки отклоняются
   - Операции с несуществующими задачами возвращают ошибки
   - Эти ошибки становятся исключениями JavaScript на фронтенде

4. **Создайте пользовательский интерфейс фронтенда**

   Теперь давайте построим фронтенд. Здесь мы будем вызывать наши методы Go и отображать интерфейс. Мы используем чистый JavaScript, чтобы сохранить простоту и показать, как работают биндинги напрямую.

   Замените `frontend/src/main.js`:

   ```javascript title="frontend/src/main.js"
   import {TodoService} from "../bindings/changeme";

   async function loadTodos() {
       const todos = await TodoService.GetAll();
       const list = document.getElementById('todo-list');

       list.innerHTML = todos.map(todo => `
           <div class="todo ${todo.completed ? 'completed' : ''}">
               <input type="checkbox"
                      ${todo.completed ? 'checked' : ''}
                      onchange="toggleTodo(${todo.id})">
               <span>${todo.title}</span>
               <button onclick="deleteTodo(${todo.id})">Delete</button>
           </div>
       `).join('');
   }

   window.addTodo = async () => {
       const input = document.getElementById('todo-input');
       const title = input.value.trim();

       if (title) {
           await TodoService.Add(title);
           input.value = '';
           await loadTodos();
       }
   }

   window.toggleTodo = async (id) => {
       await TodoService.Toggle(id);
       await loadTodos();
   }

   window.deleteTodo = async (id) => {
       await TodoService.Delete(id);
       await loadTodos();
   }

   // Load todos on startup
   loadTodos();
   ```

   **Что здесь происходит:**

   **Импорт биндингов:**
   - `import {TodoService} from "../bindings/changeme"` — подгружает автоматически сгенерированные биндинги Go
   - Примечание: `changeme` будет вашим реальным именем модуля из `go.mod`

   **Функция `loadTodos()`:**
   - Вызывает `TodoService.GetAll()` для получения всех задач из Go
   - Строит HTML для каждой задачи с помощью шаблонных строк
   - Динамически добавляет/удаляет класс `completed` для стилизации
   - Использует атрибуты `onclick` для подключения кнопок к нашим функциям
   - Объединяет весь HTML и внедряет его в DOM

   **Функции CRUD:**
   - `addTodo()` — проверяет ввод, вызывает метод Go `Add`, обновляет список
   - `toggleTodo(id)` — вызывает метод Go `Toggle`, обновляет список
   - `deleteTodo(id)` — вызывает метод Go `Delete`, обновляет список
   - Все функции асинхронны, так как вызовы Go возвращают Promises**Зачем привязывать к окну:**
- `window.addTodo = ...` делает функции доступными из HTML-атрибутов `onclick`
- Это простой паттерн для vanilla JS (фреймворки обрабатывают это иначе)
- В продакшене вы, возможно, будете использовать правильное делегирование событий

**Паттерн обновления:**
- После каждой мутации (добавление/переключение/удаление) мы снова вызываем `loadTodos()`
- Это гарантирует, что UI остаётся синхронизированным с состоянием Go
- Альтернатива: пусть методы Go возвращают новое состояние, чтобы избежать второго вызова

## Как это работает

### Потокобезопасное управление состоянием

`sync.RWMutex` обеспечивает безопасный одновременный доступ:

```go
func (t *TodoService) GetAll() []Todo {
    t.mu.RLock()  // Блок чтения — допускается несколько читателей
    defer t.mu.RUnlock()
    return t.todos
}

func (t *TodoService) Add(title string) (*Todo, error) {
    t.mu.Lock()  // Блок записи — эксклюзивный доступ
    defer t.mu.Unlock()
    // ... мутации
}
```

**Почему это важно:**
- Несколько вызовов с фронтенда могут происходить одновременно
- Операции чтения не блокируют друг друга
- Операции записи получают эксклюзивный доступ
- `defer` гарантирует, что блоки всегда освобождаются

### Обработка ошибок

Сервис возвращает ошибки для недопустимых операций:

```go
func (t *TodoService) Add(title string) (*Todo, error) {
    if title == "" {
        return nil, errors.New("title cannot be empty")
    }
    // ...
}
```

На фронтенде вы можете перехватить их:

```javascript
try {
    await TodoService.Add(title);
} catch (err) {
    alert('Error: ' + err);
}
```

### Синхронизация состояния

После каждой мутации мы перезагружаем полный список:

```javascript
window.addTodo = async () => {
    await TodoService.Add(title);  // Мутация
    await loadTodos();             // Обновление
}
```

**Альтернативный подход:** Возвращайте обновленный список из каждого метода, чтобы избежать второго вызова.

## Улучшения

### Добавление статистики

Добавьте это в `todoservice.go`:

```go
type TodoStats struct {
    Total     int `json:"total"`
    Completed int `json:"completed"`
    Active    int `json:"active"`
}

func (t *TodoService) GetStats() TodoStats {
    t.mu.RLock()
    defer t.mu.RUnlock()

    stats := TodoStats{
        Total: len(t.todos),
    }

    for _, todo := range t.todos {
        if todo.Completed {
            stats.Completed++
        } else {
            stats.Active++
        }
    }

    return stats
}
```

Отобразите на фронтенде:

```javascript
async function loadTodos() {
    const [todos, stats] = await Promise.all([
        TodoService.GetAll(),
        TodoService.GetStats()

    document.getElementById('stats').textContent =
        `${stats.active} active, ${stats.completed} completed`;

    // ... рендеринг задач
}
```

### Добавление "Очистить выполненные"

```go
func (t *TodoService) ClearCompleted() int {
    t.mu.Lock()
    defer t.mu.Unlock()

    removed := 0
    newTodos := []Todo{}

    for _, todo := range t.todos {
        if !todo.Completed {
            newTodos = append(newTodos, todo)
        } else {
            removed++
        }
    }

    t.todos = newTodos
    return removed
}
```

### Добавление сохранения данных

Для производственных приложений обычно добавляют сохранение данных в базу. См. руководство [Интеграция с базами данных](/guides/patterns/database) для примеров с SQLite, PostgreSQL и т.д.

## Сборка для производства

Когда вы будете готовы распространять свое приложение TODO, соберите его для производства:

```bash
wails3 build
```

Это создает оптимизированный нативный исполняемый файл в `build/bin/`:
- Компилирует ваш код на Go с оптимизациями
- Собирает ваш фронтенд для производства
- Упаковывает все в один исполняемый файл
- Результирующее приложение обычно занимает 10-20 МБ (по сравнению с 150 МБ+ у Electron)

Вы можете запустить исполняемый файл напрямую — не требуется среда выполнения, не нужно запускать серверы. Это настоящее нативное приложение.
