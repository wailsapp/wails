title: Меню

---

Wails v3 предоставляет мощную систему меню, которая позволяет создавать как меню приложения, так и контекстные меню. Это руководство поможет вам разобраться в различных функциях и возможностях системы меню.

### Создание меню

Для создания нового меню используйте метод `New()` из менеджера Menus:

```go
menu := app.Menu.New()
```

### Добавление пунктов меню

Wails поддерживает несколько типов пунктов меню, каждый из которых выполняет свою задачу:

#### Обычные пункты меню
Обычные пункты меню являются базовыми элементами построения меню. Они отображают текст и могут выполнять действия при нажатии:

```go
menuItem := menu.Add("Click Me")
```

#### Флажки (Checkboxes)
Пункты меню с флажками обеспечивают переключаемое состояние, что полезно для включения/отключения функций или настроек:

```go
checkbox := menu.AddCheckbox("My checkbox", true)  // true = изначально отмечен
```

#### Группы радиокнопок
Группы радиокнопок позволяют пользователю выбрать один вариант из набора взаимоисключающих вариантов. Они автоматически создаются, когда радиопункты располагаются рядом друг с другом:

```go
menu.AddRadio("Option 1", true)   // true = изначально выбран
menu.AddRadio("Option 2", false)
menu.AddRadio("Option 3", false)
```

#### Разделители
Разделители — это горизонтальные линии, которые помогают группировать пункты меню в логические блоки:

```go
menu.AddSeparator()
```

#### Вложенные меню
Вложенные меню — это подменю, которые появляются при наведении курсора или нажатии на пункт меню. Они полезны для организации сложных структур меню:

```go
submenu := menu.AddSubmenu("File")
submenu.Add("Open")
submenu.Add("Save")
```

#### Объединение меню
Меню можно добавить в другое меню, приложив его в конец или в начало.
```go
menu := app.Menu.New()
menu.Add("First Menu")

secondaryMenu := app.Menu.New()
secondaryMenu.Add("Second Menu")

// вставить 'secondaryMenu' после 'menu'
menu.Append(secondaryMenu)

// вставить 'secondaryMenu' перед 'menu'
menu.Prepend(secondaryMenu)

// обновить меню
menu.Update()
```

:::note
По умолчанию методы `prepend` и `append` разделяют состояние с исходным меню. Если вы хотите создать новое меню с собственным состоянием,
вызовите метод `.Clone()` для меню.

Например: `menu.Append(secondaryMenu.Clone())`
:::

#### Очистка меню
В некоторых случаях лучше создать совершенно новое меню, если вы работаете с переменным количеством пунктов меню.

Это очистит все пункты в существующем меню и позволит добавить новые.

```go
menu := app.Menu.New()
menu.Add("Waiting for update...")

// после определённой логики меню должно быть обновлено
menu.Clear()
menu.Add("Update complete!")
menu.Update()
```

:::note
Очистка меню удаляет только пункты верхнего уровня. Хотя вложенные меню станут невидимыми, они всё ещё будут занимать память,
поэтому внимательно управляйте вашими меню.
:::

#### Удаление меню

Если вы хотите очистить и освободить меню, используйте метод `Destroy()`:

```go
menu := app.Menu.New()
menu.Add("Waiting for update...")

// после определённой логики меню должно быть удалено
menu.Destroy()
```


### Свойства пунктов меню

Пункты меню имеют несколько настраиваемых свойств:

| Свойство      | Метод                      | Описание                                              |
|---------------|----------------------------|-------------------------------------------------------|
| Label         | `SetLabel(string)`         | Устанавливает отображаемый текст                      |
| Enabled       | `SetEnabled(bool)`         | Включает/отключает пункт                              |
| Checked       | `SetChecked(bool)`         | Устанавливает состояние отметки (для флажков/радиокнопок) |
| Tooltip       | `SetTooltip(string)`       | Устанавливает текст подсказки                         |
| Hidden        | `SetHidden(bool)`          | Показывает/скрывает пункт                             |
| Accelerator   | `SetAccelerator(string)`   | Устанавливает сочетание клавиш                        |

### Состояния пунктов меню

Пункты меню могут находиться в разных состояниях, которые управляют их видимостью и интерактивностью:

#### Видимость

Пункты меню могут динамически показываться или скрываться с помощью метода `SetHidden()`:

```go
menuItem := menu.Add("Dynamic Item")

// Скрыть пункт меню
menuItem.SetHidden(true)

// Показать пункт меню
menuItem.SetHidden(false)

// Проверить текущую видимость
isHidden := menuItem.Hidden()
```

Скрытые пункты меню полностью удаляются из меню до момента их повторного отображения. Это полезно для контекстных пунктов, которые должны появляться только в определённых состояниях приложения.

#### Состояние доступности

Пункты меню могут быть включены или отключены с помощью метода `SetEnabled()`:

```go
menuItem := menu.Add("Save")

// Отключить пункт меню
menuItem.SetEnabled(false)  // Пункт становится серым и некликабельным

// Включить пункт меню
menuItem.SetEnabled(true)   // Пункт снова становится кликабельным

// Проверить текущее состояние доступности
isEnabled := menuItem.Enabled()
```

Отключенные пункты меню остаются видимыми, но становятся серыми и некликабельными. Это обычно используется для индикации того, что действие в данный момент недоступно, например:
- Отключение "Save" (Сохранить), когда нет изменений для сохранения
- Отключение "Copy" (Копировать), когда ничего не выделено
- Отключение "Undo" (Отменить), когда нечего отменять

#### Динамическое управление состоянием

Вы можете комбинировать эти состояния с обработчиками событий для создания динамических меню:

```go
saveMenuItem := menu.Add("Save")

// Изначально отключить пункт меню Save
saveMenuItem.SetEnabled(false)

// Включить Save только при наличии несохранённых изменений
documentChanged := func() {
    saveMenuItem.SetEnabled(true)
    menu.Update()  // Не забудьте обновить меню после изменения состояний
}

// Отключить Save после сохранения
documentSaved := func() {
    saveMenuItem.SetEnabled(false)
    menu.Update()
}
```

### Обработка событий

Пункты меню могут реагировать на события нажатия с помощью метода `OnClick`:

```go
menuItem.OnClick(func(ctx *application.Context) {
    // Обработка события нажатия
    println("Menu item clicked!")
})
```

Контекст предоставляет информацию о нажатом пункте меню:

```go
menuItem.OnClick(func(ctx *application.Context) {
    // Получить нажатый пункт меню
    clickedItem := ctx.ClickedMenuItem()
    // Получить его текущее состояние
    isChecked := clickedItem.Checked()
})
```

### Пункты меню на основе ролей

Wails предоставляет набор предопределённых ролей меню, которые автоматически создают пункты меню со стандартной функциональностью. Ниже приведены поддерживаемые роли меню:

#### Полные структуры меню
Эти роли создают целые структуры меню с общей функциональностью:

| Роль          | Описание                                                    | Примечания по платформам |
|---------------|-------------------------------------------------------------|--------------------------|
| `AppMenu`     | Меню приложения с пунктами About, Services, Hide/Show и Quit | Только macOS             |
| `EditMenu`    | Стандартное меню Edit с пунктами Undo, Redo, Cut, Copy, Paste и т.д. | Все платформы            |
| `ViewMenu`    | Меню View с элементами Reload, Zoom и управлением полноэкранным режимом | Все платформы            |
| `WindowMenu`  | Управление окнами (Minimise, Zoom и т.д.)                   | Все платформы            |
| `HelpMenu`    | Меню Help со ссылкой "Learn More" на сайт Wails             | Все платформы            |------|-------------|------|
| `About` | Показать диалог «О приложении» | Все платформы |
| `Hide` | Скрыть приложение | Только macOS |
| `HideOthers` | Скрыть другие приложения | Только macOS |
| `UnHide` | Показать скрытое приложение | Только macOS |
| `CloseWindow` | Закрыть текущее окно | Все платформы |
| `Minimise` | Свернуть окно | Все платформы |
| `Zoom` | Развернуть окно | Только macOS |
| `Front` | Вывести окно на передний план | Только macOS |
| `Quit` | Закрыть приложение | Все платформы |
| `Undo` | Отменить последнее действие | Все платформы |
| `Redo` | Повторить последнее действие | Все платформы |
| `Cut` | Вырезать выделение | Все платформы |
| `Copy` | Копировать выделение | Все платформы |
| `Paste` | Вставить из буфера обмена | Все платформы |
| `PasteAndMatchStyle` | Вставить с сохранением стиля | Только macOS |
| `SelectAll` | Выделить всё | Все платформы |
| `Delete` | Удалить выделение | Все платформы |
| `Reload` | Перезагрузить текущую страницу | Все платформы |
| `ForceReload` | Принудительно перезагрузить текущую страницу | Все платформы |
| `ToggleFullscreen` | Переключить полноэкранный режим | Все платформы |
| `ResetZoom` | Сбросить уровень масштабирования | Все платформы |
| `ZoomIn` | Увеличить масштаб | Все платформы |
| `ZoomOut` | Уменьшить масштаб | Все платформы |

Ниже приведён пример, демонстрирующий использование как полных меню, так и отдельных ролей:

```go
menu := app.Menu.New()

// Добавить полные структуры меню
menu.AddRole(application.AppMenu)    // Только macOS
menu.AddRole(application.EditMenu)   // Общие операции редактирования
menu.AddRole(application.ViewMenu)   // Управление видом
menu.AddRole(application.WindowMenu) // Управление окнами

// Добавить отдельные пункты на основе ролей в пользовательское меню
fileMenu := menu.AddSubmenu("File")
fileMenu.AddRole(application.CloseWindow)
fileMenu.AddSeparator()
fileMenu.AddRole(application.Quit)
```

## Меню приложения

Меню приложения — это меню, которые появляются в верхней части окна вашего приложения (Windows/Linux) или в верхней части экрана (macOS).

### Поведение меню приложения

Когда вы устанавливаете меню приложения с помощью `app.Menu.Set()`, оно становится главным меню на macOS.
На Windows/Linux меню устанавливаются для каждого окна отдельно.

```go
app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title: "Custom Menu Window",
    Windows: application.WindowsWindow{
        Menu: customMenu,  // Переопределить меню приложения для этого окна
    },
})
```

Ниже приведён полный пример, демонстрирующий это различное поведение меню:

```go
func main() {
    app := application.New(application.Options{})

    // Создать меню приложения
    appMenu := app.Menu.New()
    fileMenu := appMenu.AddSubmenu("File")
    fileMenu.Add("New").OnClick(func(ctx *application.Context) {
        // Это будет доступно во всех окнах, если не переопределено
        window := app.Windows.Current()
        window.SetTitle("New Window")
    })
    
    // Установить как меню приложения — это для macOS
    app.Menu.Set(appMenu)

    // Окно с пользовательским меню на Windows
    customMenu := app.Menu.New()
    customMenu.Add("Custom Action")
    app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title: "Custom Menu",
        Windows: application.WindowsWindow{
            Menu: customMenu,
        },
    })

    app.Run()
}
```

## Контекстные меню

Контекстные меню — это всплывающие меню, которые появляются при щелчке правой кнопкой мыши по элементам вашего приложения. Они обеспечивают быстрый доступ к соответствующим действиям для выбранного элемента.

### Контекстное меню по умолчанию

Контекстное меню по умолчанию — это встроенное контекстное меню WebView, которое предоставляет системные операции, такие как:
- Копирование, вырезание и вставка для работы с текстом
- Управление выделением текста
- Опции проверки орфографии

#### Управление контекстным меню по умолчанию

Вы можете управлять появлением контекстного меню по умолчанию с помощью CSS-свойства `--default-contextmenu`:

```html
<!-- Всегда показывать контекстное меню по умолчанию -->
<div style="--default-contextmenu: show">
    <input type="text" placeholder="Щелкните правой кнопкой для операций с текстом"/>
    <textarea>Стандартные операции с текстом доступны здесь</textarea>
</div>

<!-- Скрыть контекстное меню по умолчанию -->
<div style="--default-contextmenu: hide">
    <div class="custom-component">Только пользовательское контекстное меню</div>
</div>

<!-- Умное поведение контекстного меню (по умолчанию) -->
<div style="--default-contextmenu: auto">
    <!-- Показывает меню по умолчанию, когда текст выделен или в полях ввода -->
    <p>Выделите этот текст, чтобы увидеть меню по умолчанию</p>
    <input type="text" placeholder="Меню по умолчанию для операций ввода"/>
</div>
```

:::note
Эта функция будет работать корректно только после [инициализации](../../learn/runtime#initialisation) среды выполнения (runtime).
:::

#### Поведение вложенных контекстных меню

При использовании свойства `--default-contextmenu` на вложенных элементах применяются следующие правила:

1. Дочерние элементы наследуют настройку контекстного меню своего родителя, если она не переопределена явно
2. Наиболее специфичное (ближайшее) значение имеет приоритет
3. Значение `auto` можно использовать для сброса к поведению по умолчанию

Пример поведения вложенных контекстных меню:

```html
<!-- Родитель устанавливает hide -->
<div style="--default-contextmenu: hide">
    <!-- Это наследует hide -->
    <p>Здесь нет контекстного меню</p>
    
    <!-- Это переопределяет на show -->
    <div style="--default-contextmenu: show">
        <p>Контекстное меню показано здесь</p>
        
        <!-- Это наследует show -->
        <span>Также имеет контекстное меню</span>
        
        <!-- Это сбрасывает к автоматическому поведению -->
        <div style="--default-contextmenu: auto">
            <p>Показывает меню только при выделении текста</p>
        </div>
    </div>
</div>
```

### Пользовательские контекстные меню

Пользовательские контекстные меню позволяют предоставлять действия, специфичные для приложения, которые относятся к элементу, по которому щелкнули. Они особенно полезны для:
- Операций с файлами в файловом менеджере
- Инструментов обработки изображений
- Пользовательских действий в сетке данных
- Операций, специфичных для компонентов

#### Создание пользовательского контекстного меню

При создании пользовательского контекстного меню вы предоставляете уникальный идентификатор (имя), который связывает меню с HTML-элементами:

```go
// Создать контекстное меню с идентификатором "imageMenu"
contextMenu := application.NewContextMenu()
app.ContextMenus.Add("imageMenu", contextMenu)
```

Параметр name ("imageMenu" в этом примере) служит уникальным идентификатором, который будет использоваться для:
1. Связывания HTML-элементов с этим конкретным контекстным меню
2. Определения того, какое меню должно отображаться при щелчке правой кнопкой мыши
3. Обновления меню и его очистки

#### Контекстные данные

При обработке событий контекстного меню вы можете получить доступ как к выбранному пункту меню, так и к связанным с ним контекстным данным:

```go
contextMenu.Add("Process").OnClick(func(ctx *application.Context) {
    // Получить выбранный пункт меню
    menuItem := ctx.ClickedMenuItem()
    
    // Получить контекстные данные в виде строки
    contextData := ctx.ContextMenuData()
    
    // Проверить, отмечен ли пункт меню (для флажков/радиокнопок)
    isChecked := ctx.IsChecked()
    
    // Использовать данные
    if contextData != "" {
        processItem(contextData)
    }
})
```

Контекстные данные передаются из свойства `--custom-contextmenu-data` HTML-элемента и доступны в обработчике щелчка через `ctx.ContextMenuData()`. Это особенно полезно, когда:

- Работаете со списками или сетками, где каждому элементу требуется уникальная идентификация
- Обрабатываете операции над конкретными компонентами или элементами
- Передаёте состояние или метаданные из фронтенда на бэкенд

#### Управление контекстным меню

После внесения изменений в контекстное меню вызовите метод `Update()`, чтобы применить изменения:

```go
contextMenu.Update()
```

Когда контекстное меню больше не нужно, вы можете его уничтожить:

```go
contextMenu.Destroy()
```
:::danger[Предупреждение] 
После вызова `Destroy()` повторное использование ссылки на контекстное меню приведёт к панике.
:::

### Пример из реальной практики: Галерея изображений

Ниже приведён полный пример реализации пользовательского контекстного меню для галереи изображений:

```go
// Бэкенд: Создать контекстное меню```go
imageMenu := application.NewContextMenu()
app.ContextMenus.Add("imageMenu", imageMenu)

// Add relevant operations
imageMenu.Add("View Full Size").OnClick(func(ctx *application.Context) {
    // Get the image ID from context data
    if imageID := ctx.ContextMenuData(); imageID != "" {
        openFullSizeImage(imageID)
    }
})

imageMenu.Add("Download").OnClick(func(ctx *application.Context) {
    if imageID := ctx.ContextMenuData(); imageID != "" {
        downloadImage(imageID)
    }
})

imageMenu.Add("Share").OnClick(func(ctx *application.Context) {
    if imageID := ctx.ContextMenuData(); imageID != "" {
        showShareDialog(imageID)
    }
})
```

```html
<!-- Frontend: Image gallery implementation -->
<div class="gallery">
    <!-- Each image container with context menu -->
    <div class="image-container" 
         style="--custom-contextmenu: imageMenu; --custom-contextmenu-data: img_123">
        <img src="/images/img_123.jpg" alt="Gallery Image"/>
        <span class="caption">Nature Photo</span>
    </div>
    
    <div class="image-container"
         style="--custom-contextmenu: imageMenu; --custom-contextmenu-data: img_124">
        <img src="/images/img_124.jpg" alt="Gallery Image"/>
        <span class="caption">City Photo</span>
    </div>
</div>
```

В этом примере:
1. Контекстное меню создается с идентификатором "imageMenu"
2. Каждый контейнер изображения связывается с меню с помощью `--custom-contextmenu: imageMenu`
3. Каждый контейнер передает свой идентификатор изображения в качестве контекстных данных через `--custom-contextmenu-data`
4. Бэкенд получает идентификатор изображения в обработчиках кликов и может выполнять специфичные операции
5. Одно и то же меню используется для всех изображений, но контекстные данные указывают, с каким именно изображением нужно работать

Этот паттерн особенно полезен для:
- Таблиц данных, где строки требуют специфичных операций
