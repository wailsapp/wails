Wails CLI предоставляет обширный набор команд, которые помогут вам разрабатывать, собирать и поддерживать ваши приложения Wails.

## Основные команды

Основные команды используются для создания проекта, разработки и сборки.

Все команды CLI имеют следующий формат: `wails3 <command>`.

### `init`
Инициализирует новый проект Wails. Во время этой инициализации выполняется команда `go mod tidy`, чтобы привести пакеты проекта в актуальное состояние. Это можно обойти, используя флаг `-skipgomodtidy` с командой `init`.

```bash
wails3 init [flags]
```

#### Флаги
| Флаг                  | Описание          | Значение по умолчанию                  |
|-----------------------|----------------------|--------------------------|
| `-p`                  | Имя пакета         | `main`                   |
| `-t`                  | Имя или URL шаблона | `vanilla`                |
| `-n`                  | Имя проекта         |                          |
| `-d`                  | Директория проекта    | `.`                      |
| `-q`                  | Подавление вывода      | `false`                  |
| `-l`                  | Список шаблонов       | `false`                  |
| `-git`                | URL Git-репозитория   |                          |
| `-productname`        | Имя продукта         | `My Product`             |
| `-productdescription` | Описание продукта  | `My Product Description` |
| `-productversion`     | Версия продукта      | `0.1.0`                  |
| `-productcompany`     | Название компании         | `My Company`             |
| `-productcopyright`   | Уведомление об авторских правах     | ` now, My Company`       |
| `-productcomments`    | Комментарии к файлу        | `This is a comment`      |
| `-productidentifier`  | Идентификатор продукта   |                          |
| `-skipgomodtidy`      | Пропустить go mod tidy     | `false`                   |

Флаг `-git` принимает различные форматы URL Git:
- HTTPS: `https://github.com/username/project`
- SSH: `git@github.com:username/project` или `ssh://git@github.com/username/project`
- Протокол Git: `git://github.com/username/project`
- Файловая система: `file:///path/to/project.git`

При указании этот флаг:
1. Инициализирует git-репозиторий в директории проекта
2. Устанавливает указанный URL в качестве удалённого origin
3. Обновляет имя модуля в `go.mod`, чтобы оно соответствовало URL репозитория
4. Добавляет все файлы

### `dev`
Запускает приложение в режиме разработки. Это даст вам живой просмотр вашего фронтенд-кода, и вы сможете вносить изменения и видеть их отражение в работающем приложении без необходимости пересобирать всё приложение. Изменения в вашем Go-коде также будут обнаружены, и приложение автоматически пересоберётся и перезапустится.

```bash
wails3 dev [flags]
```

#### Флаги
| Флаг      | Описание          | Значение по умолчанию              |
|-----------|----------------------|----------------------|
| `-config` | Путь к файлу конфигурации     | `./build/config.yml` |
| `-port`   | Порт сервера разработки Vite | `9245`               |
| `-s`      | Включить HTTPS         | `false`              |


    :::note
    Это эквивалентно запуску `wails3 task dev` и выполняет задачу `dev` в основном Taskfile проекта. Вы можете настроить это, отредактировав файл `Taskfile.yml`.
    :::


### `build`
Собирает отладочную версию вашего приложения. По умолчанию собирается для текущей платформы и архитектуры.

```bash
wails3 build [flags] [CLI variables...]
```

#### Флаги
| Флаг     | Описание                                              | Значение по умолчанию |
|----------|----------------------------------------------------------|---------|
| `-tags`  | Дополнительные теги сборки Go (через запятую)               |         |

Вы можете передать переменные CLI для настройки сборки:
```bash
wails3 build PLATFORM=linux CONFIG=production
```

Используйте флаг `-tags` для передачи пользовательских тегов сборки Go:
```bash
# Сборка с поддержкой GTK4 на Linux
wails3 build -tags gtk4

# Сборка в режиме сервера (без GUI)
wails3 build -tags server

# Несколько тегов
wails3 build -tags gtk4,customtag
```

Теги передаются как `EXTRA_TAGS` в базовый Taskfile.

    :::note
    Это эквивалентно запуску `wails3 task build`, который выполняет задачу `build` в основном Taskfile проекта. Любые переменные CLI, переданные в `build`, передаются в базовую задачу. Вы можете настроить процесс сборки, отредактировав файл `Taskfile.yml`.
    :::

### `package`

Создаёт пакеты, специфичные для платформы, для распространения.

```bash
wails3 package [CLI variables...]
```

Вы можете передать переменные CLI для настройки упаковки:
```bash
wails3 package VERSION=2.0.0 OUTPUT=myapp.pkg
```

#### Типы пакетов

Следующие типы пакетов доступны для каждой платформы:

| Платформа | Тип пакета                              |
|----------|-------------------------------------------|
| Windows  | `.exe`                                    |
| macOS    | `.app`,                                   |
| Linux    | `.AppImage`, `.deb`, `.rpm`, `.archlinux` |

    :::note
     Это эквивалентно `wails3 task package`, который выполняет задачу `package` в основном Taskfile проекта. Любые переменные CLI, переданные в `package`, передаются в базовую задачу. Вы можете настроить процесс упаковки, отредактировав файл `Taskfile.yml`.
    :::


### `task`
Выполняет задачи, определённые в Taskfile.yml вашего проекта. Это встроенная версия [Taskfile](https://taskfile.dev), которая позволяет определять и выполнять пользовательские задачи сборки, тестирования и развёртывания.

```bash
wails3 task [taskname] [CLI variables...] [flags]
```

#### Переменные CLI
Вы можете передавать переменным задачи в формате `KEY=VALUE`:
```bash
wails3 task build PLATFORM=linux CONFIG=production
wails3 task deploy ENV=staging VERSION=1.2.3
```

Эти переменные можно получить в вашем Taskfile.yml с помощью синтаксиса шаблонов Go:
```yaml
tasks:
  build:
    cmds:
      - echo "Building for {{.PLATFORM | default "darwin"}}"
      - echo "Config: {{.CONFIG | default "debug"}}"
```

#### Флаги
| Флаг         | Описание                               | Значение по умолчанию  |
|--------------|-------------------------------------------|----------|
| `-h`         | Показывает использование Task                          | `false`  |
| `-i`         | Создаёт новый Taskfile.yml                | `false`  |
| `-list`      | Выводит список задач с описаниями             | `false`  |
| `-list-all`  | Выводит все задачи (с описаниями или без) | `false`  |
| `-json`      | Форматирует список задач как JSON                 | `false`  |
| `-status`    | Завершается с ненулевым кодом, если задача не актуальна | `false`  |
| `-f`         | Принудительно выполняет задачу, даже если она актуальна | `false`  |
| `-w`         | Включает режим наблюдения для указанной задачи     | `false`  |
| `-v`         | Включает подробный режим                      | `false`  |
| `-version`   | Выводит версию Task                       | `false`  |
| `-s`         | Отключает эхо (вывод команд)                          | `false`  |
| `-p`         | Выполняет задачи параллельно                | `false`  |
| `-dry`       | Компилирует и выводит задачи без выполнения | `false`  |
| `-summary`   | Показывает сводку о задаче                | `false`  |
| `-x`         | Передаёт код выхода задачи    | `false`  |
| `-dir`       | Устанавливает директорию выполнения               |          |
| `-taskfile`  | Выбирает, какой Taskfile запустить              |          |
| `-output`    | Устанавливает стиль вывода: [interleaved|group|prefixed] |          |
| `-c`         | Цветной вывод (включён по умолчанию)       | `true`   |
| `-C`         | Ограничивает количество задач, выполняемых параллельно |          |
| `-interval`  | Интервал для отслеживания изменений (в секундах) |          |

```bash

# Вывод всех доступных задач

# Выполнение нескольких задач параллельно-------------|-------------------|-------------|

| `-names`    | Использовать имена вместо ID | `false`             |

### `generate icons`
Генерирует иконки приложения.

```bash
wails3 generate icons [flags]
```

#### Флаги
| Flag                   | Описание                                          | Значение по умолчанию |
|------------------------|--|----------|
| `-input`               | Входной PNG файл                                   | Обязательно           |

| `-macassetdir`         | Выходная директория для ресурсов Mac (Assets.car + icns) |                       |

#### Icon Composer (macOS)

На macOS 26+ вы можете использовать файлы `.icon` Icon Composer для генерации `Assets.car` и `icons.icns`:

```bash
wails3 generate icons -iconcomposerinput build/appicon.icon -macassetdir build
```

Это компилирует файл `.icon` с помощью команды Apple `actool`. Требуется Xcode с версией `actool` 26 или новее.

При использовании Icon Composer установите `cfBundleIconName` в вашем `build/config.yml` так, чтобы он соответствовал имени файла `.icon` (без расширения):

```yaml
info:
  cfBundleIconName: "appicon"
```

Если не установлено и существует `Assets.car`, по умолчанию используется `"appicon"`.

## Команды инструментов

Команды инструментов предоставляют утилиты для разработки и отладки. Все команды инструментов используют базовую команду: `wails3 tool <command>`.

### `tool checkport`
Проверяет, открыт ли порт. Полезно для проверки, запущен ли vite.

```bash
wails3 tool checkport [flags]
```

#### Флаги
| Flag    | Описание   | Значение по умолчанию |
|--|-----------|-----------|
| `-port` | Порт для проверки | `9245`      |
| `-host` | Хост для проверки | `localhost` |

### `tool watcher`
Следит за файлами и запускает команду при их изменении.

```bash

### `tool cp`
Копирует файлы.

```bash

```

### `tool buildinfo`
Отображает информацию о сборке приложения.

```bash

```

### `tool version`
Увеличивает семантическую версию в соответствии с указанными флагами.

```bash
wails3 tool version [flags]
```

#### Флаги
| Flag          | Описание                                      | Значение по умолчанию |
|---------------|--------------------------------------------------|---------|
| `-v`          | Текущая версия для увеличения                          |         |
| `-major`      | Увеличение основной версии                               | `false` |
| `-minor`      | Увеличение минорной версии                               | `false` |
| `-patch`      | Увеличение патч-версии                               | `false` |
| `-prerelease` | Увеличение версии предварительного релиза (например, alpha.5 до alpha.6) | `false` |

Команда следует порядку приоритета: major > minor > patch > prerelease. Она сохраняет префикс "v", если он присутствует во входной версии, а также любые компоненты предварительного релиза и метаданных.

Пример использования:
```bash
wails3 tool version -v 1.2.3 -major      # Вывод: 2.0.0
wails3 tool version -v v1.2.3 -minor     # Вывод: v1.3.0
wails3 tool version -v 1.2.3-alpha -patch # Вывод: 1.2.4-alpha
wails3 tool version -v v3.0.0-alpha.5 -prerelease # Вывод: v3.0.0-alpha.6
```

## Команды обновления

Команды обновления помогают управлять и обновлять ресурсы проекта. Все команды обновления используют базовую команду: `wails3 update <command>`.

### `update cli`
Обновляет Wails CLI до новой версии.

```bash
wails3 update cli [flags]
```

#### Флаги
| Flag        | Описание                     | Значение по умолчанию |
|-------------|---------------------------------|---------|
| `-pre`      | Обновление до последней предварительной версии    | `false` |
| `-version`  | Обновление до конкретной версии      |         |
| `-nocolour` | Отключение цветного вывода          | `false` |

Команда `update cli` позволяет обновить установку Wails CLI. По умолчанию она обновляется до последней стабильной версии.
Вы можете использовать флаг `-pre` для обновления до последней предварительной версии или указать конкретную версию с помощью флага `-version`.

После обновления не забудьте обновить файл go.mod вашего проекта, чтобы использовать ту же версию:
```bash
require github.com/wailsapp/wails/v3 v3.x.x
```

Базовая команда: `wails3`

## Вспомогательные команды

Вспомогательные команды предоставляют полезные сокращения для распространенных задач. Используйте эти команды напрямую с базовой командой: `wails3 <command>`.

### `docs`
Открывает документацию Wails в вашем браузере по умолчанию.

```bash
wails3 docs
```

### `releasenotes`
Показывает примечания к выпуску для текущей или указанной версии.

```bash
wails3 releasenotes [flags]
```

#### Флаги
| Flag | Описание                       | Значение по умолчанию |
|------|-----------------------------------|---------|
| `-v` | Версия, для которой нужно показать примечания к выпуску |         |
| `-n` | Отключение цветного вывода             | `false` |

### `version`
Выводит текущую версию Wails.

```bash
wails3 version
```

### `sponsor`
Открывает страницу спонсирования Wails в вашем браузере по умолчанию.

```bash
wails3 sponsor
