В приложениях на Go паники могут возникать во время выполнения, когда происходит что-то непредвиденное. Это руководство объясняет, как обрабатывать паники как в общем коде Go, так и конкретно в вашем приложении Wails.

## Понимание паник в Go

Прежде чем переходить к обработке паник, специфичной для Wails, важно понять, как работают паники в Go:

1. Паники предназначены для неисправимых ошибок, которые не должны происходить при нормальной работе
2. Когда паника происходит в горутине, затрагивается только эта горутина
3. Паники можно обработать с помощью `defer` и `recover()`

Вот базовый пример обработки паник в Go:

```go
func doSomething() {
    // Отложенные функции выполняются даже при возникновении паники
    defer func() {
        if r := recover(); r != nil {
            fmt.Printf("Recovered from panic: %v\n", r)
        }
    }()
    
    // Ваш код, который может вызвать панику
    panic("something went wrong")
}
```

Более подробную информацию о паниках и recover в Go см. в [Go Blog: Defer, Panic, and Recover](https://go.dev/blog/defer-panic-and-recover).

## Обработка паник в Wails

Wails автоматически обрабатывает паники, возникающие в методах ваших сервисов при их вызове из фронтенда. Это означает, что вам не нужно добавлять обработку паник в эти методы — Wails перехватит панику и обработает её через настроенный вами обработчик паник.

Обработчик паник специально предназначен для перехвата:
- Паник в привязанных методах сервисов, вызванных из фронтенда
- Внутренних паник среды выполнения Wails

Для других сценариев, таких как фоновые горутины или автономный код Go, вы должны обрабатывать паники самостоятельно, используя стандартные механизмы обработки паник Go.

## Структура PanicDetails

Когда возникает паника, Wails фиксирует важную информацию о ней в структуре `PanicDetails`:

```go
type PanicDetails struct {
    StackTrace     string    // Трассировка стека места возникновения паники. Может быть усечена для предоставления большего контекста
    Error          error     // Ошибка, вызвавшая панику
    Time           time.Time // Время возникновения паники
    FullStackTrace string    // Полная трассировка стека, включая кадры среды выполнения
}
```

Эта структура предоставляет исчерпывающую информацию о панике:
- `StackTrace`: Отформатированная строка, показывающая стек вызовов, приведший к панике
- `Error`: Фактическая ошибка или сообщение паники
- `Time`: Точное время возникновения паники
- `FullStackTrace`: Полная трассировка стека, включая кадры среды выполнения

:::note[Паники в коде сервиса]

Когда паники перехватываются в вашем коде сервиса после вызова из фронтенда, трассировка стека усечается, чтобы сосредоточиться именно на том месте в вашем коде, где произошла паника.
Если вы хотите увидеть полную трассировку стека, вы можете использовать поле `FullStackTrace`.

:::

## Обработчик паник по умолчанию

Если вы не укажете пользовательский обработчик паник, Wails использует свой обработчик по умолчанию, который выводит информацию об ошибке в отформатированном сообщении журнала, а затем завершает работу.
Например:

```
************************ FATAL ******************************
* There has been a catastrophic failure in your application *
********************* Error Details *************************
panic error: oh no! something went wrong deep in my service! :(
main.(*WindowService).call2
	at E:/wails/v3/examples/panic-handling/main.go:23
main.(*WindowService).call1
	at E:/wails/v3/examples/panic-handling/main.go:19
main.(*WindowService).GeneratePanic
	at E:/wails/v3/examples/panic-handling/main.go:15
*************************************************************
```

## Пользовательский обработчик паник

Вы можете реализовать собственный обработчик паник, установив опцию `PanicHandler` при создании приложения. Вот пример:

```go
app := application.New(application.Options{
    Name: "My App",
    PanicHandler: func(panicDetails *application.PanicDetails) {
        fmt.Printf("*** Custom Panic Handler ***\n")
        fmt.Printf("Time: %s\n", panicDetails.Time)
        fmt.Printf("Error: %s\n", panicDetails.Error)
        fmt.Printf("Stacktrace: %s\n", panicDetails.StackTrace)
        fmt.Printf("Full Stacktrace: %s\n", panicDetails.FullStackTrace)
        
        // Вы также можете:
        // - Логировать в файл
        // - Отправлять в сервис отчетов об ошибках
        // - Показать пользователю дружественное диалоговое окно с ошибкой
        // - Попытаться восстановить или перезапустить приложение
    },
})
```

Полный рабочий пример обработки паник в приложении Wails см. в примере panic-handling в `v3/examples/panic-handling`.

## Дополнительные замечания

Помните, что обработчик паник Wails предназначен специально для управления паниками в привязанных методах и внутренними ошибками среды выполнения. Для других частей вашего приложения следует использовать стандартные шаблоны обработки ошибок Go и механизмы обработки паник там, где это уместно. Как и во всех приложениях на Go, лучше предотвращать паники посредством надлежащей обработки ошибок, где это возможно.
