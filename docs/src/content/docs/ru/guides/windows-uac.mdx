Контроль учётных записей (UAC) в Windows определяет права выполнения вашего приложения Wails. По умолчанию приложения Wails v3 включают явную конфигурацию UAC в свой манифест Windows, что обеспечивает единообразное поведение на разных компьютерах.

## Уровни выполнения UAC

Приложения Windows могут запрашивать различные уровни выполнения через свой манифест. Wails v3 автоматически включает конфигурацию UAC с уровнем выполнения по умолчанию, который вы можете настроить в зависимости от потребностей вашего приложения.

### Доступные уровни выполнения

| Уровень | Описание | Сценарий использования |
|-------|-------------|----------|
| `asInvoker` | Запускается с теми же правами, что и родительский процесс | По умолчанию для большинства приложений |
| `highestAvailable` | Запускается с максимальными правами, доступными пользователю | Приложениям, которым может потребоваться повышенный доступ |
| `requireAdministrator` | Всегда требует прав администратора | Системные утилиты, установщики |

### Конфигурация по умолчанию

Приложения Wails v3 включают конфигурацию UAC по умолчанию в свой манифест Windows:

```xml
<trustInfo xmlns="urn:schemas-microsoft-com:asm.v3">
    <security>
        <requestedPrivileges>
            <requestedExecutionLevel level="asInvoker" uiAccess="false"/>
        </requestedPrivileges>
    </security>
</trustInfo>
```

Эта конфигурация гарантирует, что ваше приложение:
- Запускается с теми же правами, что и запускающий процесс
- По умолчанию не требует повышения привилегий
- Работает единообразно на разных компьютерах
- Не вызывает запросы UAC для обычных пользователей

## Настройка конфигурации UAC

Поскольку Wails v3 поощряет пользователей настраивать свои сборочные ресурсы, вы можете изменить конфигурацию UAC, отредактировав шаблон манифеста Windows напрямую.

### Поиск шаблона манифеста

Шаблон манифеста Windows находится по адресу:
```
build/windows/wails.exe.manifest
```

### Изменение уровня выполнения

Чтобы изменить уровень выполнения, отредактируйте атрибут `level` в элементе `requestedExecutionLevel`:

```xml title="build/windows/wails.exe.manifest"
<trustInfo xmlns="urn:schemas-microsoft-com:asm.v3">
    <security>
        <requestedPrivileges>
            <requestedExecutionLevel level="requireAdministrator" uiAccess="false"/>
        </requestedPrivileges>
    </security>
</trustInfo>
```

### Примеры

#### Стандартное приложение (по умолчанию)
Большинство приложений должно использовать уровень `asInvoker` по умолчанию:

```xml
<requestedExecutionLevel level="asInvoker" uiAccess="false"/>
```

#### Системная утилита
Приложениям, которым нужен повышенный доступ, когда он доступен:

```xml
<requestedExecutionLevel level="highestAvailable" uiAccess="false"/>
```

#### Административный инструмент
Приложениям, которым всегда требуются права администратора:

```xml
<requestedExecutionLevel level="requireAdministrator" uiAccess="false"/>
```

## Доступ к интерфейсу пользователя (UI Access)

Атрибут `uiAccess` управляет тем, может ли ваше приложение взаимодействовать с элементами интерфейса с более высокими привилегиями. В большинстве случаев он должен оставаться равным `false`.

Устанавливайте значение `true` только если вашему приложению необходимо:
- Отправлять ввод другим приложениям
- Управлять интерфейсом других приложений
- Получать доступ к элементам интерфейса процессов с более высокими привилегиями

:::caution[Требования к UI Access]
Установка `uiAccess="true"` требует, чтобы ваше приложение:
- Было подписано цифровым сертификатом от доверенного центра сертификации
- Было установлено в защищённое место (Program Files или Windows\System32)
:::

## Сборка с настройками UAC

После изменения шаблона манифеста соберите ваше приложение как обычно:

```bash
wails3 build
```

Процесс сборки автоматически внедрит вашу пользовательскую конфигурацию UAC в исполняемый файл.

## Проверка конфигурации UAC

Вы можете проверить, что ваши настройки UAC правильно внедрены, с помощью инструмента `go-winres`:

```bash
go-winres extract --in your-app.exe --out extracted-resources/
```

Затем просмотрите извлечённый файл манифеста, чтобы подтвердить наличие вашей конфигурации UAC.

:::tip[Сохранение манифеста]
В отличие от некоторых других фреймворков, конфигурация UAC Wails v3 внедряется непосредственно в исполняемый файл во время компиляции, что гарантирует её сохранение при копировании приложения на другие машины.
:::

## Устранение неполадок

### Запросы UAC не появляются
Если вы установили `requireAdministrator`, но не видите запросов UAC:
- Убедитесь, что манифест правильно внедрён в ваш исполняемый файл
- Проверьте, что вы не запускаете его из уже повышенного процесса
- Убедитесь, что синтаксис манифеста является допустимым XML

### Приложение не запускается
Если ваше приложение не запускается после изменений UAC:
- Проверьте синтаксис манифеста на наличие ошибок XML
- Убедитесь, что значение уровня выполнения допустимо
- Попробуйте вернуться к `asInvoker`, чтобы изолировать проблему

### Непоследовательное поведение на разных машинах
Если поведение UAC различается на разных компьютерах:
- Убедитесь, что манифест внедрён в исполняемый файл (а не находится отдельно)
- Проверьте, что исполняемый файл не был изменён после сборки
- Убедитесь, что настройки UAC Windows включены на целевом компьютере
