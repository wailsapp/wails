Wails v3 позволяет предоставить собственный транспортный слой IPC, сохраняя при этом все сгенерированные привязки и коммуникацию событий. Это позволяет заменить стандартный транспорт на основе HTTP fetch на WebSockets, пользовательские протоколы или любой другой механизм передачи данных.

## Обзор

По умолчанию Wails использует HTTP-запросы fetch из фронтенда для взаимодействия с бэкендом через `/wails/runtime`. API пользовательского транспорта позволяет:

- Заменить HTTP-транспорт на WebSockets, gRPC или любой другой пользовательский протокол
- Сохранить полную совместимость с генерацией кода Wails
- Сохранить все существующие привязки, события, диалоги и другие функции Wails
- Реализовать собственное управление соединениями, аутентификацию и обработку ошибок


## Использование

### 1. Реализация интерфейса Transport

Создайте пользовательский транспорт, реализовав интерфейс `Transport`:

```go
package main

import (
    "context"
    "github.com/wailsapp/wails/v3/pkg/application"
)

type MyCustomTransport struct {
    // Your fields
}

func (t *MyCustomTransport) Start(ctx context.Context, processor *application.MessageProcessor) error {
    // Initialize your transport (WebSocket server, gRPC server, etc.)
    // When you receive requests, call processor.HandleRuntimeCallWithIDs()
    return nil
}

func (t *MyCustomTransport) Stop() error {
    // Clean up your transport
    return nil
}
```

### 2. Настройка приложения

Передайте свой пользовательский транспорт в параметры приложения:

```go
func main() {
    app := application.New(application.Options{
        Name: "My App",
        Transport: &MyCustomTransport{},
        // ... other options
    })

    err := app.Run()
    if err != nil {
        log.Fatal(err)
    }
}
```

### 3. Изменение фронтенд-рантайма

Если используется пользовательский транспорт, вам необходимо изменить фронтенд-рантайм, чтобы использовать ваш транспорт вместо HTTP fetch. Реализуйте интерфейс `RuntimeTransport`, который будет использоваться для обработки запросов:

```typescript
const { setTransport } = await import('/wails/runtime.js');

class MyRuntimeTransport {
  call(objectID: number, method: number, windowName: string, args: any): Promise<any> {
    // TODO: implement IPC call with your transport protocol

    return resp;
  }
}

const myTransport = new MyRuntimeTransport();
setTransport(myTransport);
```

## Примечания

- Стандартный HTTP-транспорт продолжает работать, если не указан пользовательский транспорт
- Сгенерированные привязки остаются неизменными — изменяется только транспортный слой
- События, диалоги, буфер обмена и все остальные функции Wails работают прозрачно
- Вы несете ответственность за обработку ошибок, логику переподключения и безопасность в вашем пользовательском транспорте
- Пример WebSocket приведен для демонстрации и может потребовать доработки для использования в production

## Справочник API

### Интерфейс Transport

```go
type Transport interface {
    Start(ctx context.Context, messageProcessor *application.MessageProcessor) error
    Stop() error
}
```

### Интерфейс AssetServerTransport (необязательно)

Для развертывания в браузере или когда вы хотите обслуживать как ресурсы, так и IPC через ваш пользовательский транспорт, реализуйте интерфейс `AssetServerTransport`:

```go
type AssetServerTransport interface {
    Transport

    // ServeAssets configures the transport to serve assets alongside IPC.
    // The assetHandler is Wails' internal asset server that handles all assets,
    // runtime.js, capabilities, flags, etc.
    ServeAssets(assetHandler http.Handler) error
}
```

**Когда следует реализовывать этот интерфейс:**
- Запуск приложения в браузере вместо webview
- Обслуживание ресурсов по HTTP вместе с вашим пользовательским транспортом IPC
- Создание сетевых приложений

**Пример реализации:**

```go
func (t *MyTransport) ServeAssets(assetHandler http.Handler) error {
    mux := http.NewServeMux()

    // Mount your IPC endpoint
    mux.HandleFunc("/my/ipc/endpoint", t.handleIPC)

    // Mount Wails asset server for everything else
    mux.Handle("/", assetHandler)

    // Start HTTP server
    t.httpServer.Handler = mux
    go t.httpServer.ListenAndServe()

    return nil
}
```

При вызове `ServeAssets()` параметр `assetHandler` предоставляет:
- Все статические ресурсы (HTML, CSS, JS, изображения и т.д.)
- `/wails/runtime.js` — библиотека рантайма Wails
