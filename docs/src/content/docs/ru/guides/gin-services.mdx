# Использование Gin для сервисов

Веб-фреймворк Gin является популярным выбором для создания HTTP-сервисов на Go. С Wails v3 вы можете легко интегрировать сервисы на основе Gin в ваше приложение, предоставляя мощный способ обработки HTTP-запросов, реализации RESTful API и обслуживания веб-контента.

Это руководство проведет вас через создание сервиса на основе Gin, который может быть подключен к определенному маршруту в вашем приложении Wails. Мы создадим полный пример, демонстрирующий, как:

1. Создать сервис на основе Gin
2. Реализовать интерфейс Wails Service
3. Настроить маршруты и промежуточное ПО (middleware)
4. Интегрироваться с системой событий Wails
5. Взаимодействовать с сервисом из фронтенда

## Предварительные требования

Перед началом убедитесь, что у вас есть:

- Установленный Wails v3
- Базовые знания Go и фреймворка Gin
- Знание концепций HTTP и RESTful API

Вам нужно добавить фреймворк Gin в ваш проект:

```bash
go get github.com/gin-gonic/gin
```

## Создание сервиса на основе Gin

Начнем с создания сервиса Gin, который реализует интерфейс Wails Service. Наш сервис будет управлять коллекцией пользователей и предоставлять конечные точки API для получения и создания записей пользователей.

### 1. Определите модели данных

Сначала определите структуры данных, с которыми будет работать ваш сервис:

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User представляет пользователя в системе
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData представляет данные, отправляемые в событиях
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. Создайте структуру сервиса

Затем определите структуру сервиса, которая будет содержать ваш Gin-роутер и любое состояние, которое сервис должен поддерживать:

```go
// GinService реализует сервис Wails, использующий Gin для обработки HTTP
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService создает новый экземпляр GinService
func NewGinService() *GinService {
	// Создаем новый роутер Gin
	ginEngine := gin.New()

	// Добавляем промежуточное ПО
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// Настраиваем маршруты
	service.setupRoutes()

	return service
}
```

### 3. Реализуйте интерфейс сервиса

Реализуйте необходимые методы для интерфейса Wails Service:

```go
// ServiceName возвращает имя сервиса
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup вызывается при запуске сервиса
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// Сохраняем экземпляр приложения для дальнейшего использования
	s.app = application.Get()

	// Регистрируем обработчик событий, который может быть вызван из фронтенда
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// Логируем данные события
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// Отправляем событие обратно во фронтенд
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown вызывается при остановке сервиса
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// Освобождаем ресурсы, если необходимо
	return nil
}
```

### 3. Реализуйте интерфейс http.Handler

Чтобы сделать ваш сервис подключаемым к определенному маршруту, реализуйте интерфейс `http.Handler`. Этот единственный метод, `ServeHTTP`, является шлюзом для всех HTTP-запросов к вашему сервису. Он делегирует обработку запросов роутеру Gin, позволяя использовать все мощные возможности Gin для маршрутизации и промежуточного ПО.

```go
// ServeHTTP реализует интерфейс http.Handler
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// Все запросы направляются в роутер Gin
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. Настройте ваши маршруты

Определите маршруты вашего API в отдельном методе для лучшей организации. Этот подход делает ваш код чистым и упрощает понимание структуры вашего API. Роутер Gin предоставляет удобный API для определения маршрутов, включая поддержку групп маршрутов, которые помогают организовывать связанные конечные точки.

```go
// setupRoutes настраивает маршруты API
func (s *GinService) setupRoutes() {
	// Конечная точка с базовой информацией
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// Группа пользователей
	users := s.ginEngine.Group("/users")
	{
		// Получить всех пользователей
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// Получить пользователя по ID
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// Создать нового пользователя
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// Устанавливаем ID и время создания
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// Добавляем в срез пользователей
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// Отправляем событие для уведомления о новом пользователе
			s.app.Event.Emit("user-created", newUser)
		})

		// Удалить пользователя
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// Удаляем пользователя из среза
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. Создайте собственное промежуточное ПО

Вы можете создать собственное промежуточное ПО Gin для улучшения вашего сервиса. Функции промежуточного ПО в Gin выполняются в порядке их добавления в роутер и могут выполнять такие задачи, как логирование, аутентификация и обработка ошибок. Этот пример показывает простое промежуточное ПО для логирования, которое записывает детали запроса.

```go
// LoggingMiddleware является промежуточным ПО Gin, которое логирует детали запроса
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// Запускаем таймер
		start := time.Now()

		// Обрабатываем запрос
		c.Next()

		// Вычисляем задержку
		latency := time.Since(start)

		// Логируем детали запроса
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## Регистрация вашего сервисаДля использования вашего сервиса на базе Gin в приложении Wails необходимо зарегистрировать его в приложении и указать маршрут, по которому он должен быть подключён. Это делается при создании экземпляра приложения Wails. Указанный вами маршрут становится базовым путём для всех конечных точек (эндпоинтов), определённых в вашем роутере Gin.

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

В этом примере сервис Gin подключён по маршруту `/api`. Это означает, что если в вашем роутере Gin определена конечная точка `/info`, она будет доступна по адресу `/api/info` в вашем приложении. Такой подход позволяет логично организовывать конечные точки API и избегать конфликтов с другими частями вашего приложения.

## Интеграция с системой событий Wails

Одной из мощных особенностей использования Gin с сервисами Wails является возможность бесшовной интеграции с системой событий Wails. Это позволяет осуществлять связь в реальном времени между вашим бэкенд-сервисом и фронтендом.

В методе `ServiceStartup` вашего сервиса вы можете зарегистрировать обработчики событий для обработки событий, поступающих с фронтенда:

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

Вы также можете отправлять события на фронтенд из ваших маршрутов Gin или других частей сервиса:

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## Интеграция с фронтендом

Для взаимодействия с вашим сервисом Gin из фронтенда вам потребуется импортировать Wails runtime, выполнять HTTP-запросы к конечным точкам вашего API и использовать систему событий Wails для связи в реальном времени.

Для использования в продакшене рекомендуется использовать пакет `@wailsio/runtime` вместо прямого импорта `/wails/runtime.js`. Это обеспечивает типобезопасность, лучшую поддержку в IDE, управление версиями через npm и совместимость с современным инструментарием JavaScript.

Установите пакет:
```bash
npm install @wailsio/runtime
```

Затем используйте его в своём коде:
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## Заключительные мысли

Интеграция веб-фреймворка Gin с сервисами Wails v3 предоставляет мощный и гибкий подход к созданию модульных и поддерживаемых веб-приложений. Используя возможности маршрутизации и промежуточного программного обеспечения Gin вместе с системой событий Wails, вы можете создавать богатые интерактивные приложения с четким разделением ответственности.

Полный пример кода для этого руководства можно найти в репозитории Wails в директории `v3/examples/gin-service`.
