Механизм блокировки однократного запуска предотвращает одновременное выполнение нескольких экземпляров вашего приложения.
Это полезно для приложений, которые предназначены для открытия файлов из командной строки или из файлового менеджера операционной системы.


## Использование

Чтобы включить функциональность однократного запуска в вашем приложении, передайте структуру `SingleInstanceOptions` при создании приложения:

```go
app := application.New(application.Options{
    // ... другие параметры ...
    SingleInstance: &application.SingleInstanceOptions{
        UniqueID: "com.myapp.unique-id",
        OnSecondInstanceLaunch: func(data application.SecondInstanceData) {
            log.Printf("Запущен второй экземпляр с аргументами: %v", data.Args)
            log.Printf("Рабочий каталог: %s", data.WorkingDir)
            log.Printf("Дополнительные данные: %v", data.AdditionalData)
        },
        // Необязательно: передать дополнительные данные второму экземпляру
        AdditionalData: map[string]string{
            "launchtime": time.Now().String(),
        },
    },
})
```

Структура `SingleInstanceOptions` имеет следующие поля:

- `UniqueID`: Уникальный идентификатор вашего приложения. Это должна быть уникальная строка, обычно в нотации обратного домена (например, "com.company.appname").
- `EncryptionKey`: Необязательный массив из 32 байт для шифрования данных, передаваемых между экземплярами с помощью AES-256-GCM. Если предоставлен как ненулевой массив, вся коммуникация между экземплярами будет зашифрована.
- `OnSecondInstanceLaunch`: Функция обратного вызова, которая вызывается при запуске второго экземпляра вашего приложения. Обратный вызов получает структуру `SecondInstanceData`, содержащую:
    - `Args`: Аргументы командной строки, переданные второму экземпляру
    - `WorkingDir`: Рабочий каталог второго экземпляра
    - `AdditionalData`: Любые дополнительные данные, переданные от второго экземпляра (если предоставлены)
- `AdditionalData`: Необязательная карта пар ключ-значение типа string, которая будет передана первому экземпляру при запуске последующих экземпляров

:::danger[Предупреждение]
Функция однократного запуска реализует необязательный протокол шифрования с использованием AES-256-GCM. Без включенного шифрования
данные, передаваемые между экземплярами, не являются защищенными. При использовании функции однократного запуска без шифрования
ваше приложение должно рассматривать любые данные, переданные ему из обратного вызова второго экземпляра, как ненадежные.
Вы должны проверить, что полученные аргументы являются допустимыми и не содержат вредоносных данных.
:::

### Безопасная коммуникация

Чтобы включить безопасную коммуникацию между экземплярами, предоставьте 32-байтовый ключ шифрования. Этот ключ должен быть одинаковым для всех экземпляров вашего приложения:

```go
// Определите ваш ключ шифрования (должен быть ровно 32 байта)
var encryptionKey = [32]byte{
    0x00, 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07,
    0x08, 0x09, 0x0a, 0x0b, 0x0c, 0x0d, 0x0e, 0x0f,
    0x10, 0x11, 0x12, 0x13, 0x14, 0x15, 0x16, 0x17,
    0x18, 0x19, 0x1a, 0x1b, 0x1c, 0x1d, 0x1e, 0x1f,
}

// Используйте ключ в SingleInstanceOptions
SingleInstance: &application.SingleInstanceOptions{
    UniqueID: "com.myapp.unique-id",
    // Включить шифрование для коммуникации между экземплярами
    EncryptionKey: encryptionKey,
    // ... другие параметры ...
}
```

:::tip[Рекомендации по безопасности]
- Используйте уникальный ключ для вашего приложения
- Храните ключ в безопасном месте, если загружаете его из конфигурации
- Не используйте пример ключа, показанный выше — создайте свой собственный!
:::

### Управление окнами

При обработке запуска второго экземпляра вы часто захотите вывести окно вашего приложения на передний план. Вы можете сделать это, используя метод `Focus()` окна. Если ваше окно свернуто, возможно, потребуется сначала восстановить его:

```go

    var mainWindow *application.WebviewWindow

    SingleInstance: &application.SingleInstanceOptions{
        // Другие параметры...
        OnSecondInstanceLaunch: func(data application.SecondInstanceData) {
            // Сфокусируйте окно, если необходимо
            if mainWindow != nil {
                mainWindow.Restore()
                mainWindow.Focus()
            }
        },
    }
```
