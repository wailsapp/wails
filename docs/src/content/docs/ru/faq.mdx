## Общие вопросы

### Что такое Wails?

Wails — это фреймворк для создания десктопных приложений с использованием Go и веб-технологий. Он обеспечивает нативную интеграцию с операционной системой, позволяя при этом разрабатывать пользовательский интерфейс с помощью HTML, CSS и JavaScript.

### Чем Wails отличается от Electron?

**Wails:**
- ~10 МБ потребления памяти
- ~15 МБ размера бинарного файла
- Нативная производительность
- Бэкенд на Go

**Electron:**
- ~100 МБ+ потребления памяти
- ~150 МБ+ размера бинарного файла
- Накладные расходы Chromium
- Бэкенд на Node.js

### Готов ли Wails к использованию в продакшене?

Да! Wails v3 подходит для создания продакшн-приложений. Многие компании используют Wails для своих десктопных приложений.

### Какие платформы поддерживает Wails?

- Windows (7+)
- macOS (10.13+)
- Linux (GTK3)

## Разработка

### Нужно ли знать Go?

Базовые знания Go полезны, но не обязательны. Вы можете начать с простых сервисов и изучать язык по мере продвижения.

### Могу ли я использовать свой любимый фронтенд-фреймворк?

Да! Wails работает с:
- Чистым JavaScript (Vanilla JavaScript)
- React
- Vue
- Svelte
- Любым фреймворком, который собирает проект в HTML/CSS/JS

### Как вызвать функции Go из JavaScript?

Используйте биндинги:

```go
// Go
func (s *Service) Greet(name string) string {
    return "Hello " + name
}
```

```javascript
// JavaScript
import { Greet } from './bindings/myapp/service'
const message = await Greet("World")
```

### Могу ли я использовать TypeScript?

Да! Wails автоматически генерирует определения типов TypeScript.

### Как отлаживать моё приложение?

Используйте инструменты разработчика в браузере:
- Правый клик → «Просмотреть код» (Inspect Element)
- Или включите инструменты разработчика в настройках окна

## Сборка и распространение

### Как собрать приложение для продакшена?

```bash
wails3 build
```

Ваше приложение будет находиться в папке `build/bin/`.

### Могу ли я кросс-компилировать?

Да! Собирайте для других платформ:

```bash
wails3 build -platform windows/amd64
wails3 build -platform darwin/universal
wails3 build -platform linux/amd64
```

### Как создать установщик?

Используйте специфичные для платформы инструменты:
- Windows: NSIS или WiX
- macOS: Создание DMG-образа
- Linux: DEB/RPM пакеты

См. [Руководство по установщикам](/guides/installers).

### Как подписать моё приложение кодом?

**macOS:**
```bash
codesign --deep --force --sign "Developer ID" MyApp.app
```

**Windows:**
Используйте SignTool с вашим сертификатом.

## Возможности

### Могу ли я создавать несколько окон?

Да! Wails v3 имеет нативную поддержку нескольких окон:

```go
window1 := app.Window.New()
window2 := app.Window.New()
```

### Поддерживает ли Wails системный трей?

Да! Создавайте приложения с иконкой в системном трее:

```go
tray := app.SystemTray.New()
tray.SetIcon(iconBytes)
tray.SetMenu(menu)
```

### Могу ли я использовать нативные диалоги?

Да! Wails предоставляет нативные диалоги:

```go
path, _ := app.Dialog.OpenFile().
    SetTitle("Select File").
    PromptForSingleSelection()
```

### Поддерживает ли Wails автоматические обновления?

Wails не включает функциональность автоматических обновлений, но вы можете реализовать её самостоятельно или использовать сторонние библиотеки.

## Производительность

### Почему мой бинарный файл такой большой?

Бинарные файлы Go включают в себя среду выполнения. Чтобы уменьшить размер:

```bash
wails3 build -ldflags "-s -w"
```

### Как улучшить производительность?

- Используйте пагинацию для больших наборов данных
- Кэшируйте ресурсоёмкие операции
- Используйте события для обновлений
- Оптимизируйте фронтенд-бандл
- Профилируйте ваш код

См. [Руководство по производительности](/guides/performance).

### Поддерживает ли Wails hot reload?

Да! Используйте режим разработки:

```bash
wails3 dev
```

## Устранение неполадок

### Мои биндинги не работают

Пересгенерируйте биндинги:

```bash
wails3 generate bindings
```

### Окно не появляется

Проверьте, вызвали ли вы `Show()`:

```go
window := app.Window.New()
window.Show()  // Не забудьте это!
```

### События не срабатывают

Убедитесь, что имена событий совпадают точно:

```go
// Go
app.Event.Emit("my-event", data)

// JavaScript
OnEvent("my-event", handler)  // Должно совпадать
```

### Сборка завершается ошибкой

Распространённые решения:
- Запустите `go mod tidy`
- Проверьте конфигурацию в `wails.json`
- Убедитесь, что фронтенд собран: `cd frontend && npm run build`
- Обновите Wails: `go install github.com/wailsapp/wails/v3/cmd/wails3@latest`

## Миграция

### Стоит ли мигрировать с v2 на v3?

v3 предлагает:
- Лучшую производительность
- Поддержку нескольких окон
- Улучшенный API
- Улучшенный опыт разработки

См. [Руководство по миграции](/migration/v2-to-v3).

### Будет ли поддерживаться v2?

Да, v2 будет получать критические обновления.

### Могу ли я запускать v2 и v3 параллельно?

Да, они используют разные пути импорта.
