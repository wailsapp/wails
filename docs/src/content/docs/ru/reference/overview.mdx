## Соглашения по API

<details>
<summary><strong>Соглашения по Go API</strong> - Для разработчиков, начинающих знакомство с Go</summary>

#### Именование

- **Типы**: PascalCase (например, `WebviewWindow`)
- **Методы**: PascalCase (например, `SetTitle()`)
- **Опции**: Структуры в PascalCase (например, `WindowOptions`)
- **Константы**: PascalCase (например, `WindowStartStateMaximised`)

#### Обработка ошибок

Большинство методов, которые могут завершиться ошибкой, возвращают `error` в качестве последнего возвращаемого значения:

```go
window, err := app.Window.New()
if err != nil {
    log.Fatal(err)
}
```

#### Контекст

Методы, которые могут блокироваться или требуют отмены, принимают `context.Context`:

```go
err := app.RunWithContext(ctx)
```

#### Паттерн опций

Конфигурация использует структуры опций:

```go
app := application.New(application.Options{
    Name: "My App",
    Description: "A demo application",
    Services: []application.Service{
        application.NewService(&MyService{}),
    },
})
```

</details>

### Соглашения по JavaScript API

#### Именование

- **Функции**: camelCase (например, `setTitle()`)
- **Константы**: SCREAMING_SNAKE_CASE (например, `WINDOW_EVENT_FOCUS`)

#### Асинхронность по умолчанию

Все вызовы методов Go возвращают Promises:

```javascript
// Async/await (рекомендуется)
const result = await MyService.DoSomething()

// Цепочка промисов
MyService.DoSomething()
    .then(result => console.log(result))
    .catch(error => console.error(error))
```

#### Обработка ошибок

Ошибки Go становятся исключениями JavaScript:

```javascript
try {
    await MyService.MightFail()
} catch (error) {
    console.error('Go error:', error)
}
```

#### Безопасность типов

Определения TypeScript генерируются автоматически:

```typescript
// Полная типизация
import { Greet } from './bindings/GreetService'

const message: string = await Greet("World")
```

## Различия между платформами

Некоторые API ведут себя по-разному на разных платформах:

| Функция | Windows | macOS | Linux |
|---------|---------|-------|-------|
| **Меню приложения** | Панель меню окна | Глобальная панель меню | Панель меню окна |
| **Системный трей** | Область уведомлений | Панель меню | Системный трей |
| **Док (Dock)** | Н/Д | ✅ Доступно | Н/Д |
| **Файловые диалоги** | Нативные | Нативные | Нативные (GTK) |
| **Прозрачность** | ✅ Полная | ✅ Полная | ⚠️ Ограниченная |

Поведение, специфичное для платформы, документируется в каждом разделе API.

## Версионирование

Wails v3 следует семантическому версионированию:

- **Major** (v3.x.x): Ломающие изменения (breaking changes)
- **Minor** (v3.x.x): Новые функции, обратно совместимые
- **Patch** (v3.x.x): Исправления ошибок, обратно совместимые

**Текущий статус:** Alpha (API стабилен, продолжаются уточнения)

## Политика устаревания (Deprecation)

Когда API устаревают:

1. **Помечаются в документации** уведомлением об устаревании
2. **Предоставляется альтернатива** с руководством по миграции
3. **Поддерживаются в течение 1 мажорной версии** перед удалением
4. **Предупреждения компилятора** (где возможно)

## Стабильность API

### Стабильные APIs ✅

Эти API стабильны и безопасны для использования в продакшене:

- Основные API приложения
- Управление окнами
- Система меню
- Система событий
- Файловые диалоги
- Биндинги сервисов

### Нестабильные APIs ⚠️

Эти API могут измениться до финального релиза:

- Некоторые расширенные опции окон
- Специфичные для платформы функции
- Экспериментальные функции

Нестабильные API помечены в документации.
