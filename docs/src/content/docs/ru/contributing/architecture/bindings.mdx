## Обзор архитектуры

Система привязок Wails состоит из трех основных компонентов:

1. **Сбор (Collection)**: Анализирует код Go для извлечения информации о сервисах, моделях и других объявлениях
2. **Конфигурация (Configuration)**: Управляет настройками и параметрами процесса генерации привязок
3. **Рендеринг (Rendering)**: Генерирует JavaScript/TypeScript код на основе собранной информации

  - collect/     # Анализ пакетов и извлечение информации
  - config/      # Структуры и интерфейсы конфигурации
  - render/      # Генерация кода для JS/TS
</FileTree>

## Процесс сбора

Процесс сбора отвечает за анализ пакетов Go и извлечение информации о сервисах, моделях и других объявлениях. Этим занимается пакет `collect`.

### Ключевые компоненты

- **Collector**: Управляет информацией о пакетах и кэширует собранные данные
- **Package**: Представляет анализируемый пакет Go и хранит собранные сервисы, модели и директивы
- **Service**: Собирает информацию о типах сервисов и их методах
- **Model**: Собирает подробную информацию о типах моделей, включая поля, значения и параметры типа
- **Directive**: Парсит и интерпретирует директивы `//wails:` в исходном коде Go

### Поток сбора

1. Collector сканирует пакеты Go, указанные в проекте
2. Он определяет типы сервисов (структуры с методами, которые будут доступны на фронтенде)
3. Для каждого сервиса он собирает информацию о его методах
4. Он определяет типы моделей (структуры, используемые в качестве параметров или возвращаемых значений в методах сервисов)
5. Для каждой модели он собирает информацию о ее полях и параметрах типа
6. Он обрабатывает любые директивы `//wails:`, найденные в коде

## Процесс рендеринга

Процесс рендеринга отвечает за генерацию JavaScript/TypeScript кода на основе собранной информации. Этим занимается пакет `render`.

### Ключевые компоненты

- **Renderer**: Координирует рендеринг файлов сервисов, моделей и индексных файлов
- **Module**: Представляет один сгенерированный модуль JavaScript/TypeScript
- **Templates**: Текстовые шаблоны, используемые для генерации кода

### Поток рендеринга

1. Для каждого сервиса Renderer генерирует файл JavaScript/TypeScript с функциями, дублирующими методы сервиса
2. Для каждой модели Renderer генерирует класс JavaScript/TypeScript, дублирующий структуру модели
3. Renderer генерирует индексные файлы, которые повторно экспортируют все сервисы и модели
4. Renderer применяет любые пользовательские инъекции кода, указанные директивами `//wails:inject`

## Отображение типов

Одним из самых важных аспектов системы привязок является то, как типы Go отображаются в типы JavaScript/TypeScript. Вот краткое описание отображения:

| Тип Go | Тип JavaScript | Тип TypeScript |
|---------|----------------|----------------|
| `bool` | `boolean` | `boolean` |
| `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64` | `number` | `number` |
| `string` | `string` | `string` |
| `[]byte` | `Uint8Array` | `Uint8Array` |
| `[]T` | `Array<T>` | `T[]` |
| `map[K]V` | `Object` | `Record<K, V>` |
| `struct` | `Object` | Пользовательский класс |
| `interface{}` | `any` | `any` |
| `*T` | `T \| null` | `T \| null` |
| `func` | Не поддерживается | Не поддерживается |
| `chan` | Не поддерживается | Не поддерживается |

## Система директив

Система привязок поддерживает несколько директив, которые можно использовать для настройки сгенерированного кода. Эти директивы добавляются в виде комментариев в вашем коде Go.

### Доступные директивы

- `//wails:inject`: Вставляет пользовательский JavaScript/TypeScript код в сгенерированные привязки
- `//wails:include`: Включает дополнительные файлы вместе с сгенерированными привязками
- `//wails:internal`: Помечает тип или метод как внутренний, предотвращая его экспорт на фронтенд
- `//wails:ignore`: Полностью игнорирует метод во время генерации привязок
- `//wails:id`: Задает пользовательский ID для метода, переопределяя стандартный ID на основе хеша

### Обработка директив

1. На фазе сбора Collector идентифицирует и парсит директивы в коде Go
2. Директивы сохраняются вместе с соответствующими объявлениями (сервисами, методами, моделями и т.д.)
3. На фазе рендеринга Renderer применяет директивы для настройки сгенерированного кода

## Расширенные возможности

### Условная генерация кода

Система привязок поддерживает условную генерацию кода с использованием двухсимвольного префикса условия для директив `include` и `inject`:

```
<language><style>:<content>
```

Где:
- `<language>` может быть:
  - `*` - И JavaScript, и TypeScript
  - `j` - Только JavaScript
  - `t` - Только TypeScript

- `<style>` может быть:
  - `*` - И классы, и интерфейсы
  - `c` - Только классы
  - `i` - Только интерфейсы

Например:
```go
//wails:inject j*:console.log("JavaScript only");
//wails:inject t*:console.log("TypeScript only");
```

### Пользовательские ID методов

По умолчанию методы идентифицируются по ID на основе хеша. Однако вы можете указать пользовательский ID с помощью директивы `//wails:id`:

```go
//wails:id 42
func (s *Service) CustomIDMethod() {}
```

Это может быть полезно для поддержания совместимости при рефакторинге кода.

## Вопросы производительности

Генератор привязок разработан с учетом эффективности, но есть несколько моментов, которые стоит иметь в виду:

1. Первый запуск будет медленнее, так как формируется кэш пакетов для сканирования
2. Последующие запуски будут быстрее, так как используется кэшированная информация
3. Генератор обрабатывает все пакеты в проекте, что может занимать много времени для больших проектов
4. Вы можете использовать флаг `-clean`, чтобы очистить выходную директорию перед генерацией

## Отладка

Если вы столкнулись с проблемами при генерации привязок, вы можете использовать флаг `-v`, чтобы включить вывод отладочной информации:

```bash
wails3 generate bindings -v
```

Это предоставит подробную информацию о процессе сбора и рендеринга, что поможет выявить источник проблемы.
