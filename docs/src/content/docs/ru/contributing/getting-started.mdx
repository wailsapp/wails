## Добро пожаловать, участник!

Спасибо за ваш интерес к участию в разработке Wails! Это руководство поможет вам сделать свой первый вклад.

## Предварительные требования

Перед началом убедитесь, что у вас установлено:

- **Go 1.25+** ([скачать](https://go.dev/dl/))
- **Node.js 20+** и **npm** ([скачать](https://nodejs.org/))
- **Git**, настроенный с вашим аккаунтом GitHub
- Базовое знакомство с Go и JavaScript/TypeScript

### Требования для конкретных платформ

**macOS:**
- Xcode Command Line Tools: `xcode-select --install`

**Windows:**
- Рекомендуется MSYS2 или аналогичная среда, похожая на Unix
- WebView2 runtime (обычно предустановлен в Windows 11)

**Linux:**
- `gcc`, `pkg-config`, `libgtk-3-dev`, `libwebkit2gtk-4.0-dev`
- Установка через: `sudo apt install build-essential pkg-config libgtk-3-dev libwebkit2gtk-4.0-dev` (Debian/Ubuntu)

## Обзор процесса внесения вклада

Типичный рабочий процесс внесения вклада включает следующие шаги:

1. **Fork & Clone** - Создайте собственную копию репозитория Wails
2. **Setup** - Соберите Wails CLI и проверьте свою среду
3. **Branch** - Создайте ветку с функцией для своих изменений
4. **Develop** - Внесите изменения, следуя нашим стандартам кодирования
5. **Test** - Запустите тесты, чтобы убедиться, что всё работает
6. **Commit** - Зафиксируйте изменения с понятными сообщениями в формате conventional commit
7. **Submit** - Откройте pull request для рецензирования
8. **Iterate** - Отвечайте на обратную связь и вносите корректировки
9. **Merge** - После одобрения ваши изменения станут частью Wails!

## Пошаговое руководство

Выберите тип вашего вклада:

<Tabs>
<TabItem label="Исправление ошибок">

<Steps>

1. **Найдите или сообщите об ошибке**

   - Проверьте, не сообщалась ли уже об этой ошибке в [GitHub Issues](https://github.com/wailsapp/wails/issues)
   - Если нет, создайте новый issue с шагами для воспроизведения
   - Дождитесь подтверждения перед началом работы

2. **Создайте Fork и клонируйте**

   Создайте fork репозитория на [github.com/wailsapp/wails/fork](https://github.com/wailsapp/wails/fork)

   Клонируйте свой fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/wails.git
   cd wails
   git remote add upstream https://github.com/wailsapp/wails.git
   ```

3. **Соберите и проверьте**

   Соберите Wails и убедитесь, что можете воспроизвести ошибку:
   ```bash
   cd v3
   go build -o ../wails3 ./cmd/wails3

   # Воспроизведите ошибку, чтобы понять её
   ```

4. **Создайте ветку для исправления ошибки**

   Создайте ветку для вашего исправления:
   ```bash
   git checkout -b fix/issue-123-window-crash
   ```

5. **Исправьте ошибку**

   - Внесите минимальные изменения, необходимые для исправления ошибки
   - Не рефакторите неиспользуемый код
   - Добавьте или обновите тесты, чтобы предотвратить регрессию

   ```bash
   # Внесите свои изменения
   # Добавьте тесты в файлы *_test.go
   ```

6. **Протестируйте исправление**

   Запустите тесты, чтобы убедиться, что исправление работает:
   ```bash
   go test ./...

   # Протестируйте конкретный пакет
   go test ./pkg/application -v

   # Запустите с детектором гонок
   go test ./... -race
   ```

7. **Зафиксируйте своё исправление**

   Зафиксируйте с понятным сообщением:
   ```bash
   git commit -m "fix: предотвращение падения окна при закрытии во время инициализации

   Fixes #123"
   ```

8. **Отправьте Pull Request**

   Отправьте и создайте PR:
   ```bash
   git push origin fix/issue-123-window-crash
   ```

   В описании вашего PR:
   - Объясните ошибку и её первопричину
   - Опишите ваше исправление
   - Укажите ссылку на issue: "Fixes #123"
   - Включите поведение до и после

9. **Ответьте на обратную связь**

   Устраните замечания рецензентов и обновите ваш PR при необходимости.

<TabItem label="Улучшения">

<Steps>

1. **Обсудите функцию**

   - Откройте [GitHub Discussion](https://github.com/wailsapp/wails/discussions) или issue
   - Опишите, что вы хотите добавить и зачем
   - Дождитесь обратной связи от мейнтейнеров перед реализацией
   - Убедитесь, что это соответствует целям Wails

2. **Создайте Fork и клонируйте**

   Создайте fork репозитория на [github.com/wailsapp/wails/fork](https://github.com/wailsapp/wails/fork)

   Клонируйте свой fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/wails.git
   cd wails
   git remote add upstream https://github.com/wailsapp/wails.git
   ```

3. **Настройте среду разработки**

   Соберите Wails и проверьте свою среду:
   ```bash
   cd v3
   go build -o ../wails3 ./cmd/wails3

   # Запустите тесты, чтобы убедиться, что всё работает
   go test ./...
   ```

4. **Создайте ветку с функцией**

   Создайте описательную ветку:
   ```bash
   git checkout -b feat/window-transparency-support
   ```

5. **Реализуйте функцию**

   - Следуйте нашим [Стандартам кодирования](/ru/contributing/standards)
   - Держите изменения сосредоточенными на функции
   - Пишите чистый, документированный код
   - Добавьте комплексные тесты

   ```bash
   # Пример: Добавление нового метода окна
   # 1. Добавьте в интерфейс window.go
   # 2. Реализуйте в файлах платформ (darwin, windows, linux)
   # 3. Добавьте тесты
   # 4. Обновите документацию
   ```

6. **Тщательно протестируйте**

   Протестируйте вашу функцию:
   ```bash
   # Юнит-тесты
   go test ./pkg/application -v

   # Интеграционный тест - создайте тестовое приложение
   cd ..
   ./wails3 init -n feature-test
   cd feature-test
   # Добавьте код, использующий вашу новую функцию
   ../wails3 dev
   ```

7. **Документируйте вашу функцию**

   - Добавьте docstrings ко всем публичным API
   - Обновите соответствующую документацию в `/docs`
   - Добавьте примеры, если это применимо

8. **Зафиксируйте с соблюдением формата**

   Используйте conventional commits:
   ```bash
   git commit -m "feat: добавление поддержки прозрачности окна

   - Добавлен метод SetTransparent() в Window API
   - Реализовано для macOS, Windows и Linux
   - Добавлены тесты и документация

   Closes #456"
   ```

9. **Отправьте Pull Request**

   Отправьте и создайте PR:
   ```bash
   git push origin feat/window-transparency-support
   ```

   В вашем PR:
   - Опишите функцию и варианты использования
   - Покажите примеры или скриншоты
   - Перечислите все breaking changes
   - Укажите ссылку на обсуждение/issue

10. **Вносите корректировки на основе рецензии**

    Мейнтейнеры могут запросить изменения. Будьте терпеливы и сотрудничайте.

</Steps>

<Steps>

1. **Определите потребности в документации**

   - Нашли устаревшую документацию при использовании Wails?
   - Заметили отсутствующие примеры или объяснения?
   - Хотите исправить опечатки или улучшить ясность?
   - Проверьте [issue документации](https://github.com/wailsapp/wails/labels/documentation)

2. **Создайте Fork и клонируйте**

   Создайте fork репозитория на [github.com/wailsapp/wails/fork](https://github.com/wailsapp/wails/fork)

   Клонируйте свой fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/wails.git
   cd wails
   git remote add upstream https://github.com/wailsapp/wails.git
   ```

3. **Настройте среду документации**

   Документация находится в `/docs` и собирается с помощью Astro:
   ```bash
   cd docs
   npm install
   npm run dev
   ```

   Откройте http://localhost:4321/ для предварительного просмотра изменений в реальном времени.

4. **Создайте ветку документации**

   Создайте ветку для ваших изменений:
   ```bash
   git checkout -b docs/improve-window-api-examples
   ```

5. **Внесите свои изменения**

   Файлы документации находятся в `/docs/src/content/docs/`:
   ```bash
   # Редактируйте MDX файлы
   # Проверьте предварительный просмотр в браузере
   # Убедитесь, что форматирование корректно
   ```

   **Лучшие практики:**
   - Используйте ясный, лаконичный язык
   - Включайте практические примеры кода
   - Добавляйте ссылки на связанные разделы
   - Проверяйте орфографию и грамматику
   - Тестируйте все примеры кода

6. **Проверьте свои изменения**

   Проверьте предварительный просмотр и убедитесь, что:
   - Ссылки работают корректно
   - Примеры кода точны
   - Форматирование отображается правильно
   - Нет сломанных изображений или ссылок

7. **Зафиксируйте изменения документации**

   Зафиксируйте с понятным сообщением:
   ```bash
   git commit -m "docs: добавление практических примеров в руководство по Window API

   - Добавлены примеры позиционирования окна
   - Включен раздел с общими шаблонами
   - Исправлены сломанные ссылки на Event API"
   ```

8. **Отправьте Pull Request**

   Отправьте и создайте PR:
   ```bash
   git push origin docs/improve-window-api-examples
   ```В вашем Pull Request:
   - Опишите, какие документы вы улучшили
   - Объясните, как изменение помогает пользователям
   - Добавьте скриншоты, если изменения визуальные

9. **Учтите замечания при ревью**

   Pull Request'ы в документацию обычно быстро проходят ревью и сливаются!
