## Добро пожаловать в техническую документацию Wails v3

Этот раздел **не** посвящен правилам сообщества или тому, как открыть pull-request.
Вместо этого он погружается в **то, как устроен Wails v3**, чтобы вы могли быстро
сориентироваться в кодовой базе и уверенно приступить к разработке.

Планируете ли вы исправлять время выполнения, расширять CLI, создавать новые шаблоны или
просто понять внутреннее устройство, следующие страницы предоставят вам необходимую техническую
информацию.

На диаграмме показан **сквозной поток**:

1. **CLI** управляет генерацией, сервером разработки, компиляцией и упаковкой.  
2. **Система привязок** создает связующий код, позволяющий **Web Frontend** вызывать **Go Backend**.  
3. Во время разработки **Asset Server** проксирует запросы к серверу разработки фреймворка; в продакшене он обслуживает встроенные файлы.  
4. Во время выполнения **Desktop Runtime** управляет окнами и API ОС, а **Bridge** передает сообщения между Go и JavaScript.

## Что охватывает эта документация

| Тема | Зачем это нужно |
| ----- | -------------- |
| **Codebase Layout** | Карта директорий `/v3` и взаимодействие модулей. |
| **Runtime Internals** | Управление окнами, системные API, процессор сообщений и платформенные адаптеры. |
| **Asset & Dev Server** | Как веб-ресурсы обслуживаются в режиме разработки и встраиваются в продакшене. |
| **Build & Packaging Pipeline** | Workflow на основе Taskfile, кросс-платформенная компиляция и генерация установщиков. |
| **Binding System** | Пайплайн статического анализа, генерирующий типобезопасные привязки Go⇄TS. |
| **Template System** | Архитектура генератора, обеспечивающая работу `wails init -t <framework>`. |
| **Testing & CI** | Инструментарий для юнит/интеграционных тестов, GitHub Actions, рекомендации по детектору гонок. |
| **Extending Wails** | Добавление сервисов, шаблонов или подкоманд CLI. |

Каждая последующая страница подробно рассматривает эти области с конкретными примерами кода,
диаграммами и ссылками на соответствующие исходные файлы.
