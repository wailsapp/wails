Wails поставляется с **системой шаблонов**, которая позволяет команде `wails3 init` создавать готовый к запуску проект для **любого** современного фронтенд-стека (React, Svelte, Vue, Solid, Vanilla JS …).

На этой странице рассматриваются следующие вопросы:

1. Структура директории шаблонов  
2. Как CLI выбирает и рендерит шаблоны  
3. Пошаговое создание нового шаблона  
4. Обновление или переопределение существующих шаблонов  
5. Устранение неполадок и лучшие практики  

---

## 1. Где хранятся шаблоны

```
v3/
└── internal/templates/
    ├── _common/           # Файлы, копируемые в КАЖДЫЙ проект (main.go, .gitignore)
    ├── base/              # Шаблон "plain Go" только для бэкенда
    ├── react/             # React + Vite
    ├── react-ts/
    ├── svelte/
    ├── vue/
    ├── ... (другие)
    └── templates.go       # Реестр и вспомогательные функции
```

* **`_common/`** – универсальный шаблонный код (Go-модули, Taskfile, заглушка README,  
  настройки VS Code).  
* **Папки фреймворков** – содержат **frontend/** и любые специфичные для стека конфигурации.  
* Имена папок соответствуют **ID шаблона**, который вы передаете в CLI  
  (`wails3 init -t react-ts`).

> Вся эта директория компилируется в бинарный файл CLI с помощью `go:embed`, поэтому пользователи
> могут создавать проекты автономно (без подключения к интернету).

## 2. Как `wails3 init` использует шаблоны

Цепочка вызовов:

```
cmd/wails3/init.go
       │
       ▼
internal/commands/init.go  ← парсинг флагов, целевая директория
       │
       ▼
internal/templates.Load()  ← templates.go
       │
       ▼
Template.CopyTo(dest)      ← копирование в файловую систему + текстовые подстановки
```

### Подстановки

Плейсхолдеры, заключенные в `{{ }}`, заменяются в момент копирования:

| Плейсхолдер | Пример | Источник |
|-------------|---------|--------|
| `{{ProjectName}}` | `myapp` | Флаг CLI / имя директории |
| `{{ModulePath}}`  | `github.com/me/myapp` | если указан `--module` |
| `{{WailsVersion}}`| `v3.0.0` | скомпилированная константа |

Вы можете добавить **любой** плейсхолдер; просто убедитесь, что ему присваивается значение в
`internal/commands/init.go`.

### Хуки после копирования

Каждый шаблон может поставляться с **Taskfile**, содержащим задачу `deps`.  
После копирования CLI выполняет:

```
task --taskfile Taskfile.yml deps
```

Типичные задачи:

* `go mod tidy`
* `npm install`
* инициализация git и первое коммитирование (опционально)

Поведение хуков определено в `_common/Taskfile.yml`, но может быть переопределено для каждого шаблона.

## 3. Создание нового шаблона

> Пример: Добавление шаблона **Qwik-TS**

### 3.1 Директория и ID

```
internal/templates/qwik-ts/
```

Имя папки = ID шаблона. Сохраняйте его в формате **kebab-case**.

### 3.2 Минимальный набор файлов

```
qwik-ts/
├── README.md              # Отображается в CLI с флагом -list
├── frontend/              # Ваш веб-проект
│   ├── src/
│   ├── package.json
│   └── vite.config.ts
├── Taskfile.yml           # Задачи deps и dev
└── go/                    # Дополнительные файлы Go (опционально)
```

Начните с копирования `react-ts` и удаления лишних файлов.

### 3.3 Обновление плейсхолдеров

Поиск и замена:

* `myapp` → `{{ProjectName}}`
* `github.com/you/myapp` → `{{ModulePath}}`

### 3.4 Регистрация (опционально)

Если папка существует, **templates.go автоматически обнаруживает её** через `embed`.  
Добавляйте явный код только если вам нужна **пользовательская валидация**:

```go
func (t *Template) Validate() error {
    // проверка Node ≥ 20 для этого шаблона
}
```

### 3.5 Тестирование

```bash
wails3 init -n demo -t qwik-ts
cd demo
wails3 dev
```

Убедитесь, что:

* Сервер разработки запускается  
* Генерируются привязки `wailsjs`  
* Работает hot-reload

## 4. Изменение существующих шаблонов

1. Редактируйте файлы в `internal/templates/<id>/`.
2. Выполните `go generate ./v3/...` **или** просто `go run ./v3/cmd/wails3 –help`;
   директива embed автоматически перекомпилируется.
3. Обновите **версии зависимостей** в `frontend/package.json` и `Taskfile.yml`.
4. Обновите `README.md` внутри шаблона — пользователи видят его через  
   `wails3 init -t react --help`.

### Типичные изменения

| Задача | Где |
|------|-------|
| Изменить порт сервера разработки | `frontend/Taskfile.yml` (`vite dev --port {{DevPort}}`) |
| Добавить переменные окружения | тот же Taskfile или `.env` |
| Заменить менеджер пакетов JS | Замените `npm` на `pnpm` в Taskfile |

## 5. Советы по созданию шаблонов

* **Делайте фронтенд универсальным** – избегайте ссылок на специфичные для Wails глобальные переменные;
  среда выполнения внедряется автоматически.
* **Не включайте скомпилированные артефакты** – исключите `node_modules`, `dist`, `.DS_Store` через
  `.tmplignore`.
* **Документируйте требования** – версия Node, дополнительные CLI-инструменты и т.д.
* **Избегайте ломающих изменений** – создавайте *новый* ID шаблона, если изменения значительны.

## 6. Устранение неполадок

| Симптом | Причина | Решение |
|---------|-------|-----|
| `unknown template id` | Несоответствие имени папки | Проверьте `wails3 init -list` |
| Плейсхолдеры не заменены | Отсутствие записи в карте `Data` | Отредактируйте `internal/commands/init.go` |
| Сервер разработки открывает пустую страницу | Неверный порт `DevPort` | Убедитесь, что Taskfile выводит порт в stdout |
| Фронтенд не собирается в prod | Забытый путь `base` в vite | Установите `base: "./"` в `vite.config.ts` |

Добавьте `--verbose` к `wails3 init`, чтобы вывести информацию о каждом скопированном файле и выполненной подстановке.

---

## 7. Карта ключевых исходных файлов

| Файл | Ответственность |
|------|----------------|
| `internal/templates/templates.go` | Внедряет файловую систему шаблонов, возвращает `[]Template` |
| `internal/templates/<id>/**` | Фактическое содержимое шаблона |
| `internal/commands/init.go` | Связующее звено CLI: выбирает шаблон, заполняет плейсхолдеры |
| `internal/commands/generate_template.go` | Утилита для *экспорта* живого проекта обратно в шаблон (удобно для обновлений) |

---

## 8. Итоги

* Шаблоны находятся в **`internal/templates/`** и встроены в CLI.
* `wails3 init -t <id>` копирует `_common` + выбранный шаблон, выполняет
  подстановки, а затем запускает хук Taskfile **deps**.
* Создание шаблона так же просто, как **создание папки**, добавление файлов и
  использование плейсхолдеров.
* Система **расширяема** и **самодостаточна** — идеально подходит для обмена
  пользовательскими стеками с вашей командой или сообществом.
