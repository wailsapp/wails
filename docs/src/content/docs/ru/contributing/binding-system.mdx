> «Привязки» (Bindings) — это **контракт с проверкой типов**, который позволяет писать:

```go
msg, err := chatService.Send("Hello")
```

на Go *и*

```ts
const msg = await window.backend.ChatService.Send("Hello")
```

на TypeScript **без написания ручного связующего кода**.  
В этом документе подробно разбирается, как работает эта магия: от **статического анализа**
на этапе сборки, через **генерацию кода**, до **моста времени выполнения**, который
перемещает байты через WebView.

---

## 1. Обзор за 30 секунд

| Этап | Компонент | Выходные данные |
|-------|-----------|--------|
| **Анализ** | `internal/generator/analyse.go` | AST в памяти экспортируемых Go-структур, методов, параметров, типов возвращаемых значений |
| **Генерация** | `internal/generator/render/*.tmpl` | • `pkg/application/bindings.go` (Go)<br />• `frontend/src/wailsjs/**.ts` (определения TS)<br />• `runtime/desktop/@wailsio/runtime/internal/bindings.json` |
| **Время выполнения** | `pkg/application/messageprocessor_call.go` + JS runtime (`invoke.ts`) | JSON-сообщения через нативный мост WebView |

Поток управляется CLI `wails3`:

```
wails3 generate  ─┬─> generator.Collect()   // парсинг Go-пакетов
                  ├─> generator.Analyse()   // построение семантической модели
                  └─> generator.Render()    // запись файлов + `go fmt`
```

## 2. Статический анализ

### Точка входа

```
internal/generator/analyse.go
```

`Analyse(cfg generator.Config)`:

1. Рекурсивно загружает указанные Go-пакеты с помощью `go/packages`.
2. Обходит *типы* и *значения* в спецификациях каждого файла AST.
3. Фильтрует **экспортируемые** символы (с заглавной буквы) вне пакетов `internal/`.
4. Записывает:
   * Тип получателя (`struct`, `interface`)
   * Имя метода
   * Список параметров (имя, тип, указатель, вариативность)
   * Кортеж возвращаемых значений (значения + наличие ошибки)
5. Нормализует типы в **сериализуемый набор**  
   (`int`, `float64`, `string`, `[]byte`, срезы, карты, структуры, указатели).

Неподдерживаемые типы приводят к **ошибке компиляции**, чтобы ошибки обнаруживались на раннем этапе.

### Модель

```go
type Method struct {
    ID       uint32 // стабильный хеш для поиска во время выполнения
    Name     string
    Params   []Param
    Results  []Result
    Receiver *Struct // nil для пакетных функций
}
```

Детерминированный хеш **FNV-1a** (`internal/hash/fnv.go`) от
`<Receiver>.<Name>(<Signature>)` становится *ID метода*, используемым во время выполнения.

## 3. Генерация кода

### Шаблоны

`internal/generator/render/` содержит файлы text/template:

| Шаблон | Цель |
|----------|--------|
| `go_bindings.tmpl` | `pkg/application/bindings.go` |
| `ts_bindings.tmpl` | `frontend/wailsjs/go/models.d.ts` |
| `ts_index.tmpl`    | `frontend/wailsjs/index.ts` |

Добавляйте или изменяйте шаблоны здесь, чтобы настроить генерацию.

### Выходные данные Go

`bindings.go` регистрирует таблицу поиска:

```go
var bindings = []application.BoundMethod{
    {ID: 0x7A1201D3, Name: "ChatService.Send", Call: chatServiceSend},
}

func chatServiceSend(ctx context.Context, in []byte) ([]byte, error) {
    var req struct{ Msg string }
    if err := json.Unmarshal(in, &req); err != nil { return nil, err }
    res, err := chatService.Send(req.Msg)
    return json.Marshal(res), err
}
```

Ключевые моменты:

* **Отсутствие рефлексии** во время выполнения → высокая производительность.
* Маршалинг/унмаршалинг выполняется **для каждого метода** с использованием сгенерированных обёрток-структур.

### Выходные данные TypeScript

```ts
export namespace backend {
  export namespace ChatService {
    function Send(msg: string): Promise<string>;
  }
}
```

* Выдаётся как **ES-модули**, поэтому любой сборщик может выполнить tree-shaking.
* ID методов встраиваются в автоматически сгенерированный `bindings.json` для JS
  runtime.

## 4. Протокол вызова во время выполнения

### Сторона JavaScript

```ts
import { System } from "@wailsio/runtime";

await System.invoke(0x7a1201d3 /* ChatService.Send */, ["Hello"]);
```

Реализация: `runtime/desktop/@wailsio/runtime/invoke.ts`

1. Упаковывает `{t:"c", id:<methodID>, p:[...args]}` в JSON.
2. Вызывает `window.external.invoke(payload)` (WebView2) или эквивалент.
3. Возвращает `Promise`, который разрешается или отклоняется в зависимости от ответа.

### Сторона Go

1. `messageprocessor_call.go` получает JSON.
2. Ищет `methodID` в срезе `bindings`.
3. Выполняет сгенерированный заглушку (`chatServiceSend`).
4. Сериализует `{result, error}` обратно в JS.

### Отображение ошибок

| Go | JavaScript |
|----|------------|
| `error == nil` | `Promise` разрешается с результатом |
| `errors.New(...)` | `Promise` отклоняется с `{message, stack, code}` |

Код отображения находится в `runtime/desktop/@wailsio/runtime/errors.ts`.

## 5. Вызов JavaScript из Go

*Browser → Go* рассмотрено выше.  
*Go → Browser* использует **События** или **Eval**:

```go
window.Eval(`alert("Hi")`)
app.Publish("chat:new-message", msg)
```

Генерация привязок односторонняя (методы Go).  
Для функций, экспортируемых в JS, используйте `runtime.EventsOn` в JS и `application.Publish`
из Go.

## 6. Расширение и устранение неполадок

### Добавление пользовательских сериализаторов

* Реализуйте интерфейс `generator.TypeConverter`.
* Зарегистрируйте его в `generator.Config.Converters`.
* При необходимости обновите десериализацию в JS runtime.

### Ошибка неподдерживаемого типа

```
error: field "Client" uses unsupported type: chan struct{}
```

→ оберните канал в структуру с открытым API или переработайте дизайн.

### Рассинхронизация версий

Привязки перегенерируются при **каждом** запуске `wails3 dev` / `wails3 build`.  
Если IntelliSense IDE показывает устаревшие заглушки, удалите `frontend/wailsjs` и выполните повторную сборку.

### Советы по производительности

* Предпочитайте **значения** получателей для небольших структур, чтобы уменьшить выделения памяти.
* Избегайте передачи больших срезов байтов через мост; вместо этого используйте `application.FileServer`.
* Группируйте несколько быстрых вызовов в один метод, чтобы минимизировать задержку моста.

## 7. Карта ключевых файлов

| Задача | Файл |
|---------|------|
| Точка входа статического анализа | `internal/generator/analyse.go` |
| конвейер рендеринга | `internal/generator/generate.go` |
| Активы шаблонов | `internal/generator/render/*.tmpl` |
| Таблица привязок Go | `pkg/application/bindings.go` (сгенерировано) |
| Обработчик вызовов | `pkg/application/messageprocessor_call.go` |

| Отображение ошибок | `runtime/desktop/@wailsio/runtime/errors.ts` |

Держите эту шпаргалку под рукой, когда будете отслеживать ошибку моста.

---

## 8. Резюме

1. **Генератор** сканирует ваш Go-код → семантическая модель.  
2. **Шаблоны** выдают **заглушки Go** + **определения TypeScript**.  
3. **Обработчик сообщений** выполняет заглушки во время выполнения.  
4. **JS Runtime** оборачивает всё это в идиоматические промисы.

Всё это без рефлексии, без серверов IPC и без того, чтобы вам приходилось писать хотя бы
строку шаблонного кода. Это и есть система привязок Wails v3. Вперёд, связывайте!
