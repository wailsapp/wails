Wails позволяет пользователям перетаскивать файлы из операционной системы (файлового менеджера, рабочего стола) в ваше приложение. В отличие от HTML5 drag-and-drop, который работает только внутри браузера, это дает вам доступ к реальным путям файлов на диске.

## Включение перетаскивания файлов

По умолчанию перетаскивание файлов отключено. Чтобы включить его, установите `EnableFileDrop: true` в параметрах окна:

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:          "My App",
    Width:          800,
    Height:         600,
    EnableFileDrop: true,
})
```

Когда `EnableFileDrop` имеет значение `false` (по умолчанию), файлы, перетаскиваемые из ОС, блокируются — они не откроются в webview и не вызовут никаких событий. Это предотвращает случайную навигацию, когда пользователи перетаскивают файлы поверх вашего приложения.

## Определение зон сброса

Зоны сброса указывают Wails, какие элементы должны принимать файлы. Файлы, сброшенные вне зоны сброса, игнорируются.

Добавьте атрибут `data-file-drop-target` к любому элементу:

```html
<div id="upload" class="drop-zone" data-file-drop-target>
    Drop files here
</div>
```

Вы можете иметь несколько зон сброса. `id` и CSS-классы элемента передаются в ваш Go-код, поэтому вы можете обрабатывать сброс по-разному в зависимости от того, куда попадают файлы.

## Стилизация наведения при перетаскивании

Когда файлы перетаскиваются над зоной сброса, Wails добавляет класс `file-drop-target-active`. Это позволяет предоставить визуальную обратную связь, чтобы пользователи знали, куда они могут сбросить файлы:

```css
.drop-zone {
    border: 2px dashed #ccc;
    padding: 40px;
    text-align: center;
    transition: all 0.2s ease;
}

.drop-zone.file-drop-target-active {
    border-color: #007bff;
    background-color: rgba(0, 123, 255, 0.1);
}
```

Класс автоматически удаляется, когда файлы покидают зону или сбрасываются.

## Обнаружение сброшенных файлов

Когда файлы сбрасываются на действительную зону сброса, Wails генерирует событие `WindowFilesDropped`. Контекст события содержит полные пути к файловой системе всех сброшенных файлов:

```go
import "github.com/wailsapp/wails/v3/pkg/events"

window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    
    for _, file := range files {
        fmt.Println("Dropped:", file)
    }
})
```

Пути являются абсолютными, например, `/home/user/documents/report.pdf` или `C:\Users\Name\Documents\report.pdf`.

## Получение информации о зоне сброса

Когда у вас есть несколько зон сброса, вы можете узнать, какая из них приняла файлы, используя `DropTargetDetails()`:

```go
window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    details := event.Context().DropTargetDetails()
    
    fmt.Printf("Dropped on element: id=%s, classes=%v\n", 
        details.ElementID, details.ClassList)
    fmt.Printf("Position: x=%d, y=%d\n", details.X, details.Y)
})
```

Это позволяет направлять файлы к разным обработчикам:

```go
switch details.ElementID {
case "images":
    handleImageUpload(files)
case "documents":
    handleDocumentUpload(files)
}
```

## Полный пример

**Go:**

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:          "File Uploader",
    EnableFileDrop: true,
})

window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    details := event.Context().DropTargetDetails()
    
    // Send to frontend
    app.Event.Emit("files-dropped", map[string]any{
        "files":   files,
        "target":  details.ElementID,
    })
})
```

**HTML:**

```html
<div id="images" class="drop-zone" data-file-drop-target>
    Drop images here
</div>

<div id="documents" class="drop-zone" data-file-drop-target>
    Drop documents here
</div>

<style>
    .drop-zone {
        border: 2px dashed #ccc;
        border-radius: 8px;
        padding: 40px;
        text-align: center;
        margin: 20px;
        transition: all 0.2s ease;
    }
    
    .drop-zone.file-drop-target-active {
        border-color: #007bff;
        background-color: rgba(0, 123, 255, 0.1);
    }
</style>
```

## Сброс во всем окне

Если вы хотите, чтобы файлы можно было сбрасывать в любом месте вашего приложения, добавьте атрибут к элементу body:

```html
<body data-file-drop-target>
    <!-- Your app content -->
</body>
```

Вы можете использовать CSS-оверлей, чтобы указать, что всё окно является зоной сброса:

```css
body.file-drop-target-active::after {
    content: "Drop files anywhere";
    position: fixed;
    inset: 0;
    display: flex;
    align-items: center;
    justify-content: center;
    font-size: 24px;
    color: #007bff;
    background: rgba(255, 255, 255, 0.9);
    pointer-events: none;
}
```

## Комбинирование с HTML Drag & Drop

Вы можете использовать как внешнее перетаскивание файлов, так и внутреннее HTML drag-and-drop в одном приложении. Когда `EnableFileDrop` имеет значение `true`, Wails перехватывает внешние перетаскивания файлов, но позволяет внутренним HTML5 перетаскиваниям проходить нормально.

Чтобы различать их в обработчиках зон сброса HTML, проверьте, содержит ли перетаскивание файлы:

```javascript
zone.addEventListener('dragenter', (e) => {
    // Skip external file drags - Wails handles these
    if (e.dataTransfer?.types.includes('Files')) {
        return;
    }
    // Handle internal HTML5 drags
    zone.classList.add('drag-over');
});

zone.addEventListener('drop', (e) => {
    // Skip external file drops - Wails handles these
    if (e.dataTransfer?.types.includes('Files')) {
        return;
    }
    e.preventDefault();
    zone.classList.remove('drag-over');
    // Handle internal drop
});
```

Это гарантирует, что ваши обработчики сброса HTML реагируют только на внутренние перетаскивания (например, перемещение элементов списка), в то время как Wails обрабатывает внешние сбросы файлов отдельно через событие `WindowFilesDropped`.
