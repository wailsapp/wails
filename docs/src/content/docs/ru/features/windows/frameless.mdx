## Окна без рамки

Wails предоставляет **поддержку окон без рамки** с областями перетаскивания на основе CSS и нативным поведением платформы. Удалите нативную заголовок платформы для полного контроля над оформлением окна, пользовательскими дизайнами и уникальным пользовательским опытом, сохраняя при этом необходимую функциональность, такую как перетаскивание, изменение размера и системные элементы управления.

## Быстрый старт

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:     "Frameless App",
    Width:     800,
    Height:    600,
    Frameless: true,
})
```

**CSS для перетаскиваемой панели заголовка:**

```css
.titlebar {
    --wails-draggable: drag;
    height: 40px;
    background: #333;
}

.titlebar button {
    --wails-draggable: no-drag;
}
```

**HTML:**

```html
<div class="titlebar">
    <span>My Application</span>
    <button onclick="window.close()">×</button>
</div>
```

**Вот и всё!** У вас есть пользовательская панель заголовка.

## Создание окон без рамки

### Базовое окно без рамки

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Frameless: true,
    Width:     800,
    Height:    600,
})
```

**Что вы получаете:**
- Нет панели заголовка
- Нет границ окна
- Нет системных кнопок
- Прозрачный фон (опционально)

**Что вам нужно реализовать:**
- Область перетаскивания
- Кнопки закрытия/сворачивания/разворачивания
- Маржеры изменения размера (если окно изменяет размер)

### С прозрачным фоном

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Frameless:      true,
    BackgroundType: application.BackgroundTypeTransparent,
})
```

**Сценарии использования:**
- Скругленные углы
- Пользовательские формы
- Оверлей-окна
- Экраны приветствия (Splash screens)

## Области перетаскивания

### Перетаскивание на основе CSS

Используйте CSS-свойство `--wails-draggable`:

```css
/* Область перетаскивания */
.titlebar {
    --wails-draggable: drag;
}

/* Элементы, не подлежащие перетаскиванию, внутри области перетаскивания */
.titlebar button {
    --wails-draggable: no-drag;
}
```

**Значения:**
- `drag` - Область можно перетаскивать
- `no-drag` - Область нельзя перетаскивать (даже если родительский элемент можно)

### Полный пример панели заголовка

```html
<div class="titlebar">
    <div class="title">My Application</div>
    <div class="controls">
        <button class="minimize">−</button>
        <button class="maximize">□</button>
        <button class="close">×</button>
    </div>
</div>
```

## Системные кнопки

### Реализация кнопок Закрыть/Свернуть/Развернуть

**Сторона Go:**

```go
type WindowControls struct {
    window *application.WebviewWindow
}

func (wc *WindowControls) Minimise() {
    wc.window.Minimise()
}

func (wc *WindowControls) Maximise() {
    if wc.window.IsMaximised() {
        wc.window.UnMaximise()
    } else {
        wc.window.Maximise()
    }
}

func (wc *WindowControls) Close() {
    wc.window.Close()
}
```

**Сторона JavaScript:**

```javascript
import { Minimise, Maximise, Close } from './bindings/WindowControls'

document.querySelector('.minimize').addEventListener('click', Minimise)
document.querySelector('.maximize').addEventListener('click', Maximise)
document.querySelector('.close').addEventListener('click', Close)
```

**Или используйте методы runtime:**

```javascript
import { Window } from '@wailsio/runtime'

document.querySelector('.minimize').addEventListener('click', () => Window.Minimise())
document.querySelector('.maximize').addEventListener('click', () => Window.Maximise())
document.querySelector('.close').addEventListener('click', () => Window.Close())
```

### Переключение состояния разворачивания

Отслеживайте состояние разворачивания для иконки кнопки:

```javascript
import { Window } from '@wailsio/runtime'

async function toggleMaximise() {
    const isMaximised = await Window.IsMaximised()

    if (isMaximised) {
        await Window.Restore()
    } else {
        await Window.Maximise()
    }

    updateMaximiseButton()
}

async function updateMaximiseButton() {
    const isMaximised = await Window.IsMaximised()
    const button = document.querySelector('.maximize')
    button.textContent = isMaximised ? '❐' : '□'
}
```

## Маржеры изменения размера

### Изменение размера на основе CSS

Wails предоставляет автоматические маржеры изменения размера для окон без рамки:

```css
/* Включить изменение размера на всех краях */
body {
    --wails-resize: all;
}

/* Или на конкретных краях */
.resize-top {
    --wails-resize: top;
}

.resize-bottom {
    --wails-resize: bottom;
}

.resize-left {
    --wails-resize: left;
}

.resize-right {
    --wails-resize: right;
}

/* Углы */
.resize-top-left {
    --wails-resize: top-left;
}

.resize-top-right {
    --wails-resize: top-right;
}

.resize-bottom-left {
    --wails-resize: bottom-left;
}

.resize-bottom-right {
    --wails-resize: bottom-right;
}
```

**Значения:**
- `all` - Изменение размера со всех краев
- `top`, `bottom`, `left`, `right` - Конкретные края
- `top-left`, `top-right`, `bottom-left`, `bottom-right` - Углы
- `none` - Без изменения размера

### Пример маржера изменения размера

```html
<div class="window">
    <div class="titlebar">...</div>
    <div class="content">...</div>
    <div class="resize-handle resize-bottom-right"></div>
</div>
```

```css
.resize-handle {
    position: absolute;
    width: 16px;
    height: 16px;
}

.resize-bottom-right {
    --wails-resize: bottom-right;
    bottom: 0;
    right: 0;
    cursor: nwse-resize;
}
```

## Полный пример

Вот готовый к использованию пример окна без рамки:

**Go:**

```go
package main

import (
    _ "embed"
    "github.com/wailsapp/wails/v3/pkg/application"
)

//go:embed frontend/dist
var assets embed.FS

func main() {
    app := application.New(application.Options{
        Name: "Frameless App",
    })

    window := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:     "Frameless Application",
        Width:     1000,
        Height:    700,
        MinWidth:  800,
        MinHeight: 600,
        Frameless: true,
        
        Assets: application.AssetOptions{
            Handler: application.AssetFileServerFS(assets),
        },
        
        Mac: application.MacOptions{
            TitleBarAppearsTransparent: true,
            InvisibleTitleBarHeight:    40,
        },
        
        Windows: application.WindowsOptions{
            DisableFramelessWindowDecorations: false,
        },
    })

    window.Center()
    window.Show()

    app.Run()
}
```

```javascript
import { Window } from '@wailsio/runtime'

// Кнопка сворачивания
document.querySelector('.minimize').addEventListener('click', () => {
    Window.Minimise()
})

// Кнопка разворачивания/восстановления
const maximiseBtn = document.querySelector('.maximize')
maximiseBtn.addEventListener('click', async () => {
    const isMaximised = await Window.IsMaximised()

    if (isMaximised) {
        await Window.Restore()
    } else {
        await Window.Maximise()
    }

    updateMaximiseButton()

// Кнопка закрытия

    Window.Close()
})

// Обновление иконки кнопки разворачивания
async function updateMaximiseButton() {

    maximiseBtn.title = isMaximised ? 'Restore' : 'Maximise'

// Начальное состояние
updateMaximiseButton()
```

## Лучшие практики

### ✅ Что делать

- **Обеспечьте область перетаскивания** — Пользователи должны иметь возможность перемещать окно
- **Реализуйте системные кнопки** — Закрытие, сворачивание, разворачивание
- **Установите минимальный размер** — Предотвращает появление нерабочих макетов
- **Тестируйте на всех платформах** — Поведение может отличаться
- **Используйте CSS для областей перетаскивания** — Гибко и удобно поддерживать
- **Обеспечьте визуальную обратную связь** — Состояния наведения на кнопках

### ❌ Чего не делать

- **Не забывайте о маржерах изменения размера** — Если окно можно изменять по размеру
- **Не делайте всё окно областью перетаскивания** — Это блокирует взаимодействие
- **Не забывайте про no-drag на кнопках** — Они не будут работать---
