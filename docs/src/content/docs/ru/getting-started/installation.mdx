## Поддерживаемые платформы

- Windows 10/11 AMD64/ARM64
- macOS 10.15+ AMD64 (возможна разработка под macOS 10.13+)
- macOS 11.0+ ARM64
- Ubuntu 24.04 AMD64/ARM64 (другие дистрибутивы Linux также могут работать)

## Зависимости

Для работы Wails требуются следующие общие зависимости, которые необходимо установить перед началом:

<Tabs>
  <TabItem label="Go (не ниже 1.24)" icon="seti:go">

    Скачайте Go со [страницы загрузок Go](https://go.dev/dl/).

    Обязательно следуйте официальной [инструкции по установке Go](https://go.dev/doc/install). Кроме того, убедитесь, что переменная среды `PATH` содержит путь к вашей директории `~/go/bin`. Перезапустите терминал и выполните следующие проверки:

    - Проверьте корректность установки Go: `go version`
    - Убедитесь, что `~/go/bin` находится в переменной PATH
        - Mac / Linux: `echo $PATH | grep go/bin`
        - Windows: `$env:PATH -split ';' | Where-Object { $_ -like '*\go\bin' }`

  </TabItem>

  <TabItem label="npm (необязательно)" icon="seti:npm">

    Хотя для работы Wails установка npm не требуется, большинство встроенных шаблонов используют его.

    Скачайте установщик последней версии Node со [страницы загрузок Node](https://nodejs.org/en/download/). Рекомендуется использовать последнюю стабильную версию, так как именно она чаще всего используется при тестировании.

    Выполните `npm --version` для проверки.

      :::note
      Если вы предпочитаете другой менеджер пакетов вместо npm, вы можете использовать его. Вам потребуется обновить Taskfiles проекта для его использования.
      :::

  </TabItem>

</Tabs>

## Специфичные для платформы зависимости

Вам также необходимо установить зависимости, специфичные для вашей платформы:

<Tabs syncKey="platform">
  <TabItem label="Mac" icon="apple">

    Для работы Wails требуются инструменты командной строки Xcode. Их можно установить, выполнив:

    ```sh
    xcode-select --install
    ```

  </TabItem>
  <TabItem label="Windows" icon="seti:windows">

    Для работы Wails требуется установленный [WebView2 Runtime](https://developer.microsoft.com/en-us/microsoft-edge/webview2/). На большинстве систем Windows он уже установлен. Вы можете проверить это с помощью команды `wails doctor`.

  </TabItem>
  <TabItem label="Linux" icon="linux">

    Для Linux требуются стандартные инструменты сборки `gcc`, а также `gtk3` и `webkit2gtk`. Выполните <code>wails doctor</code> после установки, чтобы узнать, как установить необходимые зависимости. Если ваш дистрибутив или менеджер пакетов не поддерживается, пожалуйста, сообщите нам об этом в Discord.

  </TabItem>

</Tabs>

## Установка

Чтобы установить CLI Wails с помощью Go Modules, выполните следующие команды:

```shell
go install -v github.com/wailsapp/wails/v3/cmd/wails3@latest
```

Если вы хотите установить последнюю версию для разработки, выполните следующие
команды:

```shell
git clone https://github.com/wailsapp/wails.git
cd wails
cd v3/cmd/wails3
go install
```

При использовании версии для разработки все сгенерированные проекты будут использовать
директиву [replace](https://go.dev/ref/mod#go-mod-file-replace) Go, чтобы обеспечить
использование версии Wails для разработки.

## Проверка системы

Выполнение команды `wails3 doctor` проверит, установлены ли у вас правильные
зависимости. Если нет, она подскажет, чего не хватает, и поможет устранить
возникшие проблемы.

## Команда `wails3` отсутствует?

Если ваша система сообщает, что команда `wails3` отсутствует, проверьте
следующее:

- Убедитесь, что вы правильно следовали приведенному выше `руководству по установке Go` и
  что директория `go/bin` находится в переменной среды `PATH`.
- Закройте/откройте текущие терминалы, чтобы применить новую переменную `PATH`.
