## Démarrage rapide

<Tabs syncKey="os">
  <TabItem label="macOS" icon="apple">
```bash
# Installer Wails
go install github.com/wailsapp/wails/v3/cmd/wails3@latest

# Créer votre application
wails3 init -n myapp -t vanilla

# Lancer avec rechargement à chaud
cd myapp && wails3 dev
```
  </TabItem>
  
  <TabItem label="Windows" icon="seti:windows">
```powershell
# Installer Wails
go install github.com/wailsapp/wails/v3/cmd/wails3@latest

# Créer votre application
wails3 init -n myapp -t vanilla

# Lancer avec rechargement à chaud
cd myapp; wails3 dev
```
  </TabItem>
  
  <TabItem label="Linux" icon="linux">

  </TabItem>

## Pourquoi Wails ?

<CardGrid>
  <Card title="Des performances que les utilisateurs remarquent" icon="rocket">
    - Binaires d'environ 15 Mo contre 150 Mo pour Electron
    - Mémoire de base d'environ 10 Mo contre 100 Mo+
    - Temps de démarrage < 0,5 s contre 2-3 s
    - Rendu natif utilisant le WebView du système d'exploitation
    - Aucun surcoût de navigateur intégré
  </Card>
  
  <Card title="Expérience développeur" icon="setting">
    - Un seul codebase Go pour toutes les plateformes
    - N'importe quel framework web - React, Vue, Svelte
    - Rechargement à chaud pendant le développement
    - Liaisons générées automatiquement pour appeler facilement Go depuis Javascript
    - IPC en mémoire. Aucun port réseau
  </Card>
  
  <Card title="Prêt pour la production" icon="approve-check">
    - Fenêtres multiples avec cycles de vie
    - Menus natifs et zone de notification système
    - Boîtes de dialogue de fichiers natives à la plateforme
    - Intégration système et raccourcis
    - Outils de signature de code et de packaging
  </Card>
  
  <Card title="Natif multi-plateforme" icon="laptop">
    - Un seul codebase pour Windows, macOS, Linux
    - Fonctionnalités spécifiques à la plateforme si nécessaire
    - Aucun compromis sur l'expérience utilisateur
    - Déploiement sur toutes les plateformes depuis un seul build
    - Support mobile bientôt disponible...
  </Card>
</CardGrid>
