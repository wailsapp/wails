# Utilisation de Gin pour les Services

Le framework web Gin est un choix populaire pour construire des services HTTP en Go. Avec Wails v3, vous pouvez facilement intégrer des services basés sur Gin dans votre application, offrant ainsi un moyen puissant de gérer les requêtes HTTP, d'implémenter des API RESTful et de servir du contenu web.

Ce guide vous guidera étape par étape pour créer un service basé sur Gin qui peut être monté sur une route spécifique de votre application Wails. Nous construirons un exemple complet qui démontre comment :

1. Créer un service basé sur Gin
2. Implémenter l'interface Service de Wails
3. Configurer les routes et les middlewares
4. Intégrer avec le système d'événements de Wails
5. Interagir avec le service depuis le frontend

## Prérequis

Avant de commencer, assurez-vous d'avoir :

- Wails v3 installé
- Des connaissances de base en Go et sur le framework Gin
- Une familiarité avec les concepts HTTP et les API RESTful

Vous devrez ajouter le framework Gin à votre projet :

```bash
go get github.com/gin-gonic/gin
```

## Création d'un service basé sur Gin

Commençons par créer un service Gin qui implémente l'interface Service de Wails. Notre service gérera une collection d'utilisateurs et fournira des points de terminaison (endpoints) d'API pour récupérer et créer des enregistrements d'utilisateurs.

### 1. Définir vos modèles de données

D'abord, définissez les structures de données avec lesquelles votre service travaillera :

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User représente un utilisateur dans le système
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData représente les données envoyées dans les événements
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. Créer la structure de votre service

Ensuite, définissez la structure du service qui contiendra votre routeur Gin et tout état que votre service doit maintenir :

```go
// GinService implémente un service Wails qui utilise Gin pour la gestion HTTP
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService crée une nouvelle instance de GinService
func NewGinService() *GinService {
	// Crée un nouveau routeur Gin
	ginEngine := gin.New()

	// Ajoute des middlewares
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// Définit les routes
	service.setupRoutes()

	return service
}
```

### 3. Implémenter l'interface Service

Implémentez les méthodes requises pour l'interface Service de Wails :

```go
// ServiceName retourne le nom du service
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup est appelé lorsque le service démarre
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// Stocke l'instance de l'application pour une utilisation ultérieure
	s.app = application.Get()

	// Enregistre un gestionnaire d'événements qui peut être déclenché depuis le frontend
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// Journalise les données de l'événement
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// Émet un événement en retour vers le frontend
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown est appelé lorsque le service s'arrête
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// Nettoyage des ressources si nécessaire
	return nil
}
```

### 3. Implémenter l'interface http.Handler

Pour rendre votre service montable sur une route spécifique, implémentez l'interface `http.Handler`. Cette méthode unique, `ServeHTTP`, est la passerelle pour toutes les requêtes HTTP vers votre service. Elle délègue la gestion des requêtes au routeur Gin, vous permettant d'utiliser toutes les fonctionnalités puissantes de Gin pour le routage et les middlewares.

```go
// ServeHTTP implémente l'interface http.Handler
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// Toutes les requêtes vont au routeur Gin
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. Configurer vos routes

Définissez vos routes d'API dans une méthode séparée pour une meilleure organisation. Cette approche garde votre code propre et facilite la compréhension de la structure de votre API. Le routeur Gin fournit une API fluide pour définir les routes, y compris le support des groupes de routes, qui aident à organiser les points de terminaison (endpoints) liés.

```go
// setupRoutes configure les routes de l'API
func (s *GinService) setupRoutes() {
	// Endpoint d'information basique
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// Groupe Users
	users := s.ginEngine.Group("/users")
	{
		// Récupérer tous les utilisateurs
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// Récupérer un utilisateur par ID
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// Créer un nouvel utilisateur
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// Définir l'ID et l'heure de création
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// Ajouter au slice users
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// Émettre un événement pour notifier de l'utilisateur créé
			s.app.Event.Emit("user-created", newUser)
		})

		// Supprimer un utilisateur
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// Supprimer l'utilisateur du slice
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. Créer des middlewares personnalisés

Vous pouvez créer des middlewares Gin personnalisés pour améliorer votre service. Les fonctions middleware dans Gin sont exécutées dans l'ordre où elles sont ajoutées au routeur et peuvent effectuer des tâches telles que la journalisation, l'authentification et la gestion des erreurs. Cet exemple montre un middleware de journalisation simple qui enregistre les détails de la requête.

```go
// LoggingMiddleware est un middleware Gin qui journalise les détails de la requête
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// Démarre le chronomètre
		start := time.Now()

		// Traite la requête
		c.Next()

		// Calcule la latence
		latency := time.Since(start)

		// Journalise les détails de la requête
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## Enregistrement de votre servicePour utiliser votre service basé sur Gin dans une application Wails, vous devez l'enregistrer auprès de l'application et spécifier la route sur laquelle il doit être monté. Cela se fait lors de la création de l'instance de l'application Wails. La route que vous spécifiez devient le chemin de base pour tous les points de terminaison définis dans votre routeur Gin.

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

Dans cet exemple, le service Gin est monté sur la route `/api`. Cela signifie que si votre routeur Gin a un point de terminaison défini comme `/info`, il sera accessible à `/api/info` dans votre application. Cette approche vous permet d'organiser logiquement les points de terminaison de votre API et d'éviter les conflits avec d'autres parties de votre application.

## Intégration avec le système d'événements Wails

L'une des fonctionnalités puissantes de l'utilisation de Gin avec les services Wails est la capacité de s'intégrer de manière transparente au système d'événements Wails. Cela permet une communication en temps réel entre votre service backend et le frontend.

Dans la méthode `ServiceStartup` de votre service, vous pouvez enregistrer des gestionnaires d'événements pour traiter les événements provenant du frontend :

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

Vous pouvez également émettre des événements vers le frontend depuis vos routes Gin ou d'autres parties de votre service :

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## Intégration Frontend

Pour interagir avec votre service Gin depuis le frontend, vous devrez importer le runtime Wails, effectuer des requêtes HTTP vers les points de terminaison de votre API et utiliser le système d'événements Wails pour la communication en temps réel.

Pour une utilisation en production, il est recommandé d'utiliser le package `@wailsio/runtime` au lieu d'importer directement `/wails/runtime.js`. Cela garantit la sécurité des types, un meilleur support IDE, la gestion des versions via npm et la compatibilité avec les outils JavaScript modernes.

Installez le package :
```bash
npm install @wailsio/runtime
```

Ensuite, utilisez-le dans votre code :
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## Réflexions finales

L'intégration du framework web Gin avec les Services de Wails v3 offre une approche puissante et flexible pour construire des applications web modulaires et maintenables. En tirant parti des capacités de routage et de middleware de Gin ainsi que du système d'événements de Wails, vous pouvez créer des applications riches et interactives avec une séparation claire des responsabilités.

Le code complet de l'exemple de ce guide est disponible dans le dépôt Wails sous `v3/examples/gin-service`.
