Wails v3 est un **framework desktop full-stack** composé d'un runtime Go,
d'un pont JavaScript, d'une chaîne d'outils orientée tâches et d'une collection de templates qui
vous permettent de **publier** des applications natives propulsées par les technologies web modernes.

Cette page présente la *vue d'ensemble* à travers quatre diagrammes :

1. **Architecture globale** – comment chaque sous-système est connecté  
2. **Flux d'exécution** – ce qui se passe lorsque JS appelle Go et vice-versa  
3. **Développement vs Production** – deux modes du serveur d'actifs  
4. **Implémentations par plateforme** – où se trouve le code spécifique à l'OS

## 4 · Séparation du runtime spécifique à la plateforme

**Fichiers runtime par OS**

**[Placeholder du diagramme de la séparation par plateforme]**

Chaque fonctionnalité suit ce schéma :

1. **Interface commune** dans `pkg/application`  
2. **Entrée du processeur de messages** dans `pkg/application/messageprocessor_*.go`  
3. **Implémentation** par OS sous `internal/runtime/*.go` protégée par des tags de compilation  

Une fonctionnalité manquante sur un OS doit retourner `ErrCapability` et enregistrer
la disponibilité via `internal/capabilities`.
