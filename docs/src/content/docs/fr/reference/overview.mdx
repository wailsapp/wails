## Conventions de l'API

<details>
<summary><strong>Conventions de l'API Go</strong> - Pour les développeurs débutants avec Go</summary>

#### Nommage

- **Types** : PascalCase (par ex., `WebviewWindow`)
- **Méthodes** : PascalCase (par ex., `SetTitle()`)
- **Options** : Structs en PascalCase (par ex., `WindowOptions`)
- **Constantes** : PascalCase (par ex., `WindowStartStateMaximised`)

#### Gestion des erreurs

La plupart des méthodes qui peuvent échouer retournent `error` comme dernière valeur de retour :

```go
window, err := app.Window.New()
if err != nil {
    log.Fatal(err)
}
```

#### Contexte

Les méthodes qui peuvent bloquer ou nécessiter un annulation acceptent `context.Context` :

```go
err := app.RunWithContext(ctx)
```

#### Modèle d'options

La configuration utilise des structs d'options :

```go
app := application.New(application.Options{
    Name: "My App",
    Description: "A demo application",
    Services: []application.Service{
        application.NewService(&MyService{}),
    },
})
```

</details>

### Conventions de l'API JavaScript

#### Nommage

- **Fonctions** : camelCase (par ex., `setTitle()`)
- **Constantes** : SCREAMING_SNAKE_CASE (par ex., `WINDOW_EVENT_FOCUS`)

#### Asynchrone par défaut

Tous les appels de méthodes Go retournent des Promises :

```javascript
// Async/await (recommandé)
const result = await MyService.DoSomething()

// Chaîne de Promises
MyService.DoSomething()
    .then(result => console.log(result))
    .catch(error => console.error(error))
```

#### Gestion des erreurs

Les erreurs Go deviennent des exceptions JavaScript :

```javascript
try {
    await MyService.MightFail()
} catch (error) {
    console.error('Go error:', error)
}
```

#### Sécurité des types

Les définitions TypeScript sont générées automatiquement :

```typescript
// Entièrement typé
import { Greet } from './bindings/GreetService'

const message: string = await Greet("World")
```

## Différences entre plateformes

Certaines APIs se comportent différemment selon les plateformes :

| Fonctionnalité | Windows | macOS | Linux |
|---------|---------|-------|-------|
| **Menu de l'application** | Barre de menu de la fenêtre | Barre de menu globale | Barre de menu de la fenêtre |
| **Barre d'état système** | Zone de notification | Barre de menu | Barre d'état système |
| **Dock** | N/A | ✅ Disponible | N/A |
| **Dialogues de fichiers** | Natif | Natif | Natif (GTK) |
| **Transparence** | ✅ Complète | ✅ Complète | ⚠️ Limitée |

Les comportements spécifiques à chaque plateforme sont documentés dans chaque section de l'API.

## Gestion des versions

Wails v3 suit le versionnement sémantique :

- **Majeur** (v3.x.x) : Changements non rétrocompatibles
- **Mineur** (v3.x.x) : Nouvelles fonctionnalités, rétrocompatible
- **Correctif** (v3.x.x) : Corrections de bugs, rétrocompatible

**Statut actuel :** Alpha (API stable, affinement en cours)

## Politique de dépréciation

Lorsque des APIs sont dépréciées :

1. **Marquées dans la documentation** avec un avis de dépréciation
2. **Alternative fournie** avec un guide de migration
3. **Maintenues pendant 1 version majeure** avant suppression
4. **Avertisseurs du compilateur** (si possible)

## Stabilité de l'API

### APIs stables ✅

Ces APIs sont stables et sûres pour une utilisation en production :

- APIs principales de l'application
- Gestion des fenêtres
- Système de menus
- Système d'événements
- Dialogues de fichiers
- Bindings de services

### APIs instables ⚠️

Ces APIs peuvent changer avant la version finale :

- Certaines options avancées de fenêtre
- Fonctionnalités spécifiques à la plateforme
- Fonctionnalités expérimentales

Les APIs instables sont marquées dans la documentation.
