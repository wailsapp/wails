Le runtime frontend Wails est la bibliothèque standard pour les applications Wails. Il fournit
un certain nombre de fonctionnalités qui peuvent être utilisées dans vos applications, notamment :

- Gestion des fenêtres
- Boîtes de dialogue
- Intégration du navigateur
- Presse-papiers
- Menus
- Informations système
- Événements
- Menus contextuels
- Écrans
- WML (Wails Markup Language)

Le runtime est nécessaire pour l'intégration entre Go et le frontend. Il existe 2
façons d'intégrer le runtime :

- En utilisant le package `@wailsio/runtime`
- En utilisant un bundle pré-construit

## Utilisation du package npm

Le package `@wailsio/runtime` est un package JavaScript qui permet d'accéder au
runtime Wails depuis le frontend. Il est utilisé par tous les gabarits standards
et est la méthode recommandée pour intégrer le runtime dans votre application.
En utilisant le package `@wailsio/runtime`, vous n'inclurez que les parties du runtime que vous utilisez.

Le package est disponible sur npm et peut être installé en utilisant :

```shell
npm install --save @wailsio/runtime
```

## Utilisation d'un bundle pré-construit

Certains projets n'utiliseront pas de bundler JavaScript et pourraient préférer utiliser une
version bundle pré-construite du runtime. Cette version peut être générée localement
en utilisant la commande suivante :

```shell
wails3 generate runtime
```

La commande produira un fichier `runtime.js` (et `runtime.debug.js`) dans le répertoire courant.
Ce fichier est un module ES qui peut être importé par les scripts de votre application
comme le package npm, mais l'API est également exportée vers l'objet global window,
donc pour des applications plus simples, vous pouvez l'utiliser comme suit :

```html
<html>
    <head>
        <script type="module" src="./runtime.js"></script>
        <script>
            window.onload = function () {
                wails.Window.SetTitle("A new window title");
            }
        </script>
    </head>
    <!--- ... -->
</html>
```

:::caution
Il est important d'inclure l'attribut `type="module"` sur la balise `<script>` qui charge le runtime
et d'attendre que la page soit entièrement chargée avant d'appeler l'API,
car les scripts avec l'attribut `type="module"` s'exécutent de manière asynchrone.
:::

## Initialisation

En plus des fonctions de l'API, le runtime fournit un support pour les menus contextuels et le déplacement des fenêtres.
Ces fonctionnalités ne fonctionneront correctement qu'après que le runtime a été initialisé.
Même si vous n'utilisez pas l'API, assurez-vous d'inclure une instruction d'importation avec effets secondaires
quelque part dans votre code frontend :

```javascript
import "@wailsio/runtime";
```

Votre bundler devrait détecter la présence d'effets secondaires et inclure
tout le code d'initialisation requis dans la construction.

:::note
Si vous préférez le bundle pré-construit, ajouter une balise script comme montré ci-dessus suffit.
:::

## Plugin Vite pour les événements typés

Le runtime inclut un plugin Vite qui permet le support du HMR (Hot Module Replacement) pour les événements typés pendant le développement.

### Configuration

Ajoutez le plugin à votre `vite.config.ts` :

```typescript
import { defineConfig } from 'vite'
import wails from '@wailsio/runtime/plugins/vite'

export default defineConfig({
  plugins: [wails()],
})
```

### Avantages

- **Rechargement automatique** : Les liaisons d'événements sont automatiquement régénérées et rechargées lorsque vous exécutez `wails3 generate bindings`
- **Mode développement** : Fonctionne de manière transparente avec `wails3 dev` pour des mises à jour instantanées
- **Sécurité des types** : Support complet de TypeScript avec autocomplétion et vérification des types

### Utilisation avec l'enregistrement d'événements

Enregistrez vos événements dans Go :

```go
type UserData struct {
    ID   string
    Name string
}

func init() {
    application.RegisterEvent[UserData]("user-updated")
}
```

Générez les liaisons :

```bash
wails3 generate bindings

# Ou, pour inclure les définitions TypeScript
wails3 generate bindings -ts

# Pour plus d'options, voir :
wails3 generate bindings -help
```

Utilisez les événements typés dans votre frontend :

```typescript
import { Events } from '@wailsio/runtime'
import { UserUpdated } from './bindings/events'

// Événement typé avec autocomplétion
Events.Emit(UserUpdated({
    ID: "123",
    Name: "John Doe"
}))
```

## Référence de l'API

Le runtime est organisé en modules, chacun fournissant des fonctionnalités spécifiques. Importez uniquement ce dont vous avez besoin :

```javascript
import { Events, Window, Clipboard } from '@wailsio/runtime'
```

### Événements

Système d'événements pour la communication entre Go et JavaScript.

#### On()

Enregistrer un rappel pour un événement.

```typescript
function On(eventName: string, callback: (event: WailsEvent) => void): () => void
function On<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**Retourne :** Fonction de désabonnement

**Exemple :**

```javascript
import { Events } from '@wailsio/runtime'

// Écoute d'événement basique
const unsubscribe = Events.On('user-logged-in', (event) => {
    console.log('User:', event.data.username)
})

// Avec événements typés (TypeScript)
import { UserLogin } from './bindings/events'

Events.On(UserLogin, (event) => {
    // event.data est typé comme UserLoginData
    console.log('User:', event.data.username)
})

// Plus tard : unsubscribe()
```

#### Once()

Enregistrer un rappel qui s'exécute une seule fois.

```typescript
function Once(eventName: string, callback: (event: WailsEvent) => void): () => void
function Once<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**Exemple :**

```javascript
import { Events } from '@wailsio/runtime'

Events.Once('app-ready', () => {
    console.log('App initialized')
})
```

#### Emit()

Émettre un événement vers le backend Go ou d'autres fenêtres.

```typescript
function Emit(name: string, data?: any): Promise<boolean>
function Emit<T>(event: Event<T>): Promise<boolean>
```

**Retourne :** Promise qui se résout à `true` si l'événement a été annulé, `false` sinon

**Exemple :**

```javascript
import { Events } from '@wailsio/runtime'

// Émission d'événement basique
const wasCancelled = await Events.Emit('button-clicked', { buttonId: 'submit' })

// Avec événements typés (TypeScript)
import { UserLogin } from './bindings/events'

const cancelled = await Events.Emit(UserLogin({
    UserID: "123",
    Username: "john_doe",
    LoginTime: new Date().toISOString()
}))

if (cancelled) {
    console.log('Login was cancelled by a hook')
}
```

:::note
La valeur de retour indique si l'événement a été annulé par un hook. La plupart des événements ne peuvent pas être annulés et retourneront toujours `false`.
:::

#### Off()

Supprimer les écouteurs d'événements.

```typescript
function Off(...eventNames: string[]): void
```

**Exemple :**

```javascript
import { Events } from '@wailsio/runtime'

Events.Off('user-logged-in', 'user-logged-out')
```

#### OffAll()

Supprimer tous les écouteurs d'événements.

```typescript
function OffAll(): void
```

### Système

Méthodes système de bas niveau pour la communication directe avec le backend.

#### invoke()

Envoie un message brut directement au backend. Cela contourne le système de liaison standard et est géré par le `RawMessageHandler` dans les options de votre application.

```typescript
function invoke(message: any): void
```

**Exemple :**

```javascript
import { System } from '@wailsio/runtime'

// Envoyer un message brut au backend
System.invoke('my-custom-message')

// Envoyer des données structurées en JSON
System.invoke(JSON.stringify({ action: 'update', value: 42 }))
```

:::caution
Il s'agit d'une fonction « fire-and-forget » sans valeur de retour. Utilisez les événements pour recevoir des réponses du backend.
:::

Pour plus de détails, consultez le [Guide des messages bruts](/fr/guides/raw-messages).

### WML (Wails Markup Language)

WML fournit des attributs déclaratifs pour les actions courantes. Ajoutez des attributs aux éléments HTML :

#### Attributs

**wml-event** - Émet un événement lors du clic

```html<button wml-event="save-clicked">Enregistrer</button>
```

**wml-window** - Appelle une méthode de fenêtre

```html
<button wml-window="Close">Fermer la fenêtre</button>
<button wml-window="Minimise">Minimiser</button>
```

**wml-target-window** - Spécifie la fenêtre cible pour wml-window

```html
<button wml-window="Show" wml-target-window="settings">
    Afficher les paramètres
</button>
```

**wml-openurl** - Ouvre une URL dans le navigateur

```html
<a href="#" wml-openurl="https://wails.io">Visiter Wails</a>
```

**wml-confirm** - Affiche une boîte de dialogue de confirmation avant l'action

```html
<button wml-window="Close" wml-confirm="Are you sure you want to close?">
    Fermer
</button>
```

**Exemple :**

```html
<div>
    <button wml-event="save-clicked">Enregistrer</button>
    <button wml-window="Minimise">Minimiser</button>
    <button wml-window="Close" wml-confirm="Fermer la fenêtre ?">Fermer</button>
    <a href="#" wml-openurl="https://github.com/wailsapp/wails">GitHub</a>
</div>
```

## Exemple complet

```javascript
import { Events, Window, Clipboard, Dialogs, Screens } from '@wailsio/runtime'

// Écouter les événements provenant de Go
Events.On('data-updated', (event) => {
    console.log('Data:', event.data)
    updateUI(event.data)
})

// Gestion des fenêtres
document.getElementById('center-btn').addEventListener('click', async () => {
    await Window.Center()
})

document.getElementById('fullscreen-btn').addEventListener('click', async () => {
    const isFullscreen = await Window.IsFullscreen()
    if (isFullscreen) {
        await Window.UnFullscreen()
    } else {
        await Window.Fullscreen()
    }
})

// Opérations du presse-papiers
document.getElementById('copy-btn').addEventListener('click', async () => {
    await Clipboard.SetText('Copied from Wails!')
})

// Dialogue avec confirmation
document.getElementById('delete-btn').addEventListener('click', async () => {
    const result = await Dialogs.Question({
        Title: 'Confirm',
        Message: 'Delete this item?',
        Buttons: [
            { Label: 'Delete' },
            { Label: 'Cancel', IsDefault: true }
        ]
    })

    if (result === 'Delete') {
        await Events.Emit('delete-item', { id: currentItemId })
    }
})

// Informations sur les écrans
const screens = await Screens.GetAll()
console.log(`Detected ${screens.length} screen(s)`)
screens.forEach(screen => {
    console.log(`- ${screen.Name}: ${screen.Size.Width}x${screen.Size.Height}`)
})
```

## Bonnes pratiques

### ✅ À faire

- **Importer sélectivement** - Importez uniquement ce dont vous avez besoin
- **Gérer les promesses** - Toutes les méthodes retournent des promesses
- **Utiliser WML pour les actions simples** - Plus propre que JavaScript
- **Vérifier les valeurs de retour** - Surtout pour les dialogues
- **Se désabonner des événements** - Nettoyez lorsque vous avez terminé

### ❌ À ne pas faire

- **N'oubliez pas le await** - La plupart des méthodes sont asynchrones
- **Ne bloquez pas l'interface utilisateur** - Utilisez correctement async/await
- **N'ignorez pas les erreurs** - Gérez toujours les rejets

## Support TypeScript

Le runtime inclut des définitions TypeScript complètes :

```typescript
import { Events, Window } from '@wailsio/runtime'

Events.On('custom-event', (event) => {
    // TypeScript connaît event.data, event.name, event.sender
    console.log(event.data)
})

// Toutes les méthodes sont entièrement typées
const size: { width: number, height: number } = await Window.Size()
```
