2. **Créer le service TODO**

   Le service TODO gérera l'état de notre application et fournira des méthodes pour les opérations CRUD. Contrairement à un serveur web où chaque requête est isolée, les applications de bureau peuvent avoir plusieurs opérations concurrentes, nous avons donc besoin d'une gestion d'état thread-safe.

   Supprimez `greetservice.go` et créez un nouveau fichier `todoservice.go` :

   ```go title="todoservice.go"
   package main

   import (
       "errors"
       "sync"
   )

   type Todo struct {
       ID        int    `json:"id"`
       Title     string `json:"title"`
       Completed bool   `json:"completed"`
   }

   type TodoService struct {
       todos  []Todo
       nextID int
       mu     sync.RWMutex
   }

   func NewTodoService() *TodoService {
       return &TodoService{
           todos:  []Todo{},
           nextID: 1,
       }
   }

   func (t *TodoService) GetAll() []Todo {
       t.mu.RLock()
       defer t.mu.RUnlock()
       return t.todos
   }

   func (t *TodoService) Add(title string) (*Todo, error) {
       if title == "" {
           return nil, errors.New("title cannot be empty")
       }

       t.mu.Lock()
       defer t.mu.Unlock()

       todo := Todo{
           ID:        t.nextID,
           Title:     title,
           Completed: false,
       }
       t.todos = append(t.todos, todo)
       t.nextID++

       return &todo, nil
   }

   func (t *TodoService) Toggle(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i := range t.todos {
           if t.todos[i].ID == id {
               t.todos[i].Completed = !t.todos[i].Completed
               return nil
           }
       }
       return errors.New("todo not found")
   }

   func (t *TodoService) Delete(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i, todo := range t.todos {
           if todo.ID == id {
               t.todos = append(t.todos[:i], t.todos[i+1:]...)
               return nil
           }
       }
       return errors.New("todo not found")
   }
   ```

   **Ce qui se passe ici :**

   **La structure `Todo` :**
   - Définit la forme de nos données avec les champs ID, Title et Completed
   - Les balises `json:` indiquent à Go comment convertir cette structure en JSON pour le frontend
   - Chaque champ est exporté (en majuscule) afin que le générateur de liaisons puisse le voir

   **La structure `TodoService` :**
   - `todos []Todo` - un slice contenant tous nos éléments TODO
   - `nextID int` - suit le prochain ID à attribuer (simule l'incrémentation automatique)
   - `mu sync.RWMutex` - un mutex en lecture/écriture pour un accès thread-safe

   **Sécurité des threads avec `sync.RWMutex` :**
   - Les applications de bureau peuvent avoir plusieurs opérations concurrentes provenant de l'interface utilisateur
   - `RLock()` permet plusieurs lecteurs simultanés (par exemple, plusieurs appels `GetAll`)
   - `Lock()` donne un accès exclusif pour les écritures (par exemple, `Add`, `Toggle`, `Delete`)
   - `defer` garantit que les verrous sont libérés même si la fonction retourne prématurément ou plante

   **Les méthodes :**
   - `GetAll()` - Retourne tous les todos (utilise un verrou de lecture car nous ne modifions pas les données)
   - `Add(title)` - Crée un nouveau todo, valide l'entrée, incrémente l'ID
   - `Toggle(id)` - Inverse le statut de complétion d'un todo
   - `Delete(id)` - Supprime un todo du slice

   **Gestion des erreurs :**
   - Nous retournons `error` comme dernière valeur en suivant les conventions de Go
   - Les titres vides sont rejetés
   - Les opérations sur des todos inexistants retournent des erreurs
   - Ces erreurs deviennent des exceptions JavaScript dans le frontend

4. **Créer l'interface utilisateur frontend**

   Construisons maintenant le frontend. C'est ici que nous appellerons nos méthodes Go et afficherons l'interface utilisateur. Nous utilisons JavaScript vanilla pour garder les choses simples et vous montrer comment les liaisons fonctionnent directement.

   Remplacez `frontend/src/main.js` :

   ```javascript title="frontend/src/main.js"
   import {TodoService} from "../bindings/changeme";

   async function loadTodos() {
       const todos = await TodoService.GetAll();
       const list = document.getElementById('todo-list');

       list.innerHTML = todos.map(todo => `
           <div class="todo ${todo.completed ? 'completed' : ''}">
               <input type="checkbox"
                      ${todo.completed ? 'checked' : ''}
                      onchange="toggleTodo(${todo.id})">
               <span>${todo.title}</span>
               <button onclick="deleteTodo(${todo.id})">Delete</button>
           </div>
       `).join('');
   }

   window.addTodo = async () => {
       const input = document.getElementById('todo-input');
       const title = input.value.trim();

       if (title) {
           await TodoService.Add(title);
           input.value = '';
           await loadTodos();
       }
   }

   window.toggleTodo = async (id) => {
       await TodoService.Toggle(id);
       await loadTodos();
   }

   window.deleteTodo = async (id) => {
       await TodoService.Delete(id);
       await loadTodos();
   }

   // Load todos on startup
   loadTodos();
   ```

   **Ce qui se passe ici :**

   **Importation des liaisons :**
   - `import {TodoService} from "../bindings/changeme"` - importe les liaisons Go auto-générées
   - Remarque : `changeme` sera votre nom de module réel issu de `go.mod`

   **La fonction `loadTodos()` :**
   - Appelle `TodoService.GetAll()` pour récupérer tous les todos depuis Go
   - Construit le HTML pour chaque todo en utilisant des littéraux de modèle
   - Ajoute/supprime dynamiquement la classe `completed` pour le style
   - Utilise les attributs `onclick` pour connecter les boutons à nos fonctions
   - Joint tout le HTML ensemble et l'injecte dans le DOM

   **Les fonctions CRUD :**
   - `addTodo()` - Valide l'entrée, appelle la méthode Go `Add`, actualise la liste
   - `toggleTodo(id)` - Appelle la méthode Go `Toggle`, actualise la liste
   - `deleteTodo(id)` - Appelle la méthode Go `Delete`, actualise la liste
   - Toutes les fonctions sont async car les appels Go retournent des Promesses**Pourquoi attacher au window (fenêtre) :**
- `window.addTodo = ...` rend les fonctions accessibles depuis les attributs `onclick` du HTML
- Il s'agit d'un pattern simple pour le JavaScript vanilla (les frameworks gèrent cela différemment)
- En production, vous pourriez utiliser une délégation d'événements appropriée à la place

**Le pattern de rafraîchissement :**
- Après chaque mutation (ajout/basculer/suppression), nous appelons à nouveau `loadTodos()`
- Cela garantit que l'interface utilisateur reste synchronisée avec l'état Go
- Alternative : Faire en sorte que les méthodes Go retournent le nouvel état pour éviter le second appel

## Comment cela fonctionne

### Gestion d'état thread-safe

Le `sync.RWMutex` assure un accès concurrent sécurisé :

```go
func (t *TodoService) GetAll() []Todo {
    t.mu.RLock()  // Verrou en lecture - plusieurs lecteurs autorisés
    defer t.mu.RUnlock()
    return t.todos
}

func (t *TodoService) Add(title string) (*Todo, error) {
    t.mu.Lock()  // Verrou en écriture - accès exclusif
    defer t.mu.Unlock()
    // ... mutations
}
```

**Pourquoi c'est important :**
- Plusieurs appels frontend peuvent se produire simultanément
- Les opérations de lecture ne se bloquent pas mutuellement
- Les opérations d'écriture obtiennent un accès exclusif
- `defer` garantit que les verrous sont toujours libérés

### Gestion des erreurs

Le service renvoie des erreurs pour les opérations invalides :

```go
func (t *TodoService) Add(title string) (*Todo, error) {
    if title == "" {
        return nil, errors.New("title cannot be empty")
    }
    // ...
}
```

Dans le frontend, vous pouvez intercepter ces erreurs :

```javascript
try {
    await TodoService.Add(title);
} catch (err) {
    alert('Error: ' + err);
}
```

### Synchronisation de l'état

Après chaque mutation, nous rechargons la liste complète :

```javascript
window.addTodo = async () => {
    await TodoService.Add(title);  // Mutation
    await loadTodos();             // Actualisation
}
```

**Approche alternative :** Renvoyer la liste mise à jour depuis chaque méthode pour éviter le second appel.

## Améliorations

### Ajouter des statistiques

Ajoutez ceci à `todoservice.go` :

```go
type TodoStats struct {
    Total     int `json:"total"`
    Completed int `json:"completed"`
    Active    int `json:"active"`
}

func (t *TodoService) GetStats() TodoStats {
    t.mu.RLock()
    defer t.mu.RUnlock()

    stats := TodoStats{
        Total: len(t.todos),
    }

    for _, todo := range t.todos {
        if todo.Completed {
            stats.Completed++
        } else {
            stats.Active++
        }
    }

    return stats
}
```

Affichage dans le frontend :

```javascript
async function loadTodos() {
    const [todos, stats] = await Promise.all([
        TodoService.GetAll(),
        TodoService.GetStats()
    ]);

    // Afficher les statistiques
    document.getElementById('stats').textContent =
        `${stats.active} active, ${stats.completed} completed`;

    // ... afficher les todos
}
```

### Ajouter "Effacer les terminés"

```go
func (t *TodoService) ClearCompleted() int {
    t.mu.Lock()
    defer t.mu.Unlock()

    removed := 0
    newTodos := []Todo{}

    for _, todo := range t.todos {
        if !todo.Completed {
            newTodos = append(newTodos, todo)
        } else {
            removed++
        }
    }

    t.todos = newTodos
    return removed
}
```
