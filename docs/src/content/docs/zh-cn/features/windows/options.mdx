## 窗口配置选项

Wails 提供了全面的窗口配置，包含数十个用于大小、位置、外观和行为方面的选项。本参考涵盖了 Windows、macOS 和 Linux 上所有可用的选项，作为 `WebviewWindowOptions` 的**完整参考**。每一个选项、每一个平台，均附带示例和约束说明。

## WebviewWindowOptions 结构

```go
type WebviewWindowOptions struct {
    // Identity
    Name  string
    Title string
    
    // Size and Position
    Width     int
    Height    int
    X         int
    Y         int
    MinWidth  int
    MinHeight int
    MaxWidth  int
    MaxHeight int
    
    // Initial State
    Hidden          bool
    Frameless       bool
    Resizable       bool
    AlwaysOnTop     bool
    Fullscreen      bool
    Minimised       bool
    Maximised       bool
    WindowState     WindowState
    
    // Appearance
    BackgroundColour RGBA
    BackgroundType   BackgroundType
    
    // Content
    URL  string
    HTML string
    
    // Assets
    Assets application.AssetOptions
    
    // Security
    ContentProtectionEnabled bool

    // Menu
    UseApplicationMenu bool

    // Lifecycle
    OnClose   func() bool
    OnDestroy func()

    // Platform-Specific
    Mac     MacOptions
    Windows WindowsOptions
    Linux   LinuxOptions
}
```

## 核心选项

### Name

**类型：** `string`  
**默认值：** 自动生成的 UUID  
**平台：** 所有平台

```go
Name: "main-window"
```

**用途：** 用于后续查找窗口的唯一标识符。

**最佳实践：**
- 使用描述性名称：`"main"`、`"settings"`、`"about"`
- 使用短横线命名法（kebab-case）：`"file-browser"`、`"color-picker"`
- 保持简短且易记

**示例：**

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Name: "settings-window",
})

// 稍后...
settings := app.GetWindowByName("settings-window")
```

### Title

**类型：** `string`  
**默认值：** 应用程序名称  
**平台：** 所有平台

```go
Title: "My Application"
```

**用途：** 显示在标题栏和任务栏中的文本。

**动态更新：**

```go
window.SetTitle("My Application - Document.txt")
```

### Width / Height

**类型：** `int`（像素）  
**默认值：** 800 x 600  
**平台：** 所有平台  
**约束：** 必须为正数

```go
Width:  1200,
Height: 800,
```

**用途：** 窗口的初始逻辑像素大小。

**注意：**
- Wails 会自动处理 DPI 缩放
- 使用逻辑像素，而非物理像素
- 考虑最小屏幕分辨率（1024x768）

**示例尺寸：**

| 使用场景 | 宽度 | 高度 |
|----------|-------|--------|
| 小型工具 | 400 | 300 |
| 标准应用 | 1024 | 768 |
| 大型应用 | 1440 | 900 |
| 全高清 | 1920 | 1080 |

### X / Y

**类型：** `int`（像素）  
**默认值：** 屏幕居中  
**平台：** 所有平台

```go
X: 100,  // 距离左边缘 100px
Y: 100,  // 距离顶部边缘 100px
```

**用途：** 窗口的初始位置。

**坐标系：**
- (0, 0) 是主屏幕的左上角
- X 轴正方向向右
- Y 轴正方向向下

**示例：**

```go
settings := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Name: "coordinate-window",
    InitialPosition: application.WindowXY, // 使用坐标系
    X:    100,
    Y:    100,
})
```

**最佳实践：** 使用 `Center()` 将窗口居中：

```go
window := app.Window.New()
window.Center()
```

### MinWidth / MinHeight

**类型：** `int`（像素）  
**默认值：** 0（无最小值限制）  
**平台：** 所有平台

```go
MinWidth:  400,
MinHeight: 300,
```

**用途：** 防止窗口过小。

**使用场景：**
- 防止布局错乱
- 确保可用性
- 保持宽高比

**示例：**

```go
// 防止窗口小于 400x300
MinWidth:  400,
MinHeight: 300,
```

### MaxWidth / MaxHeight

**类型：** `int`（像素）  
**默认值：** 0（无最大值限制）  
**平台：** 所有平台

```go
MaxWidth:  1920,
MaxHeight: 1080,
```

**用途：** 防止窗口过大。

**使用场景：**
- 固定大小的应用程序
- 防止过度消耗资源
- 保持设计约束

## 状态选项

### Hidden

**类型：** `bool`
**默认值：** `false`
**平台：** 所有平台

```go
Hidden: true,
```

**用途：** 创建窗口但不显示它。

**使用场景：**
- 后台窗口
- 按需显示的窗口
- 启动屏（创建、加载，然后显示）
- 防止加载内容时的白屏闪烁

**平台改进：**
- **Windows：** 修复了白窗口闪烁问题 - 窗口在调用 `Show()` 之前保持不可见
- **macOS：** 完全支持
- **Linux：** 完全支持

**推荐的平滑加载模式：**

```go
// 创建隐藏窗口
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Name:   "main-window",
    Hidden: true,
    BackgroundColour: application.RGBA{R: 30, G: 30, B: 30, A: 255}, // 匹配你的主题
})

// 在隐藏状态下加载内容
// ... 内容加载 ...

// 准备好后显示（无闪烁！）
window.Show()
```

**示例：**

```go
settings := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Name:   "settings",
    Hidden: true,
})

// 需要时显示
settings.Show()
```

### Frameless

**类型：** `bool`  
**默认值：** `false`  
**平台：** 所有平台

```go
Frameless: true,
```

**用途：** 移除标题栏和窗口边框。

**使用场景：**
- 自定义窗口外观
- 启动屏
- 信息亭（Kiosk）应用程序
- 自定义设计的窗口

**重要：** 你需要自行实现：
- 窗口拖拽
- 关闭/最小化/最大化按钮
- 调整大小手柄（如果允许调整大小）

**详见 [Frameless Windows](/zh-cn/features/windows/frameless)。**

### Resizable

**类型：** `bool`  
**默认值：** `true`  
**平台：** 所有平台

```go
Resizable: false,
```

**用途：** 允许/禁止调整窗口大小。

**使用场景：**
- 固定大小的应用程序
- 启动屏
- 对话框

**注意：** 除非你也阻止了最大化/全屏操作，否则用户仍然可以最大化或进入全屏模式。

### AlwaysOnTop

**类型：** `bool`  
**默认值：** `false`  
**平台：** 所有平台

```go
AlwaysOnTop: true,
```

**用途：** 保持窗口在所有其他窗口之上。

**使用场景：**
- 浮动工具栏
- 通知
- 画中画
- 计时器

**平台说明：**
- **macOS：** 完全支持
- **Windows：** 完全支持
- **Linux：** 取决于窗口管理器

### Fullscreen

**类型：** `bool`  
**默认值：** `false`  
**平台：** 所有平台

```go
Fullscreen: true,
```

**用途：** 以全屏模式启动。

**平台行为：**
- **macOS：** 创建新的 Space（虚拟桌面）
- **Windows：** 覆盖任务栏
- **Linux：** 因桌面环境而异

**运行时切换：**

```go
window.Fullscreen()
window.UnFullscreen()
```

### Minimised / Maximised

**类型：** `bool`  
**默认值：** `false`  
**平台：** 所有平台

```go
Minimised: true,  // 启动时最小化
Maximised: true,  // 启动时最大化
```

**用途：** 设置初始窗口状态。

**注意：** 不要将两者都设置为 `true` - 行为未定义。

### WindowState

**类型：** `WindowState` 枚举  
**默认值：** `WindowStateNormal`  
**平台：** 所有平台

```go
WindowState: application.WindowStateMaximised,
```

**值：**
- `WindowStateNormal` - 普通窗口
- `WindowStateMinimised` - 最小化
- `WindowStateMaximised` - 最大化
- `WindowStateFullscreen` - 全屏
- `WindowStateHidden` - 隐藏

**优于单独的标志位：**

```go
// ✅ 推荐
WindowState: application.WindowStateMaximised,

// ❌ 避免
Maximised: true,
```

## 外观选项

### BackgroundColour

**类型：** `RGBA` 结构体  
**默认值：** 白色 (255, 255, 255, 255)  
**平台：** 所有平台

```go
BackgroundColour: application.RGBA{R: 0, G: 0, B: 0, A: 255},
```

**用途：** 内容加载前的窗口背景颜色。

**使用场景：**
- 匹配应用程序的主题
- 防止深色主题下的白屏闪烁
- 平滑的加载体验

**示例：**

```go
// Dark theme
BackgroundColour: application.RGBA{R: 30, G: 30, B: 30, A: 255},

// Light theme
BackgroundColour: application.RGBA{R: 255, G: 255, B: 255, A: 255},
```

**辅助方法：**

```go
window.SetBackgroundColour(30, 30, 30, 255)
```

### BackgroundType

**类型：** `BackgroundType` 枚举  
**默认值：** `BackgroundTypeSolid`  
**平台：** macOS, Windows (部分支持)

```go
BackgroundType: application.BackgroundTypeTranslucent,
```

**取值：**
- `BackgroundTypeSolid` - 纯色
- `BackgroundTypeTransparent` - 完全透明
- `BackgroundTypeTranslucent` - 半透明模糊

**平台支持：**
- **macOS：** 支持所有类型
- **Windows：** 透明和半透明 (Windows 11+)
- **Linux：** 仅纯色

**示例 (macOS)：**

```go
BackgroundType: application.BackgroundTypeTranslucent,
Mac: application.MacOptions{
    Backdrop: application.MacBackdropTranslucent,
},
```

## 内容选项

### URL

**类型：** `string`  
**默认值：** 空 (从 Assets 加载)  
**平台：** 所有平台

```go
URL: "https://example.com",
```

**用途：** 加载外部 URL 而不是嵌入的资源。

**使用场景：**
- 开发阶段 (从开发服务器加载)
- Web 应用程序
- 混合应用程序

**示例：**

```go
// 开发环境
URL: "http://localhost:5173",

// 生产环境
Assets: application.AssetOptions{
    Handler: application.AssetFileServerFS(assets),
},
```

### HTML

**类型：** `string`  
**默认值：** 空  
**平台：** 所有平台

```go
HTML: "<h1>Hello World</h1>",
```

**用途：** 直接加载 HTML 字符串。

**使用场景：**
- 简单窗口
- 动态生成的内容
- 测试

**示例：**

```go
HTML: `
<!DOCTYPE html>
<html>
<head><title>Simple Window</title></head>
<body><h1>Hello from Wails!</h1></body>
</html>
`,
```

### Assets

**类型：** `AssetOptions`  
**默认值：** 继承自 application  
**平台：** 所有平台

```go
Assets: application.AssetOptions{
    Handler: application.AssetFileServerFS(assets),
},
```

**用途：** 提供嵌入的前端资源。

**详情请参阅 [构建系统](/zh-cn/concepts/build-system)。**

### UseApplicationMenu

**类型：** `bool`
**默认值：** `false`
**平台：** Windows, Linux (在 macOS 上无效)

```go
UseApplicationMenu: true,
```

**用途：** 为此窗口使用应用程序菜单 (通过 `app.Menu.Set()` 设置)。

在 **macOS** 上，此选项无效，因为 macOS 始终使用屏幕顶部的全局应用程序菜单。

在 **Windows** 和 **Linux** 上，窗口默认不显示菜单。设置 `UseApplicationMenu: true` 会指示窗口使用应用程序级别的菜单，从而提供一种简单的跨平台解决方案。

**示例：**

```go
// 设置一次应用程序菜单
menu := app.NewMenu()
menu.AddRole(application.FileMenu)
menu.AddRole(application.EditMenu)
app.Menu.Set(menu)

// 所有设置 UseApplicationMenu 的窗口都将显示此菜单
app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:              "Main Window",
    UseApplicationMenu: true,
})

app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:              "Second Window",
    UseApplicationMenu: true,  // 同样获取应用菜单
})
```

**注意：**
- 如果同时设置了 `UseApplicationMenu` 和窗口特定的菜单，则窗口特定的菜单优先级更高
- 这通过消除运行时操作系统检查的需要，简化了跨平台代码
- 有关完整的菜单文档，请参阅 [应用程序菜单](/zh-cn/features/menus/application)

## 输入选项

### EnableFileDrop

**类型：** `bool`  
**默认值：** `false`  
**平台：** 所有平台

```go
EnableFileDrop: true,
```

**用途：** 启用从操作系统拖放文件到窗口。

启用后：
- 可以从文件管理器拖放文件到应用程序中
- 触发 `WindowFilesDropped` 事件并包含拖放的文件路径
- 具有 `data-file-drop-target` 属性的元素提供详细的拖放信息

**使用场景：**
- 文件上传界面
- 文档编辑器
- 媒体导入器
- 任何接受文件的应用程序

**示例：**

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:          "File Uploader",
    EnableFileDrop: true,
})

// 处理拖放的文件
window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    details := event.Context().DropTargetDetails()
    
    for _, file := range files {
        fmt.Println("Dropped:", file)
    }
})
```

**HTML 拖放区域：**

```html
<!-- 将元素标记为拖放目标 -->
<div id="upload" data-file-drop-target>
    在此处拖放文件
</div>
```

**详情请参阅 [文件拖放](/zh-cn/features/drag-and-drop/files)。**

## 安全选项

### ContentProtectionEnabled

**类型：** `bool`  
**默认值：** `false`  
**平台：** Windows (10+), macOS

```go
ContentProtectionEnabled: true,
```

**用途：** 防止捕获窗口内容。

**平台支持：**
- **Windows：** Windows 10 版本 19041+ (完全支持), 旧版本 (部分支持)
- **macOS：** 完全支持
- **Linux：** 不支持

**使用场景：**
- 银行应用程序
- 密码管理器
- 医疗记录
- 机密文档

**重要说明：**
1. 不能防止物理拍摄
2. 某些工具可能会绕过保护
3. 这是综合安全措施的一部分，而非唯一的保护手段
4. DevTools 窗口不会自动受到保护

**示例：**

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title: "Secure Window",
    ContentProtectionEnabled: true,
})

// 在运行时切换
window.SetContentProtection(true)
```

## 生命周期选项

### OnClose

**类型：** `func() bool`  
**默认值：** `nil` (始终允许关闭)  
**平台：** 所有平台

```go
OnClose: func() bool {
    // 返回 false 以取消关闭
    // 返回 true 以允许关闭
    return true
},
```

**用途：** 处理窗口关闭事件，可选择阻止关闭。

**使用场景：**
- 在关闭前确认是否有未保存的更改
- 在关闭前保存状态
- 防止意外关闭

**示例：**

```go
OnClose: func() bool {
    if hasUnsavedChanges() {
        result := showConfirmdialog("Unsaved changes. Close anyway?")
        return result == "yes"
    }
    return true
},
```

**重要：** 仅由用户操作 (点击 X) 触发，而非 `window.Destroy()`。

### OnDestroy

**类型：** `func()`  
**默认值：** `nil`  
**平台：** 所有平台

```go
OnDestroy: func() {
    // 清理代码
    fmt.Println("Window destroyed")
},
```

**用途：** 窗口销毁时的清理工作。

**使用场景：**
- 释放资源
- 关闭连接
- 更新应用程序状态

**示例：**

```go
OnDestroy: func() {
    // 关闭数据库连接
    if db != nil {
        db.Close()
    }
    
    // 从窗口列表中移除
    removeWindow(window.ID())
},
```

## 平台特定选项### Mac 选项

```go
Mac: application.MacWindow{
    TitleBar: application.MacTitleBar{
        AppearsTransparent: true,
        Hide:               false,
        HideTitle:          true,
        FullSizeContent:    true,
    },
    Backdrop:                application.MacBackdropTranslucent,
    InvisibleTitleBarHeight: 50,
    WindowLevel:             application.MacWindowLevelNormal,
    CollectionBehavior:      application.MacWindowCollectionBehaviorDefault,
},
```

**TitleBar** (`MacTitleBar`)
- `AppearsTransparent` - 使标题栏透明，内容延伸至标题栏区域
- `Hide` - 完全隐藏标题栏
- `HideTitle` - 仅隐藏标题文本
- `FullSizeContent` - 将内容扩展至窗口全尺寸

**Backdrop** (`MacBackdrop`)
- `MacBackdropNormal` - 标准不透明背景
- `MacBackdropTranslucent` - 模糊半透明背景
- `MacBackdropTransparent` - 完全透明背景

**InvisibleTitleBarHeight** (`int`)
- 不可见标题栏区域的高度（用于拖拽）
- 仅当原生标题栏拖拽区域被隐藏时才生效 —— 即窗口无边框 (`Frameless: true`) 或使用透明标题栏 (`AppearsTransparent: true`) 时
- 对带有可见标题栏的标准窗口无影响

**WindowLevel** (`MacWindowLevel`)
- `MacWindowLevelNormal` - 标准窗口层级（默认）
- `MacWindowLevelFloating` - 浮动于普通窗口之上
- `MacWindowLevelTornOffMenu` - 撕下式菜单层级
- `MacWindowLevelModalPanel` - 模态面板层级
- `MacWindowLevelMainMenu` - 主菜单层级
- `MacWindowLevelStatus` - 状态窗口层级
- `MacWindowLevelPopUpMenu` - 弹出式菜单层级
- `MacWindowLevelScreenSaver` - 屏幕保护程序层级

**CollectionBehavior** (`MacWindowCollectionBehavior`)

控制窗口在 macOS Spaces 和全屏模式下的行为。这些是位掩码值，可以使用按位或 (`|`) 组合。

**Space 行为：**
- `MacWindowCollectionBehaviorDefault` - 使用 FullScreenPrimary（默认，向后兼容）
- `MacWindowCollectionBehaviorCanJoinAllSpaces` - 窗口出现在所有 Spaces 上
- `MacWindowCollectionBehaviorMoveToActiveSpace` - 显示时移动到活动 Space
- `MacWindowCollectionBehaviorManaged` - 默认托管窗口行为
- `MacWindowCollectionBehaviorTransient` - 临时/瞬态窗口
- `MacWindowCollectionBehaviorStationary` - 在 Space 切换期间保持静止

**窗口循环：**
- `MacWindowCollectionBehaviorParticipatesInCycle` - 包含在 Cmd+` 循环中
- `MacWindowCollectionBehaviorIgnoresCycle` - 排除在 Cmd+` 循环之外

**全屏行为：**
- `MacWindowCollectionBehaviorFullScreenPrimary` - 可以进入全屏模式
- `MacWindowCollectionBehaviorFullScreenAuxiliary` - 可以覆盖全屏应用
- `MacWindowCollectionBehaviorFullScreenNone` - 禁用全屏功能
- `MacWindowCollectionBehaviorFullScreenAllowsTiling` - 允许并排平铺 (macOS 10.11+)
- `MacWindowCollectionBehaviorFullScreenDisallowsTiling` - 防止平铺 (macOS 10.11+)

**示例 - 类似 Spotlight 的窗口：**

```go
// 出现在所有 Spaces 上且可以覆盖全屏应用的窗口
Mac: application.MacWindow{
    CollectionBehavior: application.MacWindowCollectionBehaviorCanJoinAllSpaces |
                        application.MacWindowCollectionBehaviorFullScreenAuxiliary,
    WindowLevel:        application.MacWindowLevelFloating,
},
```

**示例 - 单一行为：**

```go
// 可以出现在全屏应用程序之上的窗口
Mac: application.MacWindow{
    CollectionBehavior: application.MacWindowCollectionBehaviorFullScreenAuxiliary,
},
```

### Windows 选项

```go
Windows: application.WindowsOptions{
    DisableWindowIcon:      false,
    WindowBackdropType:     application.WindowsBackdropTypeAuto,
    CustomTheme:            nil,
    DisableFramelessWindowDecorations: false,
},
```

**DisableWindowIcon** (`bool`)
- 从标题栏移除图标
- 更简洁的外观

**WindowBackdropType** (`WindowsBackdropType`)
- `WindowsBackdropTypeAuto` - 系统默认
- `WindowsBackdropTypeNone` - 无背景
- `WindowsBackdropTypeMica` - 云母材质 (Windows 11)
- `WindowsBackdropTypeAcrylic` - 亚克力材质 (Windows 11)
- `WindowsBackdropTypeTabbed` - 标签页材质 (Windows 11)

**CustomTheme** (`*WindowsTheme`)
- 自定义颜色主题
- 覆盖系统颜色

**DisableFramelessWindowDecorations** (`bool`)
- 禁用默认无边框装饰
- 用于自定义窗口外观

**示例：**

```go
Windows: application.WindowsOptions{
    WindowBackdropType: application.WindowsBackdropTypeMica,
    DisableWindowIcon:  true,
},
```

## 应用级窗口选项

某些特定于 Windows 的选项必须在应用级别进行配置，而不是针对每个窗口。这是因为 WebView2 在每个用户数据路径中共享单个浏览器环境。

### 浏览器标志

WebView2 浏览器标志控制应用程序中**所有窗口**的实验性功能和行为。这些必须在 `application.Options.Windows` 中设置：

```go
app := application.New(application.Options{
    Name: "My App",
    Windows: application.WindowsOptions{
        // 启用实验性 WebView2 功能
        EnabledFeatures: []string{
            "msWebView2EnableDraggableRegions",
        },

        // 禁用特定功能
        DisabledFeatures: []string{
            "msSmartScreenProtection",  // 始终由 Wails 禁用
        },

        // 其他 Chromium 命令行参数
        AdditionalBrowserArgs: []string{
            "--disable-gpu",
            "--remote-debugging-port=9222",
        },
    },
})
```

**EnabledFeatures** (`[]string`)
- 要启用的 WebView2 功能标志列表
- 请参阅 [WebView2 浏览器标志](https://learn.microsoft.com/en-us/microsoft-edge/webview2/concepts/webview-features-flags) 获取可用标志
- 示例：`"msWebView2EnableDraggableRegions"`

**DisabledFeatures** (`[]string`)
- 要禁用的 WebView2 功能标志列表
- Wails 会自动禁用 `msSmartScreenProtection`
- 示例：`"msExperimentalFeature"`

**AdditionalBrowserArgs** (`[]string`)
- 传递给浏览器进程的 Chromium 命令行参数
- 必须包含 `--` 前缀（例如，`"--remote-debugging-port=9222"`）
- 请参阅 [Chromium 命令行开关](https://peter.sh/experiments/chromium-command-line-switches/) 获取可用参数

:::caution[重要]
这些标志全局应用于所有窗口，因为 WebView2 在每个用户数据路径中共享单个浏览器环境。你不能为不同的窗口设置不同的浏览器标志。
:::

**完整示例：**

```go
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
)

func main() {
    app := application.New(application.Options{
        Name: "My App",
        Windows: application.WindowsOptions{
            // 启用可拖拽区域功能
            EnabledFeatures: []string{
                "msWebView2EnableDraggableRegions",
            },
            // 启用远程调试
            AdditionalBrowserArgs: []string{
                "--remote-debugging-port=9222",
            },
        },
    })

    // 所有窗口都将使用上面配置的浏览器标志
    window := app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title:  "Main Window",
        Width:  1024,
        Height: 768,
    })

    window.Show()
    app.Run()
}
```
