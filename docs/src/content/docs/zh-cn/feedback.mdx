我们欢迎（并鼓励）您提供反馈！在创建新帖子之前，请先搜索现有的问题或帖子。以下是提供反馈的不同方式：

<Tabs>
  <TabItem label="Bug 报告" icon="error">

    如果您发现了 Bug，请通过 [Discord](https://discord.gg/bdj28QNHmT) 上的 [v3 Alpha 反馈](https://discord.gg/bdj28QNHmT) 频道发帖告知我们。

    - 帖子应清楚地说明 Bug 是什么，并提供一个简单的可复现示例。如果文档不清楚*应该*发生什么，请在帖子中包含该信息。
    - 帖子应添加 `Bug` 标签。
    - 请在帖子中包含 `wails doctor` 的输出结果。
    - 如果该行为与当前文档不符（例如窗口无法正确调整大小），请执行以下操作：
      - 更新 `v3/example` 目录中的现有示例，或在 `v3/examples` 文件夹中创建一个新的示例，以清楚地展示该问题。
      - 提交一个标题为 `[v3 alpha test] <Bug 描述>` 的 [PR](https://github.com/wailsapp/wails/pulls)。
      - 请在帖子中包含 PR 的链接。

    :::caution

    _请记住_，意外行为不一定是 Bug——它可能只是没有按您的预期工作。对此类情况请使用 `建议` 标签。

    :::

  </TabItem>

  <TabItem label="修复方案" icon="approve-check">

    如果您有针对 Bug 的修复方案或对文档的更新，请执行以下操作：

    - 在 [Wails 仓库](https://github.com/wailsapp/wails) 上提交拉取请求。PR 的标题应以 `[v3 alpha]` 开头。
    - 在 [v3 Alpha 反馈](https://discord.gg/bdj28QNHmT) 频道发布帖子。
    - 帖子应添加 `PR` 标签。
    - 请在帖子中包含 PR 的链接。

  </TabItem>

  <TabItem label="建议" icon="puzzle" id="abc">

    如果您有建议，请通过 [Discord](https://discord.gg/bdj28QNHmT) 上的 [v3 Alpha 反馈](https://discord.gg/bdj28QNHmT) 频道发帖告知我们：

    - 帖子应添加 `Suggestion` 标签。

    如果您有任何问题，请随时在 [Discord](https://discord.gg/bdj28QNHmT) 上联系我们。

  </TabItem>

  <TabItem label="点赞" icon="star">

    - 帖子可以通过使用 :thumbsup: 表情符号进行“点赞”。请对您认为优先的事项进行点赞。
    - 请*不要*仅添加“+1”或“我也是”之类的评论。
    - 如果帖子中有更多内容需要补充，例如“此 Bug 也影响 ARM 构建”或“另一种选择是……”，请随时发表评论。

  </TabItem>

</Tabs>

已知问题列表及正在进行的工作可以在[这里](https://github.com/orgs/wailsapp/projects/6)找到。

## 我们希望您提供反馈的方面

- API
  - 是否易于使用？
  - 是否符合您的预期？
  - 是否缺少任何功能？
  - 是否有需要移除的部分？
  - Go 和 JS 之间是否保持一致？
- 构建系统
  - 是否易于使用？
  - 我们能否改进它？
- 示例
  - 是否清晰？
  - 是否涵盖了基础知识？
- 功能
  - 缺少哪些功能？
  - 哪些功能是不需要的？
- 文档
  - 哪些部分可以更清晰？
