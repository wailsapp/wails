## NSIS 安装程序

默认的打包格式会生成一个 NSIS 安装程序：

```bash
wails3 package GOOS=windows
```

这会运行 `wails3 task windows:package`，其执行以下操作：
1. 构建应用程序
2. 生成 WebView2 引导程序
3. 创建 NSIS 安装程序

输出文件：`build/windows/nsis/<AppName>-installer.exe`

### MSIX 包

适用于 Microsoft Store 分发或现代 Windows 部署：

```bash
wails3 package GOOS=windows FORMAT=msix
```

输出文件：`bin/<AppName>-<arch>.msix`

<Aside type="note">
MSIX 需要 `makeappx.exe`（Windows SDK）或 MSIX 工具。运行 `wails3 tool msix-install-tools` 进行设置。
</Aside>

## 自定义安装程序

NSIS 配置文件位于 `build/windows/nsis/project.nsi`。编辑此文件以自定义：

- 安装程序界面和品牌标识
- 安装目录
- 开始菜单和桌面快捷方式
- 文件关联
- 许可协议

应用程序元数据来自 `build/windows/info.json`：

```json
{
  "fixed": {
    "file_version": "1.0.0"
  },
  "info": {
    "0000": {
      "ProductVersion": "1.0.0",
      "CompanyName": "My Company",
      "FileDescription": "My Application",
      "ProductName": "MyApp"
    }
  }
}
```

## 代码签名

对可执行文件和安装程序进行签名，以避免 SmartScreen 警告：

```bash
# 使用包装器（自动检测平台）
wails3 sign GOOS=windows

# 或直接使用任务
wails3 task windows:sign
wails3 task windows:sign:installer
```

在 `build/windows/Taskfile.yml` 中配置签名：

```yaml
vars:
  SIGN_CERTIFICATE: "path/to/certificate.pfx"
  # 或者使用 Windows 存储中证书的指纹
  SIGN_THUMBPRINT: "certificate-thumbprint"
  TIMESTAMP_SERVER: "http://timestamp.digicert.com"
```

安全地存储你的证书密码：

```bash
wails3 setup signing
```

详细信息请参阅 [应用程序签名](/zh-cn/guides/build/signing)。

## 为 ARM 构建

```bash
wails3 build GOOS=windows GOARCH=arm64
wails3 package GOOS=windows GOARCH=arm64
```

## 故障排除

### 未找到 makensis

安装 NSIS：

```bash
# Windows
winget install NSIS.NSIS

# 或从 https://nsis.sourceforge.io/ 下载
```

### SmartScreen 警告

你的可执行文件未签名。请参阅上面的 [代码签名](#code-signing)。

### WebView2 缺失

安装程序包含一个 WebView2 引导程序，会在需要时下载运行时。如果需要离线安装，请从 Microsoft 下载 Evergreen 独立安装程序。
