前端路由是单页应用中切换视图的一种流行方式。本指南介绍了在使用 Wails 时，针对不同前端框架推荐的实现方法。


<Tabs syncKey="framework">
    <TabItem label="Vue" icon="seti:vue">

Vue 中推荐的路由方式是 [Hash Mode](https://next.router.vuejs.org/guide/essentials/history-mode.html#hash-mode)：

```javascript
import { createRouter, createWebHashHistory } from "vue-router";

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    //...
  ],
});
```

Hash Mode 使用 URL 的哈希部分来渲染不同的视图，通过基于哈希的 URL 格式，避免 Wails 运行时干扰路由。

    </TabItem>
    <TabItem label="Angular" icon="seti:angular">

Angular 中推荐的路由方式是 [HashLocationStrategy](https://codecraft.tv/courses/angular/routing/routing-strategies#%5Fhashlocationstrategy)：

```typescript
RouterModule.forRoot(routes, { useHash: true });
```

使用基于哈希的 URL 可确保在所有平台上与 Wails 的窗口处理机制正确配合。

    </TabItem>
    <TabItem label="React" icon="seti:react">

React 中推荐的路由方式是 [HashRouter](https://reactrouter.com/en/main/router-components/hash-router)：

```jsx
import ReactDOM from "react-dom/client";
import { HashRouter, Routes, Route } from "react-router-dom";

ReactDOM.createRoot(root).render(
  <HashRouter basename={"/"}>
    {/* The rest of your app goes here */}
    <Routes>
      <Route path="/" element={<Page0 />} />
      <Route path="/page1" element={<Page1 />} />
      <Route path="/page2" element={<Page2 />} />
      {/* more... */}
    </Routes>
  </HashRouter>
);
```

HashRouter 使用 URL 哈希而非路径，这使其在所有平台上都能与 Wails 可靠配合。

    </TabItem>
    <TabItem label="Svelte" icon="seti:svelte">

Svelte 中推荐的路由方式是 [svelte-spa-router](https://github.com/ItalyPaleAle/svelte-spa-router)：

```svelte
<script>
    import Router from "svelte-spa-router";
</script>

<Router
    routes={{
        "/": Home,
        "/products": wrap({
            asyncComponent: () => import("./routes/Products.svelte"),
        }),
        "/settings": Settings,
        "*": NotFound,
    }}
/>
```

svelte-spa-router 支持基于哈希的路由，因此与 Wails 应用兼容。

## 为什么使用哈希路由？

Wails 将你的前端嵌入到原生 WebView 窗口中。使用基于哈希的路由（`#/page` 而不是 `/page`）可避免与以下方面产生冲突：

- Wails 运行时的内部路由
- 不同平台上的原生窗口 URL 处理机制
- 从非根路径提供生产环境资源

## 故障排除

如果在生产构建中遇到前端路由问题：

- 确保你的前端构建工具已配置为输出 **hash mode** 路由
- 对于基于 Vite 的项目，在 `vite.config.js` 中添加 `base: "./"`
- 验证你的 `index.html` 是否正确处理了 SPA 路由的回退逻辑
