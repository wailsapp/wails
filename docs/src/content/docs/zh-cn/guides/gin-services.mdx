# 在 Service 中使用 Gin

Gin Web 框架是 Go 语言中构建 HTTP 服务的流行选择。借助 Wails v3，您可以轻松地将基于 Gin 的服务集成到您的应用程序中，从而提供处理 HTTP 请求、实现 RESTful API 以及提供 Web 内容的强大方式。

本指南将引导您创建一个基于 Gin 的服务，并将其挂载到 Wails 应用程序的特定路由中。我们将构建一个完整的示例，演示如何：

1. 创建基于 Gin 的服务
2. 实现 Wails Service 接口
3. 设置路由和中间件
4. 与 Wails 事件系统集成
5. 从前端与 Service 交互

## 前置条件

在开始之前，请确保您已具备：

- 已安装 Wails v3
- 具备 Go 语言和 Gin 框架的基础知识
- 熟悉 HTTP 概念和 RESTful API

您需要将 Gin 框架添加到项目中：

```bash
go get github.com/gin-gonic/gin
```

## 创建基于 Gin 的服务

让我们从创建一个实现 Wails Service 接口的 Gin 服务开始。我们的服务将管理用户集合，并提供用于检索和创建用户记录的 API 端点。

### 1. 定义数据模型

首先，定义服务将使用的数据结构：

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User 表示系统中的用户
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData 表示在事件中发送的数据
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. 创建服务结构

接下来，定义服务结构，该结构将保存您的 Gin 路由器以及服务需要维护的任何状态：

```go
// GinService 实现了一个使用 Gin 处理 HTTP 的 Wails 服务
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService 创建一个新的 GinService 实例
func NewGinService() *GinService {
	// 创建一个新的 Gin 路由器
	ginEngine := gin.New()

	// 添加中间件
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// 定义路由
	service.setupRoutes()

	return service
}
```

### 3. 实现 Service 接口

实现 Wails Service 接口所需的方法：

```go
// ServiceName 返回服务的名称
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup 在服务启动时调用
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// 存储应用程序实例以便后续使用
	s.app = application.Get()

	// 注册一个可以从前端触发的事件处理程序
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// 记录事件数据
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// 向前端发送事件
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown 在服务关闭时调用
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// 如有需要，清理资源
	return nil
}
```

### 3. 实现 http.Handler 接口

为了使您的服务可以挂载到特定路由，请实现 `http.Handler` 接口。这个单一的方法 `ServeHTTP` 是所有 HTTP 请求进入您服务的入口。它将请求处理委托给 Gin 路由器，使您能够使用 Gin 的所有强大功能进行路由和中间件处理。

```go
// ServeHTTP 实现 http.Handler 接口
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// 所有请求都转发到 Gin 路由器
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. 设置路由

在单独的方法中定义您的 API 路由，以获得更好的组织性。这种方法使代码保持整洁，并更容易理解 API 的结构。Gin 路由器提供了用于定义路由的流畅 API，包括对路由组的支持，这有助于组织相关端点。

```go
// setupRoutes 配置 API 路由
func (s *GinService) setupRoutes() {
	// 基本信息端点
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// 用户组
	users := s.ginEngine.Group("/users")
	{
		// 获取所有用户
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// 通过 ID 获取用户
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// 创建新用户
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// 设置 ID 和创建时间
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// 添加到用户切片
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// 发送事件以通知新用户
			s.app.Event.Emit("user-created", newUser)
		})

		// 删除用户
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// 从切片中移除用户
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. 创建自定义中间件

您可以创建自定义 Gin 中间件以增强您的服务。Gin 中的中间件函数按照添加到路由器的顺序执行，可以执行记录日志、身份验证和错误处理等任务。此示例显示了一个简单的记录日志中间件，用于记录请求详情。

```go
// LoggingMiddleware 是一个记录请求详情的 Gin 中间件
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// 开始计时
		start := time.Now()

		// 处理请求
		c.Next()

		// 计算延迟
		latency := time.Since(start)

		// 记录请求详情
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## 注册您的 Service要在 Wails 应用程序中使用基于 Gin 的服务，您需要将其注册到应用程序中，并指定其挂载的路由。这是在创建 Wails 应用程序实例时完成的。您指定的路由将成为 Gin 路由器中定义的所有端点的基础路径。

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

在此示例中，Gin 服务挂载在 `/api` 路由上。这意味着如果您的 Gin 路由器中定义了 `/info` 端点，它将在您的应用程序中通过 `/api/info` 访问。这种方法允许您以逻辑方式组织 API 端点，并避免与应用程序的其他部分发生冲突。

## 与 Wails 事件系统集成

将 Gin 与 Wails 服务一起使用的一个强大功能是可以无缝集成到 Wails 事件系统中。这允许后端服务与前端之间进行实时通信。

在服务的 `ServiceStartup` 方法中，您可以注册事件处理程序来处理来自前端的事件：

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

您还可以从 Gin 路由或服务中的其他部分向前端发送事件：

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## 前端集成

要从前端与您的 Gin 服务交互，您需要导入 Wails 运行时，向 API 端点发出 HTTP 请求，并使用 Wails 事件系统进行实时通信。

对于生产环境，建议使用 `@wailsio/runtime` 包，而不是直接导入 `/wails/runtime.js`。这可以确保类型安全、更好的 IDE 支持、通过 npm 进行版本管理，以及与现代 JavaScript 工具链的兼容性。

安装该包：
```bash
npm install @wailsio/runtime
```

然后在代码中使用它：
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## 结语

将 Gin Web 框架与 Wails v3 服务集成，为构建模块化、可维护的 Web 应用提供了一种强大且灵活的方法。通过利用 Gin 的路由和中间件功能，并结合 Wails 的事件系统，您可以创建具有清晰关注点分离的丰富交互式应用。

本指南的完整示例代码可以在 Wails 仓库的 `v3/examples/gin-service` 目录下找到。
