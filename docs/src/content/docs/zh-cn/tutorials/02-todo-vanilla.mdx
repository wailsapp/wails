2. **创建待办事项服务**

   待办事项服务将管理我们的应用程序状态并提供用于 CRUD 操作的方法。与每个请求都隔离的 Web 服务器不同，桌面应用可能具有多个并发操作，因此我们需要线程安全的状态管理。

   删除 `greetservice.go` 并创建一个新文件 `todoservice.go`：

   ```go title="todoservice.go"
   package main

   import (
       "errors"
       "sync"
   )

   type Todo struct {
       ID        int    `json:"id"`
       Title     string `json:"title"`
       Completed bool   `json:"completed"`
   }

   type TodoService struct {
       todos  []Todo
       nextID int
       mu     sync.RWMutex
   }

   func NewTodoService() *TodoService {
       return &TodoService{
           todos:  []Todo{},
           nextID: 1,
       }
   }

   func (t *TodoService) GetAll() []Todo {
       t.mu.RLock()
       defer t.mu.RUnlock()
       return t.todos
   }

   func (t *TodoService) Add(title string) (*Todo, error) {
       if title == "" {
           return nil, errors.New("title cannot be empty")
       }

       t.mu.Lock()
       defer t.mu.Unlock()

       todo := Todo{
           ID:        t.nextID,
           Title:     title,
           Completed: false,
       }
       t.todos = append(t.todos, todo)
       t.nextID++

       return &todo, nil
   }

   func (t *TodoService) Toggle(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i := range t.todos {
           if t.todos[i].ID == id {
               t.todos[i].Completed = !t.todos[i].Completed
               return nil
           }
       }
       return errors.New("todo not found")
   }

   func (t *TodoService) Delete(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i, todo := range t.todos {
           if todo.ID == id {
               t.todos = append(t.todos[:i], t.todos[i+1:]...)
               return nil
           }
       }
       return errors.New("todo not found")
   }
   ```

   **这里发生了什么：**

   **`Todo` 结构体：**
   - 定义了数据的形状，包含 ID、Title 和 Completed 字段
   - `json:` 标签告诉 Go 如何将此结构体转换为 JSON 以供前端使用
   - 每个字段都是导出的（大写），以便绑定生成器能够看到它们

   **`TodoService` 结构体：**
   - `todos []Todo` - 保存所有待办事项的切片
   - `nextID int` - 跟踪下一个要分配的 ID（模拟自增）
   - `mu sync.RWMutex` - 用于线程安全访问的读写互斥锁

   **使用 `sync.RWMutex` 实现线程安全：**
   - 桌面应用可能具有来自 UI 的多个并发操作
   - `RLock()` 允许同时读取（例如，多个 `GetAll` 调用）
   - `Lock()` 提供写入的独占访问权限（例如，`Add`、`Toggle`、`Delete`）
   - `defer` 确保即使函数提前返回或发生恐慌，锁也会被释放

   **这些方法：**
   - `GetAll()` - 返回所有待办事项（使用读锁，因为我们没有修改数据）
   - `Add(title)` - 创建新的待办事项，验证输入，递增 ID
   - `Toggle(id)` - 翻转待办事项的完成状态
   - `Delete(id)` - 从切片中移除待办事项

   **错误处理：**
   - 我们遵循 Go 约定，将 `error` 作为最后一个值返回
   - 空标题会被拒绝
   - 对不存在的待办事项的操作会返回错误
   - 这些错误在前端中变为 JavaScript 异常

4. **创建前端 UI**

   现在让我们构建前端。在这里我们将调用 Go 方法并显示 UI。我们使用原生 JavaScript 以保持简单，并向你展示绑定是如何直接工作的。

   替换 `frontend/src/main.js`：

   ```javascript title="frontend/src/main.js"
   import {TodoService} from "../bindings/changeme";

   async function loadTodos() {
       const todos = await TodoService.GetAll();
       const list = document.getElementById('todo-list');

       list.innerHTML = todos.map(todo => `
           <div class="todo ${todo.completed ? 'completed' : ''}">
               <input type="checkbox"
                      ${todo.completed ? 'checked' : ''}
                      onchange="toggleTodo(${todo.id})">
               <span>${todo.title}</span>
               <button onclick="deleteTodo(${todo.id})">Delete</button>
           </div>
       `).join('');
   }

   window.addTodo = async () => {
       const input = document.getElementById('todo-input');
       const title = input.value.trim();

       if (title) {
           await TodoService.Add(title);
           input.value = '';
           await loadTodos();
       }
   }

   window.toggleTodo = async (id) => {
       await TodoService.Toggle(id);
       await loadTodos();
   }

   window.deleteTodo = async (id) => {
       await TodoService.Delete(id);
       await loadTodos();
   }

   // Load todos on startup
   loadTodos();
   ```

   **这里发生了什么：**

   **导入绑定：**
   - `import {TodoService} from "../bindings/changeme"` - 引入自动生成的 Go 绑定
   - 注意：`changeme` 将是你来自 `go.mod` 的实际模块名称

   **`loadTodos()` 函数：**
   - 调用 `TodoService.GetAll()` 从 Go 获取所有待办事项
   - 使用模板字面量为每个待办事项构建 HTML
   - 动态添加/移除 `completed` 类以进行样式设置
   - 使用 `onclick` 属性将按钮连接到我们的函数
   - 将所有 HTML 连接在一起并注入到 DOM 中

   **CRUD 函数：**
   - `addTodo()` - 验证输入，调用 Go 的 `Add` 方法，刷新列表
   - `toggleTodo(id)` - 调用 Go 的 `Toggle` 方法，刷新列表
   - `deleteTodo(id)` - 调用 Go 的 `Delete` 方法，刷新列表
   - 所有函数都是异步的，因为 Go 调用返回 Promises**为什么要绑定到窗口：**
- `window.addTodo = ...` 使得函数可以从 HTML 的 `onclick` 属性中访问
- 这是一种针对 vanilla JS 的简单模式（框架会以不同的方式处理此问题）
- 在生产环境中，您可能应该使用正确的事件委托

**刷新模式：**
- 在每次修改（添加/切换/删除）之后，我们再次调用 `loadTodos()`
- 这确保 UI 与 Go 状态保持同步
- 替代方案：让 Go 方法返回新状态，以避免第二次调用

## 工作原理

### 线程安全的状态管理

`sync.RWMutex` 提供了安全的并发访问：

```go
func (t *TodoService) GetAll() []Todo {
    t.mu.RLock()  // 读锁 - 允许多个读取者
    defer t.mu.RUnlock()
    return t.todos
}

func (t *TodoService) Add(title string) (*Todo, error) {
    t.mu.Lock()  // 写锁 - 独占访问
    defer t.mu.Unlock()
    // ... 变更操作
}
```

**为什么这很重要：**
- 多个前端调用可以并发发生
- 读操作不会互相阻塞
- 写操作获得独占访问权
- `defer` 确保锁始终被释放

### 错误处理

该服务对无效操作返回错误：

```go
func (t *TodoService) Add(title string) (*Todo, error) {
    if title == "" {
        return nil, errors.New("title cannot be empty")
    }
    // ...
}
```

在前端中，您可以捕获这些错误：

```javascript
try {
    await TodoService.Add(title);
} catch (err) {
    alert('Error: ' + err);
}
```

### 状态同步

每次变更后，我们重新加载完整列表：

```javascript
window.addTodo = async () => {
    await TodoService.Add(title);  // 变更操作
    await loadTodos();             // 刷新
}
```

**替代方案：** 从每个方法返回更新后的列表，以避免第二次调用。

## 增强功能

### 添加统计信息

将此添加到 `todoservice.go`：

```go
type TodoStats struct {
    Total     int `json:"total"`
    Completed int `json:"completed"`
    Active    int `json:"active"`
}

func (t *TodoService) GetStats() TodoStats {
    t.mu.RLock()
    defer t.mu.RUnlock()

    stats := TodoStats{
        Total: len(t.todos),
    }

    for _, todo := range t.todos {
        if todo.Completed {
            stats.Completed++
        } else {
            stats.Active++
        }
    }

    return stats
}
```

在前端中显示：

```javascript
async function loadTodos() {
    const [todos, stats] = await Promise.all([
        TodoService.GetAll(),
        TodoService.GetStats()

    document.getElementById('stats').textContent =
        `${stats.active} active, ${stats.completed} completed`;

    // ... 渲染 todos
}
```

### 添加“清除已完成”

```go
func (t *TodoService) ClearCompleted() int {
    t.mu.Lock()
    defer t.mu.Unlock()

    removed := 0
    newTodos := []Todo{}

    for _, todo := range t.todos {
        if !todo.Completed {
            newTodos = append(newTodos, todo)
        } else {
            removed++
        }
    }

    t.todos = newTodos
    return removed
}
```
