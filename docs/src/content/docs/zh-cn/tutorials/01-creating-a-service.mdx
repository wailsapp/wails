Wails 中的 **服务** 是一个包含你希望在前端可用的业务逻辑的 Go 结构体。服务通过将相关功能分组在一起来保持代码的组织性。

可以将服务视为一组 JavaScript 代码可以调用的方法。在服务上定义的每个公共方法在生成绑定后都可以从前端调用。

在本教程中，我们将创建一个二维码生成器服务来演示这些概念。最后，你将了解如何创建服务、管理依赖项以及将 Go 代码连接到前端。

<br/>

<Steps>

1. ## 创建 QR 服务文件

   在你的应用程序目录中创建一个名为 `qrservice.go` 的新文件：

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **这里发生了什么：**

   - `QRService` 是一个空结构体，将保存我们的二维码生成方法
   - `NewQRService()` 是一个构造函数，用于创建服务的新实例
   - `Generate()` 是一个接受文本和尺寸的方法，然后以 PNG 字节数组形式返回二维码
   - 该方法遵循 Go 的惯例，将错误作为最后一个值返回，因此返回类型为 `([]byte, error)`
   - 我们使用 `github.com/skip2/go-qrcode` 包来处理实际的二维码生成

    <br/>

2. ## 注册服务

   仅创建服务是不够的——我们需要将其 **注册** 到 Wails 应用程序中，以便应用程序知道该服务的存在并为其生成绑定。

   注册发生在创建应用程序时的 `main.go` 中。你将服务实例传递给 `Services` 选项：

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **这里发生了什么：**

   - `application.NewService()` 包装你的服务，以便 Wails 可以管理它
   - 我们调用 `NewQRService()` 来创建服务的一个实例
   - 该服务被添加到应用程序选项的 `Services` 切片中
   - Wails 现在将扫描此服务以查找可供前端使用的公共方法

    <br/>

3. ## 安装依赖项

   我们在代码中引用了 `github.com/skip2/go-qrcode` 包，但尚未实际下载它。Go 需要知道这个依赖项并将其下载到你的项目中。

   在项目目录中，在终端运行以下命令：

   ```bash
   go mod tidy
   ```

   **这里发生了什么：**

   - `go mod tidy` 扫描你的 Go 文件中的导入语句
   - 它下载任何缺失的包（如 `go-qrcode`）并将其添加到 `go.mod` 中
   - 它还会移除任何不再使用的依赖项
   - 这确保你的项目拥有成功编译所需的所有代码

   你应该会看到输出表明二维码包已下载并添加到你的项目中。

    <br/>

4. ## 生成绑定

   要从前端调用这些方法，我们需要生成绑定。
   你可以通过在项目根目录运行 `wails generate bindings` 来完成此操作。

   :::note
   这是你第一次在项目运行此命令时，绑定生成器会对你的代码和依赖项进行彻底分析。这有时可能需要比预期更长的时间，但随后的运行会快得多。
   :::

   运行后，你应该在终端中看到类似以下内容：

   ```bash
    % wails3 generate bindings
    INFO  Processed: 337 Packages, 1 Service, 1 Method, 0 Enums, 0 Models in 740.196125ms.
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

   你应该注意到，在前端目录中，出现了一个名为 `bindings` 的新目录：

   ```bash
   frontend/
   └── bindings
       └── changeme
           ├── index.js
           └── qrservice.js
   ```

    :::tip[Pro Tip]
        当你使用 `wails3 build` 构建应用程序时，它会自动为你生成绑定并保持其最新状态。
    :::
    <br/>

5. ## 理解绑定

   让我们看看 `bindings/changeme/qrservice.js` 中生成的绑定：

   ```js title="bindings/changeme/qrservice.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT

    /**
     * QRService handles QR code generation
     * @module
     */

    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
    // @ts-ignore: Unused imports
    import {Call as $Call, Create as $Create} from "@wailsio/runtime";

    /**
     * Generate creates a QR code from the given text
     * @param {string} text
     * @param {number} size
     * @returns {Promise<string> & { cancel(): void }}
     */
    export function Generate(text, size) {
        let $resultPromise = /** @type {any} */($Call.ByID(3576998831, text, size));
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
            return $Create.ByteSlice($result);
        }));
        $typingPromise.cancel = $resultPromise.cancel.bind($resultPromise);
        return $typingPromise;
    }
   ```

    我们可以看到为 `Generate` 方法生成了绑定。参数名称已保留，
    注释也保留了。还为该方法生成了 JSDoc，以便为你的 IDE 提供类型信息。

    :::note
    不需要完全理解生成的绑定，但了解它们的工作原理很重要。
    :::

    绑定提供了：
    - 与你的 Go 方法等效的函数
    - Go 和 JavaScript 类型之间的自动转换
    - 基于 Promise 的异步操作
    - 作为 JSDoc 注释的类型信息

    :::tip[Typescript]
    绑定生成器还支持生成 TypeScript 绑定。你可以通过运行 `wails3 generate bindings -ts` 来实现。
    :::

    生成的服务由 `index.js` 文件重新导出：

   ```js title="bindings/changeme/index.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT导入 `QRService` 来自 `./qrservice.js`；
    导出 `QRService`。
   ```

    随后，你可以通过简化的导入路径 `./bindings/changeme` 来访问它，该路径仅包含你的 Go 包路径，无需指定任何文件名。

    :::note
    简化导入路径仅在使用前端构建工具（bundler）时可用。
    如果你偏好不使用构建工具的原始前端（vanilla frontend），
    则必须手动导入 `index.js` 或 `qrservice.js`。
    :::

6. ## 在前端中使用绑定

    现在我们可以从 JavaScript 调用我们的 Go 服务了！生成的绑定使这一过程变得简单且类型安全。

    更新 `frontend/src/main.js` 以使用新的绑定：

   ```js title="frontend/src/main.js"
    import { QRService } from './bindings/changeme';

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        try {
            // Generate QR code as base64
            const qrCodeBase64 = await QRService.Generate(text, 256);

            // Display the QR code
            const qrDiv = document.getElementById('qrcode');
            qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

        } catch (err) {
            console.error('Failed to generate QR code:', err);
            alert('Failed to generate QR code: ' + err);
        }
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        button.addEventListener('click', generateQR);
    }
   ```

   **这里发生了什么：**

   - 我们从生成的绑定中导入 `QRService`
   - `QRService.Generate()` 调用我们的 Go 方法——它返回一个 Promise，因此我们使用 `await`
   - Go 方法返回 `[]byte`，Wails 会自动将其转换为 JavaScript 的 base64 字符串
   - 我们使用 base64 字符串创建数据 URL 以显示 PNG 图像
   - `try/catch` 块处理来自 Go 端的任何错误（例如无效输入）
   - 如果我们的 Go 代码返回错误，Promise 将被拒绝，我们在此处捕获它

7. ## 替代方案：HTTP 处理器

    到目前为止，我们已经涵盖了以下方面：
    - 创建新的服务（Service）
    - 生成绑定（Bindings）
    - 在前端代码中使用绑定

    **为什么要使用 HTTP 处理器？**

    方法绑定非常适合数据操作，但对于提供文件、图像或其他媒体，还有另一种替代方案。与其将所有内容转换为 base64 并通过绑定发送，你可以让你的服务充当微型 Web 服务器。

    这在以下情况下非常有用：
    - 你正在提供图像、视频或大文件
    - 你想使用带有 `src` 属性的标准 HTML `<img>` 或 `<video>` 标签
    - 你需要对资源的直接 URL 访问

    如果你的服务实现了 Go 标准的 `ServeHTTP(w http.ResponseWriter, r *http.Request)` 方法，Wails 可以将其作为 HTTP 端点提供。让我们扩展我们的二维码服务以支持此功能：

    ```go title="qrservice.go" ins={4-5,37-65}
    package main

    import (
        "net/http"
        "strconv"

        "github.com/skip2/go-qrcode"
    )

    // QRService handles QR code generation
    type QRService struct {
        // We can add state here if needed
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **这里发生了什么：**

    - `ServeHTTP` 是 Go 中处理 HTTP 请求的标准接口
    - 我们从 URL 中解析查询参数（`?text=hello&size=256`）
    - 我们调用现有的 `Generate()` 方法来创建二维码
    - 我们将内容类型设置为 `image/png`，以便浏览器知道这是一张图片
    - 我们将原始 PNG 字节直接写入响应——不需要 base64！

    现在更新 `main.go` 以指定二维码服务可访问的路由：

   ```go title="main.go" ins={8-10}
    func main() {```go
        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/qrservice",
                }),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

    **这里发生了什么：**

    - 我们添加了 `application.ServiceOptions` 来配置服务的暴露方式
    - `Route: "/qrservice"` 使 HTTP 处理器可在 `/qrservice` 路径下访问
    - 现在，对 `/qrservice?text=hello` 的任何请求都将调用我们的 `ServeHTTP` 方法
    - 如果不设置 `Route`，则 HTTP 处理器功能将被禁用

    :::note
    如果您没有显式设置 `Route` 选项，
    前端将无法访问该 HTTP 处理器。
    :::

    最后，更新 `main.js` 以使用简单的图像 `src` 属性，而不是 base64 编码：

    ```js title="frontend/src/main.js"
    async function generateQR() {

        if (!text) {
            alert('Please enter some text');
            return;
        }

        const img = document.getElementById('qrcode');
        // 将图像源设置为 QR 码服务的路径，并传递文本参数
        img.src = `/qrservice?text=${encodeURIComponent(text)}`
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        if (button) {
            button.addEventListener('click', generateQR);
        } else {
            console.error('Generate button not found');
        }
    }
    ```

    **这里发生了什么：**

    - 我们移除了导入语句和 `await QRService.Generate()` 调用
    - 相反，我们只是将 `img.src` 设置为指向我们的 HTTP 端点
    - `encodeURIComponent()` 安全地转义了 URL 中的特殊字符
    - 当我们设置 `src` 时，浏览器会自动发起 HTTP GET 请求
    - 对于图像来说，这更简单且更高效——无需进行 base64 转换！

    再次运行应用程序，结果应该是相同的二维码：

8. ## 支持动态配置

    **硬编码路由的问题：**

    在上面的示例中，我们在 JavaScript 代码中使用了硬编码的路由 `/qrservice`。
    这导致您的 Go 配置与前端代码之间产生了紧密耦合。

    如果您编辑 `main.go` 并更改 `Route` 选项而未更新 `main.js`，
    应用程序将会崩溃：

    ```go title="main.go" ins={3}
            // ...
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/services/qr",
                }),
            // ...
    ```

    硬编码路由适用于简单的应用程序，但它们会使代码变得脆弱且难以维护。

    **解决方案：动态配置**

    方法绑定和 HTTP 处理器可以协同工作！我们可以使用绑定来告诉前端使用哪个路由，从而使配置动态化并消除硬编码路径。

    其工作原理如下：
    1. `ServiceStartup` 生命周期方法在应用启动时运行
    2. 我们保存从选项中配置的路由
    3. 我们添加一个 `URL()` 方法，前端可以调用它来获取正确的路由
    4. 现在，前端会向 Go 服务询问其路由，而不是猜测

    首先，实现 `ServiceStartup` 接口并添加一个新的 `URL` 方法：

    ```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
    package main

    import (
        "context"
        "net/http"
        "net/url"
        "strconv"

        "github.com/skip2/go-qrcode"
        "github.com/wailsapp/wails/v3/pkg/application"
    )

    // QRService handles QR code generation
    type QRService struct {
        route string
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // ServiceStartup runs at application startup.
    func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
        s.route = options.Route
        return nil
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    // URL returns an URL that may be used to fetch
    // a QR code with the given text and size.
    // It returns an error if the HTTP handler is not available.
    func (s *QRService) URL(text string, size int) (string, error) {
        if s.route == "" {
            return "", errors.New("http handler unavailable")
        }

        return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **这里发生了什么：**

    - 我们添加了一个 `route` 字段，用于存储从 `ServiceStartup` 中配置的路由
    - `ServiceStartup(ctx, options)` 在应用启动时调用——我们在此处保存路由
    - `URL()` 方法构建带有查询参数的完整 URL
    - 如果没有配置路由（路由为空），我们将返回错误
    - `url.QueryEscape()` 安全地对文本进行编码以用于 URL
    - 该方法将通过绑定对前端可用

    现在更新 `main.js`，使用 `URL` 方法替代硬编码路径：

    ```js title="frontend/src/main.js" ins={1,11-12}
    import { QRService } from "./bindings/changeme";

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        const img = document.getElementById('qrcode');
        // 调用 URL 方法以获取给定文本的 URL。
        img.src = await QRService.URL(text, 256);
    }```go

    }
    ```

    **这里发生了什么：**

    - 我们导入了 `QRService` 以再次使用绑定
    - 不再硬编码 `/qrservice`，而是调用 `await QRService.URL(text, 256)`
    - Go 服务会使用正确的路由和参数构建 URL
    - 现在，如果你在 `main.go` 中更改路由，前端将自动使用新路由
    - 不再需要手动同步 Go 配置和前端代码！

    它应该像之前的示例一样工作，
    但在 `main.go` 中更改服务路由
    将不再破坏前端。

    :::note
    如果 Go 方法返回非 nil 错误，
    JS 端的 Promise 将拒绝，
    并且 await 语句将抛出异常。
    :::
