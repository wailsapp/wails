Wails 前端运行时是 Wails 应用程序的标准库。它提供了许多可在应用程序中使用的功能，包括：

- 窗口管理
- 对话框
- 浏览器集成
- 剪贴板
- 菜单
- 系统信息
- 事件
- 上下文菜单
- 屏幕
- WML（Wails 标记语言）

运行时是 Go 与前端之间集成所必需的。有两种集成运行时的方式：

- 使用 `@wailsio/runtime` 包
- 使用预构建的捆绑包

## 使用 npm 包

`@wailsio/runtime` 包是一个 JavaScript 包，提供从前端访问 Wails 运行时的功能。它被所有标准模板使用，是将运行时集成到应用程序中的推荐方式。通过使用 `@wailsio/runtime` 包，您只会包含您使用的运行时部分。

该包可在 npm 上获取，并可通过以下命令安装：

```shell
npm install --save @wailsio/runtime
```

## 使用预构建的捆绑包

某些项目不使用 JavaScript 打包器，可能更倾向于使用预构建的运行时捆绑版本。可以使用以下命令在本地生成此版本：

```shell
wails3 generate runtime
```

该命令将在当前目录中输出 `runtime.js`（和 `runtime.debug.js`）文件。此文件是一个 ES 模块，可以像 npm 包一样被您的应用程序脚本导入，但 API 也导出到了全局 window 对象，因此对于更简单的应用程序，您可以按如下方式使用它：

```html
<html>
    <head>
        <script type="module" src="./runtime.js"></script>
        <script>
            window.onload = function () {
                wails.Window.SetTitle("A new window title");
            }
        </script>
    </head>
    <!--- ... -->
</html>
```

:::caution
重要的是要在加载运行时的 `<script>` 标签中包含 `type="module"` 属性，并在调用 API 之前等待页面完全加载，
因为带有 `type="module"` 属性的脚本是异步运行的。
:::

## 初始化

除了 API 函数外，运行时还提供对上下文菜单和窗口拖动的支持。
这些功能只有在运行时初始化之后才能按预期工作。
即使您不使用 API，也请确保在前端代码中的某个位置包含一个副作用导入语句：

```javascript
import "@wailsio/runtime";
```

您的打包器应检测到副作用的存在，并将所有必需的初始化代码包含在构建中。

:::note
如果您更喜欢预构建的捆绑包，添加如上所示的脚本标签即可。
:::

## 用于类型化事件的 Vite 插件

运行时包含一个 Vite 插件，可在开发期间为类型化事件启用 HMR（热模块替换）支持。

### 设置

将插件添加到您的 `vite.config.ts` 中：

```typescript
import { defineConfig } from 'vite'
import wails from '@wailsio/runtime/plugins/vite'

export default defineConfig({
  plugins: [wails()],
})
```

### 优势

- **自动重载**：当您运行 `wails3 generate bindings` 时，事件绑定会自动重新生成和重载
- **开发模式**：与 `wails3 dev` 无缝配合，实现即时更新
- **类型安全**：完整的 TypeScript 支持，包括自动补全和类型检查

### 与事件注册配合使用

在 Go 中注册您的事件：

```go
type UserData struct {
    ID   string
    Name string
}

func init() {
    application.RegisterEvent[UserData]("user-updated")
}
```

生成绑定：

```bash
wails3 generate bindings

# 或者，包含 TypeScript 定义
wails3 generate bindings -ts

# 更多选项，请参阅：
wails3 generate bindings -help
```

在前端中使用类型化事件：

```typescript
import { Events } from '@wailsio/runtime'
import { UserUpdated } from './bindings/events'

// 具有自动补全功能的类型安全事件
Events.Emit(UserUpdated({
    ID: "123",
    Name: "John Doe"
}))
```

## API 参考

运行时按模块组织，每个模块提供特定的功能。仅导入您需要的内容：

```javascript
import { Events, Window, Clipboard } from '@wailsio/runtime'
```

### Events

用于 Go 和 JavaScript 之间通信的事件系统。

#### On()

为事件注册回调函数。

```typescript
function On(eventName: string, callback: (event: WailsEvent) => void): () => void
function On<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**返回：** 取消订阅函数

**示例：**

```javascript
import { Events } from '@wailsio/runtime'

// 基本事件监听
const unsubscribe = Events.On('user-logged-in', (event) => {
    console.log('User:', event.data.username)
})

// 使用类型化事件 (TypeScript)
import { UserLogin } from './bindings/events'

Events.On(UserLogin, (event) => {
    // event.data 的类型为 UserLoginData
    console.log('User:', event.data.username)
})

// 稍后：unsubscribe()
```

#### Once()

注册仅运行一次的回调函数。

```typescript
function Once(eventName: string, callback: (event: WailsEvent) => void): () => void
function Once<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**示例：**

```javascript
import { Events } from '@wailsio/runtime'

Events.Once('app-ready', () => {
    console.log('App initialized')
})
```

#### Emit()

向 Go 后端或其他窗口发送事件。

```typescript
function Emit(name: string, data?: any): Promise<boolean>
function Emit<T>(event: Event<T>): Promise<boolean>
```

**返回：** 如果事件被取消则解析为 `true`，否则解析为 `false` 的 Promise

**示例：**

```javascript
import { Events } from '@wailsio/runtime'

// 基本事件发送
const wasCancelled = await Events.Emit('button-clicked', { buttonId: 'submit' })

// 使用类型化事件 (TypeScript)
import { UserLogin } from './bindings/events'

const cancelled = await Events.Emit(UserLogin({
    UserID: "123",
    Username: "john_doe",
    LoginTime: new Date().toISOString()
}))

if (cancelled) {
    console.log('Login was cancelled by a hook')
}
```

:::note
返回值指示事件是否被钩子取消。大多数事件无法被取消，将始终返回 `false`。
:::

### 系统

用于与后端直接通信的低级系统方法。

#### invoke()

直接向后端发送原始消息。这绕过了标准绑定系统，并由应用程序选项中的 `RawMessageHandler` 处理。

```typescript
function invoke(message: any): void
```

**示例：**

```javascript
import { System } from '@wailsio/runtime'

// 向后端发送原始消息
System.invoke('my-custom-message')

// 以 JSON 格式发送结构化数据
System.invoke(JSON.stringify({ action: 'update', value: 42 }))
```

:::caution
这是一个无返回值的“发送即忘”函数。请使用事件来接收后端的响应。
:::

更多详情，请参阅 [原始消息指南](/zh-cn/guides/raw-messages)。

### WML (Wails 标记语言)

WML 为常见操作提供声明式属性。将属性添加到 HTML 元素：

#### 属性

**wml-event** - 点击时触发事件

```html
<button wml-event="save-clicked">保存</button>
```

**wml-window** - 调用窗口方法

```html
<button wml-window="Close">关闭窗口</button>
<button wml-window="Minimise">最小化</button>
```

**wml-target-window** - 指定 wml-window 的目标窗口

```html
<button wml-window="Show" wml-target-window="settings">
    显示设置
</button>
```

**wml-openurl** - 在浏览器中打开 URL

```html
<a href="#" wml-openurl="https://wails.io">访问 Wails</a>
```

**wml-confirm** - 在操作前显示确认对话框

```html
<button wml-window="Close" wml-confirm="Are you sure you want to close?">
    关闭
</button>
```

**示例：**

```html
<div>
    <button wml-event="save-clicked">保存</button>
    <button wml-window="Minimise">最小化</button>
    <button wml-window="Close" wml-confirm="Close window?">关闭</button>
    <a href="#" wml-openurl="https://github.com/wailsapp/wails">GitHub</a>
</div>
```

## 完整示例

```javascript
import { Events, Window, Clipboard, Dialogs, Screens } from '@wailsio/runtime'

// 监听来自 Go 的事件
Events.On('data-updated', (event) => {
    console.log('Data:', event.data)
    updateUI(event.data)
})

// 窗口管理
document.getElementById('center-btn').addEventListener('click', async () => {
    await Window.Center()
})

document.getElementById('fullscreen-btn').addEventListener('click', async () => {
    const isFullscreen = await Window.IsFullscreen()
    if (isFullscreen) {
        await Window.UnFullscreen()
    } else {
        await Window.Fullscreen()
    }
})

// 剪贴板操作
document.getElementById('copy-btn').addEventListener('click', async () => {
    await Clipboard.SetText('Copied from Wails!')
})

// 带确认的对话框
document.getElementById('delete-btn').addEventListener('click', async () => {
    const result = await Dialogs.Question({
        Title: 'Confirm',
        Message: 'Delete this item?',
        Buttons: [
            { Label: 'Delete' },
            { Label: 'Cancel', IsDefault: true }
        ]
    })

    if (result === 'Delete') {
        await Events.Emit('delete-item', { id: currentItemId })
    }
})

// 屏幕信息
const screens = await Screens.GetAll()
console.log(`Detected ${screens.length} screen(s)`)
screens.forEach(screen => {
    console.log(`- ${screen.Name}: ${screen.Size.Width}x${screen.Size.Height}`)
})
```

## 最佳实践

### ✅ 建议

- **按需导入** - 仅导入所需内容
- **处理 Promise** - 所有方法均返回 Promise
- **对简单操作使用 WML** - 比 JavaScript 更简洁
- **检查返回值** - 尤其是对话框的返回值
- **取消事件订阅** - 完成后清理

### ❌ 避免

- **不要忘记 await** - 大多数方法是异步的
- **不要阻塞 UI** - 正确使用 async/await
- **不要忽略错误** - 始终处理拒绝（rejections）

## TypeScript 支持

```typescript

// 所有方法均有完整的类型定义
```
