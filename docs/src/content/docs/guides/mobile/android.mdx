## Prerequisites

- **Go** 1.21+ with CGO
- **Android SDK** (via Android Studio or command-line tools)
- **Android NDK** r26d+
- **Java JDK** 11+

### Environment Setup

Add to your shell profile:

```bash
# macOS
export ANDROID_HOME="$HOME/Library/Android/sdk"

# Linux
export ANDROID_HOME="$HOME/Android/Sdk"

export PATH="$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator"
```

## Building

```bash
# Build for device (arm64)
task android:build

# Build for emulator on Intel/AMD
task android:build ARCH=x86_64

# Package APK
task android:package

# Build and run
task android:run
```

<Aside type="tip">
Apple Silicon Macs run arm64 emulators. Use the default `ARCH=arm64`.
</Aside>

### Architectures

| Architecture | GOARCH | Use Case |
|--------------|--------|----------|
| arm64-v8a | arm64 | Modern devices, Apple Silicon emulators |
| x86_64 | amd64 | Intel/AMD emulators |

app := application.New(application.Options{

## Android-Specific Features

<Aside type="tip" title="Multiple API Options">
Android features are available through two APIs:
- **Unified API** (recommended): `app.Mobile.Vibrate()` - works cross-platform
- **Platform Manager**: `application.Android.Vibrate()` - Android-specific manager

See the [MobileManager API](/guides/mobile#mobilemanager-api) section for details on the unified API.
</Aside>

These features are available on Android via the `application.Android` manager or the unified `app.Mobile` API.

### Haptic Feedback

Trigger device vibration for tactile feedback:

<Tabs>
<TabItem label="Unified API (Recommended)">
```go
// Vibrate for 50 milliseconds
app.Mobile.Vibrate(50)

// Use haptic styles for cross-platform code
app.Mobile.HapticFeedback(application.HapticMedium)
app.Mobile.HapticFeedback(application.HapticHeavy)
```
</TabItem>
<TabItem label="Platform Manager">
```go
// Vibrate for 50 milliseconds
application.Android.Vibrate(50)

// Longer vibration for important notifications
application.Android.Vibrate(200)
```
</TabItem>
<TabItem label="Frontend">
```javascript
// Call via bound service
await myService.TriggerHaptic(50);
```
</TabItem>
</Tabs>

<Aside type="note">
Requires `android.permission.VIBRATE` in AndroidManifest.xml (included by default).
</Aside>

### Toast Notifications

Show native Android toast messages:

<Tabs>
<TabItem label="Unified API (Recommended)">
```go
// Show a short toast message
app.Mobile.Toast("Operation completed!")

// Toasts appear at the bottom of the screen
app.Mobile.Toast("File saved successfully")
```
</TabItem>
<TabItem label="Platform Manager">
```go
// Show a short toast message
application.Android.ShowToast("Operation completed!")

// Toasts appear at the bottom of the screen
application.Android.ShowToast("File saved successfully")
```
</TabItem>
</Tabs>

Toasts are non-blocking and auto-dismiss after a short duration.

### Device Information

Get detailed information about the Android device:

<Tabs>
<TabItem label="Unified API (Recommended)">
```go
// Returns typed DeviceInfo struct
info := app.Mobile.DeviceInfo()
fmt.Println(info.Platform)     // "android"
fmt.Println(info.Manufacturer) // e.g., "Google", "Samsung"
fmt.Println(info.Model)        // e.g., "Pixel 7", "SM-S918B"
fmt.Println(info.Brand)        // e.g., "google", "samsung"
fmt.Println(info.SDKVersion)   // e.g., 34 (Android 14)
fmt.Println(info.OSVersion)    // e.g., "14"
```
</TabItem>
<TabItem label="Platform Manager">
```go
// Returns JSON string with device details
infoJSON := application.Android.GetDeviceInfo()

// Parse the JSON to access fields:
// - platform: "android"
// - manufacturer: e.g., "Google", "Samsung"
// - model: e.g., "Pixel 7", "SM-S918B"
// - brand: e.g., "google", "samsung"
// - sdkVersion: e.g., 34 (Android 14)
// - release: e.g., "14"
```
</TabItem>
</Tabs>

### Screen Information

Get display metrics for the device screen:

```go
// Returns JSON with screen details
screenJSON := application.Android.GetScreenInfo()

// Available fields:
// - widthPixels: Screen width in pixels
// - heightPixels: Screen height in pixels
// - density: Display density scale factor
// - densityDpi: DPI (160, 240, 320, etc.)
// - scaledDensity: Font scaling factor
// - xdpi, ydpi: Exact physical DPI
```

### Logging

Write messages to Android logcat:

```go
// Log at different levels
application.Android.Debug("Debug message: %s", value)
application.Android.Info("Info message")
application.Android.Warn("Warning: %v", err)
application.Android.Error("Error occurred: %v", err)

// Or use the generic Log method
application.Android.Log("info", "Custom log message")
```

View logs with:
```bash
task android:logs
```

## Cross-Platform Features

These features work on Android through the unified managers (same API as desktop):

### Browser/URLs

```go
// Open URLs in default browser
app.Browser.OpenURL("https://wails.io")
```

### Clipboard

```go
// Copy text
app.Clipboard.SetText("Hello!")

// Read text
text, _ := app.Clipboard.Text()
```

### Dialogs

```go
// Show message dialog
app.Dialogs.Info("Title", "Message")
app.Dialogs.Warning("Title", "Message")
app.Dialogs.Error("Title", "Message")
```

## Troubleshooting

**INSTALL_FAILED_NO_MATCHING_ABIS**
- Architecture mismatch. Use `ARCH=arm64` for Apple Silicon, `ARCH=x86_64` for Intel/AMD emulators.

**NDK not found**
```bash
export ANDROID_NDK_HOME=$ANDROID_HOME/ndk/26.1.10909125
```

**Blank WebView**
1. Check logcat: `task android:logs`
2. Debug with Chrome DevTools
3. Verify `main_android.go` exists in build assets
