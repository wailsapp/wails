Wails v3 enables building native mobile apps using the same Go codebase and web frontend as desktop.

## How It Works

| Platform | WebView | Bridge | Output |
|----------|---------|--------|--------|
| iOS | WKWebView | CGO | .app / .ipa |
| Android | Android WebView | JNI | .apk |

Your Go code compiles to a native library, your web frontend renders in a native WebView, and Wails bridges them together.

## MobileManager API

Wails v3 provides a unified `MobileManager` API that works across both iOS and Android, as well as desktop platforms (where methods are no-ops).

### Accessing the MobileManager

```go
app := application.New(application.Options{...})

// Access mobile features via app.Mobile
app.Mobile.Vibrate(50)
app.Mobile.Toast("Hello!")
```

### Available Methods

| Method | Description | iOS | Android |
|--------|-------------|-----|---------|
| `Vibrate(ms)` | Trigger vibration | Uses haptic feedback (duration ignored) | Vibrates for specified milliseconds |
| `HapticFeedback(style)` | Native haptic feedback | Taptic Engine with style | Short vibration mapped to style |
| `Toast(message)` | Show toast notification | Not yet implemented | Native Android Toast |
| `DeviceInfo()` | Get device information | Returns DeviceInfo struct | Returns DeviceInfo struct |
| `ScreenInfo()` | Get screen metrics | Returns MobileScreenInfo | Returns MobileScreenInfo |
| `IsMobile()` | Check if mobile platform | Returns `true` | Returns `true` |

### Haptic Feedback Styles

```go
// Available styles
application.HapticLight   // Subtle feedback
application.HapticMedium  // Moderate feedback (default)
application.HapticHeavy   // Strong feedback
application.HapticSoft    // Soft, gentle feedback
application.HapticRigid   // Sharp, precise feedback

// Usage
app.Mobile.HapticFeedback(application.HapticMedium)
```

### Device Information

```go
info := app.Mobile.DeviceInfo()
fmt.Println(info.Platform)     // "android", "ios", or "desktop"
fmt.Println(info.Model)        // e.g., "Pixel 7", "iPhone 15"
fmt.Println(info.Manufacturer) // e.g., "Google", "Apple"
fmt.Println(info.OSVersion)    // e.g., "14", "17.0"
fmt.Println(info.SDKVersion)   // Android SDK level (Android only)
```

### WebView Control

Control WebView behavior at runtime:

```go
app.Mobile.SetScrollEnabled(false)           // Disable scrolling
app.Mobile.SetBounceEnabled(false)           // Disable iOS bounce/Android overscroll
app.Mobile.SetScrollIndicatorsEnabled(false) // Hide scroll bars
app.Mobile.SetBackForwardGesturesEnabled(true) // Enable swipe navigation
app.Mobile.SetLinkPreviewEnabled(true)       // Enable link previews (iOS only)
app.Mobile.SetInspectableEnabled(true)       // Enable DevTools debugging
app.Mobile.SetCustomUserAgent("MyApp/1.0")   // Set custom user agent
```

### Cross-Platform Pattern

The MobileManager uses the same pattern as other Wails managers. On desktop, all methods are safe to call but are no-ops:

```go
// This code works on all platforms
if app.Mobile.IsMobile() {
    app.Mobile.HapticFeedback(application.HapticMedium)
} else {
    // Desktop fallback if needed
}
```
