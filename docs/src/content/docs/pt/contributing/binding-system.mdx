> Os "Bindings" são o **contrato tipado** que permite escrever:

```go
msg, err := chatService.Send("Hello")
```

em Go *e*

```ts
const msg = await window.backend.ChatService.Send("Hello")
```

em TypeScript **sem código de cola manual**.  
Este documento detalha *como* essa mágica acontece, desde a **análise estática**
em tempo de build, passando pela **geração de código**, até a **ponte de runtime**
que move bytes sobre o WebView.

---

## 1. Visão Geral em 30 Segundos

| Estágio | Componente | Saída |
|-------|-----------|--------|
| **Análise** | `internal/generator/analyse.go` | AST em memória de structs exportadas do Go, métodos, parâmetros, tipos de retorno |
| **Geração** | `internal/generator/render/*.tmpl` | • `pkg/application/bindings.go` (Go)<br />• `frontend/src/wailsjs/**.ts` (defs TS)<br />• `runtime/desktop/@wailsio/runtime/internal/bindings.json` |
| **Runtime** | `pkg/application/messageprocessor_call.go` + runtime JS (`invoke.ts`) | Mensagens JSON sobre a ponte nativa do WebView |

O fluxo é orquestrado pela CLI `wails3`:

```
wails3 generate  ─┬─> generator.Collect()   // parse Go packages
                  ├─> generator.Analyse()   // constrói modelo semântico
                  └─> generator.Render()    // escreve arquivos + `go fmt`
```

## 2. Análise Estática

### Ponto de Entrada

```
internal/generator/analyse.go
```

`Analyse(cfg generator.Config)`:

1. Carrega recursivamente os pacotes Go fornecidos com `go/packages`.
2. Percorre as especificações de *tipo* e *valor* de cada arquivo AST.
3. Filtra símbolos **exportados** (iniciados com maiúscula) fora dos pacotes `internal/`.
4. Registra:
   * Tipo receptor (`struct`, `interface`)
   * Nome do método
   * Lista de parâmetros (nome, tipo, ponteiro, variádico)
   * Tupla de retorno (valores + presença de erro)
5. Normaliza tipos para um **conjunto serializável**  
   (`int`, `float64`, `string`, `[]byte`, slices, maps, structs, pointers).

Tipos não suportados produzem um **erro em tempo de compilação** para que erros sejam capturados cedo.

### Modelo

```go
type Method struct {
    ID       uint32 // hash estável para lookup no runtime
    Name     string
    Params   []Param
    Results  []Result
    Receiver *Struct // nil para funções de pacote
}
```

Um hash **FNV-1a** determinístico (`internal/hash/fnv.go`) de
`<Receiver>.<Name>(<Signature>)` torna-se o *ID do método* usado no runtime.

## 3. Geração de Código

### Templates

`internal/generator/render/` contém arquivos text/template:

| Template | Alvo |
|----------|--------|
| `go_bindings.tmpl` | `pkg/application/bindings.go` |
| `ts_bindings.tmpl` | `frontend/wailsjs/go/models.d.ts` |
| `ts_index.tmpl`    | `frontend/wailsjs/index.ts` |

Adicione ou ajuste templates aqui para personalizar a geração.

### Saída Go

`bindings.go` registra uma tabela de lookup:

```go
var bindings = []application.BoundMethod{
    {ID: 0x7A1201D3, Name: "ChatService.Send", Call: chatServiceSend},
}

func chatServiceSend(ctx context.Context, in []byte) ([]byte, error) {
    var req struct{ Msg string }
    if err := json.Unmarshal(in, &req); err != nil { return nil, err }
    res, err := chatService.Send(req.Msg)
    return json.Marshal(res), err
}
```

Pontos-chave:

* **Zero reflexão** no runtime → performance.
* Marshal/Unmarshal é **por método** com wrappers de struct gerados.

### Saída TypeScript

```ts
export namespace backend {
  export namespace ChatService {
    function Send(msg: string): Promise<string>;
  }
}
```

* Emitido como **módulos ES** para que qualquer bundler possa fazer tree-shake.
* Os IDs dos métodos são embutidos em um `bindings.json` auto-gerado para o runtime JS.

## 4. Protocolo de Invocação de Runtime

### Lado JavaScript

```ts
import { System } from "@wailsio/runtime";

await System.invoke(0x7a1201d3 /* ChatService.Send */, ["Hello"]);
```

Implementação: `runtime/desktop/@wailsio/runtime/invoke.ts`

1. Empacota `{t:"c", id:<methodID>, p:[...args]}` em JSON.
2. Chama `window.external.invoke(payload)` (WebView2) ou equivalente.
3. Retorna um `Promise` que resolve/rejeita com base na resposta.

### Lado Go

1. `messageprocessor_call.go` recebe o JSON.
2. Faz lookup do `methodID` na fatia `bindings`.
3. Executa o stub gerado (`chatServiceSend`).
4. Serializa `{result, error}` de volta para JS.

### Mapeamento de Erros

| Go | JavaScript |
|----|------------|
| `error == nil` | `Promise` resolve com o resultado |
| `errors.New(...)` | `Promise` rejeita com `{message, stack, code}` |

O código de mapeamento reside em `runtime/desktop/@wailsio/runtime/errors.ts`.

## 5. Chamando JavaScript do Go

*Browser → Go* é coberto acima.  
*Go → Browser* usa **Eventos** ou **Eval**:

```go
window.Eval(`alert("Hi")`)
app.Publish("chat:new-message", msg)
```

A geração de bindings é unidirecional (métodos Go).  
Para funções expostas por JS, use `runtime.EventsOn` no JS e `application.Publish`
do Go.

## 6. Extensão e Solução de Problemas

### Adicionando Serializadores Personalizados

* Implemente a interface `generator.TypeConverter`.
* Registre em `generator.Config.Converters`.
* Atualize a desserialização do runtime JS se necessário.

### Erro de Tipo Não Suportado

```
error: field "Client" uses unsupported type: chan struct{}
```

→ envolva o canal em uma struct com uma API exposta ou redesenhe.

### Divergência de Versão

Os bindings são regenerados em **cada** `wails3 dev` / `wails3 build`.  
Se o intellisense do IDE mostrar stubs desatualizados, exclua `frontend/wailsjs` e reconstrua.

### Dicas de Performance

* Prefira receptores de **valor** para structs pequenas para reduzir alocações.
* Evite slices de bytes grandes sobre a ponte; use `application.FileServer` em vez disso.
* Agrupe várias chamadas rápidas em um único método para minimizar a latência da ponte.

Mantenha esta folha de dicas à mão ao rastrear um bug da ponte.

---

## 8. Resumo

1. O **Gerador** escaneia seu código Go → modelo semântico.  
2. Os **Templates** emitem **stubs Go** + **definições TypeScript**.  
3. O **Processador de Mensagens** executa os stubs no runtime.  
4. O **Runtime JS** embrulha tudo em promises idiomáticas.

Tudo sem reflexão, sem servidores IPC, e sem você escrever uma única
linha de código boilerplate. Esse é o sistema de bindings do Wails v3. Vá em frente e faça bindings!
