## Bem-vindo à Documentação Técnica do Wails v3

Esta seção **não** trata de diretrizes da comunidade ou de como abrir um pull request.
Em vez disso, ela se aprofunda em **como o Wails v3 é construído** para que você possa se orientar
rapidamente no código-fonte e começar a desenvolver com confiança.

Se você planeja corrigir o runtime, estender a CLI, criar novos templates ou
simplesmente entender os componentes internos, as páginas a seguir fornecem o contexto técnico
que você precisa.

## Arquitetura de Alto Nível

<CardGrid>
  <Card title="Backend Go" icon="seti:go">
    O coração de cada aplicativo Wails é o código Go compilado em um executável nativo.
    Ele é responsável pela lógica da aplicação, integração com o sistema e operações
    críticas de desempenho.
  </Card>

  <Card title="Frontend Web" icon="seti:html">
    A interface do usuário é escrita com tecnologias web padrão (React, Vue, Svelte, Vanilla, …)

    Windows).
  </Card>

  <Card title="Camada de Ponte" icon="puzzle">
    Uma ponte em memória sem cópia permite chamadas **Go⇄JavaScript** com conversão
    automática de tipos, propagação de eventos e encaminhamento de erros.
  </Card>

  <Card title="CLI e Ferramentas" icon="terminal">
    `wails3` orquestra a criação de projetos, o servidor de desenvolvimento com recarregamento em tempo real, o empacotamento de ativos, a compilação cruzada e o empacotamento (deb, rpm, AppImage, msi, dmg…).
  </Card>
</CardGrid>

O diagrama mostra o **fluxo de ponta a ponta**:

1. A **CLI** direciona a geração, o servidor de desenvolvimento, a compilação e o empacotamento.  
2. O **Sistema de Bindings** produz código de ligação que permite que o **Frontend Web** chame o **Backend Go**.  
3. Durante o desenvolvimento, o **Servidor de Ativos** faz proxy para o servidor de desenvolvimento do framework; em produção, ele serve arquivos embutidos.  
4. Em tempo de execução, o **Desktop Runtime** gerencia janelas e APIs do sistema operacional, enquanto a **Bridge** transporta mensagens entre Go e JavaScript.
