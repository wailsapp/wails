# Usando Gin para Serviços

O framework web Gin é uma escolha popular para construir serviços HTTP em Go. Com o Wails v3, você pode integrar facilmente serviços baseados no Gin em sua aplicação, proporcionando uma maneira poderosa de lidar com requisições HTTP, implementar APIs RESTful e servir conteúdo web.

Este guia irá guiá-lo na criação de um serviço baseado no Gin que pode ser montado em uma rota específica em sua aplicação Wails. Construiremos um exemplo completo que demonstra como:

1. Criar um serviço baseado no Gin
2. Implementar a interface Service do Wails
3. Configurar rotas e middlewares
4. Integrar com o sistema de eventos do Wails
5. Interagir com o serviço a partir do frontend

## Pré-requisitos

Antes de começar, certifique-se de ter:

- Wails v3 instalado
- Conhecimento básico de Go e do framework Gin
- Familiaridade com conceitos de HTTP e APIs RESTful

Você precisará adicionar o framework Gin ao seu projeto:

```bash
go get github.com/gin-gonic/gin
```

## Criando um Serviço Baseado no Gin

Vamos começar criando um serviço Gin que implementa a interface Service do Wails. Nosso serviço irá gerenciar uma coleção de usuários e fornecer endpoints de API para recuperar e criar registros de usuários.

### 1. Defina seus Modelos de Dados

Primeiro, defina as estruturas de dados com as quais seu serviço irá trabalhar:

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User representa um usuário no sistema
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData representa dados enviados em eventos
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. Crie a Estrutura do Seu Serviço

Em seguida, defina a estrutura do serviço que irá conter seu roteador Gin e qualquer estado que seu serviço precise manter:

```go
// GinService implementa um serviço Wails que usa Gin para manipulação HTTP
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService cria uma nova instância de GinService
func NewGinService() *GinService {
	// Cria um novo roteador Gin
	ginEngine := gin.New()

	// Adiciona middlewares
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// Define as rotas
	service.setupRoutes()

	return service
}
```

### 3. Implemente a Interface do Serviço

Implemente os métodos necessários para a interface Service do Wails:

```go
// ServiceName retorna o nome do serviço
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup é chamado quando o serviço inicia
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// Armazena a instância da aplicação para uso posterior
	s.app = application.Get()

	// Registra um manipulador de eventos que pode ser acionado a partir do frontend
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// Registra os dados do evento
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// Emite um evento de volta para o frontend
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown é chamado quando o serviço é desligado
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// Limpa recursos se necessário
	return nil
}
```

### 3. Implemente a Interface http.Handler

Para tornar seu serviço montável em uma rota específica, implemente a interface `http.Handler`. Este único método, `ServeHTTP`, é a porta de entrada para todas as requisições HTTP ao seu serviço. Ele delega o tratamento da requisição ao roteador Gin, permitindo que você use todos os recursos poderosos do Gin para roteamento e middleware.

```go
// ServeHTTP implementa a interface http.Handler
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// Todas as requisições vão para o roteador Gin
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. Configure suas Rotas

Defina as rotas da sua API em um método separado para melhor organização. Esta abordagem mantém seu código limpo e facilita a compreensão da estrutura da sua API. O roteador Gin fornece uma API fluida para definir rotas, incluindo suporte para grupos de rotas, que ajudam a organizar endpoints relacionados.

```go
// setupRoutes configura as rotas da API
func (s *GinService) setupRoutes() {
	// Endpoint de informações básicas
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// Grupo de usuários
	users := s.ginEngine.Group("/users")
	{
		// Obter todos os usuários
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// Obter usuário por ID
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// Criar um novo usuário
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// Define o ID e o tempo de criação
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// Adiciona à fatia de usuários
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// Emite um evento para notificar sobre o novo usuário
			s.app.Event.Emit("user-created", newUser)
		})

		// Deletar um usuário
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// Remove o usuário da fatia
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. Crie Middleware Personalizado

Você pode criar middleware Gin personalizado para aprimorar seu serviço. As funções de middleware no Gin são executadas na ordem em que são adicionadas ao roteador e podem realizar tarefas como logging, autenticação e tratamento de erros. Este exemplo mostra um middleware de logging simples que registra detalhes da requisição.

```go
// LoggingMiddleware é um middleware Gin que registra detalhes da requisição
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// Inicia o timer
		start := time.Now()

		// Processa a requisição
		c.Next()

		// Calcula a latência
		latency := time.Since(start)

		// Registra os detalhes da requisição
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## Registrando seu ServiçoPara usar seu serviço baseado em Gin em uma aplicação Wails, você precisa registrá-lo na aplicação e especificar a rota onde ele deve ser montado. Isso é feito ao criar a instância da aplicação Wails. A rota que você especificar se tornará o caminho base para todos os endpoints definidos no seu roteador Gin.

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

Neste exemplo, o serviço Gin é montado na rota `/api`. Isso significa que, se o seu roteador Gin tiver um endpoint definido como `/info`, ele estará acessível em `/api/info` na sua aplicação. Essa abordagem permite que você organize os endpoints da sua API de forma lógica e evite conflitos com outras partes da sua aplicação.

## Integração com o Sistema de Eventos do Wails

Um dos recursos poderosos de usar Gin com Serviços Wails é a capacidade de integrar-se perfeitamente ao sistema de eventos do Wails. Isso permite comunicação em tempo real entre seu serviço de backend e o frontend.

No método `ServiceStartup` do seu serviço, você pode registrar manipuladores de eventos para processar eventos provenientes do frontend:

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

Você também pode emitir eventos para o frontend a partir das suas rotas Gin ou de outras partes do seu serviço:

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## Integração com o Frontend

Para interagir com seu serviço Gin a partir do frontend, você precisará importar o runtime do Wails, fazer requisições HTTP aos endpoints da sua API e usar o sistema de eventos do Wails para comunicação em tempo real.

Para uso em produção, é recomendável usar o pacote `@wailsio/runtime` em vez de importar diretamente `/wails/runtime.js`. Isso garante segurança de tipos, melhor suporte da IDE, gerenciamento de versões através do npm e compatibilidade com ferramentas modernas de JavaScript.

Instale o pacote:
```bash
npm install @wailsio/runtime
```

Em seguida, use-o no seu código:
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## Reflexões Finais

Integrar o framework web Gin com os Serviços do Wails v3 oferece uma abordagem poderosa e flexível para construir aplicações web modulares e de fácil manutenção. Ao aproveitar as capacidades de roteamento e middleware do Gin, juntamente com o sistema de eventos do Wails, você pode criar aplicações ricas e interativas com uma clara separação de responsabilidades.

O código de exemplo completo para este guia pode ser encontrado no repositório do Wails em `v3/examples/gin-service`.
