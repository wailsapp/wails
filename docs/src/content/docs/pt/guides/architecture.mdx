## Visão Geral

Padrões comprovados para organizar sua aplicação Wails.

## Padrão de Camada de Serviço

### Estrutura

```
app/
├── main.go
├── services/
│   ├── user_service.go
│   ├── data_service.go
│   └── file_service.go
└── models/
    └── user.go
```

### Implementação

```go
// Interface do serviço
type UserService interface {
    Create(email, password string) (*User, error)
    GetByID(id int) (*User, error)
    Update(user *User) error
    Delete(id int) error
}

// Implementação
type userService struct {
    app *application.Application
    db  *sql.DB
}

func NewUserService(app *application.Application, db *sql.DB) UserService {
    return &userService{app: app, db: db}
}
```

## Padrão de Repositório

### Estrutura

```go
// Interface do repositório
type UserRepository interface {
    Create(user *User) error
    FindByID(id int) (*User, error)
    Update(user *User) error
    Delete(id int) error
}

// Serviço usa o repositório
type UserService struct {
    repo UserRepository
}

func (s *UserService) Create(email, password string) (*User, error) {
    user := &User{Email: email}
    return user, s.repo.Create(user)
}
```

## Arquitetura Orientada a Eventos

### Bus de Eventos

```go
type EventBus struct {
    app       *application.Application
    listeners map[string][]func(interface{})
    mu        sync.RWMutex
}

func (eb *EventBus) Subscribe(event string, handler func(interface{})) {
    eb.mu.Lock()
    defer eb.mu.Unlock()
    eb.listeners[event] = append(eb.listeners[event], handler)
}

func (eb *EventBus) Publish(event string, data interface{}) {
    eb.mu.RLock()
    handlers := eb.listeners[event]
    eb.mu.RUnlock()
    
    for _, handler := range handlers {
        go handler(data)
    }
}
```

### Uso

```go
// Inscrever-se
eventBus.Subscribe("user.created", func(data interface{}) {
    user := data.(*User)
    sendWelcomeEmail(user)
})

// Publicar
eventBus.Publish("user.created", user)
```

## Injeção de Dependência

### DI Manual

```go
type App struct {
    userService *UserService
    fileService *FileService
    db          *sql.DB
}

func NewApp() *App {
    db := openDatabase()
    
    return &App{
        db:          db,
        userService: NewUserService(db),
        fileService: NewFileService(db),
    }
}
```

### Usando Wire

```go
// wire.go
//go:build wireinject

func InitializeApp() (*App, error) {
    wire.Build(
        openDatabase,
        NewUserService,
        NewFileService,
        NewApp,
    )
    return nil, nil
}
```

## Gerenciamento de Estado

### Estado Centralizado

```go
type AppState struct {
    currentUser *User
    settings    *Settings
    mu          sync.RWMutex
}

func (s *AppState) SetUser(user *User) {
    s.mu.Lock()
    defer s.mu.Unlock()
    s.currentUser = user
}

func (s *AppState) GetUser() *User {
    s.mu.RLock()
    defer s.mu.RUnlock()
    return s.currentUser
}
```

## Melhores Práticas

### ✅ Faça

- Separe as responsabilidades
- Use interfaces
- Injete dependências
- Trate os erros adequadamente
- Mantenha os serviços focados
- Documente a arquitetura

### ❌ Não faça

- Não crie objetos "deus" (god objects)
- Não acople fortemente os componentes
- Não ignore o tratamento de erros
- Não ignore a concorrência
- Não faça engenharia excessiva (over-engineering)
