# Atualizações Automáticas

O Wails v3 oferece um sistema de atualizador integrado que suporta verificação automática de atualizações, download e instalação. O atualizador inclui suporte para atualizações delta de binários (patches) para minimizar o tamanho do download.

## Início Rápido

Adicione o serviço de atualizador ao seu aplicativo:

```go title="main.go"
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "time"
)

func main() {
    // Cria o serviço de atualizador
    updater, err := application.CreateUpdaterService(
        "1.0.0", // Versão atual
        application.WithUpdateURL("https://updates.example.com/myapp/"),
        application.WithCheckInterval(24 * time.Hour),
    )
    if err != nil {
        panic(err)
    }

    app := application.New(application.Options{
        Name: "MyApp",
        Services: []application.Service{
            application.NewService(updater),
        },
    })

    // ... resto do seu app
    app.Run()
}
```

Em seguida, use-o no seu frontend:

```typescript title="App.tsx"
import { updater } from './bindings/myapp';

async function checkForUpdates() {
  const update = await updater.CheckForUpdate();

  if (update) {
    console.log(`Nova versão disponível: ${update.version}`);
    console.log(`Notas de lançamento: ${update.releaseNotes}`);

    // Baixa e aplica a atualização
    await updater.DownloadAndApply();
  }
}
```

## Opções de Configuração

O atualizador suporta várias opções de configuração:

```go
updater, err := application.CreateUpdaterService(
    "1.0.0",
    // Obrigatório: URL onde os manifests de atualização são hospedados
    application.WithUpdateURL("https://updates.example.com/myapp/"),

    // Opcional: Verificar atualizações automaticamente a cada 24 horas
    application.WithCheckInterval(24 * time.Hour),

    // Opcional: Permitir versões pré-lançamento
    application.WithAllowPrerelease(true),

    // Opcional: Canal de atualização (stable, beta, canary)
    application.WithChannel("stable"),

    // Opcional: Exigir atualizações assinadas
    application.WithRequireSignature(true),
    application.WithPublicKey("your-ed25519-public-key"),
)
```

## API do Frontend

O atualizador expõe métodos que são automaticamente vinculados ao seu frontend:

### Tipos TypeScript

```typescript
interface UpdateInfo {
  version: string;
  releaseDate: Date;
  releaseNotes: string;
  size: number;
  patchSize?: number;
  mandatory: boolean;
  hasPatch: boolean;
}

interface Updater {
  // Obtém a versão atual do aplicativo
  GetCurrentVersion(): string;

  // Verifica se há uma atualização disponível
  CheckForUpdate(): Promise<UpdateInfo | null>;

  // Baixa a atualização (emite eventos de progresso)
  DownloadUpdate(): Promise<void>;

  // Aplica a atualização baixada (reinicia o app)
  ApplyUpdate(): Promise<void>;

  // Baixa e aplica em uma única chamada
  DownloadAndApply(): Promise<void>;

  // Obtém o estado atual: idle, checking, available, downloading, ready, installing, error
  GetState(): string;

  // Obtém as informações da atualização disponível
  GetUpdateInfo(): UpdateInfo | null;

  // Obtém a última mensagem de erro
  GetLastError(): string;

  // Reseta o estado do atualizador
  Reset(): void;
}
```

### Eventos de Progresso

Ouça os eventos de progresso do download:

```typescript
import { Events } from '@wailsio/runtime';

Events.On('updater:progress', (data) => {
  console.log(`Baixado: ${data.downloaded} / ${data.total}`);
  console.log(`Progresso: ${data.percentage.toFixed(1)}%`);
  console.log(`Velocidade: ${(data.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s`);
});
```

## Exemplo Completo

Aqui está um exemplo completo com um componente React:

```tsx title="UpdateChecker.tsx"
import { useState, useEffect } from 'react';
import { updater } from './bindings/myapp';
import { Events } from '@wailsio/runtime';

interface Progress {
  downloaded: number;
  total: number;
  percentage: number;
  bytesPerSecond: number;
}

export function UpdateChecker() {
  const [checking, setChecking] = useState(false);
  const [updateInfo, setUpdateInfo] = useState<any>(null);
  const [downloading, setDownloading] = useState(false);
  const [progress, setProgress] = useState<Progress | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // Ouve eventos de progresso
    const cleanup = Events.On('updater:progress', (data: Progress) => {
      setProgress(data);
    });

    // Verifica atualizações na montagem
    checkForUpdates();

    return () => cleanup();
  }, []);

  async function checkForUpdates() {
    setChecking(true);
    setError(null);

    try {
      const info = await updater.CheckForUpdate();
      setUpdateInfo(info);
    } catch (err) {
      setError(err.message);
    } finally {
      setChecking(false);
    }
  }

  async function downloadAndInstall() {
    setDownloading(true);
    setError(null);

    try {
      await updater.DownloadAndApply();
      // O app será reiniciado automaticamente
    } catch (err) {
      setError(err.message);
      setDownloading(false);
    }
  }

  if (checking) {
    return <div>Verificando atualizações...</div>;
  }

  if (error) {
    return (
      <div>
        <p>Erro: {error}</p>
        <button onClick={checkForUpdates}>Tentar Novamente</button>
      </div>
    );
  }

  if (!updateInfo) {
    return (
      <div>
        <p>Você está atualizado! (v{updater.GetCurrentVersion()})</p>
        <button onClick={checkForUpdates}>Verificar Novamente</button>
      </div>
    );
  }

  if (downloading) {
    return (
      <div>
        <p>Baixando atualização...</p>
        {progress && (
          <div>
            <progress value={progress.percentage} max={100} />
            <p>{progress.percentage.toFixed(1)}%</p>
            <p>{(progress.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s</p>
          </div>
        )}
      </div>
    );
  }return (
    <div>
      <h3>Atualização Disponível!</h3>
      <p>A versão {updateInfo.version} está disponível</p>
      <p>Tamanho: {updateInfo.hasPatch
        ? `${(updateInfo.patchSize / 1024).toFixed(0)} KB (patch)`
        : `${(updateInfo.size / 1024 / 1024).toFixed(1)} MB`}
      </p>
      <div dangerouslySetInnerHTML={{ __html: updateInfo.releaseNotes }} />
      <button onClick={downloadAndInstall}>Baixar & Instalar</button>
      <button onClick={() => setUpdateInfo(null)}>Pular</button>
    </div>
  );
}
```

## Estratégias de Atualização

### Verificação na Inicialização

```go
func (a *App) OnStartup(ctx context.Context) {
    // Verificar atualizações após um curto atraso
    go func() {
        time.Sleep(5 * time.Second)
        info, err := a.updater.CheckForUpdate()
        if err == nil && info != nil {
            // Emitir evento para o frontend
            application.Get().EmitEvent("update-available", info)
        }
    }()
}
```

### Verificação em Segundo Plano

Configure verificações automáticas em segundo plano:

```go
updater, _ := application.CreateUpdaterService(
    "1.0.0",
    application.WithUpdateURL("https://updates.example.com/myapp/"),
    application.WithCheckInterval(6 * time.Hour), // Verificar a cada 6 horas
)
```

### Item de Menu de Verificação Manual

```go
menu := application.NewMenu()
menu.Add("Verificar Atualizações...").OnClick(func(ctx *application.Context) {
    info, err := updater.CheckForUpdate()
    if err != nil {
        application.InfoDialog().SetMessage("Erro ao verificar atualizações").Show()
        return
    }
    if info == nil {
        application.InfoDialog().SetMessage("Você está atualizado!").Show()
        return
    }
    // Exibir diálogo de atualização...
})
```

## Atualizações Delta (Patches)

As atualizações delta (patches) permitem que os usuários baixem apenas as alterações entre versões, reduzindo drasticamente o tamanho do download.

### Como Funciona

1. Ao criar uma nova versão, gere patches a partir das versões anteriores
2. Hospede os patches junto com as atualizações completas no seu servidor
3. O atualizador baixa automaticamente os patches quando disponíveis
4. Se a aplicação do patch falhar, ele retorna ao download completo

### Gerando Patches

Os patches são gerados usando o algoritmo bsdiff. Você precisará da ferramenta `bsdiff`:

```bash
# Instalar bsdiff (macOS)
brew install bsdiff

# Instalar bsdiff (Ubuntu/Debian)
sudo apt-get install bsdiff

# Gerar um patch
bsdiff old-binary new-binary patch.bsdiff
```

### Nomeação de Arquivos de Patch

Organize seus patches no seu diretório de atualização:

```
updates/
├── update.json
├── myapp-1.2.0-macos-arm64.tar.gz
├── myapp-1.2.0-windows-amd64.zip
└── patches/
    ├── 1.0.0-to-1.2.0-macos-arm64.patch
    ├── 1.1.0-to-1.2.0-macos-arm64.patch
    ├── 1.0.0-to-1.2.0-windows-amd64.patch
    └── 1.1.0-to-1.2.0-windows-amd64.patch
```

<Aside type="tip">
Mantenha os patches das últimas algumas versões. Usuários em versões muito antigas farão o download da atualização completa automaticamente.
</Aside>

## Hospedando Atualizações

### Hospedagem de Arquivos Estáticos

As atualizações podem ser hospedadas em qualquer servidor de arquivos estáticos:

- **Amazon S3** / **Cloudflare R2**
- **Google Cloud Storage**
- **GitHub Releases**
- **Qualquer CDN ou servidor web**

Exemplo de estrutura de bucket S3:

```
s3://my-updates-bucket/myapp/
├── stable/
│   ├── update.json
│   ├── myapp-1.2.0-macos-arm64.tar.gz
│   └── patches/
│       └── 1.1.0-to-1.2.0-macos-arm64.patch
└── beta/
    └── update.json
```

### Configuração de CORS

Se estiver hospedando em um domínio diferente, configure o CORS:

```json
{
  "CORSRules": [
    {
      "AllowedOrigins": ["*"],
      "AllowedMethods": ["GET"],
      "AllowedHeaders": ["*"]
    }
  ]
}
```

## Segurança

### Verificação de Checksum

Todos os downloads são verificados contra checksums SHA256 no manifesto:

```json
{
  "checksum": "sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
}
```

### Verificação de Assinatura

Para segurança adicional, habilite a verificação de assinatura:

<Steps>
1. **Gerar um par de chaves**:
   ```bash
   # Gerar par de chaves Ed25519
   openssl genpkey -algorithm Ed25519 -out private.pem
   openssl pkey -in private.pem -pubout -out public.pem
   ```

2. **Assinar seu manifesto de atualização**:
   ```bash
   openssl pkeyutl -sign -inkey private.pem -in update.json -out update.json.sig
   ```

3. **Configurar o atualizador**:
   ```go
   updater, _ := application.CreateUpdaterService(
       "1.0.0",
       application.WithUpdateURL("https://updates.example.com/myapp/"),
       application.WithRequireSignature(true),
       application.WithPublicKey("MCowBQYDK2VwAyEA..."), // Chave pública codificada em Base64
   )
   ```
</Steps>

## Melhores Práticas

### Faça

- Teste as atualizações exaustivamente antes de implantar
- Mantenha versões anteriores disponíveis para rollback
- Exiba notas de lançamento para os usuários
- Permita que os usuários pulem atualizações não obrigatórias
- Use HTTPS para todos os downloads
- Verifique checksums antes de aplicar atualizações
- Lidar com erros de forma graciosa

### Não Faça

- Forçar reinicializações imediatas sem aviso
- Pular a verificação de checksum
- Interromper os usuários durante trabalhos importantes
- Excluir a versão anterior imediatamente
- Ignorar falhas de atualização

## Solução de Problemas

### Atualização Não Encontrada

- Verifique se a URL do manifesto está correta
- Verifique se a chave da plataforma corresponde (ex: `macos-arm64`)
- Certifique-se de que a versão no manifesto é mais recente

### Falha no Download

- Verifique a conectividade de rede
- Verifique se a URL de download está acessível
- Verifique a configuração de CORS se for cross-origin

### Falha no Patch

- O atualizador retorna automaticamente ao download completo
- Certifique-se de que `bspatch` está disponível no sistema
- Verifique se o checksum do patch está correto

### Aplicativo Não Reinicia

- No macOS, certifique-se de que o app está devidamente code-signed
- No Windows, verifique se há bloqueios de arquivo
- No Linux, verifique as permissões de arquivo
