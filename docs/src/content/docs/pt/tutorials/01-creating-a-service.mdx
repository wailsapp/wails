Um **serviço** no Wails é uma struct Go que contém a lógica de negócios que você deseja disponibilizar para o seu frontend. Os serviços mantêm seu código organizado agrupando funcionalidades relacionadas.

Pense em um serviço como uma coleção de métodos que seu código JavaScript pode chamar. Cada método público no serviço se torna chamável a partir do frontend após a geração dos bindings.

Neste tutorial, criaremos um serviço de gerador de QR code para demonstrar esses conceitos. Ao final, você entenderá como criar serviços, gerenciar dependências e conectar seu código Go ao frontend.

<br/>

<Steps>

1. ## Criar o arquivo do serviço QR

   Crie um novo arquivo chamado `qrservice.go` no diretório do seu aplicativo:

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **O que está acontecendo aqui:**

   - `QRService` é uma struct vazia que conterá nossos métodos de geração de QR code
   - `NewQRService()` é uma função construtora que cria uma nova instância do nosso serviço
   - `Generate()` é um método que recebe texto e um tamanho, retornando o QR code como um array de bytes PNG
   - O método retorna `([]byte, error)` seguindo a convenção do Go de retornar erros como o último valor
   - Usamos o pacote `github.com/skip2/go-qrcode` para lidar com a geração real do QR code

    <br/>

2. ## Registrar o Serviço

   Criar um serviço não é suficiente — precisamos **registrar** ele com o aplicativo Wails para que ele saiba que o serviço existe e possa gerar bindings para ele.

   O registro acontece em `main.go` quando você cria seu aplicativo. Você passa suas instâncias de serviço para a opção `Services`:

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **O que está acontecendo aqui:**

   - `application.NewService()` envolve seu serviço para que o Wails possa gerenciá-lo
   - Chamamos `NewQRService()` para criar uma instância do nosso serviço
   - O serviço é adicionado ao slice `Services` nas opções do aplicativo
   - O Wails agora irá analisar este serviço em busca de métodos públicos para disponibilizar ao frontend

    <br/>

3. ## Instalar Dependências

   Referenciamos o pacote `github.com/skip2/go-qrcode` em nosso código, mas ainda não o baixamos de fato. O Go precisa conhecer essa dependência e baixá-la para o seu projeto.

   Execute este comando no seu terminal a partir do diretório do seu projeto:

   ```bash
   go mod tidy
   ```

   **O que está acontecendo aqui:**

   - `go mod tidy` analisa seus arquivos Go em busca de instruções de importação
   - Ele baixa quaisquer pacotes ausentes (como `go-qrcode`) e os adiciona ao `go.mod`
   - Ele também remove quaisquer dependências que não estão mais sendo usadas
   - Isso garante que seu projeto tenha todo o código necessário para compilar com sucesso

   Você deve ver uma saída indicando que o pacote de QR code foi baixado e adicionado ao seu projeto.

    <br/>

4. ## Gerar os Bindings

   Para chamar esses métodos a partir do seu frontend, precisamos gerar os bindings.
   Você pode fazer isso executando `wails generate bindings` no diretório raiz do seu projeto.

   :::note
   Na primeira vez que você executar isso em um projeto, o gerador de bindings faz uma análise minuciosa do seu código e das dependências. Isso às vezes pode demorar um pouco mais do que o esperado, no entanto, execuções subsequentes serão muito mais rápidas.
   :::

   Depois de executar isso, você deve ver algo semelhante ao seguinte no seu terminal:

   ```bash
    % wails3 generate bindings
    INFO  Processed: 337 Packages, 1 Service, 1 Method, 0 Enums, 0 Models in 740.196125ms.
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

   Você deve notar que no diretório frontend, há um novo diretório chamado `bindings`:

   ```bash
   frontend/
   └── bindings
       └── changeme
           ├── index.js
           └── qrservice.js
   ```

    :::tip[Pro Tip]
        Quando você compila seu aplicativo usando `wails3 build`, ele gerará automaticamente os bindings para você e os manterá atualizados.
    :::
    <br/>

5. ## Entendendo os Bindings

   Vamos olhar os bindings gerados em `bindings/changeme/qrservice.js`:

   ```js title="bindings/changeme/qrservice.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT

    /**
     * QRService handles QR code generation
     * @module
     */

    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
    // @ts-ignore: Unused imports
    import {Call as $Call, Create as $Create} from "@wailsio/runtime";

    /**
     * Generate creates a QR code from the given text
     * @param {string} text
     * @param {number} size
     * @returns {Promise<string> & { cancel(): void }}
     */
    export function Generate(text, size) {
        let $resultPromise = /** @type {any} */($Call.ByID(3576998831, text, size));
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
            return $Create.ByteSlice($result);
        }));
        $typingPromise.cancel = $resultPromise.cancel.bind($resultPromise);
        return $typingPromise;
    }
   ```

    Podemos ver que os bindings são gerados para o método `Generate`. Os nomes dos parâmetros foram preservados,
    assim como os comentários. O JSDoc também foi gerado para o método para fornecer informações de tipo à sua IDE.

    :::note
    Não é necessário entender completamente os bindings gerados, mas é importante entender como eles funcionam.
    :::

    Os bindings fornecem:
    - Funções equivalentes aos seus métodos Go
    - Conversão automática entre tipos Go e JavaScript
    - Operações assíncronas baseadas em Promise
    - Informações de tipo como comentários JSDoc

    :::tip[Typescript]
    O gerador de bindings também suporta a geração de bindings em Typescript. Você pode fazer isso executando `wails3 generate bindings -ts`.
    :::

    O serviço gerado é reexportado por um arquivo `index.js`:

   ```js title="bindings/changeme/index.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDITimport * as QRService from "./qrservice.js";
    export {
        QRService
    };
   ```

    Você pode então acessá-lo através do caminho de importação simplificado
    `./bindings/changeme` consistindo apenas do caminho do seu pacote Go,
    sem especificar nenhum nome de arquivo.

    :::note
    Caminhos de importação simplificados estão disponíveis apenas ao usar bundlers de frontend.
    Se você preferir um frontend vanilla que não utilize um bundler,
    você terá que importar manualmente `index.js` ou `qrservice.js`.
    :::

6. ## Use Bindings no Frontend

    Agora podemos chamar nosso serviço Go a partir do JavaScript! Os bindings gerados facilitam isso e garantem segurança de tipos.

    Atualize `frontend/src/main.js` para usar os novos bindings:

   ```js title="frontend/src/main.js"
    import { QRService } from './bindings/changeme';

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        try {
            // Generate QR code as base64
            const qrCodeBase64 = await QRService.Generate(text, 256);

            // Display the QR code
            const qrDiv = document.getElementById('qrcode');
            qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

        } catch (err) {
            console.error('Failed to generate QR code:', err);
            alert('Failed to generate QR code: ' + err);
        }
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        button.addEventListener('click', generateQR);
    }
   ```

   **O que está acontecendo aqui:**

   - Importamos `QRService` dos bindings gerados
   - `QRService.Generate()` chama nosso método Go - ele retorna uma Promise, então usamos `await`
   - O método Go retorna `[]byte`, que o Wails converte automaticamente para uma string base64 para o JavaScript
   - Criamos uma URL de dados com a string base64 para exibir a imagem PNG
   - O bloco `try/catch` lida com quaisquer erros do lado do Go (como entrada inválida)
   - Se nosso código Go retornar um erro, a Promise é rejeitada e a capturamos aqui

    Agora atualize `index.html` para usar os novos bindings na função `initializeQRGenerator`:

    ```html title="frontend/src/index.html"

7. ## Abordagem Alternativa: HTTP Handler

    Até agora, cobrimos as seguintes áreas:
    - Criando um novo Serviço
    - Gerando Bindings
    - Usando os Bindings no nosso código Frontend

    **Por que usar um HTTP handler?**

    Os bindings de métodos funcionam muito bem para operações de dados, mas há uma abordagem alternativa para servir arquivos, imagens ou outros tipos de mídia. Em vez de converter tudo para base64 e enviá-lo através dos bindings, você pode fazer com que seu serviço atue como um mini servidor web.

    Isso é útil quando:
    - Você está servindo imagens, vídeos ou arquivos grandes
    - Você deseja usar as tags HTML padrão `<img>` ou `<video>` com atributos `src`
    - Você precisa de acesso direto por URL aos recursos

    Se o seu serviço implementar o método padrão do Go `ServeHTTP(w http.ResponseWriter, r *http.Request)`, o Wails pode torná-lo acessível como um endpoint HTTP. Vamos estender nosso serviço de código QR para suportar isso:

    ```go title="qrservice.go" ins={4-5,37-65}
    package main

    import (
        "net/http"
        "strconv"

        "github.com/skip2/go-qrcode"
    )

    // QRService handles QR code generation
    type QRService struct {
        // We can add state here if needed
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **O que está acontecendo aqui:**

    - `ServeHTTP` é a interface padrão do Go para lidar com requisições HTTP
    - Analisamos os parâmetros de consulta da URL (`?text=hello&size=256`)
    - Chamamos nosso método `Generate()` existente para criar o código QR
    - Definimos o tipo de conteúdo como `image/png` para que os navegadores saibam que é uma imagem
    - Escrevemos os bytes PNG brutos diretamente na resposta - sem necessidade de base64!

    Agora atualize `main.go` para especificar a rota na qual o serviço de código QR deve estar acessível:

   ```go title="main.go" ins={8-10}
    func main() {app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/qrservice",
                }),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

    **O que está acontecendo aqui:**

    - Adicionamos `application.ServiceOptions` para configurar como o serviço é exposto
    - `Route: "/qrservice"` torna o manipulador HTTP acessível em `/qrservice`
    - Agora, qualquer solicitação para `/qrservice?text=hello` chamará nosso método `ServeHTTP`
    - Sem definir `Route`, a funcionalidade do manipulador HTTP é desativada

    :::note
    Se você não definir a opção `Route` explicitamente,
    o manipulador HTTP não estará acessível pelo frontend.
    :::

    Por fim, atualize `main.js` para usar um `src` de imagem simples em vez de codificação base64:

    ```js title="frontend/src/main.js"
    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        const img = document.getElementById('qrcode');
        // Make the image source the path to the QR code service, passing the text
        img.src = `/qrservice?text=${encodeURIComponent(text)}`
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        if (button) {
            button.addEventListener('click', generateQR);
        } else {
            console.error('Generate button not found');
        }
    }
    ```

    **O que está acontecendo aqui:**

    - Removemos a importação e a chamada `await QRService.Generate()`
    - Em vez disso, simplesmente definimos `img.src` para apontar para nosso endpoint HTTP
    - `encodeURIComponent()` escapa com segurança os caracteres especiais na URL
    - O navegador automaticamente faz uma solicitação HTTP GET quando definimos o `src`
    - Isso é mais simples e mais eficiente para imagens — nenhuma conversão base64 necessária!

    Executar o aplicativo novamente deve resultar no mesmo código QR:

    <Image src={qr1} alt="QR Code"/>

8. ## Suportando Configurações Dinâmicas

    **O problema com rotas codificadas:**

    No exemplo acima, usamos uma rota codificada `/qrservice` em nosso código JavaScript.
    Isso cria um acoplamento rígido entre sua configuração Go e seu código frontend.

    Se você editar `main.go` e alterar a opção `Route` sem atualizar `main.js`,
    o aplicativo quebrará:

    ```go title="main.go" ins={3}
            // ...
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/services/qr",
                }),
            // ...
    ```

    Rotas codificadas funcionam para aplicativos simples, mas tornam seu código frágil e mais difícil de manter.

    **A solução: Configuração dinâmica**

    Vinculações de método e manipuladores HTTP podem funcionar juntos! Podemos usar vinculações para informar ao frontend qual rota usar, tornando a configuração dinâmica e eliminando o caminho codificado.

    Veja como funciona:
    1. O método de ciclo de vida `ServiceStartup` é executado quando seu aplicativo inicia
    2. Salvamos a rota configurada das opções
    3. Adicionamos um método `URL()` que o frontend pode chamar para obter a rota correta
    4. Agora o frontend pergunta ao serviço Go qual é sua rota em vez de adivinhar

    Primeiro, implemente a interface `ServiceStartup` e adicione um novo método `URL`:

    ```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
    package main

    import (
        "context"
        "net/http"
        "net/url"
        "strconv"

        "github.com/skip2/go-qrcode"
        "github.com/wailsapp/wails/v3/pkg/application"
    )

    // QRService handles QR code generation
    type QRService struct {
        route string
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // ServiceStartup runs at application startup.
    func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
        s.route = options.Route
        return nil
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    // URL returns an URL that may be used to fetch
    // a QR code with the given text and size.
    // It returns an error if the HTTP handler is not available.
    func (s *QRService) URL(text string, size int) (string, error) {
        if s.route == "" {
            return "", errors.New("http handler unavailable")
        }

        return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **O que está acontecendo aqui:**

    - Adicionamos um campo `route` para armazenar a rota configurada de `ServiceStartup`
    - `ServiceStartup(ctx, options)` é chamado quando o aplicativo inicia — salvamos a rota aqui
    - O método `URL()` constrói a URL completa com parâmetros de consulta
    - Se nenhuma rota estiver configurada (rota vazia), retornamos um erro
    - `url.QueryEscape()` codifica com segurança o texto para uso em uma URL
    - Este método estará disponível para o frontend por meio de vinculações

    Agora atualize `main.js` para usar o método `URL` em vez de um caminho codificado:

    ```js title="frontend/src/main.js" ins={1,11-12}
    import { QRService } from "./bindings/changeme";

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        const img = document.getElementById('qrcode');
        // Invoke the URL method to obtain an URL for the given text.
        img.src = await QRService.URL(text, 256);
    }```go
export function initializeQRGenerator() {
    const button = document.getElementById('generateButton');
    if (button) {
        button.addEventListener('click', generateQR);
    } else {
        console.error('Generate button not found');
    }
}
```

**O que está acontecendo aqui:**

- Importamos `QRService` para usar os bindings novamente
- Em vez de codificar `/qrservice`, chamamos `await QRService.URL(text, 256)`
- O serviço Go constrói a URL com a rota e os parâmetros corretos
- Agora, se você alterar a rota em `main.go`, o frontend automaticamente usará a nova rota
- Não é mais necessária a sincronização manual entre a configuração do Go e o código do frontend!

Deve funcionar exatamente como no exemplo anterior,
mas alterar a rota do serviço em `main.go`
não quebrará mais o frontend.

:::note
Se um método Go retornar um erro não nulo,
a promessa no lado do JS será rejeitada
e as instruções `await` lançarão uma exceção.
:::
