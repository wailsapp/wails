Wails에서 **서비스**는 프론트엔드에서 사용할 수 있는 비즈니스 로직을 포함하는 Go 구조체입니다. 서비스는 관련 기능을 함께 그룹화하여 코드를 조직화합니다.

서비스를 JavaScript 코드에서 호출할 수 있는 메서드 모음으로 생각하세요. 서비스의 각 공개 메서드는 바인딩을 생성한 후 프론트엔드에서 호출할 수 있습니다.

이 튜토리얼에서는 이러한 개념을 시연하기 위해 QR 코드 생성 서비스를 생성합니다. 이 튜토리얼을 마치면 서비스를 생성하고, 종속성을 관리하며, Go 코드를 프론트엔드에 연결하는 방법을 이해하게 됩니다.

<br/>

<Steps>

1. ## QR 서비스 파일 생성

   애플리케이션 디렉토리에 `qrservice.go`라는 새 파일을 만듭니다:

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **여기서 일어나는 일:**

   - `QRService`는 QR 코드 생성 메서드를 보유할 빈 구조체입니다
   - `NewQRService()`는 서비스의 새 인스턴스를 생성하는 생성자 함수입니다
   - `Generate()`은 텍스트와 크기를 받아 QR 코드를 PNG 바이트 배열로 반환하는 메서드입니다
   - 이 메서드는 Go의 관례에 따라 오류를 마지막 값으로 반환하므로 `([]byte, error)`를 반환합니다
   - 실제 QR 코드 생성을 처리하기 위해 `github.com/skip2/go-qrcode` 패키지를 사용합니다

    <br/>

2. ## 서비스 등록

   서비스를 생성하는 것만으로는 충분하지 않습니다. Wails 애플리케이션에 서비스를 **등록**하여 서비스의 존재를 인지하고 해당 서비스에 대한 바인딩을 생성할 수 있도록 해야 합니다.

   등록은 애플리케이션을 생성할 때 `main.go`에서 발생합니다. 서비스 인스턴스를 `Services` 옵션에 전달합니다:

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **여기서 일어나는 일:**

   - `application.NewService()`는 Wails가 서비스를 관리할 수 있도록 서비스를 감쌉니다
   - 서비스의 인스턴스를 생성하기 위해 `NewQRService()`를 호출합니다
   - 서비스는 애플리케이션 옵션의 `Services` 슬라이스에 추가됩니다
   - Wails는 이제 프론트엔드에서 사용할 수 있는 공개 메서드를 찾기 위해 이 서비스를 스캔합니다

    <br/>

3. ## 종속성 설치

   코드에서 `github.com/skip2/go-qrcode` 패키지를 참조했지만 아직 실제로 다운로드하지 않았습니다. Go는 이 종속성에 대해 알고 프로젝트에 다운로드해야 합니다.

   프로젝트 디렉토리에서 터미널에서 다음 명령을 실행합니다:

   ```bash
   go mod tidy
   ```

   **여기서 일어나는 일:**

   - `go mod tidy`는 Go 파일의 import 문을 스캔합니다
   - 누락된 패키지(예: `go-qrcode`)를 다운로드하고 `go.mod`에 추가합니다
   - 더 이상 사용되지 않는 종속성도 제거합니다
   - 이를 통해 프로젝트가 성공적으로 컴파일하는 데 필요한 모든 코드를 갖추게 됩니다

   QR 코드 패키지가 다운로드되어 프로젝트에 추가되었음을 나타내는 출력이 표시되어야 합니다.

    <br/>

4. ## 바인딩 생성

   프론트엔드에서 이 메서드를 호출하려면 바인딩을 생성해야 합니다.
   프로젝트 루트 디렉토리에서 `wails generate bindings`를 실행하여 이를 수행할 수 있습니다.

   :::note
   프로젝트에서 이를 처음 실행할 때 바인딩 생성기는 코드와 종속성을 철저히 분석합니다. 이로 인해 예상보다 약간 더 오래 걸릴 수 있지만, 이후 실행은 훨씬 빠릅니다.
   :::

   이를 실행한 후 터미널에 다음과 유사한 내용이 표시되어야 합니다:

   ```bash
    % wails3 generate bindings
    INFO  Processed: 337 Packages, 1 Service, 1 Method, 0 Enums, 0 Models in 740.196125ms.
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

   프론트엔드 디렉토리에 `bindings`라는 새 디렉토리가 있음을 알 수 있습니다:

   ```bash
   frontend/
   └── bindings
       └── changeme
           ├── index.js
           └── qrservice.js
   ```

    :::tip[Pro Tip]
        `wails3 build`를 사용하여 애플리케이션을 빌드하면 바인딩이 자동으로 생성되고 최신 상태로 유지됩니다.
    :::
    <br/>

5. ## 바인딩 이해하기

   `bindings/changeme/qrservice.js`의 생성된 바인딩을 살펴보겠습니다:

   ```js title="bindings/changeme/qrservice.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT

    /**
     * QRService handles QR code generation
     * @module
     */

    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
    // @ts-ignore: Unused imports
    import {Call as $Call, Create as $Create} from "@wailsio/runtime";

    /**
     * Generate creates a QR code from the given text
     * @param {string} text
     * @param {number} size
     * @returns {Promise<string> & { cancel(): void }}
     */
    export function Generate(text, size) {
        let $resultPromise = /** @type {any} */($Call.ByID(3576998831, text, size));
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
            return $Create.ByteSlice($result);
        }));
        $typingPromise.cancel = $resultPromise.cancel.bind($resultPromise);
        return $typingPromise;
    }
   ```

   바인딩이 `Generate` 메서드에 대해 생성되었음을 알 수 있습니다. 매개변수 이름과 주석이 보존되었으며, IDE에 타입 정보를 제공하기 위해 메서드에 대한 JSDoc도 생성되었습니다.

   :::note
   생성된 바인딩을 완전히 이해할 필요는 없지만, 작동 방식을 이해하는 것이 중요합니다.
   :::

   바인딩은 다음을 제공합니다:
   - Go 메서드와 동일한 함수
   - Go와 JavaScript 간 자동 변환
   - Promise 기반 비동기 작업
   - JSDoc 주석으로 제공되는 타입 정보

   :::tip[Typescript]
   바인딩 생성기는 Typescript 바인딩 생성도 지원합니다. `wails3 generate bindings -ts`를 실행하여 이를 수행할 수 있습니다.
   :::

   생성된 서비스는 `index.js` 파일에서 다시 내보냅니다:

   ```js title="bindings/changeme/index.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT```js
    import * as QRService from "./qrservice.js";
    export {
        QRService
    };
   ```

    그런 다음 파일 이름 없이 Go 패키지 경로만으로 구성된 단순화된 import 경로 `./bindings/changeme`를 통해 이 서비스에 접근할 수 있습니다.

    :::note
    단순화된 import 경로는 프론트엔드 번들러를 사용할 때만 사용할 수 있습니다.
    번들러를 사용하지 않는 바닐라 프론트엔드를 선호한다면,
    `index.js` 또는 `qrservice.js`를 수동으로 import해야 합니다.
    :::

6. ## 프론트엔드에서 바인딩 사용

    이제 JavaScript에서 Go 서비스를 호출할 수 있습니다! 생성된 바인딩은 이를 쉽게 그리고 타입 안전하게 만들어줍니다.

    새로운 바인딩을 사용하도록 `frontend/src/main.js`를 업데이트하세요:

   ```js title="frontend/src/main.js"
    import { QRService } from './bindings/changeme';

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        try {
            // Generate QR code as base64
            const qrCodeBase64 = await QRService.Generate(text, 256);

            // Display the QR code
            const qrDiv = document.getElementById('qrcode');
            qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

        } catch (err) {
            console.error('Failed to generate QR code:', err);
            alert('Failed to generate QR code: ' + err);
        }
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        button.addEventListener('click', generateQR);
    }
   ```

   **여기서 일어나는 일:**

   - 생성된 바인딩에서 `QRService`를 import합니다.
   - `QRService.Generate()`은 Go 메서드를 호출합니다. Promise를 반환하므로 `await`를 사용합니다.
   - Go 메서드는 `[]byte`를 반환하며, Wails는 이를 JavaScript를 위해 자동으로 base64 문자열로 변환합니다.
   - PNG 이미지를 표시하기 위해 base64 문자열로 데이터 URL을 생성합니다.
   - `try/catch` 블록은 Go 측(잘못된 입력 등)에서 발생하는 모든 오류를 처리합니다.
   - Go 코드에서 오류를 반환하면 Promise가 reject되고 여기서 이를 catch합니다.

   이제 `initializeQRGenerator` 함수에서 새로운 바인딩을 사용하도록 `index.html`을 업데이트하세요:

7. ## 대체 접근 방식: HTTP 핸들러

    지금까지 다음 영역들을 다뤘습니다:
    - 새 서비스 생성
    - 바인딩 생성
    - 프론트엔드 코드에서 바인딩 사용

    **왜 HTTP 핸들러를 사용하나요?**

    메서드 바인딩은 데이터 작업에 매우 적합하지만, 파일, 이미지 또는 기타 미디어를 제공하기 위한 대체 접근 방식이 있습니다. 모든 것을 base64로 변환하여 바인딩을 통해 보내는 대신, 서비스를 미니 웹 서버처럼 작동하게 할 수 있습니다.

    이는 다음과 같은 경우에 유용합니다:
    - 이미지, 비디오 또는 대용량 파일을 제공할 때
    - `src` 속성과 함께 표준 HTML `<img>` 또는 `<video>` 태그를 사용하고 싶을 때
    - 리소스에 대한 직접 URL 접근이 필요할 때

    서비스에서 Go의 표준 `ServeHTTP(w http.ResponseWriter, r *http.Request)` 메서드를 구현하면 Wails가 이를 HTTP 엔드포인트로 접근 가능하게 만들 수 있습니다. QR 코드 서비스에 이를 지원하도록 확장해 보겠습니다:

    ```go title="qrservice.go" ins={4-5,37-65}
    package main

    import (
        "net/http"
        "strconv"

        "github.com/skip2/go-qrcode"
    )

    // QRService handles QR code generation
    type QRService struct {
        // We can add state here if needed
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **여기서 일어나는 일:**

    - `ServeHTTP`는 Go에서 HTTP 요청을 처리하기 위한 표준 인터페이스입니다.
    - URL에서 쿼리 매개변수(`?text=hello&size=256`)를 파싱합니다.
    - 기존 `Generate()` 메서드를 호출하여 QR 코드를 생성합니다.
    - 브라우저가 이것이 이미지임을 알 수 있도록 콘텐츠 타입을 `image/png`로 설정합니다.
    - base64 없이 raw PNG 바이트를 응답에 직접 작성합니다!

    이제 QR 코드 서비스에 접근 가능한 라우트를 지정하도록 `main.go`를 업데이트하세요:

   ```go title="main.go" ins={8-10}
    func main() {```go title="main.go"

**여기서 일어나는 일:**

- `application.ServiceOptions`를 추가하여 서비스가 노출되는 방식을 구성합니다.
- `Route: "/qrservice"`는 HTTP 핸들러를 `/qrservice`에서 접근 가능하게 만듭니다.
- 이제 `/qrservice?text=hello`로 오는 모든 요청이 우리의 `ServeHTTP` 메서드를 호출합니다.
- `Route`를 설정하지 않으면 HTTP 핸들러 기능이 비활성화됩니다.

:::note
`Route` 옵션을 명시적으로 설정하지 않으면,
프론트엔드에서 HTTP 핸들러에 접근할 수 없습니다.
:::

마지막으로 `main.js`를 업데이트하여 base64 인코딩 대신 간단한 이미지 `src`를 사용하도록 합니다:

```js title="frontend/src/main.js"
async function generateQR() {
    const text = document.getElementById('text').value;
    if (!text) {
        alert('Please enter some text');
        return;
    }

    const img = document.getElementById('qrcode');
    // Make the image source the path to the QR code service, passing the text
    img.src = `/qrservice?text=${encodeURIComponent(text)}`
}

export function initializeQRGenerator() {
    const button = document.getElementById('generateButton');
    if (button) {
        button.addEventListener('click', generateQR);
    } else {
        console.error('Generate button not found');
    }
}
```

**여기서 일어나는 일:**

- import와 `await QRService.Generate()` 호출을 제거했습니다.
- 대신 `img.src`를 HTTP 엔드포인트를 가리키도록 설정했습니다.
- `encodeURIComponent()`는 URL의 특수 문자를 안전하게 이스케이프합니다.
- `src`를 설정할 때 브라우저가 자동으로 HTTP GET 요청을 수행합니다.
- 이미지에는 이 방식이 더 간단하고 효율적입니다. base64 변환이 필요하지 않습니다!

8. ## 동적 구성 지원

**경로 하드코딩의 문제점:**

위의 예제에서 JavaScript 코드에 하드코딩된 경로 `/qrservice`를 사용했습니다.
이는 Go 구성과 프론트엔드 코드 사이의 긴밀한 결합을 만듭니다.

`main.go`를 편집하여 `Route` 옵션을 변경하고 `main.js`를 업데이트하지 않으면,
애플리케이션이 중단됩니다:

```go title="main.go" ins={3}
        // ...
            application.NewService(NewQRService(), application.ServiceOptions{
                Route: "/services/qr",
            }),
        // ...
```

하드코딩된 경로는 간단한 애플리케이션에는 작동하지만, 코드를 취약하게 만들고 유지보수를 어렵게 만듭니다.

**해결책: 동적 구성**

메서드 바인딩과 HTTP 핸들러는 함께 작동할 수 있습니다! 바인딩을 사용하여 프론트엔드에 사용할 경로를 알려줌으로써 구성을 동적으로 만들고 하드코딩된 경로를 제거할 수 있습니다.

작동 방식은 다음과 같습니다:
1. `ServiceStartup` 수명 주기 메서드는 앱이 시작될 때 실행됩니다.
2. 옵션에서 구성된 경로를 저장합니다.
3. 프론트엔드가 호출하여 올바른 경로를 가져올 수 있는 `URL()` 메서드를 추가합니다.
4. 이제 프론트엔드는 추측하는 대신 Go 서비스에 경로를 요청합니다.

먼저 `ServiceStartup` 인터페이스를 구현하고 새로운 `URL` 메서드를 추가합니다:

```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
package main

import (
    "context"
    "net/http"
    "net/url"
    "strconv"

    "github.com/skip2/go-qrcode"
    "github.com/wailsapp/wails/v3/pkg/application"
)

// QRService handles QR code generation
type QRService struct {
    route string
}

// NewQRService creates a new QR service
func NewQRService() *QRService {
    return &QRService{}
}

// ServiceStartup runs at application startup.
func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
    s.route = options.Route
    return nil
}

// Generate creates a QR code from the given text
func (s *QRService) Generate(text string, size int) ([]byte, error) {
    // Generate the QR code
    qr, err := qrcode.New(text, qrcode.Medium)
    if err != nil {
        return nil, err
    }

    // Convert to PNG
    png, err := qr.PNG(size)
    if err != nil {
        return nil, err
    }

    return png, nil
}

// URL returns an URL that may be used to fetch
// a QR code with the given text and size.
// It returns an error if the HTTP handler is not available.
func (s *QRService) URL(text string, size int) (string, error) {
    if s.route == "" {
        return "", errors.New("http handler unavailable")
    }

    return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
}

func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    // Extract the text parameter from the request
    text := r.URL.Query().Get("text")
    if text == "" {
        http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
        return
    }
    // Extract Size parameter from the request
    sizeText := r.URL.Query().Get("size")
    if sizeText == "" {
        sizeText = "256"
    }
    size, err := strconv.Atoi(sizeText)
    if err != nil {
        http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
        return
    }

    // Generate the QR code
    qrCodeData, err := s.Generate(text, size)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // Write the QR code data to the response
    w.Header().Set("Content-Type", "image/png")
    w.Write(qrCodeData)
}
```

**여기서 일어나는 일:**

- `ServiceStartup`에서 구성된 경로를 저장하기 위해 `route` 필드를 추가했습니다.
- `ServiceStartup(ctx, options)`는 앱이 시작될 때 호출되며, 여기서 경로를 저장합니다.
- `URL()` 메서드는 쿼리 매개변수가 포함된 전체 URL을 빌드합니다.
- 구성된 경로가 없는 경우(경로가 비어 있는 경우) 오류를 반환합니다.
- `url.QueryEscape()`는 URL에서 사용하기 위해 텍스트를 안전하게 인코딩합니다.
- 이 메서드는 바인딩을 통해 프론트엔드에서 사용할 수 있게 됩니다.

이제 하드코딩된 경로 대신 `URL` 메서드를 사용하도록 `main.js`를 업데이트합니다:

```js title="frontend/src/main.js" ins={1,11-12}
import { QRService } from "./bindings/changeme";

async function generateQR() {
    const text = document.getElementById('text').value;
    if (!text) {
        alert('Please enter some text');
        return;
    }

    const img = document.getElementById('qrcode');
    // Invoke the URL method to obtain an URL for the given text.
    img.src = await QRService.URL(text, 256);
}```javascript
export function initializeQRGenerator() {
    const button = document.getElementById('generateButton');
    if (button) {
        button.addEventListener('click', generateQR);
    } else {
        console.error('Generate button not found');
    }
}
```

**여기서 일어나는 일:**

- 바인딩을 다시 사용하기 위해 `QRService`를 가져옵니다.
- `/qrservice`를 하드코딩하는 대신 `await QRService.URL(text, 256)`을 호출합니다.
- Go 서비스는 올바른 라우트와 매개변수로 URL을 빌드합니다.
- 이제 `main.go`에서 라우트를 변경하면 프론트엔드가 자동으로 새 라우트를 사용합니다.
- Go 구성과 프론트엔드 코드 간의 수동 동기화가 더 이상 필요 없습니다!

이전 예제와 동일하게 작동해야 하지만, `main.go`에서 서비스 라우트를 변경해도 프론트엔드가 더 이상 깨지지 않습니다.

:::note
Go 메서드가 null이 아닌 오류를 반환하는 경우, JS 쪽의 프라미스는 거부되고 await 문은 예외를 throw합니다.
:::
