# 자동 업데이트

Wails v3는 자동 업데이트 확인, 다운로드 및 설치를 지원하는 내장 업데이터 시스템을 제공합니다. 이 업데이터는 최소한의 다운로드 크기를 위해 바이너리 델타 업데이트(패치) 지원을 포함합니다.

## 빠른 시작

애플리케이션에 업데이터 서비스를 추가합니다:

```go title="main.go"
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "time"
)

func main() {
    // 업데이터 서비스 생성
    updater, err := application.CreateUpdaterService(
        "1.0.0", // 현재 버전
        application.WithUpdateURL("https://updates.example.com/myapp/"),
        application.WithCheckInterval(24 * time.Hour),
    )
    if err != nil {
        panic(err)
    }

    app := application.New(application.Options{
        Name: "MyApp",
        Services: []application.Service{
            application.NewService(updater),
        },
    })

    // ... 나머지 앱 코드
    app.Run()
}
```

그런 다음 프론트엔드에서 사용합니다:

```typescript title="App.tsx"
import { updater } from './bindings/myapp';

async function checkForUpdates() {
  const update = await updater.CheckForUpdate();

  if (update) {
    console.log(`New version available: ${update.version}`);
    console.log(`Release notes: ${update.releaseNotes}`);

    // 다운로드 및 설치
    await updater.DownloadAndApply();
  }
}
```

## 구성 옵션

업데이터는 다양한 구성 옵션을 지원합니다:

```go
updater, err := application.CreateUpdaterService(
    "1.0.0",
    // 필수: 업데이트 매니페스트가 호스팅되는 URL
    application.WithUpdateURL("https://updates.example.com/myapp/"),

    // 선택: 24시간마다 자동으로 업데이트 확인
    application.WithCheckInterval(24 * time.Hour),

    // 선택: 프리릴리스 버전 허용
    application.WithAllowPrerelease(true),

    // 선택: 업데이트 채널 (stable, beta, canary)
    application.WithChannel("stable"),

    // 선택: 서명된 업데이트 요구
    application.WithRequireSignature(true),
    application.WithPublicKey("your-ed25519-public-key"),
)
```

| 플랫폼 | 키 |
|----------|-----|

| Windows (64-bit) | `windows-amd64` |
| Linux (64-bit) | `linux-amd64` |
| Linux (ARM64) | `linux-arm64` |

## 프론트엔드 API

업데이터는 프론트엔드에 자동으로 바인딩되는 메서드를 노출합니다:

### TypeScript 타입

```typescript
interface UpdateInfo {
  version: string;
  releaseDate: Date;
  releaseNotes: string;
  size: number;
  patchSize?: number;
  mandatory: boolean;
  hasPatch: boolean;
}

interface Updater {
  // 현재 애플리케이션 버전 가져오기
  GetCurrentVersion(): string;

  // 업데이트 가능 여부 확인
  CheckForUpdate(): Promise<UpdateInfo | null>;

  // 업데이트 다운로드 (진행률 이벤트 발생)
  DownloadUpdate(): Promise<void>;

  // 다운로드된 업데이트 적용 (앱 재시작)
  ApplyUpdate(): Promise<void>;

  // 한 번의 호출로 다운로드 및 적용
  DownloadAndApply(): Promise<void>;

  // 현재 상태 가져오기: idle, checking, available, downloading, ready, installing, error
  GetState(): string;

  // 사용 가능한 업데이트 정보 가져오기
  GetUpdateInfo(): UpdateInfo | null;

  // 마지막 오류 메시지 가져오기
  GetLastError(): string;

  // 업데이터 상태 재설정
  Reset(): void;
}
```

### 진행률 이벤트

다운로드 진행률 이벤트를 수신합니다:

```typescript
import { Events } from '@wailsio/runtime';

Events.On('updater:progress', (data) => {
  console.log(`Downloaded: ${data.downloaded} / ${data.total}`);
  console.log(`Progress: ${data.percentage.toFixed(1)}%`);
  console.log(`Speed: ${(data.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s`);
});
```

## 전체 예제

여기에 React 컴포넌트가 포함된 전체 예제가 있습니다:

```tsx title="UpdateChecker.tsx"
import { useState, useEffect } from 'react';
import { updater } from './bindings/myapp';
import { Events } from '@wailsio/runtime';

interface Progress {
  downloaded: number;
  total: number;
  percentage: number;
  bytesPerSecond: number;
}

export function UpdateChecker() {
  const [checking, setChecking] = useState(false);
  const [updateInfo, setUpdateInfo] = useState<any>(null);
  const [downloading, setDownloading] = useState(false);
  const [progress, setProgress] = useState<Progress | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    // 진행률 이벤트 수신
    const cleanup = Events.On('updater:progress', (data: Progress) => {
      setProgress(data);
    });

    // 마운트 시 업데이트 확인
    checkForUpdates();

    return () => cleanup();
  }, []);

  async function checkForUpdates() {
    setChecking(true);
    setError(null);

    try {
      const info = await updater.CheckForUpdate();
      setUpdateInfo(info);
    } catch (err) {
      setError(err.message);
    } finally {
      setChecking(false);
    }
  }

  async function downloadAndInstall() {
    setDownloading(true);
    setError(null);

    try {
      await updater.DownloadAndApply();
      // 앱이 자동으로 재시작됩니다
    } catch (err) {
      setError(err.message);
      setDownloading(false);
    }
  }

  if (checking) {
    return <div>Checking for updates...</div>;
  }

  if (error) {
    return (
      <div>
        <p>Error: {error}</p>
        <button onClick={checkForUpdates}>Retry</button>
      </div>
    );
  }

  if (!updateInfo) {
    return (
      <div>
        <p>You're up to date! (v{updater.GetCurrentVersion()})</p>
        <button onClick={checkForUpdates}>Check Again</button>
      </div>
    );
  }

  if (downloading) {
    return (
      <div>
        <p>Downloading update...</p>
        {progress && (
          <div>
            <progress value={progress.percentage} max={100} />
            <p>{progress.percentage.toFixed(1)}%</p>
            <p>{(progress.bytesPerSecond / 1024 / 1024).toFixed(2)} MB/s</p>
          </div>
        )}
      </div>
    );
  }return (
    <div>
      <h3>업데이트 가능!</h3>
      <p>Version {updateInfo.version} 업데이트가 제공됩니다</p>
      <p>크기: {updateInfo.hasPatch
        ? `${(updateInfo.patchSize / 1024).toFixed(0)} KB (패치)`
        : `${(updateInfo.size / 1024 / 1024).toFixed(1)} MB`}
      </p>
      <div dangerouslySetInnerHTML={{ __html: updateInfo.releaseNotes }} />
      <button onClick={downloadAndInstall}>다운로드 & 설치</button>
      <button onClick={() => setUpdateInfo(null)}>건너뛰기</button>
    </div>
  );
}
```

## 업데이트 전략

### 시작 시 확인

```go
func (a *App) OnStartup(ctx context.Context) {
    // 짧은 지연 후 업데이트 확인
    go func() {
        time.Sleep(5 * time.Second)
        info, err := a.updater.CheckForUpdate()
        if err == nil && info != nil {
            // 프론트엔드에 이벤트 발생
            application.Get().EmitEvent("update-available", info)
        }
    }()
}
```

### 백그라운드 확인

자동 백그라운드 확인을 구성합니다:

```go
updater, _ := application.CreateUpdaterService(
    "1.0.0",
    application.WithUpdateURL("https://updates.example.com/myapp/"),
    application.WithCheckInterval(6 * time.Hour), // 6시간마다 확인
)
```

### 수동 확인 메뉴 항목

```go
menu := application.NewMenu()
menu.Add("업데이트 확인...").OnClick(func(ctx *application.Context) {
```

## 델타 업데이트 (패치)

델타 업데이트(패치)를 사용하면 사용자는 버전 간 변경 사항만 다운로드할 수 있어 다운로드 크기를 크게 줄일 수 있습니다.

### 작동 방식

1. 새 버전을 빌드할 때 이전 버전에서 패치를 생성합니다.
2. 서버에서 전체 업데이트와 함께 패치를 호스팅합니다.
3. 업데이트 프로그램은 사용 가능한 경우 자동으로 패치를 다운로드합니다.
4. 패치 적용이 실패하면 전체 다운로드로 폴백됩니다.

### 패치 생성

패치는 `bsdiff` 알고리즘을 사용하여 생성됩니다. `bsdiff` 도구가 필요합니다:

```bash
# bsdiff 설치 (macOS)
brew install bsdiff

# bsdiff 설치 (Ubuntu/Debian)
sudo apt-get install bsdiff

# 패치 생성
bsdiff old-binary new-binary patch.bsdiff
```

### 패치 파일 이름 지정

업데이트 디렉토리에 패치를 구성합니다:

```
updates/
├── update.json
├── myapp-1.2.0-macos-arm64.tar.gz
├── myapp-1.2.0-windows-amd64.zip
└── patches/
    ├── 1.0.0-to-1.2.0-macos-arm64.patch
    ├── 1.1.0-to-1.2.0-macos-arm64.patch
    ├── 1.0.0-to-1.2.0-windows-amd64.patch
    └── 1.1.0-to-1.2.0-windows-amd64.patch
```

<Aside type="tip">
최근 몇 버전의 패치를 보관하세요. 매우 오래된 버전의 사용자는 자동으로 전체 업데이트를 다운로드합니다.
</Aside>

## 업데이트 호스팅

### 정적 파일 호스팅

업데이트는 모든 정적 파일 서버에 호스팅할 수 있습니다:

- **Amazon S3** / **Cloudflare R2**
- **Google Cloud Storage**
- **GitHub Releases**
- **모든 CDN 또는 웹 서버**

S3 버킷 구조 예시:

```
s3://my-updates-bucket/myapp/
├── stable/
│   ├── update.json
│   ├── myapp-1.2.0-macos-arm64.tar.gz
│   └── patches/
│       └── 1.1.0-to-1.2.0-macos-arm64.patch
└── beta/
    └── update.json
```

### CORS 구성

다른 도메인에 호스팅하는 경우 CORS를 구성합니다:

```json
{
  "CORSRules": [
    {
      "AllowedOrigins": ["*"],
      "AllowedMethods": ["GET"],
      "AllowedHeaders": ["*"]
    }
  ]
}
```

## 보안

### 체크섬 검증

모든 다운로드가 매니페스트의 SHA256 체크섬과 비교 검증됩니다:

```json
{
  "checksum": "sha256:e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
}
```

### 서명 검증

추가 보안을 위해 서명 검증을 활성화합니다:

<Steps>
1. **키 쌍 생성**:
   ```bash
   # Ed25519 키 쌍 생성
   openssl genpkey -algorithm Ed25519 -out private.pem
   openssl pkey -in private.pem -pubout -out public.pem
   ```

2. **업데이트 매니페스트 서명**:
   ```bash
   openssl pkeyutl -sign -inkey private.pem -in update.json -out update.json.sig
   ```

3. **업데이트 프로그램 구성**:
   ```go
   updater, _ := application.CreateUpdaterService(
       "1.0.0",
       application.WithUpdateURL("https://updates.example.com/myapp/"),
       application.WithRequireSignature(true),
       application.WithPublicKey("MCowBQYDK2VwAyEA..."), // Base64 인코딩된 공개 키
   )
   ```
</Steps>

## 모범 사례

### 수행해야 할 사항

- 배포하기 전에 업데이트를 철저히 테스트합니다.
- 롤백을 위해 이전 버전을 사용 가능하게 유지합니다.
- 사용자에게 릴리스 노트를 표시합니다.
- 사용자가 필수 아닌 업데이트를 건너뛸 수 있도록 허용합니다.
- 모든 다운로드에 HTTPS를 사용합니다.
- 업데이트 적용 전에 체크섬을 검증합니다.
- 오류를 우아하게 처리합니다.

### 수행하지 말아야 할 사항

- 경고 없이 즉시 재시작을 강요합니다.
- 체크섬 검증을 건너뜁니다.
- 중요한 작업 중 사용자를 방해합니다.
- 이전 버전을 즉시 삭제합니다.
- 업데이트 실패를 무시합니다.

## 문제 해결

### 업데이트를 찾을 수 없음

- 매니페스트 URL이 올바른지 확인합니다.
- 플랫폼 키가 일치하는지 확인합니다 (예: `macos-arm64`).
- 매니페스트의 버전이 최신인지 확인합니다.

### 다운로드 실패

- 네트워크 연결을 확인합니다.
- 다운로드 URL에 접근할 수 있는지 확인합니다.
- 크로스 오리진인 경우 CORS 구성을 확인합니다.

### 패치 실패

- 업데이트 프로그램은 자동으로 전체 다운로드로 폴백됩니다.
- 시스템에 `bspatch`가 사용 가능한지 확인합니다.
- 패치 체크섬이 올바른지 확인합니다.

### 애플리케이션이 재시작되지 않음

- macOS에서는 앱이 올바르게 코드 서명되었는지 확인합니다.
- Windows에서는 파일 잠금을 확인합니다.
- Linux에서는 파일 권한을 확인합니다.
