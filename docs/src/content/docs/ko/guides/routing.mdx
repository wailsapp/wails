프론트엔드 라우팅은 단일 페이지 애플리케이션에서 뷰를 전환하는 인기 있는 방법입니다.
이 가이드는 Wails를 사용할 때 다양한 프론트엔드 프레임워크에 권장되는 접근 방식을 다룹니다.


<Tabs syncKey="framework">
    <TabItem label="Vue" icon="seti:vue">

Vue에서 라우팅을 위한 권장 접근 방식은 [Hash Mode](https://next.router.vuejs.org/guide/essentials/history-mode.html#hash-mode)입니다:

```javascript
import { createRouter, createWebHashHistory } from "vue-router";

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    //...
  ],
});
```

Hash 모드는 URL 해시를 사용하여 다른 뷰를 렌더링하며, 해시 기반 URL 형식을 사용하여 Wails 런타임이 라우팅에 간섭하는 문제를 피합니다.

    </TabItem>
    <TabItem label="Angular" icon="seti:angular">

Angular에서 라우팅을 위한 권장 접근 방식은 [HashLocationStrategy](https://codecraft.tv/courses/angular/routing/routing-strategies#%5Fhashlocationstrategy)입니다:

```typescript
RouterModule.forRoot(routes, { useHash: true });
```

해시 기반 URL을 사용하면 모든 플랫폼에서 Wails의 창 처리와 함께 라우팅이 올바르게 작동합니다.

    </TabItem>
    <TabItem label="React" icon="seti:react">

React에서 라우팅을 위한 권장 접근 방식은 [HashRouter](https://reactrouter.com/en/main/router-components/hash-router)입니다:

```jsx
import ReactDOM from "react-dom/client";
import { HashRouter, Routes, Route } from "react-router-dom";

ReactDOM.createRoot(root).render(
  <HashRouter basename={"/"}>
    {/* The rest of your app goes here */}
    <Routes>
      <Route path="/" element={<Page0 />} />
      <Route path="/page1" element={<Page1 />} />
      <Route path="/page2" element={<Page2 />} />
      {/* more... */}
    </Routes>
  </HashRouter>
);
```

HashRouter는 경로를 대신하여 URL 해시를 사용하며, 이는 모든 플랫폼에서 Wails와 함께 신뢰성 있게 작동합니다.

    </TabItem>
    <TabItem label="Svelte" icon="seti:svelte">

Svelte에서 라우팅을 위한 권장 접근 방식은 [svelte-spa-router](https://github.com/ItalyPaleAle/svelte-spa-router)입니다:

```svelte
<script>
    import Router from "svelte-spa-router";
</script>

<Router
    routes={{
        "/": Home,
        "/products": wrap({
            asyncComponent: () => import("./routes/Products.svelte"),
        }),
        "/settings": Settings,
        "*": NotFound,
    }}
/>
```

svelte-spa-router는 해시 기반 라우팅을 지원하여 Wails 애플리케이션과 호환됩니다.

    </TabItem>
</Tabs>

## 왜 해시 라우팅인가?

Wails는 프론트엔드를 네이티브 웹뷰 창에 임베드합니다. 해시 기반 라우팅(#/page 대신 /page)을 사용하면 다음과 같은 충돌을 피할 수 있습니다:

- Wails 런타임의 내부 라우팅
- 다양한 플랫폼의 네이티브 창 URL 처리
- 루트 경로가 아닌 경로에서 제공되는 프로덕션 애셋

## 문제 해결

프로덕션 빌드에서 프론트엔드 라우팅 관련 문제가 있는 경우:

- 프론트엔드 빌드 도구가 **hash mode** 라우팅을 출력하도록 구성되어 있는지 확인하십시오
- Vite 기반 프로젝트의 경우 `vite.config.js`에 `base: "./"`를 추가하십시오
- `index.html`이 SPA 라우팅을 위해 폴백을 올바르게 처리하는지 확인하십시오
