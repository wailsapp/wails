# 서비스를 위한 Gin 사용

Gin 웹 프레임워크는 Go에서 HTTP 서비스를 구축하는 데 인기 있는 선택입니다. Wails v3를 사용하면 Gin 기반 서비스를 애플리케이션에 쉽게 통합하여 HTTP 요청 처리, RESTful API 구현 및 웹 콘텐츠 제공을 위한 강력한 방법을 제공할 수 있습니다.

이 가이드에서는 Wails 애플리케이션의 특정 경로에 마운트할 수 있는 Gin 기반 서비스를 만드는 방법을 단계별로 설명합니다. 다음 방법을 보여주는 완전한 예제를 구축합니다:

1. Gin 기반 서비스 만들기
2. Wails 서비스 인터페이스 구현
3. 경로와 미들웨어 설정
4. Wails 이벤트 시스템과 통합
5. 프론트엔드에서 서비스와 상호작용

## 사전 준비 사항

시작하기 전에 다음이 준비되어 있는지 확인하세요:

- Wails v3 설치됨
- Go와 Gin 프레임워크에 대한 기본 지식
- HTTP 개념과 RESTful API에 대한 친숙함

프로젝트에 Gin 프레임워크를 추가해야 합니다:

```bash
go get github.com/gin-gonic/gin
```

## Gin 기반 서비스 만들기

Wails 서비스 인터페이스를 구현하는 Gin 서비스를 만드는 것으로 시작해 보겠습니다. 우리의 서비스는 사용자 컬렉션을 관리하고 사용자 레코드를 검색하고 생성하기 위한 API 엔드포인트를 제공합니다.

### 1. 데이터 모델 정의

먼저 서비스가 작업할 데이터 구조를 정의합니다:

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User는 시스템의 사용자를 나타냅니다
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData는 이벤트에 전송되는 데이터를 나타냅니다
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. 서비스 구조 만들기

다음으로 Gin 라우터와 서비스가 유지해야 하는 모든 상태를 보유할 서비스 구조를 정의합니다:

```go
// GinService는 HTTP 처리에 Gin을 사용하는 Wails 서비스를 구현합니다
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService는 새로운 GinService 인스턴스를 생성합니다
func NewGinService() *GinService {
	// 새로운 Gin 라우터 생성
	ginEngine := gin.New()

	// 미들웨어 추가
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// 경로 정의
	service.setupRoutes()

	return service
}
```

### 3. 서비스 인터페이스 구현

Wails 서비스 인터페이스에 필요한 메서드를 구현합니다:

```go
// ServiceName은 서비스의 이름을 반환합니다
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup는 서비스가 시작될 때 호출됩니다
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// 나중에 사용하기 위해 애플리케이션 인스턴스 저장
	s.app = application.Get()

	// 프론트엔드에서 트리거할 수 있는 이벤트 핸들러 등록
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// 이벤트 데이터 로깅
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// 프론트엔드로 이벤트 다시 발생
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown은 서비스가 종료될 때 호출됩니다
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// 필요한 경우 리소스 정리
	return nil
}
```

### 3. http.Handler 인터페이스 구현

서비스를 특정 경로에 마운트할 수 있도록 하려면 `http.Handler` 인터페이스를 구현해야 합니다. 이 단일 메서드인 `ServeHTTP`는 서비스에 대한 모든 HTTP 요청의 진입점입니다. 이 메서드는 요청 처리를 Gin 라우터에 위임하여 라우팅과 미들웨어에 대한 Gin의 강력한 기능을 모두 사용할 수 있게 합니다.

```go
// ServeHTTP는 http.Handler 인터페이스를 구현합니다
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// 모든 요청은 Gin 라우터로 전달됩니다
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. 경로 설정

더 나은 조직화를 위해 API 경로를 별도의 메서드에서 정의합니다. 이 접근 방식은 코드를 깔끔하게 유지하고 API 구조를 이해하기 쉽게 만듭니다. Gin 라우터는 경로 그룹 지원을 포함하여 경로를 정의하기 위한 플루언트 API를 제공하며, 이는 관련 엔드포인트를 조직하는 데 도움이 됩니다.

```go
// setupRoutes는 API 경로를 구성합니다
func (s *GinService) setupRoutes() {
	// 기본 정보 엔드포인트
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// 사용자 그룹
	users := s.ginEngine.Group("/users")
	{
		// 모든 사용자 가져오기
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// ID로 사용자 가져오기
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// 새 사용자 생성
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// ID와 생성 시간 설정
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// 사용자 슬라이스에 추가
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// 새 사용자를 알리기 위해 이벤트 발생
			s.app.Event.Emit("user-created", newUser)
		})

		// 사용자 삭제
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// 슬라이스에서 사용자 제거
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. 사용자 정의 미들웨어 만들기

서비스를 향상시키기 위해 사용자 정의 Gin 미들웨어를 만들 수 있습니다. Gin의 미들웨어 함수는 라우터에 추가된 순서대로 실행되며 로깅, 인증 및 오류 처리와 같은 작업을 수행할 수 있습니다. 이 예제는 요청 세부 정보를 기록하는 간단한 로깅 미들웨어를 보여줍니다.

```go
// LoggingMiddleware는 요청 세부 정보를 로깅하는 Gin 미들웨어입니다
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// 타이머 시작
		start := time.Now()

		// 요청 처리
		c.Next()

		// 지연 시간 계산
		latency := time.Since(start)

		// 요청 세부 정보 로깅
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## 서비스 등록Wails 애플리케이션에서 Gin 기반 서비스를 사용하려면 애플리케이션에 서비스를 등록하고 마운트할 경로를 지정해야 합니다. 이는 Wails 애플리케이션 인스턴스를 생성할 때 수행됩니다. 지정된 경로는 Gin 라우터에 정의된 모든 엔드포인트의 기본 경로가 됩니다.

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

이 예제에서 Gin 서비스는 `/api` 경로에 마운트됩니다. 즉, Gin 라우터에 `/info` 엔드포인트가 정의되어 있다면 애플리케이션에서 `/api/info`로 접근할 수 있습니다. 이 접근 방식을 통해 API 엔드포인트를 논리적으로 구성하고 애플리케이션의 다른 부분과의 충돌을 피할 수 있습니다.

## Wails 이벤트 시스템 통합

Gin을 Wails 서비스와 함께 사용하는 강력한 기능 중 하나는 Wails 이벤트 시스템과 원활하게 통합할 수 있다는 점입니다. 이를 통해 백엔드 서비스와 프론트엔드 간 실시간 통신이 가능합니다.

서비스의 `ServiceStartup` 메서드에서 프론트엔드에서 오는 이벤트를 처리하기 위해 이벤트 핸들러를 등록할 수 있습니다:

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

Gin 라우트나 서비스의 다른 부분에서 프론트엔드로 이벤트를 emit할 수도 있습니다:

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## 프론트엔드 통합

프론트엔드에서 Gin 서비스와 상호 작용하려면 Wails 런타임을 가져오고 API 엔드포인트로 HTTP 요청을 보내며 실시간 통신을 위해 Wails 이벤트 시스템을 사용해야 합니다.

프로덕션 환경에서는 `/wails/runtime.js`를 직접 가져오는 대신 `@wailsio/runtime` 패키지를 사용하는 것이 권장됩니다. 이는 타입 안전성, 더 나은 IDE 지원, npm을 통한 버전 관리 및 최신 JavaScript 도구와의 호환성을 보장합니다.

패키지를 설치합니다:
```bash
npm install @wailsio/runtime
```

그런 다음 코드에서 사용합니다:
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## 마무리하며

Gin 웹 프레임워크를 Wails v3 Services와 통합하면 모듈화되고 유지보수가 용이한 웹 애플리케이션을 구축하기 위해 강력하고 유연한 접근 방식을 제공합니다. Gin의 라우팅 및 미들웨어 기능과 Wails 이벤트 시스템을 함께 활용하면 관심사의 명확한 분리를 통해 풍부하고 상호작용적인 애플리케이션을 만들 수 있습니다.

이 가이드의 전체 예제 코드는 Wails 저장소의 `v3/examples/gin-service`에서 확인할 수 있습니다.
