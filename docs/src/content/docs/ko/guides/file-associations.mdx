파일 연결을 사용하면 사용자가 파일을 열 때 애플리케이션이 특정 파일 형식을 처리할 수 있습니다. 이는 텍스트 편집기, 이미지 뷰어 또는 특정 파일 형식과 작동하는 모든 애플리케이션에 특히 유용합니다. 이 가이드에서는 Wails v3 애플리케이션에서 파일 연결을 구현하는 방법을 설명합니다.

## 개요

Wails v3에서 파일 연결 지원은 현재 다음 플랫폼에서 사용할 수 있습니다:

- Windows (NSIS 설치 패키지)
- macOS (애플리케이션 번들)

## 구성

파일 연결은 프로젝트의 `build` 디렉토리에 있는 `config.yml` 파일에서 구성됩니다.

### 기본 구성

파일 연결을 설정하려면:

1. `build/config.yml`을 엽니다.
2. `fileAssociations` 섹션 아래에 파일 연결을 추가합니다.
3. 빌드 애셋을 업데이트하려면 `wails3 update build-assets`를 실행합니다.
4. 애플리케이션 옵션에서 `FileAssociations` 필드를 설정합니다.
5. `wails3 package`를 사용하여 애플리케이션을 패키징합니다.

다음은 예제 구성입니다:

```yaml
fileAssociations:
  - ext: myapp
    name: MyApp Document
    description: MyApp Document File
    iconName: myappFileIcon
    role: Editor
  - ext: custom
    name: Custom Format
    description: Custom File Format
    iconName: customFileIcon
    role: Editor
```

### 구성 속성

| 속성        | 설명                                                         | 플랫폼   |
|------|----------------------------------------------------------|----------|
| ext         | 선행 점(.)을 제외한 파일 확장자 (예: `txt`)              | 전체     |
| name        | 파일 유형의 표시 이름                                      | 전체     |
| description | 파일 속성에 표시되는 설명                                   | Windows  |
| iconName    | 빌드 폴더 내 아이콘 파일의 이름 (확장자 제외)               | 전체     |
| role        | 이 파일 유형에 대한 애플리케이션의 역할 (예: `Editor`, `Viewer`) | macOS    |
| mimeType    | 파일의 MIME 유형 (예: `image/jpeg`)                       | macOS    |

## 파일 열기 이벤트 수신

애플리케이션에서 파일 열기 이벤트를 처리하려면 `events.Common.ApplicationOpenedWithFile` 이벤트를 수신할 수 있습니다:

```go title="main.go"
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
    "github.com/wailsapp/wails/v3/pkg/events"
)

func main() {
    app := application.New(application.Options{
        Name: "MyApp",
        FileAssociations: []string{".txt", ".md"}, // 지원되는 확장자 지정
    })

    // 파일을 사용하여 애플리케이션이 열릴 때 수신
    app.Event.OnApplicationEvent(events.Common.ApplicationOpenedWithFile, func(event *application.ApplicationEvent) {
        associatedFile := event.Context().Filename()
        application.InfoDialog().SetMessage("Application opened with file: " + associatedFile).Show()
    })

    // 창을 생성하고 앱을 실행합니다...
}

```

## 단계별 튜토리얼

간단한 텍스트 편집기를 위한 파일 연결 설정 방법을 살펴보겠습니다:

<Steps>

1. ### 아이콘 생성

    - 파일 유형에 대한 아이콘을 생성합니다 (권장 크기: 16x16, 32x32, 48x48,
      256x256)
    - 아이콘을 프로젝트의 `build` 폴더에 저장합니다.
    - `iconName` 구성에 따라 이름을 지정합니다 (예:
      `textFileIcon.png`)

    :::tip
    필요한 아이콘을 생성하려면 `wails3 generate icons`를 사용할 수 있습니다.
    자세한 정보는 `wails3 generate icons --help`를 실행하세요.
    :::

    - macOS의 경우 `create:app:bundle:` 작업에 `cp build/darwin/documenticon.icns {{.BIN_DIR}}/{{.APP_NAME}}.app/Contents/Resources`와 같은 복사 명령문을 추가합니다.

2. ### 파일 연결 구성

    파일 연결을 추가하려면 `build/config.yml` 파일을 편집합니다:

    ```yaml
    # build/config.yml
    fileAssociations:
      - ext: txt
        name: Text Document
        description: Plain Text Document
        iconName: textFileIcon
        role: Editor
    ```

3. ### 빌드 애셋 업데이트

    다음 명령을 실행하여 빌드 애셋을 업데이트합니다:

    ```bash
    wails3 update build-assets
    ```

4. ### 애플리케이션 옵션에서 파일 연결 설정

    `main.go` 파일에서 애플리케이션 옵션의 `FileAssociations` 필드를 설정합니다:

    ```go
    app := application.New(application.Options{
      Name: "MyApp",
      FileAssociations: []string{".txt", ".md"}, // 지원되는 확장자 지정
    })
    ```

    :::tip[왜 파일 확장자가 애플리케이션 구성과 config.yml 모두에 필요한가요?]

    Windows에서 파일 연결을 사용하여 파일을 열면 애플리케이션이 파일 이름을 첫 번째 인수로 사용하여 실행됩니다. 애플리케이션은 첫 번째 인수가 파일인지 명령줄 인수인지 알 수 없으므로, 애플리케이션 옵션의 `FileAssociations` 필드를 사용하여 첫 번째 인수가 연결된 파일인지 여부를 결정합니다.

    :::

5. ### 애플리케이션 패키징

    다음 명령을 사용하여 애플리케이션을 패키징합니다:

    ```bash
    wails3 package
    ```

    패키징된 애플리케이션은 `bin` 디렉토리에 생성됩니다. 그런 다음 애플리케이션을 설치하고 테스트할 수 있습니다.

    ## 추가 참고 사항

    - 아이콘은 빌드 폴더에 PNG 형식으로 제공되어야 합니다.
    - 파일 연결 테스트에는 패키징된 애플리케이션 설치가 필요합니다.

</Steps>
