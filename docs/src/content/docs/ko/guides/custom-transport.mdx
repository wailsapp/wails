Wails v3는 생성된 바인딩과 이벤트 통신을 모두 유지하면서 커스텀 IPC 전송 계층을 제공할 수 있습니다. 이를 통해 기본 HTTP fetch 기반 전송을 WebSocket, 커스텀 프로토콜 또는 기타 전송 메커니즘으로 대체할 수 있습니다.

## 개요

기본적으로 Wails는 프론트엔드에서 `/wails/runtime`을 통해 백엔드와 통신하기 위해 HTTP fetch 요청을 사용합니다. 커스텀 전송 API를 통해 다음을 수행할 수 있습니다:

- HTTP 전송을 WebSocket, gRPC 또는 기타 커스텀 프로토콜로 대체
- Wails 코드 생성과 완전한 호환성 유지
- 기존 바인딩, 이벤트, 대화상자 및 기타 Wails 기능 유지
- 자체 연결 관리, 인증 및 오류 처리 구현

## 사용법

### 1. 전송 인터페이스 구현

`Transport` 인터페이스를 구현하여 커스텀 전송을 생성합니다:

```go
package main

import (
    "context"
    "github.com/wailsapp/wails/v3/pkg/application"
)

type MyCustomTransport struct {
    // Your fields
}

func (t *MyCustomTransport) Start(ctx context.Context, processor *application.MessageProcessor) error {
    // Initialize your transport (WebSocket server, gRPC server, etc.)
    // When you receive requests, call processor.HandleRuntimeCallWithIDs()
    return nil
}

func (t *MyCustomTransport) Stop() error {
    // Clean up your transport
    return nil
}
```

### 2. 애플리케이션 구성

애플리케이션 옵션에 커스텀 전송을 전달합니다:

```go
func main() {
    app := application.New(application.Options{
        Name: "My App",
        Transport: &MyCustomTransport{},
        // ... other options
    })

    err := app.Run()
    if err != nil {
        log.Fatal(err)
    }
}
```

### 3. 프론트엔드 런타임 수정

커스텀 전송을 사용하는 경우 HTTP fetch 대신 전송을 사용하도록 프론트엔드 런타임을 수정해야 합니다. 요청을 처리하는 데 사용되는 `RuntimeTransport` 인터페이스를 구현합니다:

```typescript
const { setTransport } = await import('/wails/runtime.js');

class MyRuntimeTransport {
  call(objectID: number, method: number, windowName: string, args: any): Promise<any> {
    // TODO: implement IPC call with your transport protocol

    return resp;
  }
}

const myTransport = new MyRuntimeTransport();
setTransport(myTransport);
```

## 참고

- 커스텀 전송이 지정되지 않은 경우 기본 HTTP 전송은 계속 작동합니다
- 생성된 바인딩은 변경되지 않으며 전송 계층만 변경됩니다
- 이벤트, 대화상자, 클립보드 및 기타 모든 Wails 기능은 투명하게 작동합니다
- 커스텀 전송에서 오류 처리, 재연결 로직 및 보안을 담당합니다
- 제공된 WebSocket 예제는 시연용이며 프로덕션 사용 시 보안 강화가 필요할 수 있습니다

## API 참조

### Transport 인터페이스

```go
type Transport interface {
    Start(ctx context.Context, messageProcessor *application.MessageProcessor) error
    Stop() error
}
```

### AssetServerTransport 인터페이스 (선택사항)

브라우저 기반 배포 또는 커스텀 전송을 통해 자산과 IPC 모두를 제공하려는 경우 `AssetServerTransport` 인터페이스를 구현합니다:

```go
type AssetServerTransport interface {
    Transport

    // ServeAssets configures the transport to serve assets alongside IPC.
    // The assetHandler is Wails' internal asset server that handles all assets,
    // runtime.js, capabilities, flags, etc.
    ServeAssets(assetHandler http.Handler) error
}
```

**이 인터페이스를 구현해야 하는 경우:**
- 웹뷰 대신 브라우저에서 애플리케이션 실행
- 커스텀 IPC 전송과 함께 HTTP를 통해 자산 제공
- 네트워크 접근 가능한 애플리케이션 빌드

**구현 예시:**

```go
func (t *MyTransport) ServeAssets(assetHandler http.Handler) error {
    mux := http.NewServeMux()

    // Mount your IPC endpoint
    mux.HandleFunc("/my/ipc/endpoint", t.handleIPC)

    // Mount Wails asset server for everything else
    mux.Handle("/", assetHandler)

    // Start HTTP server
    t.httpServer.Handler = mux
    go t.httpServer.ListenAndServe()

    return nil
}
```

`ServeAssets()`가 호출되면 assetHandler는 다음을 제공합니다:
- 모든 정적 자산 (HTML, CSS, JS, 이미지 등)
- `/wails/runtime.js` - Wails 런타임 라이브러리

## 관련 항목

- `transport.go` - 핵심 전송 인터페이스 및 유형
- `messageprocessor.go` - 모든 Wails IPC를 처리하는 기본 메시지 프로세서
