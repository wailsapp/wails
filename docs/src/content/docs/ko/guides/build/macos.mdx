## 애플리케이션 번들

애플리케이션을 표준 macOS `.app` 번들로 패키징하세요:

```bash
wails3 package GOOS=darwin
```

이 명령은 다음을 포함하는 `bin/<AppName>.app`을 생성합니다:
- `Contents/MacOS/`에 있는 컴파일된 바이너리
- `Contents/Resources/`에 있는 앱 아이콘 (`icons.icns`에서, 또는 존재할 경우 자산 카탈로그 `Assets.car`에서)
- 앱 메타데이터가 포함된 `Info.plist`

### 유니버설 바이너리

Apple Silicon과 Intel Mac 모두에서 빌드하세요:

```bash
wails3 task darwin:package:universal
```

이 명령은 두 아키텍처 모두에서 네이티브로 실행되는 단일 `.app`을 생성합니다. 유니버설 바이너리는 모든 플랫폼에서 빌드할 수 있으며, Linux와 Windows에서는 `wails3 tool lipo`가 자동으로 사용됩니다.

## 번드 사용자 정의

`build/darwin/Info.plist`을 편집하여 다음을 사용자 정의할 수 있습니다:

- 번들 식별자 (`CFBundleIdentifier`)
- 앱 이름 및 버전
- 최소 macOS 버전
- 파일 연결
- URL 스키마

앱 아이콘은 `build/` 디렉토리의 애셋에서 생성됩니다. `generate:icons` 작업을 사용하세요:

```bash
wails3 task generate:icons
```

이 작업은 `build/appicon.png`를 사용하여 `darwin/icons.icns`와 `windows/icon.ico`를 생성합니다. macOS에서는 `build/appicon.icon`(Icon Composer 형식)도 제공할 수 있습니다. 이 작업은 `-iconcomposerinput appicon.icon -macassetdir darwin`을 전달하여 `.icon` 파일로부터 `Assets.car`와 `darwin/icons.icns`를 생성합니다(비macOS 플랫폼에서는 건너뜀). `Assets.car`가 존재할 경우, `Info.plist`과 `CFBundleIconName`이 적절히 업데이트되도록 `update:build-assets` 작업을 실행하세요:

```bash
wails3 task update:build-assets
```

`build/` 디렉토리에서 아이콘 명령을 수동으로 실행하려면:

```bash
cd build
wails3 generate icons -input appicon.png -macfilename darwin/icons.icns -windowsfilename windows/icon.ico -iconcomposerinput appicon.icon -macassetdir darwin
```

## 코드 서명

배포를 위해 애플리케이션에 서명하세요:

```bash
# 래퍼 사용 (플랫폼 자동 감지)
wails3 sign GOOS=darwin

# 또는 작업 직접 사용
wails3 task darwin:sign
```

`build/darwin/Taskfile.yml`에서 서명을 구성하세요:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  ENTITLEMENTS: "build/darwin/entitlements.plist"
```

### 주석 처리(Notarization)

Mac App Store 외부로 배포되는 앱의 경우 Apple은 주석 처리를 요구합니다:

```bash
wails3 task darwin:sign:notarize
```

먼저 자격 증명을 저장하세요:

```bash
wails3 signing credentials \
  --apple-id "you@email.com" \
  --team-id "TEAMID" \
  --password "app-specific-password" \
  --profile "my-notarize-profile"
```

`build/darwin/Taskfile.yml`에서 구성하세요:

```yaml
vars:
  SIGN_IDENTITY: "Developer ID Application: Your Company (TEAMID)"
  KEYCHAIN_PROFILE: "my-notarize-profile"
```

자세한 내용은 [애플리케이션 서명](/ko/guides/build/signing)을 참조하세요.

## DMG 설치 프로그램

배포용 DMG 디스크 이미지를 생성하세요:

```bash
wails3 task darwin:create:dmg
```

<Aside type="tip">
`build/darwin/dmg/` 애셋을 편집하여 DMG 배경 및 레이아웃을 사용자 정의할 수 있습니다.
</Aside>

## 문제 해결

### "앱이 손상되어 열 수 없습니다"

앱이 서명되지 않았습니다. Developer ID 인증서로 서명하거나, 사용자가 Gatekeeper를 우회할 수 있습니다:

```bash
xattr -cr /path/to/YourApp.app
```

### 주석 처리 실패

일반적인 문제:
- **잘못된 자격 증명**: `wails3 signing credentials`를 다시 실행하세요
- **하드닝된 런타임 필요**: 필요한 경우 엔터테인먼트에 `com.apple.security.cs.allow-unsigned-executable-memory`가 포함되어 있는지 확인하세요
- **누락된 타임스탬프**: 서명 프로세스에는 자동으로 타임스탬프가 포함되어야 합니다

### 크로스 컴파일된 앱이 실행되지 않음

크로스 컴파일된 macOS 바이너리는 서명되지 않았습니다. 테스트 전에 Mac으로 전송하고 서명하세요:

```bash
codesign --force --deep --sign - YourApp.app
```
