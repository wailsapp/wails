원시 메시지(Raw Messages)는 프론트엔드와 백엔드 간에 표준 바인딩 시스템을 우회하는 저수준 통신 채널을 제공합니다. 이는 편의성을 속도와 교환합니다.

## 원시 메시지를 사용해야 하는 경우

원시 메시지는 극단적인 가장자리 케이스(edge cases)에 가장 적합합니다:

- **초고빈도 업데이트** - 마이크로초 단위가 중요한 초당 수천 개의 메시지
- **사용자 정의 메시지 프로토콜** - 와이어 포맷(wire format)에 대한 완전한 제어가 필요한 경우

:::tip
거의 모든 사용 사례에서는 표준 [서비스 바인딩](/ko/features/bindings/services)을 권장합니다. 이는 타입 안전성, 자동 직렬화 및 무시할 수 있는 오버헤드로 더 나은 개발자 경험을 제공하기 때문입니다.
:::

## 백엔드 설정

애플리케이션 옵션에서 `RawMessageHandler`를 구성합니다:

```go
package main

import (
    "encoding/json"
    "fmt"

    "github.com/wailsapp/wails/v3/pkg/application"
)

func main() {
    app := application.New(application.Options{
        Name: "Raw Message Demo",
        Assets: application.AssetOptions{
            Handler: application.BundledAssetFileServer(assets),
        },
        RawMessageHandler: func(window application.Window, message string, originInfo *application.OriginInfo) {
            fmt.Printf("Raw message from window '%s': %s (origin: %+v)\n", window.Name(), message, originInfo.Origin)

            // Process the message and respond via events
            response := processMessage(message)
            window.EmitEvent("raw-response", response)
        },
    })

    app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title: "My App",
        Name:  "main",
    })

    app.Run()
}

func processMessage(message string) map[string]any {
    // Your custom message processing logic
    return map[string]any{
        "received": message,
        "status":   "processed",
    }
}
```

### 핸들러 시그니처

```go
RawMessageHandler func(window Window, message string, originInfo *application.OriginInfo)
```

| 매개변수 | 타입 | 설명 |
|-----------|------|-------------|
| `window` | `Window` | 메시지를 보낸 창 |
| `message` | `string` | 원시 메시지 내용 |
| `originInfo` | `*application.OriginInfo` | 메시지 소스에 대한 출처 정보 |

#### OriginInfo 구조체

```go
type OriginInfo struct {
	Origin      string
	TopOrigin   string
	IsMainFrame bool
}
```

| 필드 | 타입 | 설명 |
|-------|------|-------------|
| `Origin` | `string` | 메시지를 보낸 문서의 출처 URL |
| `TopOrigin` | `string` | 최상위 출처 URL (iframe의 경우 Origin과 다를 수 있음) |
| `IsMainFrame` | `bool` | 메시지가 메인 프레임에서 발생했는지 여부 |

#### 플랫폼별 사용 가능성

- **macOS**: `Origin` 및 `IsMainFrame` 제공
- **Windows**: `Origin` 및 `TopOrigin` 제공
- **Linux**: `Origin`만 제공

### 출처 검증

:::caution
핸들러에 도착했다는 이유로 메시지가 안전하다고 가정하지 마십시오. 민감한 작업이나 상태를 수정하는 작업을 처리하기 전에 출처 정보를 검증해야 합니다.
:::

**처리하기 전에 들어오는 메시지의 출처를 항상 확인하십시오.** `originInfo` 매개변수는 무단 접근을 방지하기 위해 검증해야 하는 중요한 보안 정보를 제공합니다. 악성 콘텐츠, 침해된 콘텐츠 또는 의도하지 않은 스크립트가 원시 메시지를 보낼 수 있습니다. 출처 검증 없이 신뢰할 수 없는 소스의 명령을 처리할 수 있습니다. `originInfo`를 사용하여 메시지가 예상된 소스에서 오는지 확인하십시오.

### 주요 검증 지점

- **항상 `Origin` 확인** - 출처가 예상된 신뢰할 수 있는 소스(일반적으로 로컬 자산의 경우 `wails://wails` 또는 `http://wails.localhost` 또는 앱의 특정 출처)와 일치하는지 확인합니다.
- **`IsMainFrame` 검증** (macOS) - 메시지가 iframe에서 오는지 여부를 인지하십시오. 이는 다른 보안 컨텍스트를 가진 임베디드 콘텐츠를 나타낼 수 있습니다.
- **`TopOrigin` 사용** (Windows) - 프레임 콘텐츠 처리 시 최상위 출처를 확인합니다.
- **예상치 못한 출처 거부** - 명시적으로 허용하지 않은 출처에서 오는 메시지를 거부하여 안전하게 실패합니다.


:::note
`wails:`로 시작하는 메시지는 내부 Wails 통신에 예약되어 있으며 핸들러로 전달되지 않습니다.
:::

## 프론트엔드 설정

`System.invoke()`를 사용하여 원시 메시지를 보냅니다:

```html
<!DOCTYPE html>
<html>
<head>
    <script type="module">
        import { System, Events } from '@wailsio/runtime'

        // Send raw message
        document.getElementById('send').addEventListener('click', () => {
            const message = document.getElementById('input').value
            System.invoke(message)
        })

        // Listen for response
        Events.On('raw-response', (event) => {
            console.log('Response:', event.data)
        })
    </script>
</head>
<body>
    <input type="text" id="input" placeholder="Enter message" />
    <button id="send">Send</button>
</body>
</html>
```

### 사전 빌드된 번들 사용

npm을 사용하지 않는 경우 전역 `wails` 개체를 통해 `invoke`에 접근합니다:

```html
<script type="module" src="/wails/runtime.js"></script>
<script>
    window.onload = function() {
        document.getElementById('send').onclick = function() {
            wails.System.invoke('my-message')
        }
    }
</script>
```

## 구조화된 메시지

복잡한 데이터의 경우 JSON으로 직렬화합니다:

### 프론트엔드

```javascript
import { System } from '@wailsio/runtime'

const command = {
    action: 'update',
    payload: {
        id: 123,
        value: 'new value'
    }
}

System.invoke(JSON.stringify(command))
```

### 백엔드

```go
RawMessageHandler: func(window application.Window, message string, originInfo *application.OriginInfo) {
    var cmd struct {
        Action  string `json:"action"`
        Payload struct {
            ID    int    `json:"id"`
            Value string `json:"value"`
        } `json:"payload"`
    }

    if err := json.Unmarshal([]byte(message), &cmd); err != nil {
        window.EmitEvent("error", err.Error())
        return
    }

    switch cmd.Action {
    case "update":
        // Handle update
        result := handleUpdate(cmd.Payload.ID, cmd.Payload.Value)
        window.EmitEvent("update-complete", result)
    default:
        window.EmitEvent("error", "unknown action")
    }
}
```

## 성능 비교

| 접근 방식 | 오버헤드 | 타입 안전성 | 사용 사례 |
|----------|----------|-------------|----------|
| 서비스 바인딩 | 높음 | 전체 | 범용 |
| 원시 메시지 | 최소 | 수동 | 고빈도, 성능이 중요한 경우 |

### 벤치마크 예시

원시 메시지는 간단한 페이로드의 경우 서비스 바인딩보다 훨씬 더 많은 메시지를 초당 처리할 수 있습니다:

```go
// Raw message handler - minimal overhead
RawMessageHandler: func(window application.Window, message string, originInfo *application.OriginInfo) {
    // Direct string processing, no reflection or marshaling
    counter++
}
```

## 전체 예제

간단한 명령 프로토콜을 구현하는 전체 예제입니다:### main.go

```go
package main

import (
    "embed"
    "encoding/json"
    "fmt"
    "time"

    "github.com/wailsapp/wails/v3/pkg/application"
)

//go:embed assets
var assets embed.FS

type Command struct {
    Type string          `json:"type"`
    Data json.RawMessage `json:"data"`
}

func main() {
    app := application.New(application.Options{
        Name: "Raw Message Demo",
        Assets: application.AssetOptions{
            Handler: application.BundledAssetFileServer(assets),
        },
        Mac: application.MacOptions{
            ApplicationShouldTerminateAfterLastWindowClosed: true,
        },
        RawMessageHandler: func(window application.Window, message string, originInfo *application.OriginInfo) {
            var cmd Command
            if err := json.Unmarshal([]byte(message), &cmd); err != nil {
                window.EmitEvent("error", map[string]string{"error": err.Error()})
                return
            }

            switch cmd.Type {
            case "ping":
                window.EmitEvent("pong", map[string]any{
                    "time":   time.Now().UnixMilli(),
                    "window": window.Name(),
                })
            case "echo":
                var text string
                json.Unmarshal(cmd.Data, &text)
                window.EmitEvent("echo", text)
            default:
                window.EmitEvent("error", map[string]string{
                    "error": fmt.Sprintf("unknown command: %s", cmd.Type),
                })
            }
        },
    })

    app.Window.NewWithOptions(application.WebviewWindowOptions{
        Title: "Raw Message Demo",
        Name:  "main",
        Width: 400,
        Height: 300,
    })

    app.Run()
}
```

### assets/index.html

```html
<!DOCTYPE html>
<html>
<head>
    <title>Raw Message Demo</title>
    <style>
        body { font-family: sans-serif; padding: 20px; }
        button { margin: 5px; padding: 10px 20px; }
        #output { margin-top: 20px; padding: 10px; background: #f0f0f0; }
    </style>
</head>
<body>
    <h1>Raw Message Demo</h1>

    <button id="ping">Ping</button>
    <button id="echo">Echo "Hello"</button>

    <div id="output">Waiting for response...</div>

    <script type="module">
        import { System, Events } from '@wailsio/runtime'

        const output = document.getElementById('output')

        function send(type, data) {
            System.invoke(JSON.stringify({ type, data }))
        }

        document.getElementById('ping').onclick = () => send('ping')
        document.getElementById('echo').onclick = () => send('echo', 'Hello')

        Events.On('pong', (e) => {
            output.textContent = `Pong from ${e.data.window} at ${e.data.time}`
        })

        Events.On('echo', (e) => {
            output.textContent = `Echo: ${e.data}`
        })

        Events.On('error', (e) => {
            output.textContent = `Error: ${e.data.error}`
        })
    </script>
</body>
</html>
```

## 모범 사례

### 권장 사항

- 성능이 극히 중요한 경로에서 원시 메시지를 사용하세요
- 핸들러에서 적절한 오류 처리를 구현하세요
- 응답을 프론트엔드로 보내기 위해 이벤트를 사용하세요
- 구조화된 데이터에 JSON을 고려하세요
- 차단(blocking)을 피하기 위해 메시지 처리를 빠르게 유지하세요

### 비권장 사항

- 서비스 바인딩으로도 충분할 때 원시 메시지를 사용하지 마세요
- 수신 메시지를 검증하는 것을 잊지 마세요
- 핸들러에서 장시간 실행되는 작업으로 차단하지 마세요 (고루틴 사용)
- 응답이 특정 창을 대상으로 해야 할 때 window 매개변수를 무시하지 마세요

## 다중 창 고려 사항

`window` 매개변수는 메시지를 보낸 창을 식별하므로 다음을 수행할 수 있습니다:

- 올바른 창에 응답 보내기
- 창별 동작 구현
- 디버깅을 위한 메시지 출처 추적

```go
RawMessageHandler: func(window application.Window, message string, originInfo *application.OriginInfo) {
    // 보낸 창에만 응답
    window.EmitEvent("response", result)

    // 또는 모든 창에 브로드캐스트
    app.Event.Emit("broadcast", result)
}
```
