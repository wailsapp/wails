이 가이드는 Wails 바인딩 시스템이 내부적으로 어떻게 작동하는지 설명하며, 자동 코드 생성의 메커니즘을 이해하려는 개발자에게 통찰력을 제공합니다.

## 아키텍처 개요

Wails 바인딩 시스템은 세 가지 주요 구성 요소로 구성됩니다:

1. **수집(Collection)**: Go 코드를 분석하여 서비스, 모델 및 기타 선언에 대한 정보를 추출합니다.
2. **구성(Configuration)**: 바인딩 생성 과정의 설정과 옵션을 관리합니다.
3. **렌더링(Rendering)**: 수집된 정보를 기반으로 JavaScript/TypeScript 코드를 생성합니다.

<FileTree>
- internal/generator/
  - collect/     # 패키지 분석 및 정보 추출
  - config/      # 구성 구조 및 인터페이스
  - render/      # JS/TS용 코드 생성
</FileTree>

## 수집 과정

수집 과정은 Go 패키지를 분석하고 서비스, 모델 및 기타 선언에 대한 정보를 추출하는 역할을 합니다. 이는 `collect` 패키지에 의해 처리됩니다.

### 주요 구성 요소

- **Collector**: 패키지 정보를 관리하고 수집된 데이터를 캐싱합니다.
- **Package**: 분석 중인 Go 패키지를 나타내며 수집된 서비스, 모델 및 지시문을 저장합니다.
- **Service**: 서비스 유형과 해당 메서드에 대한 정보를 수집합니다.
- **Model**: 필드, 값 및 타입 매개변수를 포함하여 모델 유형에 대한 상세 정보를 수집합니다.
- **Directive**: Go 소스 코드 내의 `//wails:` 지시문을 구문 분석하고 해석합니다.

### 수집 흐름

1. Collector가 프로젝트에서 지정된 Go 패키지를 스캔합니다.
2. 프론트엔드에 노출될 메서드가 있는 구조체(service types)를 식별합니다.
3. 각 서비스에 대해 해당 메서드에 대한 정보를 수집합니다.
4. 서비스 메서드의 매개변수 또는 반환 값으로 사용되는 구조체(model types)를 식별합니다.
5. 각 모델에 대해 해당 필드와 타입 매개변수에 대한 정보를 수집합니다.
6. 코드에서 발견된 모든 `//wails:` 지시문을 처리합니다.

## 렌더링 과정

렌더링 과정은 수집된 정보를 기반으로 JavaScript/TypeScript 코드를 생성하는 역할을 합니다. 이는 `render` 패키지에 의해 처리됩니다.

### 주요 구성 요소

- **Renderer**: 서비스, 모델 및 인덱스 파일의 렌더링을 조정합니다.
- **Module**: 생성된 단일 JavaScript/TypeScript 모듈을 나타냅니다.
- **Templates**: 코드 생성에 사용되는 텍스트 템플릿입니다.

### 렌더링 흐름

1. 각 서비스에 대해 렌더러는 서비스 메서드를 반영하는 함수가 포함된 JavaScript/TypeScript 파일을 생성합니다.
2. 각 모델에 대해 렌더러는 모델 구조체를 반영하는 JavaScript/TypeScript 클래스를 생성합니다.
3. 렌더러는 모든 서비스와 모델을 다시 내보내는(index) 파일들을 생성합니다.
4. 렌더러는 `//wails:inject` 지시문에 의해 지정된 사용자 지정 코드 주입을 적용합니다.

## 타입 매핑

바인딩 시스템의 가장 중요한 측면 중 하나는 Go 타입을 JavaScript/TypeScript 타입으로 어떻게 매핑하느냐입니다. 다음은 매핑 요약입니다:

| Go 타입 | JavaScript 타입 | TypeScript 타입 |
|---------|-----|-----|
| `bool` | `boolean` | `boolean` |
| `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64` | `number` | `number` |
| `string` | `string` | `string` |
| `[]byte` | `Uint8Array` | `Uint8Array` |
| `[]T` | `Array<T>` | `T[]` |
| `map[K]V` | `Object` | `Record<K, V>` |
| `struct` | `Object` | 사용자 지정 클래스 |
| `interface{}` | `any` | `any` |
| `*T` | `T \| null` | `T \| null` |
| `func` | 지원되지 않음 | 지원되지 않음 |
| `chan` | 지원되지 않음 | 지원되지 않음 |

## 지시문 시스템

바인딩 시스템은 생성된 코드를 사용자 지정하는 데 사용할 수 있는 여러 지시문을 지원합니다. 이러한 지시문은 Go 코드에 주석으로 추가됩니다.

### 사용 가능한 지시문

- `//wails:inject`: 생성된 바인딩에 사용자 지정 JavaScript/TypeScript 코드를 주입합니다.
- `//wails:include`: 생성된 바인딩과 함께 추가 파일을 포함합니다.
- `//wails:internal`: 유형 또는 메서드를 내부용으로 표시하여 프론트엔드에 내보내지 않도록 합니다.
- `//wails:ignore`: 바인딩 생성 동안 메서드를 완전히 무시합니다.
- `//wails:id`: 기본 해시 기반 ID를 재정의하는 메서드에 대한 사용자 지정 ID를 지정합니다.

### 지시문 처리

1. 수집 단계 동안 Collector가 Go 코드 내의 지시문을 식별하고 구문 분석합니다.
2. 지시문은 해당 선언(서비스, 메서드, 모델 등)과 함께 저장됩니다.
3. 렌더링 단계 동안 Renderer는 생성된 코드를 사용자 지정하기 위해 지시문을 적용합니다.

## 고급 기능

### 조건부 코드 생성

바인딩 시스템은 `include` 및 `inject` 지시문에 대한 두 문자 조건 접두사를 사용하여 조건부 코드 생성을 지원합니다:

```
<언어><스타일>:<내용>
```

여기서:
- `<언어>`는 다음 중 하나일 수 있습니다:
  - `*` - JavaScript와 TypeScript 모두
  - `j` - JavaScript 전용
  - `t` - TypeScript 전용

- `<스타일>`은 다음 중 하나일 수 있습니다:
  - `*` - 클래스와 인터페이스 모두
  - `c` - 클래스 전용
  - `i` - 인터페이스 전용

예를 들어:
```go
//wails:inject j*:console.log("JavaScript only");
//wails:inject t*:console.log("TypeScript only");
```

### 사용자 지정 메서드 ID

기본적으로 메서드는 해시 기반 ID로 식별됩니다. 그러나 `//wails:id` 지시문을 사용하여 사용자 지정 ID를 지정할 수 있습니다:

```go
//wails:id 42
func (s *Service) CustomIDMethod() {}
```

이는 코드 리팩토링 시 호환성을 유지하는 데 유용할 수 있습니다.

## 성능 고려 사항

바인딩 생성기는 효율적으로 설계되었지만 몇 가지 사항을 염두에 두어야 합니다:

1. 첫 번째 실행은 스캔할 패키지의 캐시를 구축하므로 느릴 수 있습니다.
2. 이후 실행은 캐시된 정보를 사용하므로 더 빠릅니다.
3. 생성기는 프로젝트의 모든 패키지를 처리하므로 대규모 프로젝트의 경우 시간이 오래 걸릴 수 있습니다.
4. 생성 전에 출력 디렉토리를 정리하려면 `-clean` 플래그를 사용할 수 있습니다.

## 디버깅

바인딩 생성 중 문제가 발생하면 `-v` 플래그를 사용하여 디버그 출력을 활성화할 수 있습니다:

```bash
wails3 generate bindings -v
```

이는 수집 및 렌더링 과정에 대한 상세 정보를 제공하여 문제의 원인을 식별하는 데 도움이 됩니다.
