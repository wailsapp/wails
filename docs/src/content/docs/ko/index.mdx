## 빠른 시작

<Tabs syncKey="os">
  <TabItem label="macOS" icon="apple">
```bash
# Wails 설치
go install github.com/wailsapp/wails/v3/cmd/wails3@latest

# 애플리케이션 생성
wails3 init -n myapp -t vanilla

# 핫 리로드와 함께 실행
cd myapp && wails3 dev
```
  </TabItem>
  
  <TabItem label="Windows" icon="seti:windows">

## 왜 Wails인가?

<CardGrid>
  <Card title="사용자가 체감하는 성능" icon="rocket">
    - ~15MB 바이너리 vs Electron의 150MB
    - ~10MB 기본 메모리 vs 100MB+
    - &lt;0.5초 시작 시간 vs 2-3초
    - OS WebView를 사용한 네이티브 렌더링
    - 번들된 브라우저 오버헤드 없음
  </Card>
  
  <Card title="개발자 경험" icon="setting">
    - 모든 플랫폼을 위한 단일 Go 코드베이스
    - 모든 웹 프레임워크 지원 - React, Vue, Svelte
    - 개발 중 핫 리로드
    - Go에서 Javascript 호출을 쉽게 할 수 있는 자동 생성 바인딩
    - 인메모리 IPC. 네트워크 포트 불필요
  </Card>
  
  <Card title="프로덕션 준비 완료" icon="approve-check">
    - 수명 주기가 있는 여러 창 지원
    - 네이티브 메뉴 및 시스템 트레이
    - 플랫폼 네이티브 파일 다이얼로그
    - 시스템 통합 및 단축키
    - 코드 서명 및 패키징 도구
  </Card>
  
  <Card title="크로스 플랫폼 네이티브" icon="laptop">
    - Windows, macOS, Linux를 위한 단일 코드베이스
    - 필요시 플랫폼별 기능 지원
    - 사용자 경험에 대한 타협 없음
    - 단일 빌드로 모든 플랫폼에 배포
    - 모바일 지원 곧 출시...
  </Card>
</CardGrid>
