이 가이드에서는 첫 번째 Wails v3 애플리케이션을 생성하는 방법을 보여줍니다. 프로젝트 설정, 빌드 및 개발 워크플로우를 다룹니다.

<br/>
<br/>

<Steps>

1.  ## 새 프로젝트 생성

    터미널을 열고 다음 명령을 실행하여 새 Wails 프로젝트를 생성합니다:

    ```bash
    wails3 init -n myfirstapp
    ```

    이 명령은 필요한 모든 파일이 포함된 `myfirstapp`이라는 새 디렉토리를 생성합니다.

      <video src={wails_init} controls></video> 

2.  ## 프로젝트 구조 탐색

    `myfirstapp` 디렉토리로 이동합니다. 여러 파일과 폴더가 있습니다:

    <FileTree>

    :::note
    Wails v3는 기본 빌드 시스템으로 [Task](https://taskfile.dev/)를 사용하지만, `make` 또는 다른 대안 빌드 시스템을 사용하는 것을 막는 것은 없습니다.
    :::

3.  ## 애플리케이션 빌드

    애플리케이션을 빌드하려면 다음을 실행합니다:

    ```bash
    wails3 build
    ```

    이 명령은 애플리케이션의 디버그 버전을 컴파일하여 새 `bin` 디렉토리에 저장합니다.

    :::note
    `wails3 build`는 `wails3 task build`의 약어이며 `Taskfile.yml`의 `build` 작업을 실행합니다.
    :::

        <video src={wails_build} controls></video> 


    빌드가 완료되면 일반 애플리케이션처럼 실행할 수 있습니다:


          </TabItem>

      <TabItem label="Windows" icon="seti:windows">


          </TabItem>

      <TabItem label="Linux" icon="linux">


          </TabItem>

    </Tabs>

    애플리케이션의 시작점이 되는 간단한 UI가 표시됩니다. 디버그 버전이므로 콘솔 창에도 로그가 표시됩니다. 이는 디버깅 목적으로 유용합니다.

4.  ## 개발 모드

    애플리케이션을 개발 모드에서도 실행할 수 있습니다. 이 모드를 사용하면 프론트엔드 코드를 변경하고 전체 애플리케이션을 다시 빌드하지 않고도 실행 중인 애플리케이션에서 변경 사항을 확인할 수 있습니다.

    1. 새 터미널 창을 엽니다.
    2. `wails3 dev`를 실행합니다. 애플리케이션이 디버그 모드로 컴파일되고 실행됩니다.
    3. 선호하는 편집기에서 `frontend/index.html`을 엽니다.
    4. 코드를 편집하여 `Please enter your name below`를 `Please enter your name below!!!`로 변경합니다.
    5. 파일을 저장합니다.

    이 변경 사항은 애플리케이션에 즉시 반영됩니다.

    백엔드 코드의 변경은 다시 빌드를 트리거합니다:

    1. `greetservice.go`를 엽니다.
    2. `return "Hello " + name + "!"`가 있는 줄을 `return "Hello there " + name + "!"`로 변경합니다.
    3. 파일을 저장합니다.

    애플리케이션은 몇 초 이내에 업데이트됩니다.

        <video src={wails_dev} controls></video>

5.  ## 애플리케이션 패키징

        애플리케이션이 배포 준비가 되면 플랫폼별 패키지를 생성할 수 있습니다:

          <Tabs syncKey="platform">

    <TabItem label="Mac" icon="apple">

              `.app` 번들을 생성하려면:

              ```bash
              wails3 package
              ```

              이는 프로덕션 빌드를 생성하고 `bin` 디렉토리에 `.app` 번들로 패키징합니다.

            </TabItem>
            <TabItem label="Windows" icon="seti:windows">

              NSIS 설치 프로그램을 생성하려면:

              ```bash
              wails3 package
              ```

              이는 프로덕션 빌드를 생성하고 `bin` 디렉토리에 NSIS 설치 프로그램으로 패키징합니다.

            </TabItem>
            <TabItem label="Linux" icon="linux">

              Wails는 Linux 배포를 위해 여러 패키지 형식을 지원합니다:

              ```bash
              # 모든 패키지 유형 생성 (AppImage, deb, rpm, Arch Linux)
              wails3 package

              # 또는 특정 패키지 유형 생성
              wails3 task linux:create:appimage  # AppImage 형식
              wails3 task linux:create:deb       # Debian 패키지
              wails3 task linux:create:rpm       # Red Hat 패키지
              wails3 task linux:create:aur       # Arch Linux 패키지
              ```

            </TabItem>

          </Tabs>

    패키징 옵션 및 구성에 대한 자세한 정보는 [패키징 가이드](/guides/packaging)를 참조하세요.

6.  ## 버전 제어 및 모듈 이름 설정

    프로젝트는 자리 표시자 모듈 이름 `changeme`으로 생성됩니다. 저장소 URL과 일치하도록 업데이트하는 것이 좋습니다:

    1. GitHub(또는 선호하는 Git 호스트)에 새 저장소를 생성합니다.
    2. 프로젝트 디렉토리에서 git을 초기화합니다:
       ```bash
       git init
       git add .
       git commit -m "Initial commit"
       ```
    3. 원격 저장소를 설정합니다 (저장소 URL로 대체):
       ```bash
       git remote add origin https://github.com/username/myfirstapp.git
       ```
    4. `go.mod`의 모듈 이름을 저장소 URL과 일치하도록 업데이트합니다:
       ```bash
       go mod edit -module github.com/username/myfirstapp
       ```
    5. 코드를 푸시합니다:
       ```bash
       git push -u origin main
       ```

    이렇게 하면 Go 모듈 이름이 Go의 모듈 이름 규칙과 일치하게 되어 코드를 공유하기 쉬워집니다.:::tip[Pro Tip]
프로젝트 생성 시 `-git` 플래그를 사용하면 초기화 단계를 모두 자동화할 수 있습니다:
```bash
wails3 init -n myfirstapp -git github.com/username/myfirstapp
```
이 명령어는 다양한 Git URL 형식을 지원합니다:
- HTTPS: `https://github.com/username/project`
- SSH: `git@github.com:username/project` 또는 `ssh://git@github.com/username/project`
- Git 프로토콜: `git://github.com/username/project`
- 파일 시스템: `file:///path/to/project.git`
:::
