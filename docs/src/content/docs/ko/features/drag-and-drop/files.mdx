Wails를 사용하면 사용자가 운영 체제(파일 관리자, 데스크톱)에서 애플리케이션으로 파일을 끌어서 놓을 수 있습니다. 브라우저 내부에서만 작동하는 HTML5 드래그 앤 드롭과 달리, 이를 통해 디스크의 실제 파일 경로에 접근할 수 있습니다.

## 파일 드롭 활성화

파일 드롭은 기본적으로 비활성화되어 있습니다. 활성화하려면 창 옵션에서 `EnableFileDrop: true`를 설정하세요:

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:          "My App",
    Width:          800,
    Height:         600,
    EnableFileDrop: true,
})
```

`EnableFileDrop`이 `false`(기본값)인 경우, OS에서 끌어서 놓은 파일이 차단됩니다. 웹뷰에서 열리지 않으며 어떤 이벤트도 트리거되지 않습니다. 이는 사용자가 파일을 애플리케이션 위로 드래그할 때 실수로 탐색되는 것을 방지합니다.

## 드롭 영역 정의

드롭 영역은 Wails에 파일을 받을 요소를 알려줍니다. 드롭 영역 외부에 파일을 놓으면 무시됩니다.

어떤 요소에도 `data-file-drop-target` 속성을 추가하세요:

```html
<div id="upload" class="drop-zone" data-file-drop-target>
    Drop files here
</div>
```

여러 드롭 영역을 가질 수 있습니다. 요소의 `id`와 CSS 클래스는 Go 코드로 전달되므로, 파일이 놓이는 위치에 따라 다르게 처리할 수 있습니다.

## 드래그 호버 스타일링

파일을 드롭 영역 위로 드래그하면 Wails는 `file-drop-target-active` 클래스를 추가합니다. 이를 통해 사용자에게 드롭 가능한 위치를 시각적으로 알려줄 수 있습니다:

```css
.drop-zone {
    border: 2px dashed #ccc;
    padding: 40px;
    text-align: center;
    transition: all 0.2s ease;
}

.drop-zone.file-drop-target-active {
    border-color: #007bff;
    background-color: rgba(0, 123, 255, 0.1);
}
```

파일이 영역을 벗어나거나 놓이면 이 클래스는 자동으로 제거됩니다.

## 놓인 파일 감지

유효한 드롭 영역에 파일이 놓이면 Wails는 `WindowFilesDropped` 이벤트를 발생시킵니다. 이벤트 컨텍스트에는 놓인 모든 파일의 전체 파일 시스템 경로가 포함됩니다:

```go
import "github.com/wailsapp/wails/v3/pkg/events"

window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    
    for _, file := range files {
        fmt.Println("Dropped:", file)
    }
})
```

경로는 `/home/user/documents/report.pdf` 또는 `C:\Users\Name\Documents\report.pdf`와 같이 절대 경로입니다.

## 드롭 대상 정보 가져오기

여러 드롭 영역이 있는 경우 `DropTargetDetails()`를 사용하여 파일을 받은 영역을 확인할 수 있습니다:

```go
window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    details := event.Context().DropTargetDetails()
    
    fmt.Printf("Dropped on element: id=%s, classes=%v\n", 
        details.ElementID, details.ClassList)
    fmt.Printf("Position: x=%d, y=%d\n", details.X, details.Y)
})
```

이를 통해 파일을 다른 핸들러로 라우팅할 수 있습니다:

```go
switch details.ElementID {
case "images":
    handleImageUpload(files)
case "documents":
    handleDocumentUpload(files)
}
```

## 전체 예제

**Go:**

```go
window := app.Window.NewWithOptions(application.WebviewWindowOptions{
    Title:          "File Uploader",
    EnableFileDrop: true,
})

window.OnWindowEvent(events.Common.WindowFilesDropped, func(event *application.WindowEvent) {
    files := event.Context().DroppedFiles()
    details := event.Context().DropTargetDetails()
    
    // Send to frontend
    app.Event.Emit("files-dropped", map[string]any{
        "files":   files,
        "target":  details.ElementID,
    })
})
```

**HTML:**

```html
<div id="images" class="drop-zone" data-file-drop-target>
    Drop images here
</div>

<div id="documents" class="drop-zone" data-file-drop-target>
    Drop documents here
</div>

<style>
    .drop-zone {
        border: 2px dashed #ccc;
        border-radius: 8px;
        padding: 40px;
        text-align: center;
        margin: 20px;
        transition: all 0.2s ease;
    }
    
    .drop-zone.file-drop-target-active {
        border-color: #007bff;
        background-color: rgba(0, 123, 255, 0.1);
    }
</style>
```

## 전체 창 드롭

애플리케이션의 어디에든 파일을 드롭할 수 있도록 하려면 body 요소에 속성을 추가하세요:

```html
<body data-file-drop-target>
    <!-- Your app content -->
</body>
```

전체 창이 드롭 대상임을 나타내기 위해 CSS 오버레이를 사용할 수 있습니다:

```css
body.file-drop-target-active::after {
    content: "Drop files anywhere";
    position: fixed;
    inset: 0;
    display: flex;
    align-items: center;
    justify-content: center;
    font-size: 24px;
    color: #007bff;
    background: rgba(255, 255, 255, 0.9);
    pointer-events: none;
}
```

## HTML 드래그 앤 드롭과 결합

애플리케이션에서 외부 파일 드롭과 내부 HTML 드래그 앤 드롭을 모두 사용할 수 있습니다. `EnableFileDrop`이 `true`인 경우, Wails는 외부 파일 드래그를 가로채지만 내부 HTML5 드래그는 정상적으로 통과시킵니다.

HTML 드롭 영역 핸들러에서 이를 구분하려면 드래그에 파일이 포함되어 있는지 확인하세요:

```javascript
zone.addEventListener('dragenter', (e) => {
    // Skip external file drags - Wails handles these
    if (e.dataTransfer?.types.includes('Files')) {
        return;
    }
    // Handle internal HTML5 drags
    zone.classList.add('drag-over');
});

zone.addEventListener('drop', (e) => {
    // Skip external file drops - Wails handles these
    if (e.dataTransfer?.types.includes('Files')) {
        return;
    }
    e.preventDefault();
    zone.classList.remove('drag-over');
    // Handle internal drop
});
```

이를 통해 HTML 드롭 핸들러는 내부 드래그(예: 목록 항목 이동)에만 응답하고, Wails는 `WindowFilesDropped` 이벤트를 통해 외부 파일 드롭을 별도로 처리합니다.
