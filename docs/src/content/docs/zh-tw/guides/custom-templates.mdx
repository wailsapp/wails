本指南將引導您完成為 Wails v3 建立自訂範本的流程。

## 為什麼我要建立自訂範本？

Wails 內建了多個預先配置的範本，讓您能快速啟動並運行應用程式。但如果您需要更客製化的設定，您可以建立符合自身需求的範本。之後，您可以將其分享給 Wails 社群，供其他人使用。

### 1. 產生範本

要建立自訂範本，您可以使用 `wails generate template` 指令：

```bash
wails3 generate template -name mytemplate
```

這會在您目前的目錄中建立一個名為 "mytemplate" 的新目錄。

`wails3 generate template` 指令支援以下選項：

| Option         | Description                                       | Default           |
|----------------|---------------------------------------------------|-------------------|
| `-name`        | 您的範本名稱（必填）                              | -                 |
| `-frontend`    | 要包含的現有前端目錄路徑                          | -                 |
| `-author`      | 範本作者                                          | -                 |
| `-description` | 範本描述                                          | -                 |
| `-helpurl`     | 範本文檔的 URL                                    | -                 |
| `-dir`         | 產生範本的目錄                                    | 目前目錄          |
| `-version`     | 範本版本                                          | v0.0.1            |

例如，要建立包含所有選項的範本：

```bash
wails3 generate template \
  -name "My Custom Template" \
  -frontend ./my-existing-frontend \
  -author "Your Name" \
  -description "A template with my preferred setup" \
  -helpurl "https://github.com/yourusername/template-docs" \
  -dir ./templates \
  -version "v1.0.0"
```

  :::tip
  使用 `-frontend` 選項會將現有的網頁前端專案複製到範本中。
  :::

### 2. 設定範本中繼資料

如果您在產生範本時未指定範本配置，您可以更新範本目錄中的 `template.json` 檔案：

```json5
{
  "name": "Your Template Name",          // 範本的顯示名稱
  "shortname": "template-shortname",     // 用於引用您的範本
  "author": "Your Name",                 // 範本作者
  "description": "Template description", // 範本描述
  "helpurl": "https://your-docs.com",    // 文件 URL
  "version": "v0.0.1",                   // 範本版本
  "schema": 3                            // 必須保持為 3 以相容於 Wails v3
}
```
:::caution
為了與 Wails v3 相容，`schema` 欄位必須保持設定為 `3`。
:::

### 3. 設定建置任務

在 `build` 目錄中有 `Taskfile.yml`，您可以在其中定義範本的建置流程。
此檔案使用 [Task](https://taskfile.dev) 進行建置自動化。關鍵步驟如下：

```yaml
tasks:
  install:frontend:deps:
    summary: Install frontend dependencies
    dir: frontend
    sources:
      - package.json
      - package-lock.json
    generates:
      - node_modules/*
    preconditions:
      - sh: npm version
        msg: "Looks like npm isn't installed. Npm is part of the Node installer: https://nodejs.org/en/download/"
    cmds:
      - npm install

  build:frontend:
    summary: Build the frontend project
    dir: frontend
    sources:
      - "**/*"
    generates:
      - dist/*
    deps:
      - task: install:frontend:deps
      - task: generate:bindings
    cmds:
      - npm run build -q


```

現在您已放置前端檔案，請使用適當的指令更新 `common/Taskfile.yml`：
   ```yaml

          - package-lock.json
        generates:

          - "**/*"
        generates:
          - dist/*
        deps:
          - task: install:frontend:deps
          - task: generate:bindings
        cmds:
          - npm run build -q
   ```

  :::note
  在此範例中，預設的 Tasks 不需要更新，因為它們使用標準的 `npm install` 和 `npm run build` 指令。
  :::

### 5. 設定 Go 應用程式

範本目錄中的預設檔案足以讓使用者開始使用。但是，您可能希望提供一些額外功能來展示範本的能力。最佳方式是將 `main.go.tmpl` 重新命名為 `main.go` 並像其他 Go 檔案一樣編輯它。完成後，在提交變更之前，請務必將其重新命名回 `main.go.tmpl`。如果您不在意擁有範本化的 `main.go` 檔案（預設範本會將專案名稱注入應用程式的 `Name` 欄位），您可以跳過此步驟。--------------------|--------------------------------|---------------------------------|
| `Name`             | 專案名稱                       | `"MyApp"`                       |
| `BinaryName`       | 生成的二進位檔名稱             | `"myapp"`                       |
| `ProductName`      | 產品名稱                       | `"My Application"`              |
| `ProductDescription` | 產品描述                     | `"An awesome application"`      |
| `ProductVersion`   | 產品版本                       | `"1.0.0"`                       |
| `ProductCompany`   | 公司名稱                       | `"My Company Ltd"`              |
| `ProductCopyright` | 版權資訊                       | `"Copyright 2024 My Company Ltd"` |
| `ProductComments`  | 額外產品註解                   | `"Built with Wails"`            |
| `ProductIdentifier`| 唯一產品識別碼                 | `"com.mycompany.myapp"`         |
| `Typescript`       | 是否使用 TypeScript            | `true` 或 `false`               |
| `WailsVersion`     | 使用的 Wails 版本              | `"3.0.0"`                       |

您可以在範本檔案中使用 Go 的模板語法來使用這些變數：

```go
// main.go.tmpl
package main

import (
    "github.com/wailsapp/wails/v3/pkg/application"
)

func main() {
    app := application.New(application.Options{
        Name:        "{{.ProductName}}",
        Description: "{{.ProductDescription}}",
    })
    // ...
}
```

    :::tip
    模板應用可以套用到範本中的任何檔案，包括 html 檔案，只要檔名中包含 `.tmpl` 即可。
    :::

### 6. 測試您的範本

要測試您的範本：

1. 使用您的範本生成專案：`wails3 init -n testproject -t path/to/your/template`
2. 執行 `wails3 build` 以生成生產環境版本，並確認 `bin` 中的二進位檔能正常運作
3. 執行 `wails3 dev` 以啟動開發伺服器。
4. 測試前端程式碼的變更是否反映在應用程式中。
5. 測試 Go 程式碼的變更是否能重新編譯並重新啟動應用程式

### 7. 分享您的範本

一旦您的範本準備就緒，您可以透過在 GitHub 上託管來與社群分享。方法如下：

1. 為您的範本建立新的 GitHub 儲存庫
2. 將您的範本程式碼推送到該儲存庫
3. 使用語義化版本控制（例如 v1.0.0）標記您的版本

使用者隨後可以使用 HTTPS URL 直接從 GitHub 使用您的範本：

```bash
wails3 init -n myapp -t https://github.com/yourusername/your-template
```

您也可以使用 URL 格式指定特定版本：

```bash
# 使用特定版本標籤
wails3 init -n myapp -t https://github.com/yourusername/your-template/releases/tag/v1.0.0

# 使用特定分支
wails3 init -n myapp -t https://github.com/yourusername/your-template/tree/main
```

在分享之前測試您的範本：

1. 將您的變更推送到 GitHub
2. 使用 HTTPS URL 建立新的測試專案：
   ```bash
   wails3 init -n testapp -t https://github.com/yourusername/your-template
   ```
3. 驗證所有檔案是否正確生成
4. 按照測試章節所述測試建置和開發工作流程

    :::note
    如果您希望其他人使用您的範本，請確保您的儲存庫是公開的。
    :::

如需更多資訊，請造訪 [Wails 文件](https://wails.io)


## 最佳實踐

讓我們談論一些關鍵實踐，這些實踐將幫助您的範本更具用處且易於維護。以下是需要關注的主要領域：

1. **讓您的範本易於理解**
   - 撰寫清晰、有幫助的 README.md，讓使用者能快速上手
   - 在設定檔中加入註解，解釋您選擇背後的「原因」
   - 展示常見自訂的範例 - 使用者很喜歡看到實際應用案例！

2. **保持依賴項的穩定**
   - 密切關注前端套件更新
   - 在 package.json 中鎖定特定版本，以避免意外情況
   - 提前告知使用者他們需要安裝什麼

3. **愛護您的範本**
   - 定期更新以保持新鮮感
   - 在分享之前進行徹底測試
   - 與 Wails 社群分享 - 我們很樂意看到您創造的成果！
