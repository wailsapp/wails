Wails 中的**服務**是一個包含您希望在前端使用的業務邏輯的 Go 結構體。服務透過將相關功能分組在一起，幫助您保持程式碼的組織性。

您可以將服務視為一組您的 JavaScript 程式碼可以呼叫的方法。在產生繫結後，服務上的每個公開方法都可以從前端呼叫。

在本教學中，我們將建立一個 QR Code 產生器服務來示範這些概念。最後，您將了解如何建立服務、管理依賴項，並將您的 Go 程式碼與前端連接。

<br/>

<Steps>

1. ## 建立 QR 服務檔案

   在您的應用程式目錄中建立一個名為 `qrservice.go` 的新檔案：

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **這裡發生了什麼：**

   - `QRService` 是一個空的結構體，將儲存我們的 QR Code 產生方法
   - `NewQRService()` 是一個建構函式，用於建立我們服務的新實例
   - `Generate()` 是一個接受文字和大小的方法，然後以 PNG 位元組陣列的形式回傳 QR Code
   - 該方法回傳 `([]byte, error)`，遵循 Go 將錯誤作為最後一個值回傳的慣例
   - 我們使用 `github.com/skip2/go-qrcode` 套件來處理實際的 QR Code 產生

    <br/>

2. ## 註冊服務

   僅建立服務是不夠的——我們需要將其**註冊**到 Wails 應用程式中，以便它知道服務的存在並能為其產生繫結。

   註冊發生在您建立應用程式時的 `main.go` 中。您將服務實例傳遞給 `Services` 選項：

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **這裡發生了什麼：**

   - `application.NewService()` 包裝您的服務，以便 Wails 可以管理它
   - 我們呼叫 `NewQRService()` 來建立我們服務的實例
   - 該服務被新增到應用程式選項中的 `Services` 切片
   - Wails 現在將掃描此服務以取得公開方法，使其可供前端使用

3. ## 安裝依賴項

   我們在程式碼中引用了 `github.com/skip2/go-qrcode` 套件，但我們還沒有實際下載它。Go 需要知道這個依賴項並將其下載到您的專案中。

   從您的專案目錄在終端機中執行此命令：

   ```bash
   go mod tidy
   ```

   **這裡發生了什麼：**

   - `go mod tidy` 掃描您的 Go 檔案中的 import 陳述式
   - 它會下載任何遺漏的套件（例如 `go-qrcode`）並將其新增到 `go.mod` 中
   - 它還會移除任何不再使用的依賴項
   - 這確保您的專案擁有成功編譯所需的所有程式碼

   您應該會看到輸出顯示 QR Code 套件已下載並新增到您的專案中。

4. ## 產生繫結

   要從前端呼叫這些方法，我們需要產生繫結。
   您可以在專案根目錄中執行 `wails generate bindings` 來完成此操作。

   :::note
   當您在專案中第一次執行此操作時，繫結產生器會對您的程式碼和依賴項進行徹底分析。這有時可能會比預期花費更長的時間，但後續執行會快得多。
   :::

   執行後，您應該會在終端機中看到類似以下的內容：

   ```bash
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

        當您使用 `wails3 build` 建構應用程式時，它會自動為您產生繫結並保持其為最新狀態。

   ```js title="bindings/changeme/qrservice.js"
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
   ```

    我們可以看到繫結是為 `Generate` 方法產生的。參數名稱已保留，
    註解也保留了。JSDoc 也已為該方法產生，以向您的 IDE 提供型別資訊。

    :::note
    不需要完全理解產生的繫結，但了解它們如何運作很重要。
    :::

    繫結提供：
    - 與您的 Go 方法等效的函式
    - Go 和 JavaScript 型別之間的自動轉換
    - 基於 Promise 的非同步操作
    - 作為 JSDoc 註解的型別資訊

    :::tip[Typescript]
    繫結產生器也支援產生 Typescript 繫結。您可以透過執行 `wails3 generate bindings -ts` 來完成此操作。
    :::

    產生的服務由 `index.js` 檔案重新匯出：

6. ## 在前端中使用繫結

    現在我們可以從 JavaScript 呼叫我們的 Go 服務了！產生的繫結讓這變得簡單且具備型別安全。

    更新 `frontend/src/main.js` 以使用新的繫結：

   ```js title="frontend/src/main.js"
    import { QRService } from './bindings/changeme';

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        try {
            // Generate QR code as base64
            const qrCodeBase64 = await QRService.Generate(text, 256);

            // Display the QR code
            const qrDiv = document.getElementById('qrcode');
            qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

        } catch (err) {
            console.error('Failed to generate QR code:', err);
            alert('Failed to generate QR code: ' + err);
        }
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        button.addEventListener('click', generateQR);
    }
   ```

   **這裡發生了什麼事：**

   - 我們從產生的繫結匯入 `QRService`
   - `QRService.Generate()` 呼叫我們的 Go 方法 — 它回傳一個 Promise，因此我們使用 `await`
   - Go 方法回傳 `[]byte`，Wails 會自動將其轉換為 JavaScript 的 base64 字串
   - 我們使用 base64 字串建立資料 URL 以顯示 PNG 影像
   - `try/catch` 區塊處理來自 Go 端的任何錯誤（例如無效的輸入）
   - 如果我們的 Go 程式碼回傳錯誤，Promise 會拒絕（reject），我們在此處捕獲它

7. ## 替代方法：HTTP Handler

    到目前為止，我們已經涵蓋了以下領域：
    - 建立新的 Service
    - 產生繫結
    - 在前端程式碼中使用繫結

    **為什麼要使用 HTTP handler？**

    方法繫結（Method bindings）對於資料操作非常有效，但對於提供檔案、影像或其他媒體，還有另一種替代方法。與其將所有內容轉換為 base64 並透過繫結傳送，你可以讓你的服務像一個迷你網頁伺服器一樣運作。

    這在以下情況很有用：
    - 你正在提供影像、影片或大型檔案
    - 你想使用標準的 HTML `<img>` 或 `<video>` 標籤並搭配 `src` 屬性
    - 你需要對資源的直接 URL 存取

    如果你的服務實作了 Go 標準的 `ServeHTTP(w http.ResponseWriter, r *http.Request)` 方法，Wails 可以將其作為 HTTP 端點提供。讓我們擴展我們的 QR Code 服務以支援此功能：

    ```go title="qrservice.go" ins={4-5,37-65}
    package main

    import (
        "net/http"
        "strconv"

        "github.com/skip2/go-qrcode"
    )

    // QRService handles QR code generation
    type QRService struct {
        // We can add state here if needed
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **這裡發生了什麼事：**

    - `ServeHTTP` 是 Go 處理 HTTP 請求的標準介面
    - 我們從 URL 解析查詢參數（`?text=hello&size=256`）
    - 我們呼叫現有的 `Generate()` 方法來建立 QR Code
    - 我們將內容類型設定為 `image/png`，以便瀏覽器知道這是影像
    - 我們將原始 PNG 位元組直接寫入回應 — 不需要 base64！

    現在更新 `main.go` 以指定 QR Code 服務可存取的路由：

8. ## 支援動態設定

**硬編碼路徑的問題：**

在上面的範例中，我們在 JavaScript 程式碼中使用了硬編碼的路徑 `/qrservice`。
這會在您的 Go 設定與前端程式碼之間建立緊密的耦合。

如果您編輯 `main.go` 並更改 `Route` 選項而未更新 `main.js`，
應用程式將會崩潰：

```go title="main.go" ins={3}
        // ...
            application.NewService(NewQRService(), application.ServiceOptions{
                Route: "/services/qr",
            }),
        // ...
```

硬編碼路徑適用於簡單的應用程式，但它們會讓程式碼變得脆弱且難以維護。

**解決方案：動態設定**

方法繫結與 HTTP 處理常式可以協同工作！我們可以使用繫結來告訴前端使用哪個路徑，從而使設定變得動態並消除硬編碼路徑。

運作方式如下：
1. `ServiceStartup` 生命週期方法會在應用程式啟動時執行
2. 我們儲存從選項中配置的路由
3. 我們新增一個 `URL()` 方法，前端可以呼叫它來取得正確的路由
4. 現在前端會向 Go 服務查詢其路由，而不是猜測

首先，實作 `ServiceStartup` 介面並新增一個新的 `URL` 方法：

```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
package main

import (
	"context"
	"net/http"
	"net/url"
	"strconv"

	"github.com/skip2/go-qrcode"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// QRService handles QR code generation
type QRService struct {
	route string
}

// NewQRService creates a new QR service
func NewQRService() *QRService {
	return &QRService{}
}

// ServiceStartup runs at application startup.
func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	s.route = options.Route
	return nil
}

// Generate creates a QR code from the given text
func (s *QRService) Generate(text string, size int) ([]byte, error) {
	// Generate the QR code
	qr, err := qrcode.New(text, qrcode.Medium)
	if err != nil {
		return nil, err
	}

	// Convert to PNG
	png, err := qr.PNG(size)
	if err != nil {
		return nil, err
	}

	return png, nil
}

// URL returns an URL that may be used to fetch
// a QR code with the given text and size.
// It returns an error if the HTTP handler is not available.
func (s *QRService) URL(text string, size int) (string, error) {
	if s.route == "" {
		return "", errors.New("http handler unavailable")
	}

	return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
}

func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// Extract the text parameter from the request
	text := r.URL.Query().Get("text")
	if text == "" {
		http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
		return
	}
	// Extract Size parameter from the request
	sizeText := r.URL.Query().Get("size")
	if sizeText == "" {
		sizeText = "256"
	}
	size, err := strconv.Atoi(sizeText)
	if err != nil {
		http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
		return
	}

	// Generate the QR code
	qrCodeData, err := s.Generate(text, size)
	if err != nil {
		http.Error(w, err.Error(), http.StatusInternalServerError)
		return
	}

	// Write the QR code data to the response
	w.Header().Set("Content-Type", "image/png")
	w.Write(qrCodeData)
}
```

- `url.QueryEscape()` 安全地編碼文字以用於 URL 中
- 此方法將透過繫結對前端可用
