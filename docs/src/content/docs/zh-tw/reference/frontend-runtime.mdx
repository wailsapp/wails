Wails 前端執行階段是 Wails 應用程式的標準函式庫。它提供了許多可在應用程式中使用的功能，包括：

- 視窗管理
- 對話方塊
- 瀏覽器整合
- 剪貼簿
- 選單
- 系統資訊
- 事件
- 右鍵選單
- 螢幕
- WML（Wails Markup Language）

執行階段是 Go 與前端之間整合所必需的。有兩種方式可以整合執行階段：

- 使用 `@wailsio/runtime` 套件
- 使用預先建構的套件

## 使用 npm 套件

`@wailsio/runtime` 套件是一個 JavaScript 套件，提供從前端存取 Wails 執行階段的功能。所有標準範本都使用此套件，也是將執行階段整合至應用程式的推薦方式。
使用 `@wailsio/runtime` 套件時，您只會包含您所使用的執行階段部分。

此套件可在 npm 上取得，並可透過以下指令安裝：

```shell
npm install --save @wailsio/runtime
```

## 使用預先建構的套件

某些專案不會使用 JavaScript 捆綁器，而可能偏好使用預先建構的執行階段版本。此版本可透過以下指令在本機產生：

```shell
wails3 generate runtime
```

此指令會在目前目錄中輸出 `runtime.js`（以及 `runtime.debug.js`）檔案。此檔案是一個 ES 模組，可像 npm 套件一樣被您的應用程式腳本匯入，但 API 也會匯出至全域 window 物件，因此對於較簡單的應用程式，您可以如下使用：

```html
<html>
    <head>
        <script type="module" src="./runtime.js"></script>
        <script>
            window.onload = function () {
                wails.Window.SetTitle("A new window title");
            }
        </script>
    </head>
    <!--- ... -->
</html>
```

:::caution
在載入執行階段的 `<script>` 標籤上包含 `type="module"` 屬性非常重要，
並且在呼叫 API 之前必須等待頁面完全載入，
因為具有 `type="module"` 屬性的腳本會非同步執行。
:::

## 初始化

除了 API 函式之外，執行階段還提供對右鍵選單和視窗拖曳支援。
這些功能只有在執行階段初始化之後才會如預期運作。
即使您不使用 API，也請務必在前端程式碼中的某個位置包含副作用匯入語句：

```javascript
import "@wailsio/runtime";
```

您的捆綁器應能偵測到副作用的存在，並將所有必要的初始化程式碼包含在建置中。

:::note
如果您偏好使用預先建構的套件，只需像上面那樣新增 script 標籤即可。
:::

## 用於類型化事件的 Vite 外掛程式

執行階段包含一個 Vite 外掛程式，可在開發期間為類型化事件啟用 HMR（熱模組替換）支援。

### 設定

將外掛程式新增至您的 `vite.config.ts`：

```typescript
import { defineConfig } from 'vite'
import wails from '@wailsio/runtime/plugins/vite'

export default defineConfig({
  plugins: [wails()],
})
```

### 優點

- **自動重新載入**：當您執行 `wails3 generate bindings` 時，事件繫結會自動重新產生並重新載入
- **開發模式**：與 `wails3 dev` 無縫運作，實現即時更新
- **類型安全**：完整的 TypeScript 支援，包含自動完成和類型檢查

### 與事件註冊搭配使用

在 Go 中註冊您的事件：

```go
type UserData struct {
    ID   string
    Name string
}

func init() {
    application.RegisterEvent[UserData]("user-updated")
}
```

產生繫結：

```bash
wails3 generate bindings

# 或，包含 TypeScript 定義
wails3 generate bindings -ts

# 更多選項，請參閱：
wails3 generate bindings -help
```

在前端使用類型化事件：

```typescript
import { Events } from '@wailsio/runtime'
import { UserUpdated } from './bindings/events'

// 類型安全的事件，具有自動完成功能
Events.Emit(UserUpdated({
    ID: "123",
    Name: "John Doe"
}))
```

## API 參考

執行階段組織成多個模組，每個模組提供特定的功能。僅匯入您需要的部分：

```javascript
import { Events, Window, Clipboard } from '@wailsio/runtime'
```

### Events

用於 Go 與 JavaScript 之間通訊的事件系統。

#### On()

註冊事件的回呼函式。

```typescript
function On(eventName: string, callback: (event: WailsEvent) => void): () => void
function On<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**傳回：** 取消訂閱函式

**範例：**

```javascript
import { Events } from '@wailsio/runtime'

// 基本事件監聽
const unsubscribe = Events.On('user-logged-in', (event) => {
    console.log('User:', event.data.username)
})

// 使用類型化事件（TypeScript）
import { UserLogin } from './bindings/events'

Events.On(UserLogin, (event) => {
    // event.data 的類型為 UserLoginData
    console.log('User:', event.data.username)
})

// 稍後：取消訂閱 unsubscribe()
```

#### Once()

註冊僅執行一次的回呼函式。

```typescript
function Once(eventName: string, callback: (event: WailsEvent) => void): () => void
function Once<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**範例：**

```javascript
import { Events } from '@wailsio/runtime'

Events.Once('app-ready', () => {
    console.log('App initialized')
})
```

#### Emit()

向 Go 後端或其他視窗發出事件。

```typescript
function Emit(name: string, data?: any): Promise<boolean>
function Emit<T>(event: Event<T>): Promise<boolean>
```

**傳回：** Promise，若事件被取消則解析為 `true`，否則為 `false`

**範例：**

```javascript
import { Events } from '@wailsio/runtime'

// 基本事件發出
const wasCancelled = await Events.Emit('button-clicked', { buttonId: 'submit' })

// 使用類型化事件（TypeScript）
import { UserLogin } from './bindings/events'

const cancelled = await Events.Emit(UserLogin({
    UserID: "123",
    Username: "john_doe",
    LoginTime: new Date().toISOString()
}))

if (cancelled) {
    console.log('Login was cancelled by a hook')
}
```

:::note
傳回值表示事件是否被鉤子取消。大多數事件無法被取消，將始終返回 `false`。
:::

#### Off()

移除事件監聽器。

```typescript
function Off(...eventNames: string[]): void
```

**範例：**

```javascript
import { Events } from '@wailsio/runtime'

Events.Off('user-logged-in', 'user-logged-out')
```

#### OffAll()

移除所有事件監聽器。

```typescript
function OffAll(): void
```

### 系統

用於與後端直接通訊的低階系統方法。

#### invoke()

直接將原始訊息傳送至後端。這會繞過標準繫結系統，並由應用程式選項中的 `RawMessageHandler` 處理。

```typescript
function invoke(message: any): void
```

**範例：**

```javascript
import { System } from '@wailsio/runtime'

// 傳送原始訊息至後端
System.invoke('my-custom-message')

// 以 JSON 格式傳送結構化資料
System.invoke(JSON.stringify({ action: 'update', value: 42 }))
```

:::caution
這是一個無返回值的 fire-and-forget 函式。請使用事件來接收後端的回應。
:::

更多詳細資訊，請參閱 [原始訊息指南](/zh-tw/guides/raw-messages)。

## 完整範例

```javascript
import { Events, Window, Clipboard, Dialogs, Screens } from '@wailsio/runtime'

// 監聽來自 Go 的事件
Events.On('data-updated', (event) => {
    console.log('Data:', event.data)
    updateUI(event.data)
})

// 視窗管理
document.getElementById('center-btn').addEventListener('click', async () => {
    await Window.Center()
})

document.getElementById('fullscreen-btn').addEventListener('click', async () => {
    const isFullscreen = await Window.IsFullscreen()
    if (isFullscreen) {
        await Window.UnFullscreen()
    } else {
        await Window.Fullscreen()
    }
})

// 剪貼簿操作
document.getElementById('copy-btn').addEventListener('click', async () => {
    await Clipboard.SetText('Copied from Wails!')
})

// 帶有確認功能的對話框
document.getElementById('delete-btn').addEventListener('click', async () => {
    const result = await Dialogs.Question({
        Title: 'Confirm',
        Message: 'Delete this item?',
        Buttons: [
            { Label: 'Delete' },
            { Label: 'Cancel', IsDefault: true }
        ]
    })

    if (result === 'Delete') {
        await Events.Emit('delete-item', { id: currentItemId })
    }
})

// 螢幕資訊
const screens = await Screens.GetAll()
console.log(`Detected ${screens.length} screen(s)`)
screens.forEach(screen => {
    console.log(`- ${screen.Name}: ${screen.Size.Width}x${screen.Size.Height}`)
})
```

## 最佳實踐

### ✅ 建議做法

- **選擇性匯入** - 僅匯入您需要的內容
- **處理 Promise** - 所有方法都會回傳 Promise
- **使用 WML 處理簡單動作** - 比 JavaScript 更簡潔
- **檢查回傳值** - 特別是對話框的回傳值
- **取消訂閱事件** - 完成後進行清理

### ❌ 避免做法

- **不要忘記 await** - 大多數方法是非同步的
- **不要阻塞 UI** - 正確使用 async/await
- **不要忽略錯誤** - 始終處理拒絕（rejections）

## TypeScript 支援

執行階段包含完整的 TypeScript 定義：

```typescript
import { Events, Window } from '@wailsio/runtime'

Events.On('custom-event', (event) => {
    // TypeScript 知道 event.data, event.name, event.sender
    console.log(event.data)
})

// 所有方法都有完整的型別定義
const size: { width: number, height: number } = await Window.Size()
```
