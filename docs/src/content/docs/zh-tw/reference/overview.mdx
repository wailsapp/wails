## API 慣例

<details>
<summary><strong>Go API 慣例</strong> - 給剛接觸 Go 的開發者</summary>

#### 命名規則

- **類型**：PascalCase（例如：`WebviewWindow`）
- **方法**：PascalCase（例如：`SetTitle()`）
- **選項**：PascalCase 結構體（例如：`WindowOptions`）
- **常數**：PascalCase（例如：`WindowStartStateMaximised`）

#### 錯誤處理

大多數可能失敗的方法會將 `error` 作為最後一個返回值：

```go
window, err := app.Window.New()
if err != nil {
    log.Fatal(err)
}
```

#### Context

可能阻塞或需要取消的方法會接受 `context.Context`：

```go
err := app.RunWithContext(ctx)
```

#### 選項模式

配置使用選項結構體：

```go
app := application.New(application.Options{
    Name: "My App",
    Description: "A demo application",
    Services: []application.Service{
        application.NewService(&MyService{}),
    },
})
```

</details>

### JavaScript API 慣例

#### 命名規則

- **函式**：camelCase（例如：`setTitle()`）
- **常數**：SCREAMING_SNAKE_CASE（例如：`WINDOW_EVENT_FOCUS`）

#### 預設為非同步

所有 Go 方法呼叫都會返回 Promise：

```javascript
// Async/await（推薦）
const result = await MyService.DoSomething()

// Promise 鏈
MyService.DoSomething()
    .then(result => console.log(result))
    .catch(error => console.error(error))
```

#### 錯誤處理

Go 錯誤會變成 JavaScript 異常：

```javascript
try {
    await MyService.MightFail()
} catch (error) {
    console.error('Go error:', error)
}
```

#### 類型安全

TypeScript 定義是自動生成的：

```typescript
// 完整類型支援
import { Greet } from './bindings/GreetService'

const message: string = await Greet("World")
```

## 平台差異

某些 API 在不同平台上的行為有所不同：

| 功能 | Windows | macOS | Linux |
|---------|---------|-------|-------|
| **應用程式選單** | 視窗選單列 | 全域選單列 | 視窗選單列 |
| **系統匣** | 通知區域 | 選單列 | 系統匣 |
| **Dock** | N/A | ✅ 可用 | N/A |
| **檔案對話框** | 原生 | 原生 | 原生 (GTK) |
| **透明度** | ✅ 完整 | ✅ 完整 | ⚠️ 有限 |

平台特定的行為已在每個 API 章節中記錄。

## 版本控制

Wails v3 遵循語意化版本控制：

- **主版本** (v3.x.x)：破壞性變更
- **次版本** (v3.x.x)：新功能，向後相容
- **修訂版本** (v3.x.x)：錯誤修復，向後相容

**目前狀態：** Alpha（API 穩定，持續優化中）

## 棄用政策

當 API 被棄用時：

1. **在文件中標記** 並附上棄用通知
2. **提供替代方案** 並附上遷移指南
3. **在移除前維持 1 個主版本**
4. **編譯器警告**（如可能）

## API 穩定性

### 穩定 API ✅

這些 API 是穩定的，可安全用於生產環境：

- 核心應用程式 API
- 視窗管理
- 選單系統
- 事件系統
- 檔案對話框
- 服務綁定

### 不穩定 API ⚠️

這些 API 在最終發布前可能會變更：

- 某些進階視窗選項
- 平台特定功能
- 實驗性功能

不穩定 API 會在文件中標記。
