このガイドでは、Wailsのバインディングシステムが内部でどのように動作するかを解説し、自動コード生成の仕組みを理解したい開発者向けに洞察を提供します。

## アーキテクチャの概要

Wailsのバインディングシステムは、以下の3つの主要なコンポーネントで構成されています。

1. **収集 (Collection)**: Goコードを解析し、サービス、モデル、その他の宣言に関する情報を抽出します
2. **設定 (Configuration)**: バインディング生成プロセスの設定とオプションを管理します
3. **レンダリング (Rendering)**: 収集された情報に基づいてJavaScript/TypeScriptコードを生成します

<FileTree>
- internal/generator/
  - collect/     # パッケージ解析と情報抽出
  - config/      # 設定構造体とインターフェース
  - render/      # JS/TS用のコード生成
</FileTree>

## 収集プロセス

収集プロセスは、Goパッケージを解析し、サービス、モデル、その他の宣言に関する情報を抽出する役割を担います。これは `collect` パッケージによって処理されます。

### 主要なコンポーネント

- **Collector**: パッケージ情報を管理し、収集したデータをキャッシュします
- **Package**: 解析対象のGoパッケージを表し、収集されたサービス、モデル、ディレクティブを保存します
- **Service**: サービス型とそのメソッドに関する情報を収集します
- **Model**: フィールド、値、型パラメータを含む、モデル型に関する詳細な情報を収集します
- **Directive**: Goソースコード内の `//wails:` ディレクティブを解析して解釈します

### 収集フロー

1. Collectorはプロジェクト内で指定されたGoパッケージをスキャンします
2. サービス型（フロントエンドに公開されるメソッドを持つstruct）を識別します
3. 各サービスについて、そのメソッドに関する情報を収集します
4. モデル型（サービスメソッドのパラメータまたは戻り値として使用されるstruct）を識別します
5. 各モデルについて、そのフィールドと型パラメータに関する情報を収集します
6. コード内に見つかった `//wails:` ディレクティブを処理します

## レンダリングプロセス

レンダリングプロセスは、収集された情報に基づいてJavaScript/TypeScriptコードを生成する役割を担います。これは `render` パッケージによって処理されます。

### 主要なコンポーネント

- **Renderer**: サービス、モデル、インデックスファイルのレンダリングを調整します
- **Module**: 生成された単一のJavaScript/TypeScriptモジュールを表します
- **Templates**: コード生成に使用されるテキストテンプレート

### レンダリングフロー

1. 各サービスについて、Rendererはサービスメソッドをミラーリングする関数を持つJavaScript/TypeScriptファイルを生成します
2. 各モデルについて、RendererはモデルstructをミラーリングするJavaScript/TypeScriptクラスを生成します
3. Rendererは、すべてのサービスとモデルを再エクスポートするインデックスファイルを生成します
4. Rendererは `//wails:inject` ディレクティブで指定されたカスタムコードインジェクションを適用します

## 型マッピング

バインディングシステムの最も重要な側面の1つは、Go型がJavaScript/TypeScript型にどのようにマッピングされるかです。以下にマッピングの概要を示します。

| Go型 | JavaScript型 | TypeScript型 |
|---------|----------------|----------------|
| `bool` | `boolean` | `boolean` |
| `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64` | `number` | `number` |
| `string` | `string` | `string` |
| `[]byte` | `Uint8Array` | `Uint8Array` |
| `[]T` | `Array<T>` | `T[]` |
| `map[K]V` | `Object` | `Record<K, V>` |
| `struct` | `Object` | カスタムクラス |
| `interface{}` | `any` | `any` |
| `*T` | `T \| null` | `T \| null` |
| `func` | サポートされていません | サポートされていません |
| `chan` | サポートされていません | サポートされていません |

## ディレクティブシステム

バインディングシステムは、生成されたコードをカスタマイズするために使用できるいくつかのディレクティブをサポートしています。これらのディレクティブは、Goコード内のコメントとして追加されます。

### 利用可能なディレクティブ

- `//wails:inject`: 生成されたバインディングにカスタムJavaScript/TypeScriptコードをインジェクトします
- `//wails:include`: 生成されたバインディングに追加ファイルを含めます
- `//wails:internal`: 型またはメソッドを内部用としてマークし、フロントエンドへのエクスポートを防ぎます
- `//wails:ignore`: バインディング生成中にメソッドを完全に無視します
- `//wails:id`: メソッドのカスタムIDを指定し、デフォルトのハッシュベースのIDを上書きします

### ディレクティブの処理

1. 収集フェーズ中に、CollectorはGoコード内のディレクティブを識別して解析します
2. ディレクティブは、対応する宣言（サービス、メソッド、モデルなど）と共に保存されます
3. レンダリングフェーズ中に、Rendererは生成されたコードをカスタマイズするためにディレクティブを適用します

## 高度な機能

### 条件付きコード生成

バインディングシステムは、`include` および `inject` ディレクティブの2文字の条件プレフィックスを使用して、条件付きコード生成をサポートします。

```
<language><style>:<content>
```

ここで:
- `<language>` は以下のいずれかです:
  - `*` - JavaScriptとTypeScriptの両方
  - `j` - JavaScriptのみ
  - `t` - TypeScriptのみ

- `<style>` は以下のいずれかです:
  - `*` - クラスとインターフェースの両方
  - `c` - クラスのみ
  - `i` - インターフェースのみ

例:
```go
//wails:inject j*:console.log("JavaScript only");
//wails:inject t*:console.log("TypeScript only");
```

### カスタムメソッドID

デフォルトでは、メソッドはハッシュベースのIDによって識別されます。ただし、`//wails:id` ディレクティブを使用してカスタムIDを指定できます。

```go
//wails:id 42
func (s *Service) CustomIDMethod() {}
```

これは、コードのリファクタリング時に互換性を維持するのに役立ちます。

## パフォーマンスに関する考慮事項

バインディングジェネレーターは効率的に設計されていますが、いくつかの点に注意が必要です。

1. 初回実行は、スキャンするパッケージのキャッシュを構築するため、遅くなります
2. 以降の実行は、キャッシュされた情報を使用するため、高速になります
3. ジェネレーターはプロジェクト内のすべてのパッケージを処理するため、大規模なプロジェクトでは時間がかかる場合があります
4. 生成前に出力ディレクトリをクリーンアップするには、`-clean` フラグを使用できます

## デバッグ

バインディング生成に問題が発生した場合は、`-v` フラグを使用してデバッグ出力を有効にできます。

```bash
wails3 generate bindings -v
```

これにより、収集およびレンダリングプロセスに関する詳細な情報が提供され、問題の原因を特定するのに役立ちます。
