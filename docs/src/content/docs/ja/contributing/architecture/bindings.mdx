このガイドでは、Wails バインディングシステムが内部でどのように動作するかを説明し、自動コード生成の仕組みを理解したい開発者にとっての洞察を提供します。

## アーキテクチャの概要

Wails バインディングシステムは、以下の3つの主要なコンポーネントで構成されています。

1. **収集 (Collection)**: Go コードを解析し、サービス、モデル、その他の宣言に関する情報を抽出します
2. **設定 (Configuration)**: バインディング生成プロセスの設定とオプションを管理します
3. **レンダリング (Rendering)**: 収集された情報に基づいて JavaScript/TypeScript コードを生成します

<FileTree>
- internal/generator/
  - collect/     # パッケージ解析と情報抽出
  - config/      # 設定構造体とインターフェース
  - render/      # JS/TS 用のコード生成
</FileTree>

## 収集プロセス

収集プロセスは、Go パッケージを解析し、サービス、モデル、その他の宣言に関する情報を抽出する役割を担います。これは `collect` パッケージによって処理されます。

### 主要なコンポーネント

- **Collector**: パッケージ情報を管理し、収集したデータをキャッシュします
- **Package**: 解析対象の Go パッケージを表し、収集されたサービス、モデル、ディレクティブを保存します
- **Service**: サービス型とそのメソッドに関する情報を収集します
- **Model**: フィールド、値、型パラメータを含む、モデル型に関する詳細な情報を収集します
- **Directive**: Go ソースコード内の `//wails:` ディレクティブを解析して解釈します

### 収集フロー

1. Collector はプロジェクト内で指定された Go パッケージをスキャンします
2. サービス型（フロントエンドに公開されるメソッドを持つ構造体）を識別します
3. 各サービスについて、そのメソッドに関する情報を収集します
4. モデル型（サービスメソッドのパラメータまたは戻り値として使用される構造体）を識別します
5. 各モデルについて、そのフィールドと型パラメータに関する情報を収集します
6. コード内に見つかった `//wails:` ディレクティブを処理します

## レンダリングプロセス

レンダリングプロセスは、収集された情報に基づいて JavaScript/TypeScript コードを生成する役割を担います。これは `render` パッケージによって処理されます。

### 主要なコンポーネント

- **Renderer**: サービス、モデル、インデックスファイルのレンダリングを調整します
- **Module**: 生成された単一の JavaScript/TypeScript モジュールを表します
- **Templates**: コード生成に使用されるテキストテンプレート

### レンダリングフロー

1. 各サービスについて、Renderer はサービスメソッドをミラーリングする関数を持つ JavaScript/TypeScript ファイルを生成します
2. 各モデルについて、Renderer はモデル構造体をミラーリングする JavaScript/TypeScript クラスを生成します
3. Renderer は、すべてのサービスとモデルを再エクスポートするインデックスファイルを生成します
4. Renderer は `//wails:inject` ディレクティブで指定されたカスタムコードインジェクションを適用します

## 型マッピング

バインディングシステムの最も重要な側面の1つは、Go 型を JavaScript/TypeScript 型にどのようにマッピングするかです。以下にマッピングの概要を示します。

| Go 型 | JavaScript 型 | TypeScript 型 |
|---------|----------------|----------------|
| `bool` | `boolean` | `boolean` |
| `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64` | `number` | `number` |
| `string` | `string` | `string` |
| `[]byte` | `Uint8Array` | `Uint8Array` |
| `[]T` | `Array<T>` | `T[]` |
| `map[K]V` | `Object` | `Record<K, V>` |
| `struct` | `Object` | カスタムクラス |
| `interface{}` | `any` | `any` |
| `*T` | `T \| null` | `T \| null` |
| `func` | サポートされていません | サポートされていません |
| `chan` | サポートされていません | サポートされていません |

## ディレクティブシステム

バインディングシステムは、生成されたコードをカスタマイズするために使用できるいくつかのディレクティブをサポートしています。これらのディレクティブは、Go コード内のコメントとして追加されます。

### 利用可能なディレクティブ

- `//wails:inject`: 生成されたバインディングにカスタム JavaScript/TypeScript コードを注入します
- `//wails:include`: 生成されたバインディングに追加ファイルを含めます
- `//wails:internal`: 型またはメソッドを内部用としてマークし、フロントエンドへのエクスポートを防ぎます
- `//wails:ignore`: バインディング生成中にメソッドを完全に無視します
- `//wails:id`: メソッドのカスタム ID を指定し、デフォルトのハッシュベースの ID をオーバーライドします

### ディレクティブの処理

1. 収集フェーズ中に、Collector は Go コード内のディレクティブを識別して解析します
2. ディレクティブは、対応する宣言（サービス、メソッド、モデルなど）と共に保存されます
3. レンダリングフェーズ中に、Renderer は生成されたコードをカスタマイズするためにディレクティブを適用します

## 高度な機能

### 条件付きコード生成

バインディングシステムは、`include` および `inject` ディレクティブの2文字の条件プレフィックスを使用して、条件付きコード生成をサポートします。

```
<language><style>:<content>
```

ここで:
- `<language>` は以下のいずれかです:
  - `*` - JavaScript と TypeScript の両方
  - `j` - JavaScript のみ
  - `t` - TypeScript のみ

- `<style>` は以下のいずれかです:
  - `*` - クラスとインターフェースの両方
  - `c` - クラスのみ
  - `i` - インターフェースのみ

例:
```go
//wails:inject j*:console.log("JavaScript only");
//wails:inject t*:console.log("TypeScript only");
```

### カスタムメソッド ID

デフォルトでは、メソッドはハッシュベースの ID によって識別されます。ただし、`//wails:id` ディレクティブを使用してカスタム ID を指定できます。

```go
//wails:id 42
func (s *Service) CustomIDMethod() {}
```

これは、コードをリファクタリングする際に互換性を維持するのに役立ちます。

## パフォーマンスに関する考慮事項

バインディングジェネレーターは効率的に設計されていますが、いくつかの点に注意が必要です。

1. 初回実行は、スキャンするパッケージのキャッシュを構築するため、遅くなります
2. 以降の実行は、キャッシュされた情報を使用するため、高速になります
3. ジェネレーターはプロジェクト内のすべてのパッケージを処理するため、大規模なプロジェクトでは時間がかかる場合があります
4. 生成前に出力ディレクトリをクリーンアップするには、`-clean` フラグを使用できます

## デバッグ

バインディング生成に問題が発生した場合は、`-v` フラグを使用してデバッグ出力を有効にできます。

```bash
wails3 generate bindings -v
```

これにより、収集およびレンダリングプロセスに関する詳細な情報が提供され、問題の原因を特定するのに役立ちます。
