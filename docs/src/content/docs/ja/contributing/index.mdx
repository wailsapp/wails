## Wails v3 技術ドキュメントへようこそ

このセクションは、コミュニティのガイドラインやプルリクエストの開放方法に関するものではありません。
その代わりに、**Wails v3 がどのように構築されているか**に深く入り込み、コードベース内で迅速に方向性を定め、自信を持って開発を始めることができるようにします。

ランタイムの修正、CLI の拡張、新しいテンプレートの作成、あるいは単に内部構造の理解を目的としていても、以下のページは必要な技術的コンテキストを提供します。

**[エンドツーエンドのフロー図のプレースホルダー]**


図は**エンドツーエンドのフロー**を示しています：

1. **CLI** が生成、開発サーバー、コンパイル、パッケージングを制御します。
2. **バインディングシステム** が、**Web フロントエンド**が**Go バックエンド**を呼び出せるようにする接着コードを生成します。
3. 開発中は**アセットサーバー**がフレームワークの開発サーバーにプロキシし、本番環境では埋め込まれたファイルを配信します。
4. 実行時、**デスクトップランタイム**がウィンドウと OS API を管理し、**ブリッジ**が Go と JavaScript の間でメッセージを中継します。

## このドキュメントで扱う内容

| トピック | 重要性 |
| ----- | -------------- |
| **コードベースの構成** | `/v3` ディレクトリのマップとモジュール間の相互作用。 |
| **ランタイムの内部構造** | ウィンドウ管理、システム API、メッセージプロセッサ、プラットフォーム用 shim。 |
| **アセットと開発サーバー** | 開発環境での Web アセットの配信方法と、本番環境での埋め込み方法。 |
| **ビルドとパッケージングパイプライン** | Taskfile ベースのワークフロー、クロスプラットフォームコンパイル、インストーラーの生成。 |
| **バインディングシステム** | 型安全な Go⇄TS バインディングを生成する静的解析パイプライン。 |
| **テンプレートシステム** | `wails init -t <framework>` を支えるジェネレーターアーキテクチャ。 |
| **テストと CI** | ユニット/統合テストのハーネス、GitHub Actions、レースデテクターのガイダンス。 |
| **Wails の拡張** | サービス、テンプレート、CLI サブコマンドの追加。 |

各後のページでは、具体的なコードサンプル、図、関連するソースファイルへの参照を用いて、これらの領域を詳しく解説します。
