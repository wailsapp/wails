あなたのフィードバックを歓迎し、積極的に促しています！新しいチケットや投稿を作成する前に、既存のものを探してください。フィードバックを提供する方法は以下の通りです。

<Tabs>
  <TabItem label="バグ" icon="error">

    バグを見つけた場合は、Discordの[v3 Alphaフィードバック](https://discord.gg/bdj28QNHmT)チャンネルに投稿して知らせてください。

    - 投稿には、バグの内容を明確に記述し、再現可能な簡単な例を含めてください。ドキュメントで*何が起こるべきか*が不明確な場合は、その旨を投稿に含めてください。
    - 投稿には`Bug`タグを付けてください。
    - 投稿に`wails doctor`の実行結果を含めてください。
    - バグが現在のドキュメントと一致しない動作（例：ウィンドウが適切にリサイズされない）である場合は、以下の手順に従ってください：
      - `v3/example`ディレクトリ内の既存の例を更新するか、問題が明確に示されるように`v3/examples`フォルダに新しい例を作成してください。
      - タイトルを`[v3 alpha test] <バグの説明>`として[PR](https://github.com/wailsapp/wails/pulls)を開いてください。
      - 投稿にPRへのリンクを含めてください。

    :::caution

    _覚えておいてください_、予期せぬ動作が必ずしもバグであるとは限りません。単に期待通りに動作していないだけかもしれません。その場合は`Suggestions`を使用してください。

    :::

  </TabItem>

  <TabItem label="修正" icon="approve-check">

    バグの修正やドキュメントの更新がある場合は、以下の手順に従ってください：

    - [Wailsリポジトリ](https://github.com/wailsapp/wails)でプルリクエストを開いてください。PRのタイトルは`[v3 alpha]`で始まる必要があります。
    - [v3 Alphaフィードバック](https://discord.gg/bdj28QNHmT)チャンネルに投稿を作成してください。
    - 投稿には`PR`タグを付けてください。
    - 投稿にPRへのリンクを含めてください。

  </TabItem>

  <TabItem label="提案" icon="puzzle" id="abc">

    提案がある場合は、Discordの[v3 Alphaフィードバック](https://discord.gg/bdj28QNHmT)チャンネルに投稿して知らせてください：

    - 投稿には`Suggestion`タグを付けてください。

    ご質問がある場合は、[Discord](https://discord.gg/bdj28QNHmT)でお気軽にお問い合わせください。

  </TabItem>

  <TabItem label="高評価" icon="star">

    - 投稿は：thumbsup:絵文字を使用して「高評価」できます。優先度の高い投稿には必ず高評価を付けてください。
    - 「+1」や「私も」といったコメントは*追加しないでください*。
    - 投稿に追加すべき情報がある場合（例：「このバグはARMビルドにも影響します」や「別のオプションとしては....」など）、お気軽にコメントしてください。

  </TabItem>

</Tabs>

既知の問題のリストと進行中の作業は[こちら](https://github.com/orgs/wailsapp/projects/6)で確認できます。

## フィードバックを求めている項目

- API
  - 使いやすいですか？
  - 期待通りに動作しますか？
  - 不足している機能はありますか？
  - 削除すべき機能はありますか？
  - GoとJS間で一貫性がありますか？
- ビルドシステム
  - 使いやすいですか？
  - 改善できますか？
- 例
  - 明確ですか？
  - 基本をカバーしていますか？
- 機能
  - 不足している機能は何ですか？
  - 不要な機能は何ですか？
- ドキュメント
  - より明確にできる点はありますか？
