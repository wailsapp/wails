フロントエンドルーティングは、シングルページアプリケーションでビューを切り替えるための一般的な方法です。
このガイドでは、Wailsを使用する際に各フロントエンドフレームワーク向けに推奨されるアプローチについて説明します。


<Tabs syncKey="framework">
    <TabItem label="Vue" icon="seti:vue">

Vueでのルーティングの推奨アプローチは[Hashモード](https://next.router.vuejs.org/guide/essentials/history-mode.html#hash-mode)です：

```javascript
import { createRouter, createWebHashHistory } from "vue-router";

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    //...
  ],
});
```

HashモードはURLのハッシュ部分を使用して異なるビューをレンダリングし、ハッシュベースのURL形式を使用することで、Wailsランタイムがルーティングに干渉する問題を回避します。

    </TabItem>
    <TabItem label="Angular" icon="seti:angular">

Angularでのルーティングの推奨アプローチは[HashLocationStrategy](https://codecraft.tv/courses/angular/routing/routing-strategies#%5Fhashlocationstrategy)です：

```typescript
RouterModule.forRoot(routes, { useHash: true });
```

ハッシュベースのURLを使用することで、すべてのプラットフォームでWailsのウィンドウ処理とルーティングが正しく機能します。

    </TabItem>
    <TabItem label="React" icon="seti:react">

Reactでのルーティングの推奨アプローチは[HashRouter](https://reactrouter.com/en/main/router-components/hash-router)です：

```jsx
import ReactDOM from "react-dom/client";
import { HashRouter, Routes, Route } from "react-router-dom";

ReactDOM.createRoot(root).render(
  <HashRouter basename={"/"}>
    {/* The rest of your app goes here */}
    <Routes>
      <Route path="/" element={<Page0 />} />
      <Route path="/page1" element={<Page1 />} />
      <Route path="/page2" element={<Page2 />} />
      {/* more... */}
    </Routes>
  </HashRouter>
);
```

HashRouterはパスの代わりにURLハッシュを使用するため、すべてのプラットフォームでWailsと確実に連携します。

    </TabItem>
    <TabItem label="Svelte" icon="seti:svelte">

Svelteでのルーティングの推奨アプローチは[svelte-spa-router](https://github.com/ItalyPaleAle/svelte-spa-router)です：

```svelte
<script>
    import Router from "svelte-spa-router";
</script>

<Router
    routes={{
        "/": Home,
        "/products": wrap({
            asyncComponent: () => import("./routes/Products.svelte"),
        }),
        "/settings": Settings,
        "*": NotFound,
    }}
/>
```

svelte-spa-routerはハッシュベースのルーティングをサポートしており、Wailsアプリケーションと互換性があります。

## なぜハッシュルーティングなのか？

WailsはフロントエンドをネイティブのWebViewウィンドウに埋め込みます。ハッシュベースのルーティング（`#/page` を `/page` の代わりに使用）は、以下の競合を回避します：

- Wailsランタイムの内部ルーティング
- 異なるプラットフォームにおけるネイティブウィンドウのURL処理
- ルート以外のパスから提供されるプロダクション用アセット

## トラブルシューティング

プロダクションビルドでフロントエンドルーティングに問題がある場合：

- フロントエンドのビルドツールが**ハッシュモード**ルーティング向けに出力されるように設定されていることを確認してください
- Viteベースのプロジェクトでは、`vite.config.js` に `base: "./"` を追加してください
- `index.html` がSPAルーティングのフォールバックを適切に処理していることを確認してください
