# Ginをサービスで使用する

Gin Webフレームワークは、GoでHTTPサービスを作成するための人気のある選択肢です。Wails v3では、Ginベースのサービスをアプリケーションに簡単に統合でき、HTTPリクエストの処理、RESTful APIの実装、Webコンテンツの提供に対して強力な方法を提供します。

このガイドでは、Wailsアプリケーションの特定のルートにマウントできるGinベースのサービスを作成する方法を順を追って説明します。以下の方法をデモンストレーションする完全な例を構築します。

1. Ginベースのサービスの作成
2. Wailsサービスインターフェースの実装
3. ルートとミドルウェアの設定
4. Wailsイベントシステムとの統合
5. フロントエンドからのサービスとの対話

## 前提条件

始める前に、以下が準備できていることを確認してください。

- Wails v3がインストールされている
- GoとGinフレームワークの基本的な知識
- HTTPの概念とRESTful APIに関する理解

プロジェクトにGinフレームワークを追加する必要があります。

```bash
go get github.com/gin-gonic/gin
```

## Ginベースのサービスの作成

Wailsサービスインターフェースを実装するGinサービスを作成することから始めましょう。このサービスはユーザーのコレクションを管理し、ユーザーレコードの取得と作成のためのAPIエンドポイントを提供します。

### 1. データモデルの定義

まず、サービスが扱うデータ構造を定義します。

```go
package services

import (
	"context"
	"net/http"
	"strconv"
	"sync"
	"time"

	"github.com/gin-gonic/gin"
	"github.com/wailsapp/wails/v3/pkg/application"
)

// User はシステム内のユーザーを表します
type User struct {
	ID        int       `json:"id"`
	Name      string    `json:"name"`
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"createdAt"`
}

// EventData はイベントで送信されるデータを表します
type EventData struct {
	Message   string `json:"message"`
	Timestamp string `json:"timestamp"`
}
```

### 2. サービス構造の作成

次に、Ginルーターとサービスが維持する必要のある状態を保持するサービス構造を定義します。

```go
// GinService はHTTP処理にGinを使用するWailsサービスを実装します
type GinService struct {
	ginEngine *gin.Engine
	users     []User
	nextID    int
	mu        sync.RWMutex
	app       *application.App
}

// NewGinService は新しいGinServiceインスタンスを作成します
func NewGinService() *GinService {
	// 新しいGinルーターを作成
	ginEngine := gin.New()

	// ミドルウェアを追加
	ginEngine.Use(gin.Recovery())
	ginEngine.Use(LoggingMiddleware())

	service := &GinService{
		ginEngine: ginEngine,
		users: []User{
			{ID: 1, Name: "Alice", Email: "alice@example.com", CreatedAt: time.Now().Add(-72 * time.Hour)},
			{ID: 2, Name: "Bob", Email: "bob@example.com", CreatedAt: time.Now().Add(-48 * time.Hour)},
			{ID: 3, Name: "Charlie", Email: "charlie@example.com", CreatedAt: time.Now().Add(-24 * time.Hour)},
		},
		nextID: 4,
	}

	// ルートを定義
	service.setupRoutes()

	return service
}
```

### 3. サービスインターフェースの実装

Wailsサービスインターフェースに必要なメソッドを実装します。

```go
// ServiceName はサービスの名前を返します
func (s *GinService) ServiceName() string {
	return "Gin API Service"
}

// ServiceStartup はサービスが開始されるときに呼び出されます
func (s *GinService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
	// 後で使用するためにアプリケーションインスタンスを保存
	s.app = application.Get()

	// フロントエンドからトリガーできるイベントハンドラーを登録
	s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
		// イベントデータをログに記録
		s.app.Logger.Info("Received event from frontend", "data", event.Data)

		// フロントエンドに戻りイベントを発行
		s.app.Event.Emit("gin-api-response",
			map[string]interface{}{
                "message": "Response from Gin API Service",
                "time":    time.Now().Format(time.RFC3339),
            },
		)
	})

	return nil
}

// ServiceShutdown はサービスがシャットダウンされるときに呼び出されます
func (s *GinService) ServiceShutdown(ctx context.Context) error {
	// 必要に応じてリソースをクリーンアップ
	return nil
}
```

### 3. http.Handlerインターフェースの実装

サービスを特定のルートにマウント可能にするために、`http.Handler`インターフェースを実装します。この単一のメソッド `ServeHTTP` は、サービスへのすべてのHTTPリクエストへのゲートウェイです。リクエスト処理をGinルーターに委譲し、ルーティングとミドルウェアのためにGinの強力な機能のすべてを使用できるようにします。

```go
// ServeHTTP はhttp.Handlerインターフェースを実装します
func (s *GinService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// すべてのリクエストがGinルーターに送信されます
	s.ginEngine.ServeHTTP(w, r)
}
```

### 4. ルートの設定

より良い構成のために、APIルートを別のメソッドで定義します。このアプローチによりコードが整理され、APIの構造を理解しやすくなります。Ginルーターは、関連するエンドポイントを整理するのに役立つルートグループのサポートを含む、ルートを定義するための流暢なAPIを提供します。

```go
// setupRoutes はAPIルートを構成します
func (s *GinService) setupRoutes() {
	// 基本情報エンドポイント
	s.ginEngine.GET("/info", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{
			"service": "Gin API Service",
			"version": "1.0.0",
			"time":    time.Now().Format(time.RFC3339),
		})
	})

	// ユーザーグループ
	users := s.ginEngine.Group("/users")
	{
		// すべてのユーザーを取得
		users.GET("", func(c *gin.Context) {
			s.mu.RLock()
			defer s.mu.RUnlock()
			c.JSON(http.StatusOK, s.users)
		})

		// IDでユーザーを取得
		users.GET("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.RLock()
			defer s.mu.RUnlock()

			for _, user := range s.users {
				if user.ID == id {
					c.JSON(http.StatusOK, user)
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})

		// 新しいユーザーを作成
		users.POST("", func(c *gin.Context) {
			var newUser User
			if err := c.ShouldBindJSON(&newUser); err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": err.Error()})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			// IDと作成時間を設定
			newUser.ID = s.nextID
			newUser.CreatedAt = time.Now()
			s.nextID++

			// ユーザースライスに追加
			s.users = append(s.users, newUser)

			c.JSON(http.StatusCreated, newUser)

			// 新しいユーザーについて通知するイベントを発行
			s.app.Event.Emit("user-created", newUser)
		})

		// ユーザーを削除
		users.DELETE("/:id", func(c *gin.Context) {
			id, err := strconv.Atoi(c.Param("id"))
			if err != nil {
				c.JSON(http.StatusBadRequest, gin.H{"error": "Invalid user ID"})
				return
			}

			s.mu.Lock()
			defer s.mu.Unlock()

			for i, user := range s.users {
				if user.ID == id {
					// ユーザーをスライスから削除
					s.users = append(s.users[:i], s.users[i+1:]...)
					c.JSON(http.StatusOK, gin.H{"message": "User deleted"})
					return
				}
			}

			c.JSON(http.StatusNotFound, gin.H{"error": "User not found"})
		})
	}
}
```

### 5. カスタムミドルウェアの作成

サービスを強化するためにカスタムGinミドルウェアを作成できます。Ginのミドルウェア関数はルーターに追加された順序で実行され、ログ記録、認証、エラー処理などのタスクを実行できます。この例では、リクエストの詳細を記録する単純なログ記録ミドルウェアを示しています。

```go
// LoggingMiddleware はリクエストの詳細をログに記録するGinミドルウェアです
func LoggingMiddleware() gin.HandlerFunc {
	return func(c *gin.Context) {
		// タイマーを開始
		start := time.Now()

		// リクエストを処理
		c.Next()

		// レイテンシーを計算
		latency := time.Since(start)

		// リクエストの詳細をログに記録
		log.Printf("[GIN] %s %s %d %s", c.Request.Method, c.Request.URL.Path, c.Writer.Status(), latency)
	}
}
```

## サービスの登録Gin ベースのサービスを Wails アプリケーションで使用するには、アプリケーションに登録し、マウントするルートを指定する必要があります。これは Wails アプリケーションのインスタンスを作成する際に行われます。指定したルートは、Gin ルーターで定義されたすべてのエンドポイントのベースパスになります。

```go
app := application.New(application.Options{
    Name:        "Gin Service Demo",
    Description: "A demo of using Gin in Wails services",
    Mac: application.MacOptions{
        ApplicationShouldTerminateAfterLastWindowClosed: true,
    },
    LogLevel: slog.LevelDebug,
    Services: []application.Service{
        application.NewServiceWithOptions(services.NewGinService(), application.ServiceOptions{
            Route: "/api",
        }),
    },
    Assets: application.AssetOptions{
        Handler: application.BundledAssetFileServer(assets),
    },
})
```

この例では、Gin サービスは `/api` ルートにマウントされています。つまり、Gin ルーターに `/info` というエンドポイントが定義されている場合、アプリケーション内では `/api/info` でアクセス可能になります。このアプローチにより、API エンドポイントを論理的に整理し、アプリケーションの他の部分との競合を回避することができます。

## Wails イベントシステムとの統合

Gin を Wails サービスと併用する際の強力な機能の一つは、Wails イベントシステムとシームレスに統合できることです。これにより、バックエンドサービスとフロントエンド間のリアルタイム通信が可能になります。

サービスの `ServiceStartup` メソッド内で、フロントエンドからのイベントを処理するためのイベントハンドラを登録できます。

```go
s.app.Event.On("gin-api-event", func(event *application.CustomEvent) {
	// Log the event data
	s.app.Logger.Info("Received event from frontend", "data", event.Data)

	// Emit an event back to the frontend
	s.app.Event.Emit("gin-api-response",
		map[string]interface{}{
			"message": "Response from Gin API Service",
			"time":    time.Now().Format(time.RFC3339),
		},
	)
})
```

また、Gin ルートやサービスの他の部分からフロントエンドへイベントを発信することもできます。

```go
// After creating a new user
s.app.Event.Emit("user-created", newUser)
```

## フロントエンドとの統合

フロントエンドから Gin サービスと対話するには、Wails ランタイムをインポートし、API エンドポイントに対して HTTP リクエストを行い、リアルタイム通信のために Wails イベントシステムを使用する必要があります。

本番環境では、`/wails/runtime.js` を直接インポートする代わりに、`@wailsio/runtime` パッケージを使用することをお勧めします。これにより、型安全性、より良い IDE のサポート、npm によるバージョン管理、および最新の JavaScript ツールチェーンとの互換性が確保されます。

パッケージをインストールします：
```bash
npm install @wailsio/runtime
```

その後、コードで使用します：
```javascript
import * as wails from '@wailsio/runtime';

// Event emission
wails.Events.Emit('gin-api-event', eventData);
```

## 結び

Gin Web フレームワークを Wails v3 Services と統合することで、モジュール化され、保守性の高い Web アプリケーションを構築するための強力かつ柔軟なアプローチが提供されます。Gin のルーティングおよびミドルウェアの機能と Wails イベントシステムを組み合わせることで、関心の分離を明確にしたリッチでインタラクティブなアプリケーションを作成することができます。

このガイドの完全な例のコードは、Wails リポジトリの `v3/examples/gin-service` にあります。
