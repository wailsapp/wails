Wails フロントエンドランタイムは、Wails アプリケーションの標準ライブラリです。これには、アプリケーションで使用できる多数の機能が含まれています。

- ウィンドウ管理
- ダイアログ
- ブラウザ統合
- クリップボード
- メニュー
- システム情報
- イベント
- コンテキストメニュー
- スクリーン
- WML (Wails Markup Language)

ランタイムは、Go とフロントエンド間の統合に必要です。ランタイムを統合する方法は 2 つあります。

- `@wailsio/runtime` パッケージを使用する
- 事前にビルドされたバンドルを使用する

## npm パッケージの使用

`@wailsio/runtime` パッケージは、フロントエンドから Wails ランタイムへのアクセスを提供する JavaScript パッケージです。これはすべての標準テンプレートで使用されており、ランタイムをアプリケーションに統合するための推奨される方法です。`@wailsio/runtime` パッケージを使用することで、必要なランタイムの部分のみをアプリケーションに含めることができます。

このパッケージは npm で利用可能で、次のコマンドでインストールできます。

```shell
npm install --save @wailsio/runtime
```

## 事前ビルドされたバンドルの使用

一部のプロジェクトでは JavaScript バンダーラーを使用せず、ランタイムの事前にビルドされたバンドル版を好む場合があります。このバージョンは、次のコマンドを使用してローカルで生成できます。

```shell
wails3 generate runtime
```

このコマンドは、現在のディレクトリに `runtime.js`（および `runtime.debug.js`）ファイルを出力します。このファイルは ES モジュールであり、npm パッケージと同様にアプリケーションのスクリプトからインポートできますが、API はグローバルな `window` オブジェクトにもエクスポートされるため、より単純なアプリケーションでは以下のように使用できます。

```html
<html>
    <head>
        <script type="module" src="./runtime.js"></script>
        <script>
            window.onload = function () {
                wails.Window.SetTitle("A new window title");
            }
        </script>
    </head>
    <!--- ... -->
</html>
```

:::caution
ランタイムを読み込む `<script>` タグに `type="module"` 属性を含め、API を呼び出す前にページが完全に読み込まれるのを待つことが重要です。`type="module"` 属性を持つスクリプトは非同期で実行されるためです。
:::

## 初期化

API 関数の他に、ランタイムはコンテキストメニューとウィンドウのドラッグのサポートを提供します。これらの機能は、ランタイムが初期化された後にのみ期待どおりに動作します。API を使用しない場合でも、フロントエンドコードのどこかに副作用（side-effect）の import 文を含めるようにしてください。

```javascript
import "@wailsio/runtime";
```

バンドラーは副作用の存在を検出し、ビルドに必要なすべての初期化コードを含めるはずです。

:::note
事前ビルドされたバンドルを好む場合、上記のようにスクリプトタグを追加するだけで十分です。
:::

## 型付きイベント用の Vite プラグイン

ランタイムには、開発中に型付きイベントの HMR（Hot Module Replacement）サポートを有効にする Vite プラグインが含まれています。

### セットアップ

プラグインを `vite.config.ts` に追加します。

```typescript
import { defineConfig } from 'vite'
import wails from '@wailsio/runtime/plugins/vite'

export default defineConfig({
  plugins: [wails()],
})
```

### メリット

- **自動リロード**: `wails3 generate bindings` を実行すると、イベントバインディングが自動的に再生成され、リロードされます
- **開発モード**: `wails3 dev` とシームレスに連携し、即時更新が可能です
- **型安全性**: オートコンプリートと型チェックを含む完全な TypeScript サポート

### イベント登録との併用

Go でイベントを登録します。

```go
type UserData struct {
    ID   string
    Name string
}

func init() {
    application.RegisterEvent[UserData]("user-updated")
}
```

バインディングを生成します。

```bash
wails3 generate bindings

# または、TypeScript 定義を含める場合
wails3 generate bindings -ts

# その他のオプションについては、以下を参照してください。
wails3 generate bindings -help
```

フロントエンドで型付きイベントを使用します。

```typescript
import { Events } from '@wailsio/runtime'
import { UserUpdated } from './bindings/events'

// オートコンプリート付きの型安全なイベント
Events.Emit(UserUpdated({
    ID: "123",
    Name: "John Doe"
}))
```

## API リファレンス

ランタイムはモジュールごとに整理されており、各モジュールが特定の機能を提供します。必要なもののみをインポートしてください。

```javascript
import { Events, Window, Clipboard } from '@wailsio/runtime'
```

### Events

Go と JavaScript の間の通信用のイベントシステム。

#### On()

イベントのコールバックを登録します。

```typescript
function On(eventName: string, callback: (event: WailsEvent) => void): () => void
function On<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**戻り値:** 購読解除関数

**例:**

```javascript
import { Events } from '@wailsio/runtime'

// 基本的なイベントリスニング
const unsubscribe = Events.On('user-logged-in', (event) => {
    console.log('User:', event.data.username)
})

// 型付きイベントを使用する場合（TypeScript）
import { UserLogin } from './bindings/events'

Events.On(UserLogin, (event) => {
    // event.data は UserLoginData として型付けされます
    console.log('User:', event.data.username)
})

// その後: unsubscribe()
```

#### Once()

1 回だけ実行されるコールバックを登録します。

```typescript
function Once(eventName: string, callback: (event: WailsEvent) => void): () => void
function Once<T>(eventType: EventType<T>, callback: (event: WailsEvent<T>) => void): () => void
```

**例:**

```javascript
import { Events } from '@wailsio/runtime'

Events.Once('app-ready', () => {
    console.log('App initialized')
})
```

#### Emit()

Go バックエンドまたは他のウィンドウにイベントを発信します。

```typescript
function Emit(name: string, data?: any): Promise<boolean>
function Emit<T>(event: Event<T>): Promise<boolean>
```

**戻り値:** イベントがキャンセルされた場合は `true`、それ以外は `false` を解決する Promise

**例:**

```javascript
import { Events } from '@wailsio/runtime'

// 基本的なイベント発信
const wasCancelled = await Events.Emit('button-clicked', { buttonId: 'submit' })

// 型付きイベントを使用する場合（TypeScript）
import { UserLogin } from './bindings/events'

const cancelled = await Events.Emit(UserLogin({
    UserID: "123",
    Username: "john_doe",
    LoginTime: new Date().toISOString()
}))

if (cancelled) {
    console.log('Login was cancelled by a hook')
}
```

:::note
戻り値は、イベントがフックによってキャンセルされたかどうかを示します。ほとんどのイベントはキャンセルできず、常に `false` を返します。
:::

#### Off()

イベントリスナーを削除します。

```typescript
function Off(...eventNames: string[]): void
```

**例:**

```javascript
import { Events } from '@wailsio/runtime'

Events.Off('user-logged-in', 'user-logged-out')
```

#### OffAll()

すべてのイベントリスナーを削除します。

```typescript
function OffAll(): void
```

### システム

バックエンドと直接通信するための低レベルシステムメソッド。

#### invoke()

バックエンドに生のメッセージを直接送信します。これは標準のバインディングシステムをバイパスし、アプリケーションオプションの `RawMessageHandler` によって処理されます。

```typescript
function invoke(message: any): void
```

**例:**

```javascript
import { System } from '@wailsio/runtime'

// バックエンドに生のメッセージを送信
System.invoke('my-custom-message')

// JSONとして構造化データを送信
System.invoke(JSON.stringify({ action: 'update', value: 42 }))
```

:::caution
これは戻り値のないファイア・アンド・フォゲット関数です。バックエンドからの応答を受け取るにはイベントを使用してください。
:::

詳細については、[Raw Messages ガイド](/ja/guides/raw-messages) を参照してください。

### WML (Wails Markup Language)

WML は一般的なアクションのための宣言的な属性を提供します。HTML要素に属性を追加します：

#### 属性

**wml-event** - クリック時にイベントを発火します

```html<button wml-event="save-clicked">保存</button>
```

**wml-window** - ウィンドウのメソッドを呼び出します

```html
<button wml-window="Close">ウィンドウを閉じる</button>
<button wml-window="Minimise">最小化</button>
```

**wml-target-window** - wml-window の対象ウィンドウを指定します

```html
<button wml-window="Show" wml-target-window="settings">
    設定を表示
</button>
```

**wml-openurl** - ブラウザで URL を開きます

```html
<a href="#" wml-openurl="https://wails.io">Wails を訪問</a>
```

**wml-confirm** - アクション前に確認ダイアログを表示します

```html
<button wml-window="Close" wml-confirm="本当に閉じますか?">
    閉じる
</button>
```

**例:**

```html
<div>
    <button wml-event="save-clicked">保存</button>
    <button wml-window="Minimise">最小化</button>
    <button wml-window="Close" wml-confirm="ウィンドウを閉じますか?">閉じる</button>
    <a href="#" wml-openurl="https://github.com/wailsapp/wails">GitHub</a>
</div>
```

## 完全な例

```javascript
import { Events, Window, Clipboard, Dialogs, Screens } from '@wailsio/runtime'

// Go からのイベントをリッスン
Events.On('data-updated', (event) => {
    console.log('データ:', event.data)
    updateUI(event.data)
})

// ウィンドウ管理
document.getElementById('center-btn').addEventListener('click', async () => {
    await Window.Center()
})

document.getElementById('fullscreen-btn').addEventListener('click', async () => {
    const isFullscreen = await Window.IsFullscreen()
    if (isFullscreen) {
        await Window.UnFullscreen()
    } else {
        await Window.Fullscreen()
    }
})

// クリップボード操作
document.getElementById('copy-btn').addEventListener('click', async () => {
    await Clipboard.SetText('Wails からコピーしました!')
})

// 確認付きダイアログ
document.getElementById('delete-btn').addEventListener('click', async () => {
    const result = await Dialogs.Question({
        Title: '確認',
        Message: 'このアイテムを削除しますか?',
        Buttons: [
            { Label: '削除' },
            { Label: 'キャンセル', IsDefault: true }
        ]
    })

    if (result === '削除') {
        await Events.Emit('delete-item', { id: currentItemId })
    }
})

// スクリーン情報
const screens = await Screens.GetAll()
console.log(`${screens.length} 個の画面を検出しました`)
screens.forEach(screen => {
    console.log(`- ${screen.Name}: ${screen.Size.Width}x${screen.Size.Height}`)
})
```

## ベストプラクティス

### ✅ 推奨

- **選択的にインポートする** - 必要なものだけをインポートする
- **プロミスを処理する** - すべてのメソッドはプロミスを返す
- **単純なアクションには WML を使用する** - JavaScript よりもクリーン
- **戻り値を確認する** - 特にダイアログの場合
- **イベントの購読解除を行う** - 完了時にクリーンアップする

### ❌ 非推奨

- **await を忘れない** - ほとんどのメソッドは非同期
- **UI をブロックしない** - async/await を適切に使用する
- **エラーを無視しない** - 常に拒否を処理する

## TypeScript サポート

ランタイムには完全な TypeScript 定義が含まれています:

```typescript
import { Events, Window } from '@wailsio/runtime'

Events.On('custom-event', (event) => {
    // TypeScript は event.data, event.name, event.sender を認識する
    console.log(event.data)
})

// すべてのメソッドは完全に型付けされている
const size: { width: number, height: number } = await Window.Size()
```
