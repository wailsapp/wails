## プロジェクトの作成

<Steps>

1. **プロジェクトの生成**

   まず、新しい Wails プロジェクトを作成します。クリーンなスタート地点を提供するデフォルトのバニラテンプレートを使用します。

   ```bash
   wails3 init -n todo-app
   cd todo-app
   ```

   これにより、基本的な構造を持つ新しいプロジェクトが作成されます。Go バックエンドはルートに、フロントエンドコードは `frontend/` ディレクトリに配置されます。

2. **TODO サービスの作成**

   TODO サービスはアプリケーションの状態を管理し、CRUD 操作用のメソッドを提供します。各リクエストが独立している Web サーバーとは異なり、デスクトップアプリでは複数の並行操作が発生する可能性があるため、スレッドセーフな状態管理が必要です。

   `greetservice.go` を削除し、新しいファイル `todoservice.go` を作成します。

   ```go title="todoservice.go"
   package main

   import (
       "errors"
       "sync"
   )

   type Todo struct {
       ID        int    `json:"id"`
       Title     string `json:"title"`
       Completed bool   `json:"completed"`
   }

   type TodoService struct {
       todos  []Todo
       nextID int
       mu     sync.RWMutex
   }

   func NewTodoService() *TodoService {
       return &TodoService{
           todos:  []Todo{},
           nextID: 1,
       }
   }

   func (t *TodoService) GetAll() []Todo {
       t.mu.RLock()
       defer t.mu.RUnlock()
       return t.todos
   }

   func (t *TodoService) Add(title string) (*Todo, error) {
       if title == "" {
           return nil, errors.New("title cannot be empty")
       }

       t.mu.Lock()
       defer t.mu.Unlock()

       todo := Todo{
           ID:        t.nextID,
           Title:     title,
           Completed: false,
       }
       t.todos = append(t.todos, todo)
       t.nextID++

       return &todo, nil
   }

   func (t *TodoService) Toggle(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i := range t.todos {
           if t.todos[i].ID == id {
               t.todos[i].Completed = !t.todos[i].Completed
               return nil
           }
       }
       return errors.New("todo not found")
   }

   func (t *TodoService) Delete(id int) error {
       t.mu.Lock()
       defer t.mu.Unlock()

       for i, todo := range t.todos {
           if todo.ID == id {
               t.todos = append(t.todos[:i], t.todos[i+1:]...)
               return nil
           }
       }
       return errors.New("todo not found")
   }
   ```

   **ここで何が起こっているか:**

   **`Todo` 構造体:**
   - ID、Title、Completed フィールドを持つデータの形状を定義します
   - `json:` タグは、Go がこの構造体をフロントエンド向けに JSON に変換する方法を指示します
   - 各フィールドはエクスポート済み（大文字始まり）であるため、バインディングジェネレーターが参照できます

   **`TodoService` 構造体:**
   - `todos []Todo` - すべての TODO アイテムを保持するスライス
   - `nextID int` - 次に割り当てる ID を追跡します（自動インクリメントをシミュレート）
   - `mu sync.RWMutex` - スレッドセーフなアクセス用の読み書きミューテックス

   **`sync.RWMutex` によるスレッドセーフ:**
   - デスクトップアプリでは、UI から複数の並行操作が発生する可能性があります
   - `RLock()` は複数のリーダーを同時に許可します（例：複数の `GetAll` 呼び出し）
   - `Lock()` は書き込みに対して排他的なアクセスを提供します（例：`Add`、`Toggle`、`Delete`）
   - `defer` は、関数が早期に終了したりパニックしたりした場合でもロックが解放されることを保証します

   **メソッド:**
   - `GetAll()` - すべての TODO を返します（データを変更しないため読み取りロックを使用）
   - `Add(title)` - 新しい TODO を作成し、入力を検証し、ID をインクリメントします
   - `Toggle(id)` - TODO の完了ステータスを反転します
   - `Delete(id)` - スライスから TODO を削除します

   **エラー処理:**
   - Go の慣習に従って、最後の値として `error` を返します
   - 空のタイトルは拒否されます
   - 存在しない TODO に対する操作はエラーを返します
   - これらのエラーはフロントエンドで JavaScript 例外になります

3. **main.go の更新**

   TODO サービスを Wails アプリケーションに登録します。`main.go` の `Services` セクションを見つけて、GreetService を TodoService に置き換えます。

   ```go title="main.go" {5}
   Services: []application.Service{
       application.NewService(NewTodoService()),
   },
   ```

   **ここで何が起こっているか:**
   - デフォルトの GreetService を削除し、代わりに TodoService を追加しています
   - `application.NewService()` は Wails がサービスを管理できるようにサービスをラップします
   - Wails はこのサービス上のすべての公開メソッドに対して JavaScript バインディングを自動的に生成します

4. **フロントエンド UI の作成**

   次にフロントエンドを構築します。ここで Go メソッドを呼び出し、UI を表示します。バインディングがどのように直接機能するかを示し、シンプルに保つためにバニラ JavaScript を使用しています。

   `frontend/src/main.js` を置き換えます。

   ```javascript title="frontend/src/main.js"
   import {TodoService} from "../bindings/changeme";

   async function loadTodos() {
       const todos = await TodoService.GetAll();
       const list = document.getElementById('todo-list');

       list.innerHTML = todos.map(todo => `
           <div class="todo ${todo.completed ? 'completed' : ''}">
               <input type="checkbox"
                      ${todo.completed ? 'checked' : ''}
                      onchange="toggleTodo(${todo.id})">
               <span>${todo.title}</span>
               <button onclick="deleteTodo(${todo.id})">Delete</button>
           </div>
       `).join('');
   }

   window.addTodo = async () => {
       const input = document.getElementById('todo-input');
       const title = input.value.trim();

       if (title) {
           await TodoService.Add(title);
           input.value = '';
           await loadTodos();
       }
   }

   window.toggleTodo = async (id) => {
       await TodoService.Toggle(id);
       await loadTodos();
   }

   window.deleteTodo = async (id) => {
       await TodoService.Delete(id);
       await loadTodos();
   }

   // Load todos on startup
   loadTodos();
   ```

   **ここで何が起こっているか:**

   **バインディングのインポート:**
   - `import {TodoService} from "../bindings/changeme"` - 自動生成された Go バインディングを取り込みます
   - 注: `changeme` は `go.mod` からの実際のモジュール名になります

   **`loadTodos()` 関数:**
   - `TodoService.GetAll()` を呼び出して Go からすべての TODO をフェッチします
   - テンプレートリテラルを使用して各 TODO の HTML を構築します
   - スタイリングのために `completed` クラスを動的に追加/削除します
   - `onclick` 属性を使用してボタンを関数に接続します
   - すべての HTML を結合して DOM に注入します

   **CRUD 関数:**
   - `addTodo()` - 入力を検証し、Go の `Add` メソッドを呼び出し、リストを更新します
   - `toggleTodo(id)` - Go の `Toggle` メソッドを呼び出し、リストを更新します
   - `deleteTodo(id)` - Go の `Delete` メソッドを呼び出し、リストを更新します
   - すべての関数は非同期です。Go の呼び出しは Promise を返すためです**ウィンドウにアタッチする理由:**
- `window.addTodo = ...` により、HTML の `onclick` 属性から関数にアクセス可能になります
- これはバニラ JS におけるシンプルなパターンです（フレームワークはこれを異なる方法で処理します）
- 本番環境では、適切なイベント委譲を使用することが推奨されます

**リフレッシュパターン:**
- 各ミューテーション（追加/切り替え/削除）の後に、再度 `loadTodos()` を呼び出します
- これにより、UI が Go の状態と同期したまま保たれます
- 代替案: Go のメソッドが新しい状態を返すようにし、2 回目の呼び出しを回避する

**モジュールスクリプト:**
- `<script type="module">` により ES6 のインポートが使用可能になります

## 仕組み

### スレッドセーフな状態管理

`sync.RWMutex` は安全な同時アクセスを提供します:

```go
func (t *TodoService) GetAll() []Todo {
    t.mu.RLock()  // 読み取りロック - 複数の読み取りを許可
    defer t.mu.RUnlock()
    return t.todos
}

func (t *TodoService) Add(title string) (*Todo, error) {
    t.mu.Lock()  // 書き込みロック - 排他アクセス
    defer t.mu.Unlock()
    // ... ミューテーション
}
```

**これが重要な理由:**
- 複数のフロントエンド呼び出しが同時に発生する可能性があります
- 読み取り操作は互いをブロックしません
- 書き込み操作は排他アクセスを取得します
- `defer` はロックが常に解放されることを保証します

### エラー処理

サービスは無効な操作に対してエラーを返します:

```go
func (t *TodoService) Add(title string) (*Todo, error) {
    if title == "" {
        return nil, errors.New("title cannot be empty")
    }
    // ...
}
```

フロントエンドでは、これらをキャッチできます:

```javascript
try {
    await TodoService.Add(title);
} catch (err) {
    alert('Error: ' + err);
}
```

### 状態の同期

各ミューテーションの後、完全なリストを再読み込みします:

```javascript
window.addTodo = async () => {
    await TodoService.Add(title);  // ミューテーション
    await loadTodos();             // 更新
}
```

**代替アプローチ:** 2 回目の呼び出しを避けるために、各メソッドから更新されたリストを返します。

## 拡張機能

### 統計情報の追加

`todoservice.go` にこれを追加します:

```go
type TodoStats struct {
    Total     int `json:"total"`
    Completed int `json:"completed"`
    Active    int `json:"active"`
}

func (t *TodoService) GetStats() TodoStats {
    t.mu.RLock()
    defer t.mu.RUnlock()

    stats := TodoStats{
        Total: len(t.todos),
    }

    for _, todo := range t.todos {
        if todo.Completed {
            stats.Completed++
        } else {
            stats.Active++
        }
    }

    return stats
}
```

フロントエンドに表示します:

```javascript
async function loadTodos() {
    const [todos, stats] = await Promise.all([
        TodoService.GetAll(),
        TodoService.GetStats()
    ]);

    // 統計情報を表示
    document.getElementById('stats').textContent =
        `${stats.active} active, ${stats.completed} completed`;

    // ... todos をレンダリング
}
```

### 「完了したものをクリア」の追加

```go
func (t *TodoService) ClearCompleted() int {
    t.mu.Lock()
    defer t.mu.Unlock()

    removed := 0
    newTodos := []Todo{}

    for _, todo := range t.todos {
        if !todo.Completed {
            newTodos = append(newTodos, todo)
        } else {
            removed++
        }
    }

    t.todos = newTodos
    return removed
}
```


これにより、`build/bin/` に最適化されたネイティブ実行ファイルが作成されます:
- Go コードを最適化してコンパイル
- フロントエンドを本番環境向けにビルド
- すべてを単一の実行ファイルにバンドル
- 結果として得られるアプリは通常 10〜20MB です（Electron の 150MB 以上と比較）

実行ファイルを直接実行できます - ランタイムは不要で、起動するサーバーもありません。真のネイティブアプリケーションです。
