Wailsにおける**サービス**は、フロントエンドから利用可能にしたいビジネスロジックを含むGoの構造体です。サービスは関連する機能をグループ化することで、コードの整理を保ちます。

サービスを、JavaScriptコードが呼び出すことができるメソッドのコレクションと考えると良いでしょう。サービス上の各公開メソッドは、バインディングを生成した後、フロントエンドから呼び出すことができます。

このチュートリアルでは、これらの概念を実証するためにQRコード生成サービスを作成します。終わるまでには、サービスの作成、依存関係の管理、Goコードとフロントエンドの接続方法を理解していることになります。

<br/>

<Steps>

1. ## QRサービスファイルの作成

   アプリケーションディレクトリに `qrservice.go` という名前の新しいファイルを作成します:

   ```go title="qrservice.go"
   package main

   import (
       "github.com/skip2/go-qrcode"
   )

   // QRService handles QR code generation
   type QRService struct {
       // We can add state here if needed
   }

   // NewQRService creates a new QR service
   func NewQRService() *QRService {
       return &QRService{}
   }

   // Generate creates a QR code from the given text
   func (s *QRService) Generate(text string, size int) ([]byte, error) {
       // Generate the QR code
       qr, err := qrcode.New(text, qrcode.Medium)
       if err != nil {
           return nil, err
       }

       // Convert to PNG
       png, err := qr.PNG(size)
       if err != nil {
           return nil, err
       }

       return png, nil
   }
   ```

   **ここで何が起こっているか:**

   - `QRService` は、QRコード生成メソッドを保持する空の構造体です
   - `NewQRService()` は、サービスの新しいインスタンスを作成するコンストラクタ関数です
   - `Generate()` は、テキストとサイズを受け取り、QRコードをPNGバイト配列として返すメソッドです
   - このメソッドは、エラーを最後の値として返すGoの規約に従って `([]byte, error)` を返します
   - 実際のQRコード生成には `github.com/skip2/go-qrcode` パッケージを使用しています

    <br/>

2. ## サービスの登録

   サービスを作成するだけでは不十分です。Wailsアプリケーションにサービスを**登録**し、サービスが存在することを知り、それ用のバインディングを生成できるようにする必要があります。

   登録は、アプリケーションを作成する際に `main.go` で行われます。サービスインスタンスを `Services` オプションに渡します:

   ```go title="main.go" ins={7-9}
    func main() {

        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService()),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

   **ここで何が起こっているか:**

   - `application.NewService()` は、Wailsが管理できるようにサービスをラップします
   - `NewQRService()` を呼び出して、サービスのインスタンスを作成します
   - サービスはアプリケーションオプションの `Services` スライスに追加されます
   - Wailsは今後、フロントエンドで利用可能にするために公開メソッドをこのサービスからスキャンします

    <br/>

3. ## 依存関係のインストール

   コード内で `github.com/skip2/go-qrcode` パッケージを参照しましたが、まだ実際にダウンロードしていません。Goはこの依存関係を認識し、プロジェクトにダウンロードする必要があります。

   プロジェクトディレクトリからターミナルで次のコマンドを実行します:

   ```bash
   go mod tidy
   ```

   **ここで何が起こっているか:**

   - `go mod tidy` は、Goファイルのインポート文をスキャンします
   - 欠落しているパッケージ（`go-qrcode` など）をダウンロードし、`go.mod` に追加します
   - また、もはや使用されていない依存関係も削除します
   - これにより、プロジェクトが正常にコンパイルするために必要なコードがすべて揃っていることが保証されます

   QRコードパッケージがダウンロードされ、プロジェクトに追加されたことを示す出力が表示されるはずです。

    <br/>

4. ## バインディングの生成

   フロントエンドからこれらのメソッドを呼び出すには、バインディングを生成する必要があります。
   プロジェクトのルートディレクトリで `wails generate bindings` を実行することでこれを行うことができます。

   :::note
   プロジェクトでこれを初めて実行する場合、バインディングジェネレーターはコードと依存関係の徹底的な分析を行います。これには予想よりも少し時間がかかる場合がありますが、その後の実行ははるかに高速になります。
   :::

   これを実行すると、ターミナルに次のような出力が表示されるはずです:

   ```bash
    % wails3 generate bindings
    INFO  Processed: 337 Packages, 1 Service, 1 Method, 0 Enums, 0 Models in 740.196125ms.
    INFO  Output directory: /Users/leaanthony/myproject/frontend/bindings
   ```

   フロントエンドディレクトリ内に `bindings` という名前の新しいディレクトリがあることに気づくはずです:

   ```bash
   frontend/
   └── bindings
       └── changeme
           ├── index.js
           └── qrservice.js
   ```

    :::tip[Pro Tip]
        `wails3 build` を使用してアプリケーションをビルドすると、バインディングが自動的に生成され、最新の状態に保たれます。
    :::
    <br/>

5. ## バインディングの理解

   `bindings/changeme/qrservice.js` 内の生成されたバインディングを見てみましょう:

   ```js title="bindings/changeme/qrservice.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT

    /**
     * QRService handles QR code generation
     * @module
     */

    // eslint-disable-next-line @typescript-eslint/ban-ts-comment
    // @ts-ignore: Unused imports
    import {Call as $Call, Create as $Create} from "@wailsio/runtime";

    /**
     * Generate creates a QR code from the given text
     * @param {string} text
     * @param {number} size
     * @returns {Promise<string> & { cancel(): void }}
     */
    export function Generate(text, size) {
        let $resultPromise = /** @type {any} */($Call.ByID(3576998831, text, size));
        let $typingPromise = /** @type {any} */($resultPromise.then(($result) => {
            return $Create.ByteSlice($result);
        }));
        $typingPromise.cancel = $resultPromise.cancel.bind($resultPromise);
        return $typingPromise;
    }
   ```

   バインディングが `Generate` メソッド用に生成されていることがわかります。パラメータ名は保持され、
   コメントも保持されています。また、IDEに型情報を提供するためにメソッド用にJSDocも生成されています。

   :::note
   生成されたバインディングを完全に理解する必要はありませんが、それらがどのように機能するかを理解することは重要です。
   :::

   バインディングは以下のものを提供します:
   - Goのメソッドに相当する関数
   - GoとJavaScriptの型間の自動変換
   - Promiseベースの非同期操作
   - JSDocコメントとしての型情報

   :::tip[Typescript]
   バインディングジェネレーターは、TypeScriptバインディングの生成もサポートしています。これを行うには、`wails3 generate bindings -ts` を実行します。
   :::

   生成されたサービスは `index.js` ファイルによって再エクスポートされます:

   ```js title="bindings/changeme/index.js"
    // @ts-check
    // Cynhyrchwyd y ffeil hon yn awtomatig. PEIDIWCH Â MODIWL
    // This file is automatically generated. DO NOT EDIT```js title="frontend/src/bindings/index.js"
    import * as QRService from "./qrservice.js";
    export {
        QRService
    };
   ```

    その後、ファイル名を指定せずに Go パッケージパスのみで構成される簡略化されたインポートパス
    `./bindings/changeme` を通じてアクセスできます。

    :::note
    簡略化されたインポートパスは、フロントエンドのバンドラーを使用している場合にのみ利用可能です。
    バンドラーを使用しないバニラのフロントエンドを好む場合は、`index.js` または `qrservice.js` を手動でインポートする必要があります。
    :::
    <br/>

6. ## フロントエンドでのバインディングの使用

    これで JavaScript から Go サービスを呼び出すことができます！生成されたバインディングにより、これは簡単かつ型安全に行えます。

    新しいバインディングを使用するように `frontend/src/main.js` を更新します：

   ```js title="frontend/src/main.js"
    import { QRService } from './bindings/changeme';

    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        try {
            // Generate QR code as base64
            const qrCodeBase64 = await QRService.Generate(text, 256);

            // Display the QR code
            const qrDiv = document.getElementById('qrcode');
            qrDiv.src = `data:image/png;base64,${qrCodeBase64}`;

        } catch (err) {
            console.error('Failed to generate QR code:', err);
            alert('Failed to generate QR code: ' + err);
        }
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        button.addEventListener('click', generateQR);
    }
   ```

   **ここで何が起こっているか：**

   - 生成されたバインディングから `QRService` をインポートします
   - `QRService.Generate()` は Go のメソッドを呼び出します - Promise を返すため、`await` を使用します
   - Go メソッドは `[]byte` を返しますが、Wails はこれを自動的に JavaScript 用の base64 文字列に変換します
   - base64 文字列を使用してデータ URL を作成し、PNG 画像を表示します
   - `try/catch` ブロックは Go 側からのエラー（無効な入力など）を処理します
   - Go コードがエラーを返した場合、Promise は拒否され、ここでキャッチされます

   次に、`initializeQRGenerator` 関数で新しいバインディングを使用するように `index.html` を更新します：

7. ## 代替アプローチ：HTTP ハンドラー

    これまでに以下の領域をカバーしました：
    - 新しいサービスの作成
    - バインディングの生成
    - フロントエンドコードでのバインディングの使用

    **なぜ HTTP ハンドラーを使用するのか？**

    メソッドバインディングはデータ操作には非常に適していますが、ファイル、画像、その他のメディアを提供するための代替アプローチがあります。すべてを base64 に変換してバインディングを通じて送信する代わりに、サービスをミニウェブサーバーのように機能させることができます。

    これは以下の場合に便利です：
    - 画像、動画、または大きなファイルを提供する場合
    - `src` 属性を持つ標準的な HTML `<img>` または `<video>` タグを使用したい場合
    - リソースへの直接 URL アクセスが必要な場合

    サービスが Go の標準 `ServeHTTP(w http.ResponseWriter, r *http.Request)` メソッドを実装している場合、Wails はそれを HTTP エンドポイントとしてアクセス可能にできます。QR コードサービスにこれをサポートさせるために拡張しましょう：

    ```go title="qrservice.go" ins={4-5,37-65}
    package main

    import (
        "net/http"
        "strconv"

        "github.com/skip2/go-qrcode"
    )

    // QRService handles QR code generation
    type QRService struct {
        // We can add state here if needed
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **ここで何が起こっているか：**

    - `ServeHTTP` は Go の標準的な HTTP リクエストを処理するためのインターフェースです
    - URL からクエリパラメータを解析します（`?text=hello&size=256`）
    - 既存の `Generate()` メソッドを呼び出して QR コードを作成します
    - コンテンツタイプを `image/png` に設定し、ブラウザがそれが画像であることを認識できるようにします
    - base64 は不要です！生の PNG バイトを直接レスポンスに書き込みます

    次に、QR コードサービスがアクセス可能なルートを指定するように `main.go` を更新します：

   ```go title="main.go" ins={8-10}
    func main() {```go title="main.go"
        app := application.New(application.Options{
            Name:        "myproject",
            Description: "A demo of using raw HTML & CSS",
            LogLevel:    slog.LevelDebug,
            Services: []application.Service{
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/qrservice",
                }),
            },
            Assets: application.AssetOptions{
                Handler: application.AssetFileServerFS(assets),
            },
            Mac: application.MacOptions{
                ApplicationShouldTerminateAfterLastWindowClosed: true,
            },
        })

        app.Window.NewWithOptions(application.WebviewWindowOptions{
            Title:  "myproject",
            Width:  600,
            Height: 400,
        })

        // Run the application. This blocks until the application has been exited.
        err := app.Run()

        // If an error occurred while running the application, log it and exit.
        if err != nil {
            log.Fatal(err)
        }
    }
   ```

    **ここで何が起こっているか:**

    - `application.ServiceOptions` を追加して、サービスがどのように公開されるかを設定します
    - `Route: "/qrservice"` は、HTTP ハンドラーを `/qrservice` でアクセス可能にします
    - これで、`/qrservice?text=hello` へのすべてのリクエストが `ServeHTTP` メソッドを呼び出します
    - `Route` を設定しないと、HTTP ハンドラーの機能は無効になります

    :::note
    `Route` オプションを明示的に設定しない場合、
    フロントエンドから HTTP ハンドラーにアクセスできません。
    :::

    最後に、base64 エンコーディングの代わりに単純な画像 `src` を使用するよう `main.js` を更新します：

    ```js title="frontend/src/main.js"
    async function generateQR() {
        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;
        }

        const img = document.getElementById('qrcode');
        // Make the image source the path to the QR code service, passing the text
        img.src = `/qrservice?text=${encodeURIComponent(text)}`
    }

    export function initializeQRGenerator() {
        const button = document.getElementById('generateButton');
        if (button) {
            button.addEventListener('click', generateQR);
        } else {
            console.error('Generate button not found');
        }
    }
    ```

    **ここで何が起こっているか:**

    - インポートと `await QRService.Generate()` 呼び出しを削除しました
    - その代わりに、`img.src` を HTTP エンドポイントを指すように設定するだけです
    - `encodeURIComponent()` は URL 内の特殊文字を安全にエスケープします
    - `src` を設定すると、ブラウザは自動的に HTTP GET リクエストを行います
    - これは画像にとってより単純で効率的です - base64 変換は不要です！

    アプリケーションを再度実行すると、同じ QR コードが表示されます：

    <Image src={qr1} alt="QR Code"/>

8. ## 動的構成のサポート

    **ハードコードされたルートの問題点:**

    上記の例では、JavaScript コード内でハードコードされたルート `/qrservice` を使用しました。
    これにより、Go の構成とフロントエンド コードの間に密結合が生じます。

    `main.go` を編集して `Route` オプションを変更し、`main.js` を更新しないと、
    アプリケーションは壊れます：

    ```go title="main.go" ins={3}
            // ...
                application.NewService(NewQRService(), application.ServiceOptions{
                    Route: "/services/qr",
                }),
            // ...
    ```

    ハードコードされたルートは単純なアプリケーションでは機能しますが、コードを脆くし、保守を困難にします。

    **解決策：動的構成**

    メソッド バインディングと HTTP ハンドラーは一緒に動作できます！バインディングを使用して、フロントエンドに使用するルートを知らせ、構成を動的にし、ハードコードされたパスを排除します。

    仕組みは次のとおりです：
    1. `ServiceStartup` ライフサイクル メソッドはアプリの起動時に実行されます
    2. オプションから構成されたルートを保存します
    3. フロントエンドが呼び出して正しいルートを取得できる `URL()` メソッドを追加します
    4. これでフロントエンドは、推測するのではなく Go サービスにルートを問い合せます

    まず、`ServiceStartup` インターフェースを実装し、新しい `URL` メソッドを追加します：

    ```go title="qrservice.go" ins={4,6,10,15,23-27,46-55}
    package main

    import (
        "context"
        "net/http"
        "net/url"
        "strconv"

        "github.com/skip2/go-qrcode"
        "github.com/wailsapp/wails/v3/pkg/application"
    )

    // QRService handles QR code generation
    type QRService struct {
        route string
    }

    // NewQRService creates a new QR service
    func NewQRService() *QRService {
        return &QRService{}
    }

    // ServiceStartup runs at application startup.
    func (s *QRService) ServiceStartup(ctx context.Context, options application.ServiceOptions) error {
        s.route = options.Route
        return nil
    }

    // Generate creates a QR code from the given text
    func (s *QRService) Generate(text string, size int) ([]byte, error) {
        // Generate the QR code
        qr, err := qrcode.New(text, qrcode.Medium)
        if err != nil {
            return nil, err
        }

        // Convert to PNG
        png, err := qr.PNG(size)
        if err != nil {
            return nil, err
        }

        return png, nil
    }

    // URL returns an URL that may be used to fetch
    // a QR code with the given text and size.
    // It returns an error if the HTTP handler is not available.
    func (s *QRService) URL(text string, size int) (string, error) {
        if s.route == "" {
            return "", errors.New("http handler unavailable")
        }

        return fmt.Sprintf("%s?text=%s&size=%d", s.route, url.QueryEscape(text), size), nil
    }

    func (s *QRService) ServeHTTP(w http.ResponseWriter, r *http.Request) {
        // Extract the text parameter from the request
        text := r.URL.Query().Get("text")
        if text == "" {
            http.Error(w, "Missing 'text' parameter", http.StatusBadRequest)
            return
        }
        // Extract Size parameter from the request
        sizeText := r.URL.Query().Get("size")
        if sizeText == "" {
            sizeText = "256"
        }
        size, err := strconv.Atoi(sizeText)
        if err != nil {
            http.Error(w, "Invalid 'size' parameter", http.StatusBadRequest)
            return
        }

        // Generate the QR code
        qrCodeData, err := s.Generate(text, size)
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }

        // Write the QR code data to the response
        w.Header().Set("Content-Type", "image/png")
        w.Write(qrCodeData)
    }
    ```

    **ここで何が起こっているか:**

    - `ServiceStartup` から構成されたルートを保存するための `route` フィールドを追加しました
    - `ServiceStartup(ctx, options)` はアプリ起動時に呼び出され、ここでルートを保存します
    - `URL()` メソッドは、クエリ パラメータを含む完全な URL を構築します
    - ルートが構成されていない場合（ルートが空の場合）、エラーを返します
    - `url.QueryEscape()` は、URL で使用するためにテキストを安全にエンコードします
    - このメソッドは、バインディングを通じてフロントエンドから利用可能になります

    次に、ハードコードされたパスの代わりに `URL` メソッドを使用するよう `main.js` を更新します：

    ```js title="frontend/src/main.js" ins={1,11-12}
    import { QRService } from "./bindings/changeme";

        const text = document.getElementById('text').value;
        if (!text) {
            alert('Please enter some text');
            return;

        const img = document.getElementById('qrcode');
        // Invoke the URL method to obtain an URL for the given text.

**ここで何が起こっているか：**

- `QRService` をインポートして、バインディングを再び使用します
- `/qrservice` をハードコードする代わりに、`await QRService.URL(text, 256)` を呼び出します
- Go サービスは、正しいルートとパラメータを使用して URL を構築します
- `main.go` でルートを変更した場合、フロントエンドは自動的に新しいルートを使用します
- Go の設定とフロントエンド コード間の手動同期は不要になりました！
